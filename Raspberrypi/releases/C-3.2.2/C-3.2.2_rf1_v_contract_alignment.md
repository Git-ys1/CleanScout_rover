## 树莓派端当前已提供的能力

本轮完成后，树莓派端已提供以下最小联调能力：

1. OpenRF1 USB 串口桥
2. `/cmd_vel -> W,a,b,c,d` 实时下发链
3. `/rf1/vel -> /odom` 轻量积分器
4. 标准 `/imu/data`
5. 真实雷达 `/scan`
6. `rosbridge_websocket`，端口 `9090`
7. 一键清场脚本
8. 一键启动脚本

## 当前联调合同

### 1. rosbridge 入口

树莓派侧当前提供：

```text
ws://<树莓派IP>:9090
```

本轮固定：

1. transport：`rosbridge`
2. 端口：`9090`
3. 场景：局域网本地联调

本轮不处理：

1. 公网暴露
2. NAT 穿透
3. FRP / VPN / ECS

### 2. topic 合同冻结

V 线 backend 可直接使用以下 topic，不需要再改名：

```text
/cmd_vel
/odom
/imu/data
/scan
```

树莓派内部还提供：

```text
/rf1/vel
/rf1/pwm
/rf1/enc
/rf1/ack
/rf1/raw_rx
/rf1/status
```

### 3. RF1 串口契约冻结

当前 RF1 侧口径为：

1. 物理接入：OpenRF1 板载 USB -> 树莓派 USB
2. 串口设备优先：`/dev/serial/by-id/usb-1a86_USB_Serial-if00-port0`
3. 备选：`/dev/ttyUSB1`
4. 波特率：`115200`
5. 主命令：`W,a,b,c,d`
6. 单位：四轮目标速度，`m/s`
7. 控制发送：`50Hz`
8. 板端回传：`ACK / VEL / PWM / ENC / DBG`

### 4. V 线安全限幅冻结

供第一轮真实联调使用的安全阈值如下：

1. `vx ∈ [-0.20, 0.20] m/s`
2. `vy ∈ [-0.15, 0.15] m/s`
3. `wz ∈ [-0.35, 0.35] rad/s`
4. hold timeout：`400ms`

即：

1. backend 可以直接发布 `/cmd_vel`
2. 树莓派侧会做限幅
3. 若 `/cmd_vel` 超过 `400ms` 不刷新，树莓派会自动发送全零目标

## V 线如何接入

### 1. backend 侧参数

建议直接使用：

```text
ROS_TRANSPORT=rosbridge
ROSBRIDGE_URL=ws://<树莓派IP>:9090
```

### 2. 订阅 topic

V 线可直接订阅：

1. `/odom`
2. `/imu/data`
3. `/scan`

### 3. 发布 topic

V 线直接发布：

1. `/cmd_vel`

不需要再做：

1. topic 改名
2. mock 适配层 topic 重写
3. ticks/s 到 m/s 的额外转换

### 4. 联调建议顺序

建议 V 线按下面顺序切换：

1. 浏览器或 backend 先连 `ws://<树莓派IP>:9090`
2. 确认 `/odom / imu/data / scan` 可订阅
3. 用低速 `/cmd_vel` 试发
4. 观察底盘响应和 `/odom` 变化
