# C-2.3.2 Fan PWM Minimal Verify

## 文档定位

- 任务：`C-2.3.2B`
- 当前目标：树莓派直控风机的最小验证

## 当前冻结接线

### 主供电

1. 风机 12V 主供电继续走小车 12V 主电源
2. 继电器只负责切 12V 总开关

### 控制脚

1. `GPIO17 / Pin 11 -> 继电器 IN`
2. `GPIO18 / Pin 12 -> 风机 A PWM`
3. `GPIO19 / Pin 35 -> 风机 B PWM`
3. `GPIO23 -> 前级 FG`
4. `GPIO24 -> 后级 FG`

## 当前冻结控制口径

1. `relay enable` 与 `PWM duty` 分层
2. `GPIO17` 为双风机共享继电器总开关
3. `GPIO18` 为风机 A 独立 PWM
4. `GPIO19` 为风机 B 独立 PWM
5. `enable=false` 时：继电器断开，PWM=0
6. `enable=true` 时：先闭合继电器，延时约 `0.2s`，再分别输出风机 A / B PWM
7. 停机顺序：先将 A/B PWM 都置 0，再断继电器

## 当前 GPIO23/24 上拉判断

当前 FG 冻结口径为：

1. 风机 A：`FG_in -> GPIO23`, `FG_out -> GPIO24`
2. 风机 B：`FG_in -> GPIO25`, `FG_out -> GPIO16`

且每路按任务书口径应上拉到 `3.3V`。

树莓派 GPIO 可以启用**内部上拉输入模式**，所以本轮已用内部上拉做最小尝试。

但当前冻结结论是：

1. 这可以用于最小验证
2. 内部上拉更弱
3. 如果 FG 脉冲后续不稳，仍建议回到外部 `4.7k ~ 10k` 上拉到 `3.3V`

## 当前实现前提

本轮软件实现冻结为 `pigpio` 路线，因为目标是稳定输出 `25 kHz PWM`。

当前系统若无以下组件，则本轮实测会阻塞：

1. `pigpio` Python 模块
2. `pigpiod` 守护进程

## 当前验收标准

1. `GPIO17` 能控制继电器吸合/释放
2. `GPIO18` 能输出 `25 kHz PWM`
3. 风机能在 `0 / 30 / 60 / 100%` 下观察到启停与转速变化
4. 本轮不依赖 RViz
5. 本轮不把风机重新塞回 UNO 主线

## 本轮实测结果

本轮已实际完成两步硬件验证。

### 1. 继电器验证

先执行纯 `GPIO17` 翻转测试，已实际听到继电器吸合/释放。

结论：

1. `GPIO17 -> 继电器 IN` 已通
2. 继电器控制链可用

### 2. 双风机 PWM 验证

当前双风机验证序列为：

```text
verify1 relay on, no pwm
verify2 fan A only 60%
verify3 fan B only 60%
verify4 fan A 30%, fan B 70%
stop all
```

现场最终反馈为：

```text
完全一致
```

这说明四段验证均符合任务书要求：

1. 只吸继电器时，两只风机不乱跑
2. 只给风机 A 时，只有风机 A 变化
3. 只给风机 B 时，只有风机 B 变化
4. `A=30%, B=70%` 时存在明显转速差

单风机方案到此已升级为双风机方案。

### 3. FG 最小尝试

本轮已补做双风机 FG 读取尝试：

1. 已确认脚本侧使用的是 `GPIO.IN + GPIO.PUD_UP`
2. 已确认不是“上拉输出”错误配置
3. 已做过 `FALLING` 计数版本
4. 已做过 `BOTH` 边沿计数版本

本轮最后一次复测结果为：

```text
counts phase1: {'a_in': 0, 'a_out': 0, 'b_in': 0, 'b_out': 0}
counts phase2: {'a_in': 0, 'a_out': 0, 'b_in': 0, 'b_out': 0}
```

因此当前冻结结论为：

1. FG 已做最小软件侧复查与复测
2. 本轮未稳定复现可用脉冲
3. FG 不阻塞当前版本收口
4. 后续如继续推进，优先结合外部上拉和接线复核再做下一轮

### 4. 单风机历史记录保留

此前单风机最小验证也已成功，记录保留，不删除，作为当前双风机版本的演进参考。

```text
relay on
set fan pwm percent=0
set fan pwm percent=30
set fan pwm percent=60

```

在单风机早期测试中，第一次风机无反应，后经现场确认原因为：


```text
开始转了，顺利收工
```

这说明当前最小主线已经成立：

1. 继电器可切 12V 总开关
2. PWM 输出后风机可启动
3. 当前接线与控制链总体有效
