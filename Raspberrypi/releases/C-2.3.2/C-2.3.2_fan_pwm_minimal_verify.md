# C-2.3.2 Fan PWM Minimal Verify

## 文档定位

- 任务：`C-2.3.2B`
- 当前目标：树莓派直控风机的最小验证

## 当前冻结接线

### 主供电

1. 风机 12V 主供电继续走小车 12V 主电源
2. 继电器只负责切 12V 总开关

### 控制脚

1. `GPIO17 / Pin 11 -> 继电器 IN`
2. `GPIO18 / Pin 12 -> 风机 PWM`
3. `GPIO23 -> 前级 FG`
4. `GPIO24 -> 后级 FG`

## 当前冻结控制口径

1. `relay enable` 与 `PWM duty` 分层
2. `enable=false` 时：继电器断开，PWM=0
3. `enable=true` 时：先闭合继电器，延时约 `0.2s`，再输出 `25 kHz PWM`
4. 停机顺序：先 `PWM=0`，再断继电器

## 当前 GPIO23/24 上拉判断

当前黄色/白色 FG 线已直接接到 `GPIO23/24`，未外接上拉电阻。

树莓派 GPIO 可以启用**内部上拉输入模式**，所以本轮可先用内部上拉做最小尝试。

但当前冻结结论是：

1. 这可以用于最小验证
2. 内部上拉更弱
3. 如果 FG 脉冲后续不稳，仍建议回到外部 `4.7k ~ 10k` 上拉到 `3.3V`

## 当前实现前提

本轮软件实现冻结为 `pigpio` 路线，因为目标是稳定输出 `25 kHz PWM`。

当前系统若无以下组件，则本轮实测会阻塞：

1. `pigpio` Python 模块
2. `pigpiod` 守护进程

## 当前验收标准

1. `GPIO17` 能控制继电器吸合/释放
2. `GPIO18` 能输出 `25 kHz PWM`
3. 风机能在 `0 / 30 / 60 / 100%` 下观察到启停与转速变化
4. 本轮不依赖 RViz
5. 本轮不把风机重新塞回 UNO 主线
