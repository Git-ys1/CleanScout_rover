# C-2.3.2 Fan PWM Minimal Verify

## 文档定位

- 任务：`C-2.3.2B`
- 当前目标：树莓派直控风机的最小验证

## 当前冻结接线

### 主供电

1. 风机 12V 主供电继续走小车 12V 主电源
2. 继电器只负责切 12V 总开关

### 控制脚

1. `GPIO17 / Pin 11 -> 继电器 IN`
2. `GPIO18 / Pin 12 -> 风机 PWM`
3. `GPIO23 -> 前级 FG`
4. `GPIO24 -> 后级 FG`

## 当前冻结控制口径

1. `relay enable` 与 `PWM duty` 分层
2. `enable=false` 时：继电器断开，PWM=0
3. `enable=true` 时：先闭合继电器，延时约 `0.2s`，再输出 `25 kHz PWM`
4. 停机顺序：先 `PWM=0`，再断继电器

## 当前 GPIO23/24 上拉判断

当前黄色/白色 FG 线已直接接到 `GPIO23/24`，未外接上拉电阻。

树莓派 GPIO 可以启用**内部上拉输入模式**，所以本轮可先用内部上拉做最小尝试。

但当前冻结结论是：

1. 这可以用于最小验证
2. 内部上拉更弱
3. 如果 FG 脉冲后续不稳，仍建议回到外部 `4.7k ~ 10k` 上拉到 `3.3V`

## 当前软件实现

新增包：

```text
Raspberrypi/catkin_ws/src/csrpi_fan_bridge
```

包含：

1. `fan_pwm_test.py`
2. `fan_bridge.py`

## 当前实现前提

本轮软件实现冻结为 `pigpio` 路线，因为目标是稳定输出 `25 kHz PWM`。

当前系统若无以下组件，则本轮实测会阻塞：

1. `pigpio` Python 模块
2. `pigpiod` 守护进程

## 当前验收标准

1. `GPIO17` 能控制继电器吸合/释放
2. `GPIO18` 能输出 `25 kHz PWM`
3. 风机能在 `0 / 30 / 60 / 100%` 下观察到启停与转速变化
4. 本轮不依赖 RViz
5. 本轮不把风机重新塞回 UNO 主线

## 本轮实测结果

本轮已实际完成两步硬件验证。

### 1. 继电器验证

先执行纯 `GPIO17` 翻转测试，已实际听到继电器吸合/释放。

结论：

1. `GPIO17 -> 继电器 IN` 已通
2. 继电器控制链可用

### 2. 风机 PWM 验证

随后执行：

```text
relay on
set fan pwm percent=0
set fan pwm percent=30
set fan pwm percent=60
set fan pwm percent=100
stop fan
```

第一次测试时风机无反应，后经现场确认原因为：

```text
风机 12V 主电源当时未打开
```

补上主电源后再次执行同一序列，现场结果反馈为：

```text
开始转了，顺利收工
```

这说明当前最小主线已经成立：

1. 继电器可切 12V 总开关
2. PWM 输出后风机可启动
3. 当前接线与控制链总体有效

## 当前阶段结论

本轮 `C-2.3.2B` 已完成最小通过：

1. 继电器主线可控
2. 风机在补上 12V 总电源后能启动
3. 树莓派已完成“继电器 enable + PWM duty”两层控制的最小验证
