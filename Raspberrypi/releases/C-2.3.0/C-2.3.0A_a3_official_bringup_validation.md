# C-2.3.0A A3 Official Bringup Validation

## 文档定位

- 任务：`C-2.3.0A`
- 执行侧：树莓派端
- 目标：仅验证 `A3 -> 官方 rplidar_ros -> /scan` 链路
- 范围外：不接导航、不接 SLAM、不接底盘运动、不接 UNO 控制

## 当前 bench 结论

本轮已完成 RPLIDAR A3 在树莓派 Noetic 工作区中的官方驱动直连验证，结果为通过。

## 设备枚举结果

当前 A3 转接板被系统识别为：

```text
Silicon Labs CP2102 USB to UART Bridge
```

识别信息：

- `VID:PID = 10c4:ea60`
- `Serial = bde26819d978564a98616dc8161b0fd6`
- 当前设备节点：`/dev/ttyUSB0`

## 固定别名结果

已安装 udev 规则：

```text
/etc/udev/rules.d/99-clblidar.rules
```

规则内容：

```udev
SUBSYSTEM=="tty", ATTRS{idVendor}=="10c4", ATTRS{idProduct}=="ea60", ATTRS{serial}=="bde26819d978564a98616dc8161b0fd6", SYMLINK+="clblidar"
```

固定别名已成功生效：

```text
/dev/clblidar -> /dev/ttyUSB0
```

## 官方 bringup 命令

本轮通过口径使用的是官方 A3 launch，而不是项目内自定义 launch：

```bash
roslaunch rplidar_ros rplidar_a3.launch serial_port:=/dev/clblidar serial_baudrate:=256000
```

## 波特率验证结论

这是本轮最关键的硬件约束之一。

### 115200 结果

当转接板拨在 `115200` 时：

1. `rplidarNode` 可以启动
2. `/scan` 不出现
3. 节点约 5 秒后退出
4. 该口径不能通过 A3 官方 bringup

### 256000 结果

当转接板切到 `256000` 并热插拔后：

1. `rplidarNode` 成功启动
2. `/scan` 成功出现
3. 节点保持运行
4. bench bringup 通过

因此，当前 A3 的有效工作口径应冻结为：

```text
serial_port=/dev/clblidar
serial_baudrate=256000
```

## /scan 实测结果

ROS 观测结果：

1. `/scan` 存在
2. `rostopic hz /scan` 稳定在约 `12 Hz`
3. `rostopic echo -n 1 /scan` 成功输出完整 `sensor_msgs/LaserScan`

实际测得频率片段：

```text
average rate: 11.920
average rate: 11.995
average rate: 11.992
average rate: 12.011
average rate: 12.007
```

LaserScan 中已实际看到：

1. `header.frame_id = laser`
2. `angle_min`
3. `angle_max`
4. `angle_increment`
5. `scan_time`
6. `ranges`
7. `intensities`

## 通过判定

任务书要求的四条通过标准，当前都已满足：

1. `/dev/clblidar` 固定别名成功
2. 官方 `rplidar_ros rplidar_a3.launch` 能起
3. `/scan` 可见且有稳定频率
4. 整个过程不依赖 `clbrobot` 自定义 launch

因此：

```text
C-2.3.0A 通过
```

## 当前冻结约束

本轮需要明确冻结以下约束，避免后续回退到错误口径：

1. A3 首测必须先走官方 `rplidar_ros`，不先走 `clbrobot` 自定义 launch
2. 当前有效别名是 `/dev/clblidar`
3. 当前有效波特率是 `256000`
4. `115200` 不是本轮 A3 通过口径
5. 本轮只确认雷达扫描链，不把导航层、SLAM 层、底盘控制层混进同一次验证

## 对后续 C-2.3.0B 的直接意义

现在已经满足回写项目内入口的前提条件：

1. 官方 A3 bringup 已通过
2. `/scan` 已稳定可用
3. 项目内下一步应新增或回写 A3 专用入口
4. 该入口必须默认：
   - `serial_port:=/dev/clblidar`
   - `serial_baudrate:=256000`

## 一句话总结

本轮已经证明：

```text
树莓派 Noetic 工作区可通过官方 rplidar_ros 直接打通 A3 雷达链；
关键前提是使用 /dev/clblidar 固定别名，并将转接板与 launch 统一在 256000 波特率口径。
```
