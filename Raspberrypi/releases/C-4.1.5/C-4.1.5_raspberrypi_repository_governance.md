ROS Noetic 工作区。保留现有文件位置和启动链，仅通过 README 明确每一层职责、
当前入口、历史兼容项、风险和维护规则。

## 为什么没有移动 shell 文件

当前多个树莓派脚本仍包含：

- `/home/clbrobot/Work/CleanScout_rover/Raspberrypi/catkin_ws` 固定路径
- 对同级环境、清理和启动脚本的调用
- 实车部署和人工操作形成的既有入口

直接迁入 `shfiles/` 会同时影响部署路径、脚本调用、交接文档和现场操作习惯。
本轮先完成可读性治理；后续若要移动，应先把路径改为基于 `BASH_SOURCE` 的相对定位，
再为旧入口提供兼容包装并完成 PC/树莓派双端回归。

删除动作应在远端设备、systemd、crontab、桌面快捷方式和交接文档引用核验后单独进行。

## 已知事项

1. 一些旧 launch 和脚本仍保存早期几何参数，不应绕过当前顶层入口直接用于实车。
2. 两张地图 YAML 使用机器绝对路径，跨设备运行前需要检查 `image` 字段。
3. 顶层脚本彻底分目录是后续独立重构任务，不与本轮文档发布混合。
4. 凭据治理不属于本轮运行行为零变化范围，后续应统一迁移到环境变量或本机密钥配置。
