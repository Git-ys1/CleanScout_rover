## 1. 发布结论

1. 雷达后方物理遮挡会让 costmap 把未观测区域长期当作旧自由空间，也会让纯激光
   scan matcher 在部分朝向发生明显失真。
2. TEB 原配置确实会对后方目标输出负 `vx`，因此当前硬件必须禁止自主倒车。
3. 麦克纳姆横移和前向斜向运动继续保留；执行层只硬裁负 `vx`，不裁 `vy`。
4. 新车体二维外形封存为长 `0.244 m`、宽 `0.255 m`，暂按 `base_link` 位于几何
   中心处理。
5. 新车体无 IMU 实测 `K_M` 封存为 `0.342587 m`，命令侧与 PC 里程计侧统一。

完整原始证据见
[`../../docs/teb_rear_blind_sector_and_km_recalibration.md`](../../docs/teb_rear_blind_sector_and_km_recalibration.md)。

## 2. 雷达与 TEB 风险证据

`47` 帧、每帧 `1440` 束的实测中，主连续盲区约
`+115.6 deg -> 180 deg -> -103.2 deg`，宽约 `141 deg`，并不是边界整齐的
`180 deg` 软件裁剪。局部 costmap 后半区 `1759` 个栅格中有 `1728` 个自由栅格、
`0` 个未知栅格。

将 move_base 输出隔离到无人订阅话题后，旧 TEB 对车后 `0.30 m` 目标输出过
`vx=-0.10 m/s`；旧日志还显示优化器使用点机器人模型，并会启动旋转清障。

本版调整：

| 层级 | 发布参数 |
| --- | --- |
| TEB 初始化 | `allow_init_with_backwards_motion=false` |
| TEB 反向 | `max_vel_x_backwards=0.05`，仅满足优化器数值约束 |
| TEB 全向 | `max_vel_y=0.10`、`acc_lim_y=0.15`、`weight_kinematics_nh=1.0` |
| TEB 倒车惩罚 | `weight_kinematics_forward_drive=1000.0` |
| 执行安全门 | `min_vx=0`、`allow_lateral=true` |
| 恢复行为 | `clearing_rotation_allowed=false` |
| 激光清障 | `inf_is_valid=false` |

TEB 的权重不是硬约束，隔离测试中仍见约 `-0.00514 m/s` 的微小负速度，因此
`cmd_vel_safety_gate` 是不可删除的最终约束。

## 3. 新车体二维几何

costmap 与 TEB 使用完全相同的多边形：

```yaml
[[-0.122, -0.1275], [-0.122, 0.1275],
 [ 0.122,  0.1275], [ 0.122, -0.1275]]
```

URDF `base_link` 盒体同步为 `0.244 x 0.255 m`。车体高度、轮心坐标和雷达 TF 没有
新实测值，本版不推算、不伪造；后续若测得 `base_link` 偏置，costmap、TEB 和 URDF
必须同时更新。

## 4. K_M 标定

标定全程使用四轮编码器线速度积分和纯激光 scan matcher，明确关闭 IMU 与 odom
辅助。最终采用左右最一致的 `+/-0.20 rad/s`、`12 s` 对称样本：

| 方向 | 轮速积分 | 雷达角度 | 单向 `K_M` |
| --- | ---: | ---: | ---: |
| 左 | `+0.790819 m` | `+132.710 deg` | `0.341426 m` |
| 右 | `-0.778883 m` | `-129.815 deg` | `0.343773 m` |

左右差约 `0.7%`，绝对角度加权结果为 `0.342586545 m`，发布值取
`0.342587 m`。代回后左转的编码器 odom 与激光角只差 `0.268 deg`。

部分后续右转样本因半视场匹配分别少算 `6.38 deg`、`19.98 deg`，已作为异常样本
剔除并记录。它们说明未来仍应使用外部角度基准交叉复核，而不是说明生产 odom 需要
跟随每次 scan matcher 漂移。

## 5. 自动与实车验证

| 检查 | 结果 |
| --- | --- |
| YAML/XML/xacro/shell/Python 语法 | 通过 |
| `catkin_make` | 通过，27 个包全部构建成功 |
| `csrpi_base_bridge` 测试 | 6 个通过，0 错误、0 失败 |
| TEB 插件运行时 | polygon footprint 已加载，全向与倒车约束参数均正确 |
| 安全门倒车 + 横移注入 | 输入 `(-0.10,+0.08,+0.30)`，输出 `(0,+0.08,+0.30)` |
| 安全门前向斜移注入 | 输入 `(+0.10,-0.08,0)`，两轴原样通过 |
| 板端默认 `K_M` | 不传覆盖参数启动，参数服务器为 `0.342587` |
| 停车状态 | `/rf1/vel` 连续三帧四轮全零 |

安全门运行时注入前先停止 RF1 转换器，因此横移验证没有驱动车轮；K_M 标定则是本轮
明确授权的低速实车左右转向。

## 6. 板端同步边界

为避免覆盖树莓派现有脏工作区，本轮只对板端做最小更新：

1. 同步纯 ASCII `cmd_vel_safety_gate.py`。
2. 将活动硬件入口、RF1 最小 launch 和命令转换器中的默认 `K_M` 改为
   `0.342587`。
3. 不向树莓派同步带中文注释的 PC 导航 YAML、TEB 参数或 URDF。

云端 `main` 是完整源码基线；板端后续应在安排好停机窗口后按正式发布提交整体更新。

## 7. 已知限制与下一步

1. 后向没有可靠实时感知，自主倒车不得恢复。
2. 横移与斜向已保留，但侧向回波存在缺口，仍需开放场地软障碍回归。
3. 本轮没有执行自主平移整路线，不能替代完整 AMCL/TEB 导航验收。
4. 车体高度、`base_link` 偏置、轮心位置与雷达 TF 仍待机械测量。
5. 不同地面、电量和负载下的 `K_M` 需要后续外部角度基准交叉检查。

## 8. 回滚

代码发布后使用：

```bash
git revert <C-4.1.8-commit>
```

回滚会恢复旧 footprint、旧 `K_M` 和允许 TEB 倒车的配置，不应作为当前硬件的长期
运行方案。
