# C-4.1.3 TEB 导航新基线发布记录

## 发布信息

- 日期：2026-06-06
- 目标分支：`main`
- 发布类型：ROS Noetic 实车 TEB 导航阶段基线
- 前序基线：`C-4.1.2`
- 基线定位：自本版本起，PC 完整导航默认使用 TEB 局部规划器

## 阶段结论

本轮将已经完成实车联调且表现稳定的 TEB 导航链路正式封存为
`C-4.1.3`。传统 `TrajectoryPlannerROS` 链路继续保留为显式回退入口，
但不再是默认 PC 导航模式。

正式控制链路：

```text
move_base/TebLocalPlannerROS
  -> /cmd_vel_nav
  -> cmd_vel_safety_gate
  -> /cmd_vel
  -> RF1
```

正式定位与里程计链路：

```text
/scan + /map + /odom
  -> AMCL + move_base
  -> map -> odom -> base_link
```

TEB 仍以当前实车验证值进行优化；安全门保持最终执行侧硬限制：

```text
max_wz=0.50 rad/s
timeout=0.4 s
```

因此实际前进速度不会超过 `0.20 m/s`。本轮不提高安全门上限。

## 启动入口

默认启动：

```bash
cd /home/yusu/Work/CleanScout_rover/Raspberrypi/catkin_ws
./start_pc_full_navigation.sh
```

默认值：

```text
NAV_LAUNCH=navigation_406_rf1_teb.launch
ODOM_TOPIC=/odom
ODOM_K_M=0.1987
```

传统规划器回退：

```bash
NAV_LAUNCH=navigation_406_rf1.launch ./start_pc_full_navigation.sh
```

两个导航入口现在共享 `map_file`、`cmd_vel_topic`、`odom_frame_id` 和
`odom_topic` 参数契约，切换编码器里程计或激光匹配里程计时不会再把
局部规划器的订阅话题固定在 `/odom`。

## 本轮实现

1. 新增独立的 TEB 导航 launch、move_base 配置和局部规划器参数。
2. `start_pc_full_navigation.sh` 默认切换到 TEB，并保留
   `NAV_LAUNCH` 回退能力。
3. 保留 RF1 编码器里程计 `ODOM_K_M=0.1987` 基线。
4. 将里程计 frame 与 topic 一起透传到局部规划器。
5. 补齐 `clbrobot` 对 `teb_local_planner` 的运行依赖声明。
6. 封存 TEB、Costmap Converter 和 MBF 的准确源码修订。
7. 同步传统规划器的近期速度与仿真窗口参数，作为回退配置。

## 已知事项

1. TEB 优化上限 `max_vel_x=0.25 m/s` 高于安全门
   `max_vx=0.20 m/s`；安全门是最终执行上限，后续若统一两侧模型需
   重新做完整路线回归。
2. 在线快照中 AMCL 曾报告一次 `Standard deviation: Too large`
   的一级诊断警告；未观察到导航节点退出，但后续长路线应继续记录
   AMCL 协方差和重定位次数。
3. 本次发布验证不启动 `pigpiod`，其报错不属于导航基线阻断项。
4. 工作空间根目录的 `csm/` 和重复 `move_base_flex/` 是本地依赖下载
   残留，不进入 `C-4.1.3`；正式源码只位于 `catkin_ws/src/`。
