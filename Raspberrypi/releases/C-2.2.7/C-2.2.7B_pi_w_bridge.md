# C-2.2.7B 树莓派 W 协议桥说明

## 1. 本轮定位

本轮是树莓派端协议纠偏轮，不是 `ROS` 导航接管轮。

当前必须先把树莓派侧从：

```text
CMD,vx,vy,wz
```

纠正到：

```text
W,w1,w2,w3,w4
```

## 2. 当前主桥职责

当前主桥应拆为两层：

### 2.1 `wheel_bridge.py`

当前主节点，只负责：

- 持久串口连接 `/dev/csr_uno`
- 等待 `CSR_UNO_READY`
- 以固定频率 `20Hz` 持续发送 `W,w1,w2,w3,w4`
- 读取并记录：
  - `ACK:W`
  - `ENC,...`
  - `PID,...`
  - `ERR:...`

### 2.2 `cmdvel_to_wheels.py`

本轮只建骨架，不默认启用。

它未来负责：

- 订阅 `/cmd_vel`
- 根据 `D_EFF / WB / TW / CPR_X1_EST` 把 `vx/vy/wz` 转成四轮目标 `ticks/s`

## 3. 当前禁止事项

本轮禁止：

- 继续向 `UNO` 发 `CMD,vx,vy,wz`
- 在本轮把 `cmd_vel -> W` 一次做满
- 在本轮启动 `EKF / SLAM / 导航`

## 4. 第一阶段人工测试输入

当前建议先直接人工发轮级目标值：

- 纯前进：
  - `W,100,100,100,100`
- 纯后退：
  - `W,-100,-100,-100,-100`
- 原地旋转：
  - 先试 `W,100,100,-100,-100`
- 横移：
  - 先试 `W,100,-100,100,-100`

符号以实测轮向校正，不在本轮假装已经最终冻结。

## 5. 当前验收依据

当前树莓派端通过标准：

- 能收到 `CSR_UNO_READY`
- 能发送 `W,...`
- 能收到 `ACK:W`
- 能解析 `ENC,...`
- 能解析 `PID,...`

当前必须明确：

- 本轮不再向 `UNO` 发 `CMD,...`
- `ACK:W + ENC + PID` 才是当前验收依据

## 6. 当前参数口径

当前几何参数：

- `D_MECH = 80`
- `D_EFF = 78`
- `WB = 0.16535`
- `TW = 0.17850`
- `K = 0.171925`
- `CPR_X1_EST = 260`

这些参数只用于第二阶段 `cmd_vel -> W`，不是本轮发送 `W` 的前提。
