# C-2.2.6B Raspberry Pi Serial Bridge Execution

## 本轮目标

- 在树莓派 `Noetic` 工作区内新增最小桥接包 `csrpi_base_bridge`
- 用持久串口连接替代一次一条 shell 调试
- 订阅 `/cmd_vel` 并持续向 UNO 下发 `CMD,<vx>,<vy>,<wz>`
- 实时读取 `READY / ACK / ENC / PID / ERR`
- 为后续 `/odom`、EKF、导航接入保留桥接入口，但本轮不直接启用

## 运行环境

- Raspberry Pi
- Ubuntu 20.04 / ROS Noetic
- Python 3
- `rospy`
- `pyserial`
- UNO 设备入口：`/dev/csr_uno`
- 波特率：`115200`

## 当前冻结协议

树莓派下发：

```text
CMD,<vx>,<vy>,<wz>
```

UNO 回传：

```text
CSR_UNO_READY
ACK,...
ENC,...
PID,...
ERR,...
```

## 节点行为

1. 启动后尝试打开 `/dev/csr_uno`
2. 等待 `CSR_UNO_READY` 或 `READY`
3. 成功后持续保持串口连接，不再每条命令重连
4. 订阅 `/cmd_vel`
5. 以 `20Hz` 持续发送最近一帧 `vx / vy / wz`
6. 实时解析串口返回并发布到调试话题

## 调试话题

- `/csr_base/raw_serial_line`
- `/csr_base/encoder_debug`
- `/csr_base/pid_debug`
- `/csr_base/bridge_status`


发送测试速度：

```bash
rostopic pub /cmd_vel geometry_msgs/Twist '{linear: {x: 0.10, y: 0.00, z: 0.00}, angular: {x: 0.0, y: 0.0, z: 0.0}}' -r 1
```

## 验证目标

至少确认以下日志/现象：

1. 串口已连接
2. 收到 `CSR_UNO_READY`
3. 下发 `/cmd_vel` 后收到 `ACK`
4. 轻推电机或车体时能看到 `ENC,...`

## 已知边界

- 本轮不发布正式 `/odom`
- 本轮不启动 EKF
- 本轮不接导航栈
- `ENC` 与 `PID` 当前先发布为通用数组调试话题，不定义复杂自定义消息
- 桥接层当前是过渡层，不替代未来统一 bringup
