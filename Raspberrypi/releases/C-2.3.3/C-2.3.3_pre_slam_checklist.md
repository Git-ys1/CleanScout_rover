## 通过标准

### /scan

检查方式：

```bash
rostopic hz /scan
```

通过标准：

1. 频率稳定
2. 数据持续刷新
3. 雷达转动与实物环境变化时，数据变化合理

### /imu/data

检查方式：

```bash
rostopic hz /imu/data
```

通过标准：

1. 频率稳定
2. 数据持续刷新
3. 轻微转动车体时，姿态变化方向合理

### /odom

检查方式：

```bash
rostopic echo -n 1 /odom
```

通过标准：

1. `/odom` 必须存在
2. 落地低速前进/后退/原地转时，里程计变化方向合理
3. 不要求第一轮就绝对精确，但不能完全没有输出或明显乱跳

### /tf

检查方式：

1. `rostopic echo -n 1 /tf`
2. 分布式 RViz 观察

通过标准：

1. `odom -> base_*` 存在
2. `base_footprint -> imu_link` 存在
3. `base_link -> laser` 或等价链存在
4. 坐标方向不明显颠倒
