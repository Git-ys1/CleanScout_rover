## 阶段结论

`C-4.1.6` 将 CleanScout 默认工作网络从随机手机热点迁移到地址稳定的随身 Wi-Fi，
并保留旧热点逻辑作为显式回退模式。

| 角色 | 随身 Wi-Fi 固定地址 |
| --- | --- |
| 网关 | `192.168.8.1` |
| Raspberry Pi | `192.168.8.108` |
| Orange Pi | `192.168.8.148` |
| Ubuntu PC | `192.168.8.222` |

## 新旧逻辑

| 模式 | 状态 | Pi / ROS master | PC / local backend |
| --- | --- | --- | --- |
| `portable_wifi` | 新默认 | `192.168.8.108` | `192.168.8.222` |
| `phone_hotspot` | 旧兼容 | 当前子网 + `.84` | 当前子网 + `.190` |

默认模式不再根据 PC 当前地址猜测树莓派地址。临时回到手机热点时，必须显式设置
`CLEANSCOUT_NETWORK_MODE=phone_hotspot`。

## 代码变化

- 新增 `catkin_ws/cleanscout_network.sh`，集中维护网络模式、固定地址和旧尾号规则。
- `use_cleanscout_pc.sh` 默认将 `ROS_MASTER_URI` 指向 `192.168.8.108:11311`。
- `use_cleanscout_pi.sh` 继续从默认路由读取本机 IP，兼容两类网络。
- 当前硬件主入口和 4 个 edge 入口统一通过共享脚本解析本地 backend。
- edge relay 节点与 launch 的直接启动默认值更新为 `192.168.8.222`。
- 新增 `catkin_ws/NETWORK.md`，明确地址、覆盖变量和自动连接优先级边界。

## 发布门禁

- 对全部变更 shell 执行 `bash -n`。
- 验证默认 `portable_wifi` 和旧 `phone_hotspot` 两种地址解析。
- 验证 `CLEANSCOUT_PI_HOST` 显式覆盖优先级。
- 解析 edge relay XML，并编译检查 Python 语法。
- 在 Pi 临时只启动 `roscore`，从 PC 通过默认配置读取 `/rosout` 与 Noetic 版本，随后关闭并确认 `11311` 释放。
- 未启动底盘、雷达、IMU、风机、导航或运动话题。

## 已知事项

1. 树莓派板端仓库包含大量历史修改和 catkin 生成物，本轮不覆盖、不拉取。
2. 随身 Wi-Fi 固定地址依赖当前设备分配保持不变；若路由器恢复出厂设置，应重新核验三台地址。
3. NetworkManager AP 优先级与仓库运行模式是两个独立概念，详见 `catkin_ws/NETWORK.md`。
4. Wi-Fi 密码只保存在设备 NetworkManager 配置中，不进入 Git。
