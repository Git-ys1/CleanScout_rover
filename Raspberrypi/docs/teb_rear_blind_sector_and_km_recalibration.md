# TEB 后向盲区、新车体几何与 K_M 实车封存记录

## 1. 本轮边界

| 项目 | 状态 |
| --- | --- |
| 日期 | 2026-07-20 |
| ROS | Ubuntu / ROS Noetic / 分布式 ROS1 |
| PC | `192.168.8.222` |
| Raspberry Pi | `192.168.8.108` |
| 雷达 | RPLIDAR，后方由机械结构物理遮挡 |
| IMU | 已退出活动栈，本轮完全不使用 |
| 新车体外形 | 长 `0.244 m`、宽 `0.255 m` |
| 运动 | 低速原地左转、右转；未执行自主平移导航 |

本轮回答三个问题：后向遮挡怎样影响 costmap/TEB、是否禁止倒车，以及新车体
`K_M` 应采用什么值。最终结论是：禁止自主倒车，保留麦克纳姆横移和斜向运动，
并封存无 IMU 实测 `K_M=0.342587 m`。

车体高度、四轮中心位置和雷达 `x/y/z/yaw` 本轮没有新测量值，不能从外形长宽推导，
因此继续保留旧记录并明确标记为待复测项。

## 2. 雷达有效视场

连续采集 `47` 帧 `/scan`，每帧 `1440` 束：

| 指标 | 实测 |
| --- | --- |
| `/scan` 频率 | 约 `12.49 Hz` |
| `/rf1/vel` 频率 | 约 `10.02 Hz` |
| 最小有效量测 | `0.419 m` |
| 主连续近零回波区 | 约 `+115.6 deg -> 180 deg -> -103.2 deg` |
| 主连续盲区宽度 | 约 `141 deg` |

物理挡板按后向 `180 deg` 设计，但 ROS 数据不是边界整齐的半圆裁剪：两侧存在零碎
漏光，前视区也有由环境和机械结构造成的缺口。`+60..+75 deg` 的有效回波约
`4.7%`，所以横移虽然保留，也不能把侧向边界视为完整感知区域。

`inf_is_valid=false` 只能禁止无穷远量测作为自由射线清空盲区，不会自动把盲区标成
障碍或未知区域。

## 3. costmap 与 TEB 风险复现

旧配置运行时，局部 costmap 以机器人朝向划分后的统计为：

| 半区 | 总栅格 | 自由 | 未知 | 致命障碍 |
| --- | ---: | ---: | ---: | ---: |
| 后方 | 1759 | 1728 | 0 | 31 |
| 前方 | 1841 | 1540 | 0 | 301 |

后半区约 `98.2%` 被当作自由空间，而不是未知空间。这证明“RViz 后方没有点”不等于
“TEB 认为后方不可通行”。

随后把 move_base 输出隔离到无人订阅的 `/cmd_vel_teb_audit`，向车后 `0.30 m`
发送目标，实车执行链没有收到该命令。旧 TEB 实际输出：

```text
vx min = -0.10000 m/s
vy max = +0.02280 m/s
```

运行日志同时证明：

1. 旧 TEB 未配置 `footprint_model`，优化器把整车当作点模型。
2. `clearing_rotation_allowed=true` 时，失败恢复确实启动旋转清障。
3. `costmap_obstacles_behind_robot_dist=0.60` 只筛选已进入 costmap 的障碍，不能补盲。

因此当前硬件条件下必须禁止自主倒车，并关闭失败后的自动旋转清障。正常路径转向、
原地调头、横移和 `vx/vy` 组合斜向能力仍然保留；运行前必须人工确认旋转扫掠空间，
横移时还要重点观察盲区边界回波。

## 4. 新车体二维几何

新车体实测外形长 `0.244 m`、宽 `0.255 m`。本轮按 `base_link` 位于外形几何中心
建立二维边界：

```yaml
[[-0.122, -0.1275],
 [-0.122,  0.1275],
 [ 0.122,  0.1275],
 [ 0.122, -0.1275]]
```

该多边形已同步写入 costmap footprint 与 TEB `footprint_model`，URDF 的
`base_link` 盒体也更新为 `0.244 x 0.255 m`。这是当前唯一可执行的中心假设；若后续
测得 `base_link` 相对外壳存在偏置，三处必须一起改成非对称边界。

本轮没有修改旧的车体高度、轮心坐标和雷达 TF，因为这些量不能由外形长宽安全推算。

## 5. 无 IMU K_M 标定

标定以四轮实测线速度积分作为路程项，以纯激光 scan matcher 作为角度项：

```text
K_M = wheel_integral / lidar_angle
```

scan matcher 明确设置 `use_imu=false`、`use_odom=false`，每次从零轮速开始，脚本退出
时连续发布停车命令。首轮旧参数 `0.1987 m` 下的样本为：

| 方向 | 命令 | 时长 | 轮速积分 | 雷达角度 | `K_M` |
| --- | ---: | ---: | ---: | ---: | ---: |
| 左 | `+0.15 rad/s` | `8 s` | `+0.168782 m` | `+30.126 deg` | `0.321002 m` |
| 右 | `-0.15 rad/s` | `8 s` | `-0.164662 m` | `-28.379 deg` | `0.332439 m` |
| 左 | `+0.20 rad/s` | `12 s` | `+0.398765 m` | `+71.657 deg` | `0.318845 m` |
| 右 | `-0.20 rad/s` | `12 s` | `-0.403059 m` | `-67.712 deg` | `0.341058 m` |

首轮绝对角度加权候选为 `0.328724 m`。继续提高候选并做大角度对称复核后，得到本轮
左右最一致的一组：

| 方向 | 命令 | 时长 | 轮速积分 | 雷达角度 | 单向 `K_M` |
| --- | ---: | ---: | ---: | ---: | ---: |
| 左 | `+0.20 rad/s` | `12 s` | `+0.790819 m` | `+132.710 deg` | `0.341426 m` |
| 右 | `-0.20 rad/s` | `12 s` | `-0.778883 m` | `-129.815 deg` | `0.343773 m` |

左右差约 `0.7%`，按绝对角度加权：

```text
K_M = (0.790819 + 0.778883) / rad(132.710 + 129.815)
    = 0.342586545 m
    -> 0.342587 m
```

把 `0.342587 m` 代回命令侧和 odom 后，再做 `+0.20 rad/s`、`12 s` 左转，编码器
里程计角与激光角分别为 `123.879 deg` 和 `123.611 deg`，误差 `0.268 deg`。

后续两次右转中，scan matcher 分别少算约 `6.38 deg` 和 `19.98 deg`，而轮速积分
保持稳定。这些样本不参与 `K_M`，它们证明后向遮挡会让纯激光标定在某些朝向失真。
因此 `0.342587 m` 是当前实车工作值，但以后具备外部角度基准时仍应交叉复核；异常
scan matcher 结果不得直接覆盖生产参数。

## 6. 安全收口

| 层级 | 调整 |
| --- | --- |
| TEB 轨迹 | `allow_init_with_backwards_motion=false` |
| TEB 速度 | 反向上限仅保留数值稳定所需 `0.05`；横移 `max_vel_y=0.10` |
| TEB 权重 | 保留全向模型，使用高前向权重强烈惩罚倒车 |
| TEB footprint | 显式加载与 costmap 一致的新车体多边形 |
| Homotopy | `delete_detours_backwards=true` |
| move_base | `clearing_rotation_allowed=false` |
| 执行安全门 | 默认 `min_vx=0`、`allow_lateral=true`，硬裁倒车但保留横移 |
| LaserScan | 显式 `inf_is_valid=false` |

修改后的隔离后向目标仍可能让 TEB 优化器给出约 `-0.00514 m/s` 的微小负 `vx`，
说明高权重不是硬约束，执行层 `min_vx=0` 不得删除。安全门回归必须同时证明：负
`vx` 被裁为零，非零 `vy` 保持通过。

树莓派安全门更新后，在 RF1 转换器停用的隔离条件下注入
`vx=-0.10, vy=+0.08, wz=+0.30`，实测输出为 `(0.00, +0.08, +0.30)`；再注入
`vx=+0.10, vy=-0.08`，两轴原样通过。该测试证明自主倒车被硬裁，纯横移和前向
斜向运动没有被误伤。后向斜向仍包含负 `vx`，不属于允许动作。

PC 端实际启动 TEB 后，日志明确报告 polygon footprint 已加载；动态参数服务器读到
`max_vel_y=0.10`、`acc_lim_y=0.15`、`weight_kinematics_nh=1.0`、
`weight_kinematics_forward_drive=1000.0` 和
`allow_init_with_backwards_motion=false`。失败恢复仅执行清图，没有再启动自动旋转。

## 7. 后续验收门槛

1. 测量车体相对 `base_link` 的前、后、左、右真实偏置，以及车体高度。
2. 测量四轮中心位置与雷达 `x/y/z/yaw`，再同步校正 URDF；不要由外壳尺寸推算。
3. 在开放场地完成前进、横移、斜向、原地转向和无倒车路线回归。
4. 在盲区边缘放置软障碍，验证横移和转向时不会把缺失回波误判为安全空间。
5. 有外部角度基准后交叉复核 `K_M=0.342587 m`，重点检查不同电量和地面负载。

若以后需要恢复自主倒车，必须先增加可靠后向传感器或等效碰撞保护，再显式放开
安全门并重新验收整条导航链。横移与斜向能力当前保留，不属于待恢复功能。
