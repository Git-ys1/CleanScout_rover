先启动成熟导航第二模式：


启动默认测试路线（当前默认是 `lab_test_2point`）：
```bash

```

查询路线执行状态：
```bash
rosservice call /get_route_status
```

取消当前路线：
```bash
rosservice call /cancel_route
```

当前路线文件位置：

用途：

- 顺序发送 `move_base` action goal
- 用于先做 `home -> point_A -> home` 的人工验证路线

七、导航第二模式下手动发速度（找起步阈值）

用途：
- 在成熟导航第二模式已经启动的前提下
- 电脑端直接向 `/cmd_vel_nav` 发送一个固定速度
- 用于验证“规划器给出的某个速度，车到底动不动”

命令格式：
```bash
cd /home/yusu/Work/CleanScout_rover/Raspberrypi/catkin_ws
./send_nav_cmd.sh vx [vy] [wz] [seconds] [topic]
```

说明：
- 默认 topic 是 `/cmd_vel_nav`
- 默认持续时间是 2 秒
- 这个脚本适用于当前成熟导航第二模式，不是树莓派最小 `/cmd_vel` 自检链

前进 2 秒（测试前进起步阈值）：
```bash
cd /home/yusu/Work/CleanScout_rover/Raspberrypi/catkin_ws
./send_nav_cmd.sh 0.05 0 0 2
```

再大一点前进 2 秒：
```bash
./send_nav_cmd.sh 0.08 0 0 2
```

原地旋转 2 秒：
```bash
./send_nav_cmd.sh 0 0 0.20 2
```

横移 2 秒（如果当前链路允许）：
```bash
./send_nav_cmd.sh 0 0.05 0 2
```

如果你想直接对 `/cmd_vel` 发：
```bash
./send_nav_cmd.sh 0.08 0 0 2 /cmd_vel
```
