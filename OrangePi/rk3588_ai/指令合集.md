# RK3588 AI 指令合集（精简版）

只保留当前机械臂追踪常用命令。C-5.0.9 阶段追踪目标仍固定为 `bottle`，默认启用 000 yaw + 001 lift + 003 pitch 三轴组合：

- 000 yaw：左右追踪。
- 003 pitch：上下快速细调。
- 001 lift：只在上下误差较大时低频粗调，避免和 003 抢同一个误差。

C-5.0.9 默认启动前先回标准初始化位姿：`0=1500,1=1907,2=1900,3=900,4=1500,5=1500`。C-5.0.5 的 000 yaw、C-5.0.6 的 001 lift、C-5.0.7 的 003 pitch 命令保留作单轴回归。

003 pitch 方向基线沿用 C-5.0.7：`invert_pitch=false`，`pitch_pwm_sign=-1`。不要在没有真实动作复核的情况下改驱动层 `pitch_pwm_sign`。

C-5.0.9 当前实测推荐参数偏“稳”：`control_rate_hz=12`、`duration_ms=150`、`dead_zone_px=42`、`error_filter_alpha=0.6`。如果继续调快，优先小步调 `max_yaw_delta`，不要直接乱改方向符号。

香橙派固定 IP：`192.168.8.148`。

```bash
ssh orangepi@192.168.8.148
```

标准初始化位姿：`0=1500,1=1907,2=1900,3=900,4=1500,5=1500`。如果实物方位不对，不要直接恢复这组值，先用交互模式重新调。

只保存当前实物姿态：

如果你已经手动调到了一个好姿态，只想把它保存为追踪启动姿态，用这个。

```text

save 0-5
```

移动到历史记录姿态并保存：

注意：这条会先让 0~5 号舵机实际运动到下面这组固定 PWM，然后再把这组位置保存成启动姿态。它不是“保存当前实物姿态”。

```bash
cd ~/rk3588_ai/arm_tracking_demo
~/rk3588_ai/rknn_lite_env/bin/python3 tools/arm_servo_tune.py --set 0=1500,1=1907,2=1900,3=900,4=1500,5=1500 --duration_ms 1500 --settle_s 0.8 --save-start-pose --ids 0-5
```

## 6. 三轴组合真实追踪

这条会在启动追踪前自动回标准初始化位姿。

```bash
cd ~/rk3588_ai/arm_tracking_demo

  --model_path ~/rk3588_ai/models/official_yolo11.rknn \
  --target rk3588 \
  --camera 0 \
  --track_class bottle \
  --control_axes yaw,lift,pitch \
  --serial_port /dev/ttyUSB0 \
  --dry_run false \
  --enable_arm \
  --print_cmd
```

观察点：

- 常规小误差下命令包应主要包含 000 和 003。
- 当目标长期明显偏上/偏下时，命令包才低频加入 001。
- 退出时应看到 000/001/003 的 `PDST`，不应出现 002/004/005。

## 11. 最小排障

```bash
# pyserial 缺失
~/rk3588_ai/rknn_lite_env/bin/python3 -m pip install pyserial

# 只测 yaw / pitch
cd ~/rk3588_ai/arm_tracking_demo
~/rk3588_ai/rknn_lite_env/bin/python3 tools/test_one_joint_yaw.py --serial_port /dev/ttyUSB0 --enable_arm --dry_run false --yaw 0.03
~/rk3588_ai/rknn_lite_env/bin/python3 tools/test_one_joint_pitch.py --serial_port /dev/ttyUSB0 --enable_arm --dry_run false --pitch 0.03
```

现场原则：先 `read 0-5`，再小步 `nudge`，确认能动后再跑追踪。不要一上来大幅改 `001/002/003`。
