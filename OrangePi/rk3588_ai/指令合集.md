# RK3588 AI 机械臂现场指令

以下命令对应当前非 ROS 动态闭环抓取代码。香橙派地址为 `192.168.8.148`，机械臂
串口为 `/dev/ttyUSB0`。同一时间只能运行一个占用 D435 或串口的程序。

## 答辩现场：先用这一组（2026-07-20）

先登录香橙派，后面的命令都在香橙派 SSH 终端执行：

```bash
ssh orangepi@192.168.8.148
cd ~/rk3588_ai/arm_grasp_pipeline
```

### 一键到 bottle 前方（当前推荐）

```bash
bash tools/run_bottle_stage.sh oneclick
```

它会自动执行 `003=620` 桌面视角初始化、寻找 bottle、000/003 视觉居中、有效深度锁定、
预抓取后重新观测，再按每步 8 mm 接近；最后停在闭合 TCP 距目标中心约 6 mm 的正前方。
当前 Servo005 回读故障只在这条“张开夹爪、到瓶前”为止的路径中豁免；Servo000～004
仍逐步 PRAD 验证，004 必须为 1500。该命令**不执行最后 6 mm 插入、不闭爪、不抬升、
不自动 PDST**。

### 分段运行（上一段失败时定位/接续）

1. 只观察，不动机械臂：

```bash
bash tools/run_bottle_stage.sh observe
```

确认终端持续识别 `bottle`，并且远距深度大于约 0.17 m。

2. 回到 003=620 后只做视觉居中，不前伸：

```bash
bash tools/run_bottle_stage.sh center
```

成功标志是 `CENTER_READY`；失败会保持最后姿态。

3. 走到约 110 mm 的重新观察位，并保存本次目标坐标：

```bash
bash tools/run_bottle_stage.sh pregrasp
```

成功标志是 `PREGRASP_READY`。它会把刚才的有效远距目标锁保存到
`~/rk3588_ai/debug_logs/bottle_demo_latest_lock.json`。

4. 从当前预抓取姿态接续 8 mm 小步，停在 bottle 前 6 mm；不会重新初始化：

```bash
bash tools/run_bottle_stage.sh approach
```

这一步要求第 3 步保存的目标锁不超过 900 秒，并会先重新识别和读取 Servo000～004；
目标丢失、切换、RGB 跳变、IK 或任一姿态关节回读失败都会停止。

5. Servo005 回读恢复后才允许严格闭爪+验证抬升：

```bash
bash tools/run_bottle_stage.sh full
```

`full` 不带 Servo005 豁免，当前 005 仍无回读时预期安全停止。因此本次答辩先演示
`oneclick` 或 `pregrasp` + `approach` 的真实视觉闭环到瓶前。

若看到“已有程序占用 D435/串口”，先确认旧程序是否仍在运行：

```bash
pgrep -af 'bottle_grasp_demo.py|d435_yolo_grasp.py'
```

不要同时开两个阶段。所有阶段退出时机械臂保持最后姿态，未发送 PDST。

## 0. Bottle 摆放与现场检查

- 推荐把 bottle 的抓取区域放在 Servo000 旋转轴正前方约 **0.35～0.38 m**，优先
  取约 `0.36 m`；横向先靠近机械臂正中。水平完整抓取的实测/IK 下限约为
  `0.34 m`，配置工作空间上限为 `0.39 m`，不要贴近机械臂摆放。
- 在 003=`620` 的桌面初始视角下，bottle 的可见表面 D435 深度应至少约
  `0.20 m`。小于 `0.17 m` 时深度不可信，程序必须停止使用深度继续接近。
- 垫高不是固定要求。只要初始画面能识别到 bottle，视觉伺服会调 000/003 将其移到
  画面中心；当前抓取高度补偿已配置为向上 `40 mm`。
- 保证瓶身前方的水平前伸路径无垫块、桌沿或车体干涉，004 必须保持 `1500`，急停或
  断电手段必须在手边。

## 2. 答辩一键抓取

先只读检查配置和 IK；该命令不打开相机、串口，也不运动机械臂：

```bash
cd ~/rk3588_ai/arm_grasp_pipeline && ~/rk3588_ai/rknn_lite_env/bin/python3 tools/bottle_grasp_demo.py --model_path ~/rk3588_ai/models/official_yolo11.rknn --check_config
```

确认摆放和急停条件后，整段复制粘贴即可启动真实演示：

```bash
cd ~/rk3588_ai/arm_grasp_pipeline && source ~/rk3588_ai/scripts/use_realsense_rsusb.sh && RUN=~/rk3588_ai/debug_logs/bottle-demo-$(date +%Y%m%d-%H%M%S) && mkdir -p "$RUN" && set -o pipefail && ~/rk3588_ai/rknn_lite_env/bin/python3 tools/bottle_grasp_demo.py --model_path ~/rk3588_ai/models/official_yolo11.rknn --dry_run false --enable_arm --serial_port /dev/ttyUSB0 --full_lift true --save_dir "$RUN" --no_show 2>&1 | tee "$RUN/console.log"
```

这条命令自动执行：

```text
六轴低速到 [1500,1909,1968,620,1500,1112]
→ bottle/cup 视觉居中
→ 远距有效深度锁定
→ 110 mm、工具俯角 40° 的可测深观察位
→ 动作后重新识别/测深/PRAD
→ 每次 8 mm RGB 守护接近并渐进放平
→ 距目标 30/22/14/6 mm 时保持 0°，仅水平前伸
→ CLOSE
→ 15 mm 验证抬升
→ 额外 50 mm LIFT
```

每次动作都发送/保持六轴并要求完整 `PRAD` 到位。近距深度失效后，只允许在已有远距
深度锁和同一目标 RGB 关联仍成立时按配置小步接近；最后 30 mm 起只能保持 0° 水平
前伸。目标丢失、切换、横向/高度误差、
PRAD 或 IK 异常都会停止。退出保持最后已回读姿态，`automatic_PDST=false`。
控制台最终必须查看 `BOTTLE_DEMO_SUMMARY`；即使闭爪和 PWM 到位，也要以瓶子确实离开
垫块/桌面的现场现象确认成功。

## 5. 五阶段实机验收命令

这些是任务书要求的分阶段验收入口。它们互不自动串联；每条都重新建立目标观测和日志。
运动阶段必须人工看守，只有确认上一阶段现场姿态安全后才运行下一条。

### observe：动态坐标只读，不运动

```bash

  --mode observe --target_class bottle --dry_run false --serial_port /dev/ttyUSB0 \
```

## 8. 失败后先看什么

每次运行的目录中：

- `console.log`：最终 `SUMMARY` / `BOTTLE_DEMO_SUMMARY` 和停止原因；
- `events.jsonl`：每次新图像、深度、实时 PWM、动态矩阵、IK 和到位回读；
- 图像/深度文件：目标是否完整、是否切换、D435 是否进入近距盲区。

串口发送成功、动作执行结束或 Servo005 到达 `2000` 都不等于抓取成功。任一阶段失败
后不要直接重跑后续阶段；先让瓶子和路径恢复安全，再从 observe/center 或一键入口重来。
