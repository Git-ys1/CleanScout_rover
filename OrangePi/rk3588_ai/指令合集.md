# RK3588 AI 指令合集

本文件只放板端常用命令。默认在 Orange Pi 5 Max 上执行，工程目录为 `~/rk3588_ai`。

## 0. 进入环境

```bash
cd ~/rk3588_ai
source ~/rk3588_ai/rknn_lite_env/bin/activate
```

如果从普通 SSH 终端启动窗口到 NoMachine 桌面，先执行：

```bash
export DISPLAY=:0
export XAUTHORITY=$HOME/.Xauthority
export XDG_RUNTIME_DIR=/run/user/1000
export DBUS_SESSION_BUS_ADDRESS=unix:path=/run/user/1000/bus
```

## 1. 摄像头枚举检查

更换鱼眼、Astra Pro 或延长线后，先确认系统真的识别到摄像头：

```bash
lsusb
ls -l /dev/video*
v4l2-ctl --list-devices
dmesg | tail -n 80
```

如果摄像头节点存在但脚本打不开，检查是否被旧进程占用：

```bash
fuser -v /dev/video0 /dev/video1
pgrep -a -f 'yolo11_camera.py|yolo_arm_track.py'
```

确认是旧测试进程后再结束，例如：

```bash
kill <PID>
```

实时观察插拔日志：

```bash
dmesg -w
```

正常情况下应能看到 UVC/摄像头相关设备，并出现 `/dev/video0` 或其它 `/dev/videoN` 输入节点。`/dev/video-dec0`、`/dev/video-enc0` 是编解码节点，不是摄像头输入。

## 4. 机械臂串口检查

```bash
cd ~/rk3588_ai/arm_tracking_demo
~/rk3588_ai/rknn_lite_env/bin/python3 tools/scan_serial.py
```

当前实测 USB-CH340 下位机串口为 `/dev/ttyUSB0`。如果扫描结果不同，后续命令里的 `--serial_port` 要改成实际端口。

远程桌面手动调机械臂初始姿态，优先用这个交互工具：

```bash
cd ~/rk3588_ai/arm_tracking_demo
~/rk3588_ai/rknn_lite_env/bin/python3 tools/arm_servo_tune.py
```

进入后常用输入：

```text
read 0-5
set 2=1900,3=900 1500
nudge 2=-50 1000
nudge 3=50 1000
save 0-5
quit
```

含义：

- `read 0-5`：读取当前 0 到 5 号舵机位置。
- `set 2=1900,3=900 1500`：把 002/003 转到指定 PWM，用时 1500ms。
- `nudge 2=-50 1000`：在当前位置基础上让 002 减小 50，用时 1000ms。
- `save 0-5`：读取当前 000 到 005，并保存为追踪启动姿态。

也可以不用进入交互模式，直接发一次命令：

```bash
~/rk3588_ai/rknn_lite_env/bin/python3 tools/arm_servo_tune.py \
  --set 2=1900,3=900 \
  --duration_ms 1500
```

一键保存当前姿态为启动姿态：

```bash
cd ~/rk3588_ai/arm_tracking_demo
~/rk3588_ai/rknn_lite_env/bin/python3 tools/save_arm_start_pose.py --ids 0-5
```

如果某个舵机能动作但偶尔读不回位置，例如 `003` 的 `PRAD` 为空，可以直接把你最终确认的数值写入：

```bash
~/rk3588_ai/rknn_lite_env/bin/python3 tools/save_arm_start_pose.py \
  --values 0=1500,1=1907,2=1900,3=900,4=1500,5=1500
```

如果是在 `tools/arm_servo_tune.py` 交互模式里调出来的，直接输入 `save 0-5` 即可；工具会优先读真实位置，读不到时使用本次会话里最后一次发给该舵机的目标值。

保存会更新 `config/arm_track_config.yaml`：

- `driver.tracking_pose_pwms`
- `driver.tracking_pose_stages`
- 如果保存了 `003`，会同步更新 `driver.pitch_pwm_neutral`

原配置会自动备份为 `config/arm_track_config.yaml.bak`。

安全读回探针：

```bash
~/rk3588_ai/rknn_lite_env/bin/python3 tools/bus_servo_probe.py \
  --serial_port /dev/ttyUSB0 \
  --read position \
  --ids 0-5
```

完整读 ID、版本、位置、温度/电压：

```bash
~/rk3588_ai/rknn_lite_env/bin/python3 tools/bus_servo_probe.py \
  --serial_port /dev/ttyUSB0 \
  --read all \
  --ids 0-5
```

注意：`PRAD/PID/PVER/PRTV` 是总线舵机指令表里的正式读回命令；如果 OrangePi 侧 `rx_len=0`，不要先判断为下位机协议不支持，应优先检查串口端口、转接线线序、DTR/RTS、电平路径，以及当前 CH340 是否真的接到电脑上位机软件可读写的同一条总线链路。

## 5. 舵机命令 dry-run

只打印命令，不真实动作：

```bash
cd ~/rk3588_ai/arm_tracking_demo
~/rk3588_ai/rknn_lite_env/bin/python3 tools/test_arm_driver_dryrun.py --print_cmd
```

期望只看到追踪实际占用的 `000/003` 两个舵机帧：

```text
{#000P...T0200!#003P...T0200!}
```

第一版视觉追踪只控制 `000`（yaw）和 `003`（pitch），不会每帧重写
`001/002/004/005`。

## 6. 单关节真实小幅测试

先测底座 yaw：

```bash
cd ~/rk3588_ai/arm_tracking_demo
~/rk3588_ai/rknn_lite_env/bin/python3 tools/test_one_joint_yaw.py \
  --serial_port /dev/ttyUSB0 \
  --enable_arm \
  --dry_run false \
  --yaw 0.03 \
  --duration_ms 500 \
  --hold_s 1.0
```

再测 pitch：

```bash
~/rk3588_ai/rknn_lite_env/bin/python3 tools/test_one_joint_pitch.py \
  --serial_port /dev/ttyUSB0 \
  --enable_arm \
  --dry_run false \
  --pitch 1.23 \
  --duration_ms 500 \
  --hold_s 1.0
```

如果单关节测试看不出动作，可直接发单个舵机命令排除视觉算法：

```bash
~/rk3588_ai/rknn_lite_env/bin/python3 tools/bus_servo_probe.py \
  --serial_port /dev/ttyUSB0 \
  --timeout 0.2 \
  --delay 1.2 \
  --command '#000P1600T1000!' \
  --command '#000P1500T1000!'
```

2026-06-11 已在实机验证：

- `#000PVER!` 返回 `#000P Servo V20240126G!`。
- `#000PRAD!` 可以返回当前位置。
- `000` 号舵机按 `1550 -> 1450 -> 1500` 往返，三次位置读回完全一致。

不要同时运行 `bus_servo_probe.py`、单关节脚本和真实追踪程序。它们会争用
同一个 `/dev/ttyUSB0`；当前代码已启用 Linux 串口独占锁，第二个程序应直接失败，
而不是悄悄拿走另一个程序的回包。

相机已经绑在机械臂末端时，可以用画面位移客观验证 `000/003` 是否真实运动：

```bash
~/rk3588_ai/rknn_lite_env/bin/python3 tools/test_camera_arm_motion.py \
  --serial_port /dev/ttyUSB0 \
  --camera 0 \
  --enable_arm \
  --dry_run false \
  --pwm_offset 100
```

工具会依次保存基准、yaw 偏转、pitch 偏转三张图，最后恢复 `000/003`
到配置中的初始位置。安全限制为最大 `+/-150`。

无需真实检测目标，也可以验证整条视觉伺服控制链是否会累积并产生可见动作：

```bash
~/rk3588_ai/rknn_lite_env/bin/python3 tools/test_visual_servo_motion.py \
  --serial_port /dev/ttyUSB0 \
  --camera 0 \
  --enable_arm \
  --dry_run false \
  --target_x 500 \
  --target_y 360 \
  --cycles 15
```

这条命令使用固定的合成目标框，只用于硬件回归；正式追踪仍必须由
`yolo_arm_track.py` 的真实检测框驱动。

如果方向反了，先停，不要直接开闭环追踪；需要回到 `arm_tracking_demo/config/arm_track_config.yaml` 调整 `yaw_pwm_sign` 或 `pitch_pwm_sign`。

## 8. 机械臂视觉追踪真实输出

只有在摄像头画面稳定、串口扫描正常、单关节方向确认后再执行：

```bash
cd ~/rk3588_ai/arm_tracking_demo
~/rk3588_ai/rknn_lite_env/bin/python3 yolo_arm_track.py \
  --model_path ~/rk3588_ai/models/official_yolo11.rknn \
  --target rk3588 \
  --camera 0 \
  --track_class any \
  --serial_port /dev/ttyUSB0 \
  --dry_run false \
  --enable_arm \
  --print_cmd
```

真实启动默认先把承重关节慢速折回底座附近，然后再把末端相机调到用户实测保存的前视起始姿态：

1. `001=1907、002=1900`，先调整承重主臂；
2. `000=1500、003=900、004=1500、005=1500`，再调整底座/腕部/末端。

这组姿态用于避免相机装在末端后，机械臂向前长距离伸展而压垮 `003`。
动作结束并稳定后才打开摄像头。临时不需要预备姿态时追加：

```bash
--prepare_pose false
```

热键：

- `q` 或 `ESC`：退出。
- `space`：暂停/恢复机械臂输出，视觉继续跑。
- `r`：复位追踪姿态。

## 9. 常见问题

| 现象 | 优先处理 |
| --- | --- |
| `No module named serial` | 执行第 3 节安装 `pyserial` |
| `/dev/video0` 打不开 | 先跑第 1 节，确认摄像头是否枚举；杜邦线/长线容易导致 UVC 枚举失败 |
| `/dev/video0` 存在但 OpenCV 仍打不开 | 很可能被旧 `yolo11_camera.py` 占用，执行 `fuser -v /dev/video0 /dev/video1` |
| 只有 `/dev/video-dec0` / `/dev/video-enc0` | 这是编解码节点，说明没有真实摄像头输入节点 |
| 真实模式没动作 | 先确认 `tools/scan_serial.py` 看到 `/dev/ttyUSB0`，再用 `tools/bus_servo_probe.py --command '#000P1600T1000!'` 做直接动作命令 |
| 舵机方向反 | 停止真实追踪，改 `yaw_pwm_sign` / `pitch_pwm_sign` 后从单关节测试重来 |
