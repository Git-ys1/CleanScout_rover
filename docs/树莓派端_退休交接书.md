## 五、当前关键控制链

当前新阶段第二模式与建图第二模式下，统一控制链是：

```text
电脑 move_base 或 电脑建图手动控制 / 后端手动控制
    -> /cmd_vel_nav
    -> 树莓派 cmd_vel_safety_gate
    -> /cmd_vel
    -> cmdvel_to_rf1.py
    -> /rf1/wheel_target_ms
    -> rf1_serial_bridge.py
    -> OpenRF1
```

说明：

1. 后端控制与导航共用 `/cmd_vel_nav`
2. edge-relay 空闲时不再持续发送 0 速度
3. 当前后端运动控制在第二模式下使用翻转式

## 六、当前最容易误判的坑

### 坑 1：把旧联合演示链当成新阶段主线

旧链能用，不代表它就是当前架构目标。

当前主方向是：

1. 电脑参与解算
2. 树莓派减压
3. 后端和电脑共用 `/cmd_vel_nav`

### 坑 2：误把 edge-relay 空闲发 0 当成正常行为

历史上这个 bug 已修：

1. 后端空闲时不持续发 0
2. 停下时只发一次 0

### 坑 3：编码器 odom 轮序搞错

当前 `/rf1/vel` 轮序口径是：

```text
左后 左前 右后 右前
```

也就是：

```text
[LR, LF, RR, RF]
```

`rf1_vel_to_odom.py` 的反解公式必须按这个轮序理解。

### 坑 4：保存地图脚本以为只能在树莓派执行

当前 `save_map.sh` 已改：

1. 电脑端建图模式下可直接在电脑执行
2. 默认保存到当前机器工作区的 `clbrobot/maps`
3. 默认文件名带日期时间
