# C-1.1.1 Hardware Freeze

## 1. 文档定位

- 版本号：`C-1.1.1`
- 本轮性质：硬件冻结文档轮
- 冻结目标：把当前 C 线车体硬件拓扑、引脚口径、供电关系和继电器风机方案固化为正式仓库文档
- 本轮边界：不做新功能开发，不改 Tyler/受控库 API，不新增风机控制代码实现

本文件是当前 C 线整车硬件冻结主文档。若后续硬件方案变更，必须在新的版本号下另行立项，不得口头覆盖本文件。

本轮资产入口：

- 冻结图：`docs/HARDWARE/assets/C-1.1.1_wiring_freeze.png`
- 来源工程：`LCEDA/车载Arduino.eprj`
- 来源图片：`HardwarePicture/太乐一号原型机连线图.png`
- 来源图片：`HardwarePicture/继电器与吸尘风机连线图.png`

## 2. 平台基线

- 主控：Arduino UNO
- 电机扩展板：AFMotor v1 风格扩展板
- 底盘：4 路二线减速电机 + 麦克纳姆轮
- 舵机：SG90，安装在 SER1 位置
- 蓝牙：HC-06，当前仅作为下行控制通道
- 测距：HC-SR04
- 继电器：HW-803 单路光耦隔离继电器
- 风机：两路风机，当前方案仅支持上电/断电切换

硬件事实说明：

- 当前代码尚未实现左右平移控制。
- 这只说明当前底盘是麦克纳姆轮硬件安装事实，不代表当前软件已经具备全向移动能力。

## 3. 车体方向与电机冻结

文档固定口径：

- 车头超前为正方向。

当前电机安装方位冻结为：

- `M1 = 右后`
- `M2 = 右前`
- `M3 = 左前`
- `M4 = 左后`

本轮只冻结物理安装口径，不在本轮内讨论横移算法、轮速分配或新运动控制逻辑。

## 4. 硬件拓扑冻结

当前整车拓扑按以下关系冻结：

- 3S 电池作为主电源。
- 船型开关位于主电源正极总路径。
- 主电源正极在开关后分为两条主要支路：
- 一路进入 AFMotor 电机电源侧，驱动底盘电机。
- 一路进入继电器触点侧，为两路风机提供开关电源。
- UNO 负责输出底盘控制、舵机控制、蓝牙输入接收、超声测距读入和继电器触发信号。

## 5. 引脚冻结结论

当前冻结口径如下：

| 资源 | 用途 | 冻结说明 |
| --- | --- | --- |
| `D11` | `M1 PWM` | AFMotor 占用 |
| `D3` | `M2 PWM` | AFMotor 占用 |
| `D5` | `M3 PWM` | AFMotor 占用 |
| `D6` | `M4 PWM` | AFMotor 占用 |
| `D4 / D7 / D8 / D12` | AFMotor 锁存控制 | AFMotor 占用 |
| `D9` | `SG90 (SER1)` | 当前舵机冻结口 |
| `D10` | `HC-06 TX 输入侧 (SERVO_2)` | 当前蓝牙下行输入口 |
| `A0` | `HC-SR04 TRIG` | 测距触发口 |
| `A1` | `HC-SR04 ECHO` | 测距回响口 |
| `A2` | `HW-803 IN` | 继电器控制口，计划冻结 |
| `A3 / A4 / A5` | 预留 | 当前未冻结新用途 |
| `D2` | 可用未用 | 当前可扩展但未使用 |
| `D13` | 非优先扩展口 | 当前不作为优先扩展 |
| `D0 / D1` | USB/串口上传与调试 | 不纳入当前扩展方案 |

详细分组表见：

- `docs/HARDWARE/C-1.1.1_pin_budget.md`

## 6. HC-06 冻结说明

当前蓝牙链路冻结为：

- 当前是单向接法：仅 `HC-06 -> Arduino`
- 当前定位为蓝牙下行控制通道
- 当前冻结输入侧引脚：`D10`

这意味着：

- 现阶段只考虑手机或上位发送控制指令到 UNO。
- 现阶段不承担状态回传。
- 若后续需要状态回传或 AT 配置，再单独立项补 `RX` 回路和相关电平/接线方案。

## 8. 供电与继电器冻结摘要

当前继电器与供电方案冻结为：

- 继电器模块：`HW-803` 单路光耦隔离继电器
- 触发方式：高电平触发
- 控制引脚：`A2`
- 触点用途：使用 `NO / COM` 切换两路风机正极
- 当前目标：只做风机开关，不做调速

详细说明见：

- `docs/HARDWARE/C-1.1.1_power_and_relay_plan.md`
