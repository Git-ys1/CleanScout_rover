# C-1.1.1 Pin Budget

## 1. 文档目的

本文件用于冻结 Arduino UNO + AFMotor v1 风格扩展板在 `C-1.1.1` 下的引脚预算，确保所有成员能快速判断当前哪些口已占用、哪些口预留、哪些口不建议继续扩展。

## 3. SER1 / SERVO_2 口径冻结

当前必须统一以下写法：

- `SER1 = D9 = SG90`
- `SERVO_2 = D10 = HC-06 输入侧`

说明：

- 本轮文档冻结明确作废旧说法“`HC-06 TX -> D9`”。
- 旧代码注释仍保留历史口径，但不再作为当前接线依据。

## 4. 引脚扩展压力

当前 UNO + AFMotor v1 方案下，可自由扩展的余量已经较小：

- `A3 / A4 / A5` 虽预留，但后续若叠加更多传感器，模拟口会快速吃紧。
- `D2` 虽当前可用，但单点剩余数字口已经有限。
- `D13` 不建议作为优先扩展口。
- `D0 / D1` 被保留给上传与调试，当前不应抢占。

结论：

- 当前后续扩展余量有限。
- 若未来进入多传感器、状态回传、机械臂协同或更多执行器方案，可能触发主控能力或 IO 预算的重新评估。
