# C-1.2.0 C↔J 通信接线与电平冻结

## 1. 本轮边界

本轮只做软件侧最小闭环，硬件侧冻结写入报告，不扩大成硬件改造轮。

冻结前提：

- `UNO` 为 `5V`
- `F411` 为 `3.3V`
- `J` 侧本轮对外端点为 `OpenMV UART1(A9/A10)`
- 三板只共 `GND`，不共 `VCC`

## 2. F411 引脚冻结

### 2.1 F411 ↔ UNO

- `PA2 = USART2_TX`
- `PA3 = USART2_RX`

### 2.2 F411 ↔ J(OpenMV)

- `PA9 = USART1_TX`
- `PA10 = USART1_RX`

说明：

- 本轮不使用 `PA11/PA12`，避免与 `USB FS` 资源冲突
- 若后续实板确认 `PA9/PA10` 被下载链路强占，再单独评估 `USART6(PC6/PC7)` 回退

## 3. UNO ↔ F411 接线冻结

- `UNO A4 (TX_UNO) -> F411 PA3 (USART2_RX)`
- `UNO A3 (RX_UNO) <- F411 PA2 (USART2_TX)`
- `UNO GND <-> F411 GND`
- `UNO A5`：本轮预留，不参与首版 UART

说明：

- `A3/A4/A5` 在 `UNO + AFMotor v1` 组合下仍可作为通用 GPIO 使用
- 本轮 `UNO` 侧采用 `SoftwareSerial`，速率冻结为 `9600 8N1`

## 4. F411 ↔ J(OpenMV) 接线冻结

- `F411 PA9 (USART1_TX) -> OpenMV A10 (UART1_RX)`
- `F411 PA10 (USART1_RX) <- OpenMV A9 (UART1_TX)`
- `F411 GND <-> OpenMV GND`

本轮不分配：

- `PB0 / PB1 / PB4 / PB5`
- `CLK / DIO`

这些脚只保留给后续 `READY/BUSY/ABORT` 辅助硬件线候选。

## 5. 电平兼容结论

### 5.1 UNO -> F411

- `UNO` 输出为 `5V` 逻辑
- 本轮冻结的 `F411 RX = PA3` 属于 `FT (5V tolerant)` I/O
- 因此在当前冻结引脚前提下，`UNO TX -> F411 RX` 本轮不额外加分压

### 5.2 F411 -> UNO

- `F411` 输出为 `3.3V` 逻辑
- `ATmega328P` 在 `5V` 供电下可识别 `3.3V` 高电平
- 因此 `F411 TX -> UNO RX` 本轮直接连接成立

### 5.3 F411 -> OpenMV

- 两侧都是 `3.3V` UART
- 本轮无需电平转换

## 6. 必须遵守的硬件纪律

1. 三板必须共地
2. 严禁把 `UNO 5V` 和 `STM32/OpenMV 3V3` 电源直接并联
3. 若擅自改用非 `FT` 脚，当前“无需分压”的结论立即失效
4. 若后续改成直连 `ESP32 / 其他外设`，必须重新做电平审查
5. 若 `F411` 用串口下载占用 `PA9/PA10`，上板调试前必须先断开 J 侧线缆

## 7. 本轮最终冻结接线表

### UNO ↔ F411

- `UNO A4 (TX_UNO) -> F411 PA3 (USART2_RX)`
- `UNO A3 (RX_UNO) <- F411 PA2 (USART2_TX)`
- `UNO GND <-> F411 GND`

### F411 ↔ J(OpenMV)

- `F411 PA9 (USART1_TX) -> OpenMV A10 (UART1_RX)`
- `F411 PA10 (USART1_RX) <- OpenMV A9 (UART1_TX)`
- `F411 GND <-> OpenMV GND`

### 预留

- `UNO A5`
- `J PB0 / PB1 / PB4 / PB5`

## 8. 工程结论

在当前冻结引脚下：

- 不需要额外分压
- 必须共地
- 不能共 `VCC`
- `UNO` 侧补丁式串口接入成立
- `F411` 作为中转通信板的电气条件成立
- `J(OpenMV)` 以 `A9/A10` 作为最小真实闭环端点成立
