# 快速上手

这份文档只放常用启动、编译、调试、部署命令。详细架构和阶段结论看 [C-3.4.2_仓库总述与导航](./C-3.4.2_仓库总述与导航.md)。

## 0. 安全约定

- 不把真实 `edge_device_token` 写进公开仓库。
- 下文用 `<EDGE_DEVICE_TOKEN>` 作为占位符，现场调试时替换成真实值，或用环境变量注入。
- 树莓派命令默认在车载 Raspberry Pi 4B 上执行。
- Windows / 小程序命令默认在开发电脑上执行。

功能覆盖：

- 双风机 PWM
- 风机转速回传

### Edge 联调参数

后端 / 小程序侧当前统一设备 ID：

```text
edge_device_id:=csrpi-001
edge_device_token:=<EDGE_DEVICE_TOKEN>
```

`edge_device_token` 不写入公开仓库，现场从 `/etc/vline-backend.env` 或交接记录取明文值。

本地联调：

```text
edge_url:=ws://<开发电脑局域网IP>:3000/edge/ros
```

云端联调：

```text
edge_url:=wss://api.hzhhds.top/edge/ros
edge_fallback_url:=ws://<开发电脑局域网IP>:3000/edge/ros
edge_primary_failures_before_fallback:=3
```

## 4. RF1 底盘串口调试

开发电脑 PowerShell，在仓库根目录执行。

### 默认测试

```powershell
.\tools\openrf1_serial_probe.ps1
```

### 前进 / 后退

```powershell
# 低速前进
.\tools\openrf1_serial_probe.ps1 -WheelA 0.05 -WheelB 0.05 -WheelC 0.05 -WheelD 0.05

# 中速前进
.\tools\openrf1_serial_probe.ps1 -WheelA 0.15 -WheelB 0.15 -WheelC 0.15 -WheelD 0.15

# 较高速前进
.\tools\openrf1_serial_probe.ps1 -WheelA 0.25 -WheelB 0.25 -WheelC 0.25 -WheelD 0.25

# 后退
.\tools\openrf1_serial_probe.ps1 -WheelA -0.10 -WheelB -0.10 -WheelC -0.10 -WheelD -0.10
```

### 转向 / 横移

`W` 参数顺序固定为：`CN1/LR, CN2/LF, CN3/RR, CN4/RF`。

```powershell
# 左转
.\tools\openrf1_serial_probe.ps1 -WheelA -0.10 -WheelB -0.10 -WheelC 0.10 -WheelD 0.10

# 右转
.\tools\openrf1_serial_probe.ps1 -WheelA 0.10 -WheelB 0.10 -WheelC -0.10 -WheelD -0.10

# 左移
.\tools\openrf1_serial_probe.ps1 -WheelA 0.10 -WheelB -0.10 -WheelC -0.10 -WheelD 0.10

# 右移
.\tools\openrf1_serial_probe.ps1 -WheelA -0.10 -WheelB 0.10 -WheelC 0.10 -WheelD -0.10
```

## 6. 小程序本地调试

开发电脑 PowerShell，在 `vue3/` 目录执行。

### 推荐一条命令启动 H5 + 本地 backend

```powershell
cd /d F:\Project\CSc——uniapp\vue3
cmd /c npm.cmd run local:edge
```

该命令会生成本地 `public-edge` 临时 env，并分别拉起：

- H5：`http://localhost:5173`
- backend：`http://127.0.0.1:3000`
- edge：`ws://<开发电脑局域网IP>:3000/edge/ros`

```text
edge_url:=ws://<开发电脑局域网IP>:3000/edge/ros
edge_device_id:=csrpi-001

```

如果只需要手动启动 H5：

```powershell

```

当前云端后端：

```text
systemd 服务：vline-backend
```

### 首次部署

```bash
sudo mkdir -p /opt/CleanScout_rover
cd /opt/CleanScout_rover
sudo git clone https://github.com/Git-ys1/CleanScout_rover.git .

cd /opt/CleanScout_rover/vue3
sudo bash scripts/bootstrap-backend.sh
sudo bash scripts/check-backend-state.sh
```

首次部署前必须先准备好：

```text
/etc/vline-backend.env
```

其中 `EDGE_DEVICE_BOOTSTRAP_TOKEN` 只用于 seed，数据库保存 bcrypt hash，不保存明文。

### 后续更新

```bash
cd /opt/CleanScout_rover
sudo git pull --ff-only

cd /opt/CleanScout_rover/vue3
sudo bash scripts/update-backend.sh
```

如果已配置 alias，也可执行：

```bash
update-backend
```

### 部署后验收

```bash
cat /opt/vline-backend/backend/.deploy-revision
curl -s http://127.0.0.1:3000/api/system/health
curl -s https://api.hzhhds.top/api/system/health
```

公网返回必须包含 `revision` 和 `profile=public-edge`，否则不要继续做实车云端联调。

### 微信小程序后台

```text
request 合法域名：https://api.hzhhds.top
```

当前小程序前端不直连 `/edge/ros`，本轮不要求配置 socket 合法域名。

## 9. 云端前端 H5

当前 H5 托管在 Netlify：

```text
自定义域名：https://h5.hzhhds.top
Netlify 域名：https://cleanscoutrover-management.netlify.app
Netlify 项目：cleanscoutrover-management
```

前端生产 API：

```text
VITE_API_BASE_URL=https://api.hzhhds.top/api
VITE_WS_BASE_URL=
```

Netlify 构建口径：

```text
base directory: vue3
build command: npm run build:h5:production
publish directory: dist/build/h5
```

如果 Netlify base directory 配置为仓库根目录，则 publish directory 改为：

```text
vue3/dist/build/h5
```

后端 CORS 必须包含两个 H5 Origin：

```text
https://h5.hzhhds.top
https://cleanscoutrover-management.netlify.app
```
