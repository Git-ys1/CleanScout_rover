### Summary
- 在 `feature/c-0.0.1-init` 基础上新建 `feature/c-0.0.2-library-ingest`，先提交 `checkpoint: before C-0.0.2`。
- 本轮只做工程整顿，不做功能改动：保留原始 `Tyler_1_Library`，完整拷贝一份到受控库目录，并建立新的独立编译入口。
- 构建标准切换到 `arduino-cli`；`arduino-builder` 作为回退方案保留。
- 补齐第三方来源与许可证留痕（未知许可证明确标注“待核验”）。

### Implementation Changes
- 代码组织与兼容策略（按你确认的方案）：
  - 保留 `Tyler_1_Library/` 原样不改，作为上游基线痕迹。
  - 新增 `libraries/Tyler_1/`，从 `Tyler_1_Library` 完整拷贝（`Tyler_1.h/.cpp`、`keywords.txt`、`examples` 及必要说明）。
  - 不修改 `Tyler_1` 外部 API、动作语义、默认控制流程。
  - 新增独立编译入口（例如 `sketches/c002_uno_baseline/c002_uno_baseline.ino`），内容等价于当前主示例逻辑，后续标准入口统一使用该路径。
- arduino-cli 工具链标准化：
  - 新增 `tools/compile_uno_cli.ps1`：
    - 检查 `arduino-cli` 是否可用；
    - 检查/提示 `arduino:avr` core；
    - 以 `arduino:avr:uno` 编译独立入口；
    - 显式传入库路径（`libraries/Tyler_1` + `Adafruit-Motor-Shield-library-master`）避免同名歧义。
  - 可选新增 `tools/setup_arduino_cli.ps1`（推荐）：
    - `arduino-cli core update-index`
    - `arduino-cli core install arduino:avr`
  - 保留 `tools/compile_uno.ps1` 作为 IDE1.x 回退，不移除。
- VSCode 最小工作区支持：
  - 新增 `.vscode/tasks.json`：
    - 默认任务：`C-0.0.2: compile UNO (arduino-cli)`；
    - 回退任务：`compile UNO (arduino-builder)`。
  - 新增 `.vscode/settings.json` 最小配置：
    - `.ino` 语法/关联（如映射为 C++ 语义）；
    - 不把 Microsoft Arduino 扩展写成强依赖。
- 文档与第三方合规：
  - 新增 `THIRD_PARTY_NOTICES.md`，至少包含：
    - Tyler_1_Library 来源、原始页面/仓库地址、当前使用方式（原始保留 + 受控拷贝）；
    - AFMotor 来源与地址、当前使用方式；
    - 许可证字段：若仓库未见明确 LICENSE，标注“待核验”。
  - 更新 `README.md`：
    - 说明 C 线继承关系（原始基线 vs 受控库）；
    - 给出 Windows 下 arduino-cli 安装与 PATH 配置步骤；
    - 给出 UNO core 安装命令与标准编译入口；
    - 说明 VSCode 使用方式（task 一键编译）；
    - 明确本轮未改业务逻辑，仅做纳管与工具链标准化。
