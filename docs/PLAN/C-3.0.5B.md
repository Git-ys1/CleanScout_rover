# C-3.0.5B：OpenRF1 自写底层二次重建

## Summary

- 本轮从 `C-3.0.5` 的失败回溯点继续，但不再沿用其方向修正思路。
- 目标只收敛一件事：建立 `CN1~CN4` 的单通道驱动真值与编码器真值。
- 本轮不做树莓派、不做 PID、不做 `W,va,vb,vc,vd`、不做车体运动学。
- 当前工作纪律固定为：先记文档，再回填代码；代码只允许回填到 `csr_board_map.c` 的两张方向表。

## Key Changes

### 1. 记录方式

- `docs/VERIFY/C-3.0.5_openrf1_truth_table.md` 作为唯一真值记录页。
- 记录格式固定为：
  - 驱动真值：`M,<ch>,+700` 与 `M,<ch>,-700` 的实际物理现象
  - 编码器真值：`E,<ch>` 的 `count/delta`
- 记录内容只写物理现象，不写前进/后退语义。

### 2. 代码回填纪律

- 真值完整前，不允许继续改：
  - `main.c`
  - `csr_motor_drv.c`
  - `csr_encoder_drv.c`
- 真值完整后，只允许回填：
  - `_local/openrf1_keil_work_3/User/csr_board_map.c`
  - `g_csr_motor_dir_sign[4]`
  - `g_csr_encoder_dir_sign[4]`

### 3. bringup 顺序

1. 冻结 `C-3.0.5` 当前失败支线  
   当前最小协议已打通，但单通道方向真值未建立。
2. 新开 `feature/c-3.0.5b-openrf1-driver-rewrite`
3. 单通道驱动真值测试  
   `CN1~CN4` 逐路测试 `+700 / -700`，每步后 `STOP`
4. 单通道编码器真值测试  
   对实际会转的方向执行 `E,<ch>`，记录 `count/delta`
5. 回填方向表  
   只回填 `csr_board_map.c`
6. 双向一致性回归  
   要求每个通道都形成统一的 `+` / `-` 语义

## Test Plan

- 驱动测试：
  - `M,1,700` / `M,1,-700`
  - `M,2,700` / `M,2,-700`
  - `M,3,700` / `M,3,-700`
  - `M,4,700` / `M,4,-700`
  - 每步后执行 `STOP`
- 当前已知首轮证据：
  - `CN1`: `+700=反转`, `-700=不转`
  - `CN2`: `+700=不转`, `-700=正转`
  - `CN3`: `+700=正转`, `-700=不转`
  - `CN4`: `+700=不转`, `-700=反转`
- 编码器测试：
  - `E,1` / `E,2` / `E,3` / `E,4`
  - 记录 `ENC,<ch>,<count>,<delta>`

## Assumptions

- 当前工作树很脏，本轮提交只收口 `C-3.0.5 / C-3.0.5B` 相关内容。
- `feature/c-3.0.5-openrf1-min-bringup` 作为失败回溯点保留。
- `C-3.0.5B` 默认从当前失败支线分出，但不继承其驱动修正方法。
