# C-2.2.2A：UNO 基线裁剪为树莓派 USB 手动上位控制

## Summary
- 新开分支 `feature/c-2.2.2a-uno-usb-manual-trim`，先做 `checkpoint: before C-2.2.2A`。
- 本轮只改权威入口 [c002_uno_baseline.ino](f:/Project/CleanScout_rover/sketches/c002_uno_baseline/c002_uno_baseline.ino)，不改 `Tyler_1` 库。
- 目标是把当前草图裁成“单模式 + 单入口 + 单职责”版本：只接受树莓派通过 USB `Serial@115200` 发来的单字符手动控制命令，并带失联自动停车。
- 同时补一份极简说明文档，明确这轮虽然不改 `Tyler_1` 库，但**后续必须单独立项修改 `Tyler_1` 库**，把超声/舵机耦合从构造函数和库内部剥离。

## Key Changes
### 1. 草图裁剪为单模式手动控制
- 删除或停用以下整块逻辑：
  - `AUTO`
  - `GEST`
  - `CleanScoutFan`
  - `softSerial(9, 2)`
  - `bridgeSerial(A3, A4)`
  - 外部暂停桥接
  - 超声自动避障、`autoMode()`、`autoTurn()`、`turnL90()`、`turnR90()`
  - 头部舵机初始化与转头调用
- 草图只保留一个运行模式，不再保留 `systemMode` / `previousMode` / `cmdChar` 的多模式语义。
- 支持的控制字符固定为：
  - `8` 前进
  - `2` 后退
  - `4` 左转
  - `6` 右转
  - `5` 停止
  - `7` 左前
  - `9` 右前
  - `1` 右后
  - `3` 左后

### 2. 通信入口改为仅 USB Serial
- 通信入口固定为：
  - `Serial.begin(115200);`
- 只从 `Serial` 读取单字符控制命令，不再保留蓝牙和桥接串口入口。
- 解析规则固定为：
  - 只接受 `1/2/3/4/5/6/7/8/9`
  - 忽略 `\r` / `\n`
  - 其他字符直接丢弃且不刷新失联计时器
- 动作执行收敛成单一手动控制函数，不再保留旧的 `runMode()` / `manMode()` / 模式切换分支。

### 3. 失联自动停车与启动安全
- `setup()` 完成后必须显式：
  - `tyler_1.stop();`
- 新增失联保护：
  - `lastCommandAtMs`
  - 超时阈值固定为 `400ms`
- 规则固定为：
  - 收到任一有效控制字符时刷新计时器
  - 超过 `400ms` 未收到新有效字符则立即 `stop()`
  - `5` 也视为有效命令并刷新计时器

### 4. `Tyler_1` 构造占位与引脚口径
- 本轮不改库 API，但为了尽量释放旧外设脚，草图中的构造改为占位参数：
  - `Tyler_1 tyler_1(1, 0, 1, 1, 200, A5, 13, 13);`
- 本轮文档口径固定为：
  - 从主控草图中释放：`D2`、`D9`、`D10`、`A0`、`A1`、`A2`、`A3`、`A4`
  - 兼容库构造暂占位：`A5`、`D13`
- **必须在说明文档里单列“后续库改造要求”**：
  - 当前 `Tyler_1` 构造函数仍会对超声相关引脚做 `pinMode`
  - 当前 `Tyler_1` 仍内置舵机与超声耦合接口
  - 本轮只是通过占位脚绕开，不是根治
  - 后续必须单独立项修改 `Tyler_1` 库，把超声/舵机从构造参数和库内耦合中拆出，才能形成真正干净的 `C-2.x` 底层执行基线

### 5. 极简说明文档
- 新增一份极简说明文档，建议路径：
  - `docs/SOFTWARE/C-2.2.2A_uno_usb_manual_control.md`
- 文档必须写清：
  - 当前串口波特率：`115200`
  - 当前支持的字符命令
  - 当前已裁掉的功能：
    - `AUTO`
    - `GEST`
    - 蓝牙
    - `bridgeSerial`
    - 风机
    - 超声避障
    - 头部舵机初始化/转头
  - 当前释放出来的引脚：
    - `D2`、`D9`、`D10`、`A0`、`A1`、`A2`、`A3`、`A4`
  - 当前为兼容库构造保留的占位脚：
    - `A5`、`D13`
  - 当前不是最终库边界收口版本
  - 后续必须修改 `Tyler_1` 库，移除超声/舵机耦合

## Public Interfaces
- 串口公共接口固定为：
  - `USB Serial@115200`
  - 单字符命令：`1/2/3/4/5/6/7/8/9`
- 不再暴露：
  - 蓝牙控制入口
  - `bridgeSerial` ASCII 行协议
  - `AUTO/GEST` 模式命令
  - 风机和外部暂停桥接口

## Assumptions
- 本轮只裁主控草图，不改 `Tyler_1` 库，因此通过构造占位脚迁移来释放旧外设脚，而不是直接修改库实现。
- 树莓派上位手动控制默认通过 USB 串口发送单字符命令，不要求本轮兼容 ROS 消息或复杂协议。
- `400ms` 作为失联停车阈值，位于任务书允许的 `300ms~500ms` 区间内，足够保守。
- `Tyler_1` 后续库改造是**明确后续任务**，不是可选建议；本轮文档必须把这一点写给项目负责人。
