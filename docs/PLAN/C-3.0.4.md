# C-3.0.4：OpenRF1 下位机迁移基线建立计划

## 当前阶段结论

- 2026-04-16 当前阶段已通过的最小事实不是“闭环已经完成”，而是：
  - OpenRF1 板上四路底层驱动已经能在最小测试入口下完成可观测的正反轮流驱动
  - `COM14 / UART2` 已经打通，板端可回传 `CSR_RF1_READY` 与 phase/counter 调试信息
  - vendor 例程 9 的 `app_motor_run()` 不再适合作为“底层方向真值测试”入口
- 因此，当前冻结基线从“vendor 闭环现成可用”修正为：
  - vendor 底层资源继续复用
  - 底层方向/接口真值由 `_local/openrf1_keil_work_2` 的最小测试入口校准
  - 下一步从这个已打通的底层入口继续进入极简协议与运行时桥接

## Summary
- 当前正式迁移判断已锁定：`OpenRF1 (STM32F103RCT6)` 取代 `UNO` 成为底盘下位机新主线；`UNO` 不再继续做主线闭环优化，只保留历史事实和回退证据。
- 本轮实施以 **Keil / vendor 例程 9** 作为首个 bring-up 权威入口，不先迁 `CubeIDE`，避免把“工程搬家”混入“底盘稳定闭环”。
- 仓库按 **公开安全** 策略入库：不把 vendor 原工程整包提交到公开仓库；公开仓库只跟踪自研应用层、接口契约、迁移说明和集成说明。vendor 源码继续留在 [`docs/STM32F103RCT6`](/f:/Project/CleanScout_rover/docs/STM32F103RCT6) 作为本地参考依据。
- 当前唯一目标是：在电脑串口工具控制下，让 OpenRF1 板实现四轮“持续、平顺、可控、可回传”的最小闭环基线。树莓派、OpenMV、机械臂、额外传感器全部不进入本轮。

### 1.1 轮位编号真相冻结
- 当前 OpenRF1 运行时必须显式区分两套编号：
  - `A/B/C/D`：协议和 `motor_speed_set(float A,float B,float C,float D)` 使用的四轮顺序
  - `M1/M2/M3/M4`：板载电机驱动与历史硬件口径使用的四路电机编号
- 当前经实机观察冻结为：
  - `A = 左前 = M3`
  - `B = 右前 = M2`
  - `C = 左后 = M4`
  - `D = 右后 = M1`
- 等价写法：
  - `M1 = 右后`
  - `M2 = 右前`
  - `M3 = 左前`
  - `M4 = 左后`
- 后续凡是讨论方向符号、轮速回传、单轮测试或 `W,va,vb,vc,vd` 命令，必须先按这套映射换算，不允许再把 `A` 误当成 `M1`。

### 2. 底层复用口径与应用层重写方式
- `OpenRF1` 当前继续复用的底层真值收口为：
  - `y_encoder/*`
  - `y_motor/*`
  - `y_timer/*`
  - `y_usart/*`
- 当前 `app_motor.c/.h` 不再继续视为 vendor 应用层真值，而是转为我方接管的底盘运行层：
  - 读取四路编码器增量
  - 计算实时轮速
  - 执行安全限幅与最小闭环/前馈控制
  - 下发四路板载电机驱动命令
- 也就是说，本轮策略已经从“完全保留 vendor app_motor”收敛为：
  - 底层驱动继续复用
  - `app_motor + app_csr_bridge + app_chassis + app_telemetry` 统一由我方接管
- 这些底层模块在本轮仍视为“局部只读真值”，不向里面继续塞树莓派逻辑或其他杂项功能。
- 应用层全部走新模块，不沿用 vendor 的 `app_uart.c` 控制框架，不继承 PS2、蓝牙、小程序、动作组、舵机、传感器逻辑。
- 新模块职责固定为：
  - `app_csr_bridge`
    - 串口接收行缓冲
    - 解析 `W,va,vb,vc,vd`
    - 下发 `ACK:W` / `ERR:<reason>`
    - 维护最近一次命令时间
  - `app_chassis`
    - 管理四轮目标速度
    - 对接 `motor_speed_set(float A,float B,float C,float D)`
    - 提供 `set_targets / stop / timeout_watchdog`
  - `app_telemetry`
    - 周期输出 `VEL` 和 `PWM`
    - 统一格式与频率
- 新模块 API 固定为最小集合：
  - `void app_csr_bridge_init(void);`
  - `void app_csr_bridge_run(void);`
  - `void app_chassis_init(void);`
  - `void app_chassis_set_targets(float a, float b, float c, float d);`
  - `void app_chassis_stop(void);`
  - `void app_chassis_tick_20ms(void);`
  - `void app_telemetry_init(void);`
  - `void app_telemetry_tick_20ms(void);`

### 3. 运行时数据流与串口契约
- 本轮串口协议冻结为单一最小协议，不兼容 `UNO` 的 `W,w1,w2,w3,w4 ticks/s` 旧层级，也不提前做树莓派适配。
- 上位机输入固定为：
  - `W,va,vb,vc,vd`
  - `va~vd` 单位固定为 `m/s`
  - 示例：`W,0.10,0.10,0.10,0.10`
- 板端输出固定为：
  - 启动完成：`CSR_RF1_READY`
  - 合法命令：`ACK:W`
  - 非法命令：`ERR:bad_prefix` / `ERR:arg_count` / `ERR:parse_float`
  - 周期速度遥测：`VEL,rt1,rt2,rt3,rt4,tg1,tg2,tg3,tg4`
  - 周期输出遥测：`PWM,p1,p2,p3,p4`
- 命令超时固定为 `400ms`，允许实现时落在 `300~500ms` 区间，但文档默认冻结 `400ms`：
  - 超时后四轮目标清零
  - 调用 `motor_speed_set(0,0,0,0)`
  - PWM 输出回零
- 控制与遥测节拍固定为：
  - vendor `app_motor_run()` 继续保持 `20ms / 50Hz`
  - `app_chassis_tick_20ms()` 与 `app_motor_run()` 同步挂在同一 `20ms` 节拍
  - `VEL/PWM` 遥测默认 `100ms / 10Hz` 输出，避免 115200 串口被 50Hz 全量日志淹没
- 主程序挂接方式固定为：
  - `main.c` 负责初始化 `SysTick_Init()`、`app_motor_init()`、`app_chassis_init()`、`app_csr_bridge_init()`、`app_telemetry_init()`
  - `while(1)` 里只轮询 `app_csr_bridge_run()`
  - `y_timer.c` 的 `20ms` 分支里顺序执行：
    - `app_chassis_tick_20ms()`
    - `app_motor_run()`
    - `app_telemetry_tick_20ms()`

### 4. 分阶段实施顺序
- **阶段 A：固定目标闭环确认**
  - 先不接串口协议
  - 直接在应用层写固定目标速度 `0.05 / 0.10 / 0.15 m/s`
  - 验证四轮持续运转、`RT` 有数、`PWM` 有响应、无单轮明显失控
- **阶段 B：极简串口桥接**
  - 接入 `W,va,vb,vc,vd`
  - 手工串口工具验证：
    - `W,0.10,0.10,0.10,0.10`
    - `W,0.05,0.05,0.05,0.05`
    - `W,0.00,0.00,0.00,0.00`
    - 单轮差速和左右差速组合
  - 验证 `ACK:W / VEL / PWM / timeout stop`
- **阶段 C：四轮调顺**
  - 只围绕四轮目标-实际误差、PWM 顶满情况、方向符号修正做整定
  - 允许改：
    - 单轮方向符号
    - PID / PD 参数
    - 编码器比例参数
  - 禁止引入新功能、禁止提前接树莓派和 ROS

## Public Interfaces
- 电脑串口工具 -> OpenRF1：
  - `W,va,vb,vc,vd`
  - 单位：`m/s`
- OpenRF1 -> 上位机：
  - `CSR_RF1_READY`
  - `ACK:W`
  - `ERR:<reason>`
  - `VEL,rt1,rt2,rt3,rt4,tg1,tg2,tg3,tg4`
  - `PWM,p1,p2,p3,p4`
- 内部轮控接口：
  - 继续复用 `motor_speed_set(float A,float B,float C,float D)`，不改成 `tick/s`、不改成 `PWM` 命令接口
- 当前不对外暴露：
  - 树莓派协议
  - ROS `cmd_vel`
  - `odom`
  - 机械臂 / OpenMV / 传感器接口
