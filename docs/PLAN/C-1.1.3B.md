# C-1.1.3B 计划：G 指令彻底解耦修复轮

## Summary
- 基线是当前 `feature/c-1.1.3-bluetooth-fan-toggle`。实际代码里，权威入口已经把 `G` 做成一次性风机事件；`C-1.1.3B` 不再重做功能，而是做一次“彻底清残留 + 结构固化 + 验收口径冻结”的修复轮。
- 本轮冻结后的对外协议只有一套：
  - `A -> AUTO`
  - `M -> MAN`
  - `T -> GEST`
  - `G -> FAN TOGGLE EVENT`
  - `1/2/3/4/5/6/7/8/9 -> 运动控制`
- `AUTO` 下的 `G` 冻结为“无效忽略但立即消费”；绝不能改 `systemMode`，也不能切到 `GEST`。
- 上游保留目录 `Tyler_1_Library/` 不改，作为历史证据链保留；但必须在 `C-1.1.3B` 文档中明确标注它不是现行协议依据。

## Key Changes
- 命令分层在 active sketch 中显式固化：
  - 只在现行可运行入口里做协议修复：`sketches/c002_uno_baseline` 和 `libraries/Tyler_1/examples/Tyler_1`。
  - 在 sketch 内新增最小命令分类层，把输入分成三类：
    - 模式命令：`A/M/T`
    - 事件命令：`G`
    - 运动命令：`1..9`
  - `cmdChar` 只保留给模式命令和运动命令；`G` 永远不写回 `cmdChar`，只进入一次性 `fanTogglePending`。
  - 模式切换逻辑里只允许 `case 'A'`、`case 'M'`、`case 'T'`；任何 `case 'G' -> systemMode = GEST` 路径都必须从现行代码中移除。
  - `G` 处理顺序固定为：接收 -> 置位事件 -> 本轮消费一次 -> 清零；不得在后续 loop 中复用旧事件。

- 风机逻辑保持模块化，不新增第二套 A2 控制：
  - 继续复用现有 `CleanScoutFan`，不重命名现有控制权接口。
  - 保留并继续使用当前等价接口：`setControlSourceAuto`、`setControlSourceManual`、`toggleManualLatch`、`isManualLatchOn`、`getControlSource`。
  - 本轮不新增散乱 `digitalWrite(A2, ...)`，也不把风机逻辑移回 sketch。
  - `AUTO` 下收到 `G` 时，只清空 `fanTogglePending`，不触发 `toggleManualLatch`，不触发模式切换。
  - `MAN` 和 `GEST` 下收到 `G` 时，只翻转风机，不改 `systemMode`，不自动停车。

- 仓库范围内的残留口径清理：
  - 清理所有现行文档、现行注释表、兼容示例中的 `G -> GEST` 说法，统一改成 `T -> GEST`、`G -> FAN TOGGLE`。
  - 对 `Tyler_1_Library/` 不做代码改动；在新文档中单列“历史上游口径说明”，明确该目录保留旧 `G -> GEST` 只是历史基线，不作为现行协议依据。
  - `README` 的软件入口更新到 `C-1.1.3B` 文档，避免仓库首页继续指向 `C-1.1.3` 的旧冻结口径。

- 文档冻结：
  - 新增 `docs/PLAN/C-1.1.3B.md`：写问题来源、修复目标、禁止事项、验收标准。
  - 新增 `docs/SOFTWARE/C-1.1.3B_protocol_freeze.md`：给出唯一协议表，并显式写“旧协议 `G -> GEST`，新协议 `T -> GEST`，`G` 已改为事件命令”。
  - 新增 `docs/SOFTWARE/C-1.1.3B_event_vs_mode.md`：解释模式命令 vs 事件命令、为什么 `G` 必须一次性消费、为什么 `G` 不得改 `systemMode`。
  - 新增 `docs/SOFTWARE/C-1.1.3B_test_cases.md`：覆盖 `MAN/GEST/AUTO` 下的 `G` 行为、`T` 进入 `GEST`、以及“不再被 `G` 顶出手动模式”的回归项。
  - 在 `C-1.1.3B_protocol_freeze.md` 中增加“历史目录例外”段，明确 `Tyler_1_Library/` 为上游保留目录。

## Public Interfaces
- 对外蓝牙协议冻结为：
  - `A -> AUTO`
  - `M -> MAN`
  - `T -> GEST`
  - `G -> FAN TOGGLE EVENT`
  - `1..9 -> 运动控制`
- 不新增新的模式枚举，不删除 `GEST`，不改默认模式。
- `CleanScoutFan` 公共接口不重命名；沿用当前控制权模型，`C-1.1.3B` 只把该模型明确为唯一风机控制入口。

## Assumptions
- `C-1.1.3B` 是在 `C-1.1.3` 基础上的修复/冻结轮，不重新设计风机控制模型。
- `AUTO` 下的 `G` 明确冻结为“无效忽略且消费掉”，不作为自动模式风机抢占命令。
- `Tyler_1_Library/` 保留历史原样，不在本轮修改；通过新文档把它标记为非现行协议源。
- 本轮重点是“彻底解耦模式命令与事件命令”，不是扩展新功能，也不是重做自动风机策略。
