# C-5.0.3：机械臂视觉追踪任务书（简版）

## 目标

- 只做机械臂，不碰底盘、ROS、清扫。
- 目标不是“每帧全 IK”，而是先跑通“固定前向姿态 + 小范围视觉追踪 + 失目标停住”。
- 这版要能直接指导 OrangePi 和下位机怎么分工，别再写成泛泛而谈的说明书。

## 实物编号

- 按你给的实物编号，舵机从下到上是 `000~005`。
- `000~003` 是主臂链，`y_kinematics` 只负责这 4 个关节。
- `004` 是前端旋转。
- `005` 是夹爪。
- 第一版追踪里，`004/005` 先固定，不进追踪闭环。

## 现有代码的事实

- `firmware/mechanical_arm_official_baseline/User/Components/y_kinematics/y_kinematics.c` 是 000~003 的逆运动学。
- `kinematics_move()` 最后只打包 `#000`~`#003` 四个舵机命令。
- `set_servo()` 是单舵机文本命令，不是 IK。
- `OrangePi/rk3588_ai/arm_tracking_demo/00_REFERENCE_AUDIT.md` 里的参考追踪也是“固定姿态 + 少数关节直控”，不是每帧完整 IK。

## 结论

第一版应该这样做：

1. 先把机械臂摆到“朝前、可见、安全”的基准姿态。
2. 视觉框只产生中心误差，不直接驱动整条手臂重算。
3. 用少数关节做小角度修正，左右误差优先调水平轴，上下误差优先调俯仰轴。
4. 目标丢失时保持或停止，不继续追空。
5. `004/005` 只在人工旋转或抓取阶段动作。

## 这版不做什么

- 不做每帧完整 IK 闭环。
- 不做 ROS 联动。
- 不做抓取状态机。
- 不把 `004/005` 拉进第一版追踪。

## 第一版交付

- `YOLO box -> target_selector -> visual_servo -> ArmDriver`
- `visual_servo` 只输出安全的小增量。
- `ArmDriver` 只发 `yh_pwm_text` / 文本舵机命令。
- 所有关节都要有死区、限幅、丢目标停机。

## 后续再扩展

- 如果后面要让末端保持空间姿态，才把 `y_kinematics` 或完整 IK 拉进来。
- IK 用来“摆姿态 / 到位”，不是第一版逐帧找目标中心的主循环。
- 真要抓取，再把 `004/005` 接进抓取状态机。

## 验收

- 能在固定姿态下把目标维持在画面中心附近。
- 目标消失后不会继续猛追。
- `004/005` 不影响第一版追踪闭环。
- 文档里能一眼看出：追踪、姿态、抓取是三个阶段。
