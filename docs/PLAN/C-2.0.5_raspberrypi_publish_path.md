# C-2.0.5：树莓派端发布路径与交付规则

## 1. 文档定位

- 关联版本：`C-2.0.5`
- 目标：为树莓派端后续由 `opencode` 推进的代码和文档提供仓库内固定发布入口
- 本文只定义发布路径和交付规则，不定义树莓派侧具体软件实现

## 2. 发布入口

树莓派端统一发布到仓库根目录：

- `Raspberrypi/`

该目录用于承接：

- 树莓派本地服务
- 串口桥接程序
- `ROS2` 试验结果
- 启动脚本
- 配置文件
- 树莓派侧验证文档

## 3. 隔离原则

当前冻结口径：

- 树莓派端代码与主机端现有受控基线隔离推进
- `UNO + AFMotor` 当前底盘控制基线不因树莓派侧发布而被直接改写
- 树莓派端每轮改动先在 `Raspberrypi/` 内独立留痕
- 需要回写主线时，再单独立项

## 4. 每轮发布要求

树莓派端每完成一轮，必须在 `Raspberrypi/releases/` 下新建版本目录，目录名必须带版本号和主题。

示例：

- `Raspberrypi/releases/R-0.0.1_serial_bridge_bootstrap/`
- `Raspberrypi/releases/R-0.0.2_ros2_mapping_bootstrap/`

每轮目录必须至少包含：

- `README.md`

并在 `README.md` 中明确写出：

- 本轮目标
- 环境信息
- 依赖
- 启动方法
- 修改文件
- 验证结果
- 已知问题
- 下轮计划

如有代码或配置，建议按如下结构组织：

- `src/`
- `scripts/`
- `configs/`
- `logs/`
- `assets/`

## 5. Git 纪律

树莓派端发布必须遵守以下纪律：

- 一轮一目录
- 一轮一说明文档
- 不把树莓派临时文件散放到仓库根目录
- 不直接覆盖 `sketches/`、`libraries/`、`jixiebi/` 下现有受控基线
- 若本轮只产出文档，也必须放到对应版本目录下

## 6. 与 C-2.0.5 的关系

`C-2.0.5` 已冻结了树莓派上位介入一期的硬件方向：

- `RPLIDAR -> Raspberry Pi USB`
- `MPU6050 -> Raspberry Pi I2C`
- 编码器反馈进入 `C` 线底盘
- 树莓派 <-> Arduino 通过串口形成控制 / 反馈链路

因此，树莓派端后续发布的内容，默认应围绕以下趋势推进：

- 串口控制与反馈
- 传感器接入
- 上位状态采集
- `ROS2` 建图 / 导航 / 巡航试验

但这些内容的实现和验证，当前都先通过 `Raspberrypi/` 目录单独留痕，不直接侵入既有底盘控制基线。
