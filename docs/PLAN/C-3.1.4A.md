# C-3.1.4A：回退到 C-3.0.6 定时器主线并用好编码器复测

## Summary
- 当前回退锚点判断正确：`feature/c-3.0.6-openrf1-closed-loop-freeze` 是最后一版完整的原生定时器主线；`C-3.0.7 / C-3.0.8 / C-3.1.0 / C-3.1.3` 都属于围绕 `CN1/CN3` 的诊断或绕行分支，不应继续作为正式主线叠补丁。
- 本轮只做两件事：
  1. 把 OpenRF1 运行层恢复到 `C-3.0.6` 的 TIM2/3/4/5 编码器主线。
  2. 用已验证为好的编码器电机，对 `CN1` 和 `CN3` 的原生定时器输入链做交叉复测。
- 本轮不继续使用循迹口软编码器，不扩协议，不改 PI 架构，不回 UNO。

## Test Plan
- 编译与烧录回归：
  - Keil 使用回退后的 `.uvprojx` 编译，要求 `0 Error(s)`。
  - STLink 烧录后，串口看到 `CSR_RF1_READY`。
  - 控制组先确认 `CN2/CN4` 仍能返回非零计数，证明 C-3.0.6 主线已真正恢复。

- Phase A：控制组验证
  - `STOP`
  - `M,2,700`、`D,2`、`E,2`
  - `M,4,700`、`D,4`、`E,4`
  - 验收：`phaseA/phaseB` 变化，`timerCount` 非零，`ENC delta` 非零。

- Phase B：CN1 原生 TIM5 复测
  - 将一只好编码器电机接入 `CN1`。
  - 执行：
    - `STOP`
    - `M,1,700`
    - `D,1`
    - `E,1`
  - 验收：`phaseA/phaseB` 非静止，`TIM5->CNT` 非零，`ENC,1` 非零。

- Phase C：CN3 原生 TIM2 full remap 复测
  - 将一只好编码器电机接入 `CN3`。
  - 执行：
    - `STOP`
    - `M,3,700`
    - `D,3`
    - `E,3`
  - 验收：`phaseA/phaseB` 非静止，`TIM2->CNT` 非零，`ENC,3` 非零。

- Phase D：交叉验证
  - 同一只好编码器电机先接 `CN2` 或 `CN4`，确认它本身在控制组通道能计数。
  - 再将同一只电机移到 `CN1` 或 `CN3` 复测。
  - 结论规则固定为：
    - 若同一只好电机在 `CN2/CN4` 有数，在 `CN1/CN3` 也有数：写死“原生 TIM 主线软件成立，原先阻塞首因是坏编码器电机/编码器链。”
    - 若同一只好电机在 `CN2/CN4` 有数，在 `CN1/CN3` 仍无数：写死“电机好坏不是唯一因素，`CN1/CN3` 原生板级通道仍需继续追查。”

- 本轮通过标准：
  - A 档通过：`CN1` 与 `CN3` 在 C-3.0.6 主线下都恢复定时器计数。
  - B 档通过：即使未恢复，也必须通过交叉验证把结论收敛成“原生 TIM 通道问题”还是“原先编码器硬件问题”。
