# C-1.2.3：夹爪动作原语恢复与抓取链路分层恢复

## 1. 本轮目标

本轮只做一件事：先救活夹爪动作原语。

优先级冻结为：

1. 先确认 OpenMV 部署包完整
2. 再做夹爪 `OPEN -> CLOSE -> OPEN` 自检
3. 再做夹爪角度/PWM 标定
4. 再恢复纯本地抓取
5. 最后才重新挂回当前 `C↔J` 通信链路

## 2. 边界

### 2.1 允许修改

- `jixiebi/experiments/C-1.2.0_color_grasp_uart_poc/main.py`
- `jixiebi/experiments/C-1.2.0_color_grasp_uart_poc/cj_link.py`
- 实验目录新增自检/标定/部署清单文档
- 验证与计划文档

### 2.2 禁止修改

- 不改 `UNO`
- 不改 `C` 线暂停/恢复协议名
- 不切到 `J-STM32` 收口
- 不引入 `CAN / Wi-Fi / 上位机`
- 不碰卖家原始目录
- 不顺手改识别算法大框架

## 3. 冻结策略

- `main.py` 分层调试，新增：
  - `MODE_CLAW_SELFTEST`
  - `MODE_FORCE_LOCAL_GRAB`
  - `MODE_VISION_LOCAL_GRAB`
  - `MODE_CJ_LINKED`
- 夹爪角度从散写 `60 / 150` 改为统一原语：
  - `CLAW_OPEN_ANGLE`
  - `CLAW_CLOSE_ANGLE`
  - `claw_open()`
  - `claw_close()`
- `claw_selftest.py` 成为独立最小入口，不依赖识别与通信
- `main.py` 中所有抓取结果仍继续走现有 `cj_link.py` 回传，不新增旁路结果通道

## 4. 验收顺序

1. `Gate1` 运行包完整性
2. `Gate2` 夹爪自检
3. `Gate3` 夹爪标定
4. `Gate4` 纯本地抓取
5. `Gate5` 识别后本地抓取
6. `Gate6` 结果回传

只有前一 Gate 通过，才进入后一 Gate。

## 5. 当前过渡性说明

- 本轮的 `MODE_FORCE_LOCAL_GRAB` 和 `MODE_VISION_LOCAL_GRAB` 是恢复路径，不是最终系统交付形态。
- 最终目标仍然是把已经验证过的夹爪原语重新挂回 `COLOR_FOUND -> WAIT_PICK_WINDOW -> LOCAL_PICK_FALLBACK -> PICK_DONE/PICK_TIMEOUT/ARM_FAIL` 这条主线。
