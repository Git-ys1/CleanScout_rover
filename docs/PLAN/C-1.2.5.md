## 4. Key Known Fact

The selected vendor template has a complete gimbal motion path, but it does **not** already provide explicit claw open/close primitives.

What is currently known from the vendor code:

- `Servo(3)` controls pan
- `Servo(4)` controls tilt
- `Servo(1).angle(50)` triggers a known claw-side motion during startup
- `ubluetooth.py` provides the single-byte command-state organization

Therefore this round includes claw-angle calibration before freezing `claw_open()` / `claw_close()`.
