# C-5.2.0：原厂 D435 纯 Python 感知抓取基线

## 目标

- 不接 ROS，在 Orange Pi 上直接运行 D435 + RKNN YOLO11 + 机械臂抓取规划。
- 学习 `docs/00最新参考/01.树莓派源码/src/car_vision/car_vision/dynamic_space_grab.py` 的有效结构，不复制 ROS2 依赖。
- 保留现有 `arm_tracking_demo`，抓取功能独立放在 `arm_grasp_pipeline`。

## 主链路

```text
D435 aligned RGB-D
→ RKNN YOLO11 目标框
→ 目标框中心稳定与居中检查
→ ROI 深度多帧稳定检查
→ 像素/深度转相机三维坐标
→ 手眼矩阵与机械臂基座坐标
→ 工作空间检查与官方 STM32F103 机械臂 IK 对照实现
→ OPEN / PRE_GRASP / APPROACH / CLOSE / LIFT / RETURN_VERIFY
→ RF1 总线舵机 ASCII 命令
```

## 安全边界

- 默认 dry-run，不发送真实舵机命令。
- `hand_eye.calibrated=false` 时拒绝真实抓取。
- `tool_reference.calibrated=false` 时拒绝真实抓取。
- `serial.joint_pwm_calibrated=false` 时拒绝真实抓取。
- 深度采集不设置人为最近/最远距离；机械臂工作空间在坐标变换后检查。
- 相机已临时固定到机械臂末端，但支架使相机光心与夹爪 TCP 距离过大；当前安装只用于暴露问题，不视为最终手眼标定结构。
- 2026-07-13 实机预抓失败：目标识别与深度锁定成功，但机械臂把相机推进到 bottle，未张合完成抓取。
- 新支架、手眼外参、005 开闭端和接近路径未验收前，真实自动执行继续保持关闭。
- 官方 `$KMS` 只控制 000--003；004 腕部和 005 夹爪不得由 Python IK 补造。
