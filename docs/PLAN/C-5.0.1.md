四、重要架构判断

参考 ROS2 项目只用于学习，不是最终运行环境。

参考项目里的机械臂控制模式大致是：

```text
视觉节点
↓
目标中心误差
↓
PID
↓
ArmControl.set_steer()
↓
发布 /ik_states 或 /arm_states
↓
car_base.cpp 打包串口协议
↓
STM32 / 下位机控制舵机
```

我们当前项目不要照搬 ROS2 topic，而是抽象成：

```text
YOLO视觉追踪层
↓
目标中心误差层
↓
控制器层 PID/P
↓
机械臂驱动层 arm_driver
↓
OPENRF1 / STM32 串口
```

也就是说，参考项目里 `set_steer([joint0, joint1, joint2, joint3, joint4, joint5], duration)` 的思想可以借鉴，但真正发给下位机的协议必须以当前 STM32 / OPENRF1 工作区为准。

八、visual_servo.py 要求

实现二维视觉伺服控制器：

输入：

```text
frame_width
frame_height
target_box = [x1, y1, x2, y2]
current_yaw
current_pitch
```

输出：

```text
new_yaw
new_pitch
error_x
error_y
```

基本逻辑：

```python
cx = (x1 + x2) / 2
cy = (y1 + y2) / 2

error_x = cx - frame_width / 2
error_y = cy - frame_height / 2
```

第一版控制策略：

```text
yaw 根据 error_x 调整
pitch 根据 error_y 调整
```

必须有：

```text
死区 dead_zone_px
最大单步变化 max_delta
yaw 限位
pitch 限位
控制频率限制 control_rate_hz
目标丢失保持策略
连续识别确认帧数
连续丢失停止帧数
```

建议初始参数：

```yaml
dead_zone_px: 30
control_rate_hz: 10
max_yaw_delta: 0.015
max_pitch_delta: 0.015

yaw_init: 0.0
pitch_init: 1.2

yaw_min: -0.5
yaw_max: 0.5
pitch_min: 0.9
pitch_max: 1.5

kp_yaw: 0.0008
ki_yaw: 0.0
kd_yaw: 0.0

kp_pitch: 0.0008
ki_pitch: 0.0
kd_pitch: 0.0
```

注意：这些只是保守初值。真正范围必须由当前机械臂下位机工作区确认。

十一、真机测试流程

必须严格按顺序：

1. 只读审计。
2. `--dry_run` 跑视觉追踪，只在窗口显示目标中心和 yaw/pitch 计算值。
3. `--dry_run --print_cmd` 确认命令平滑，不跳变。
4. 不接机械臂，只测试串口打开和关闭。
5. 真机只测试 yaw 单关节，小范围移动。
6. 真机只测试 pitch 单关节，小范围移动。
7. 真机测试 yaw + pitch 联动。
8. 最后才运行 YOLO 目标追踪闭环。

禁止直接上来运行全关节闭环。

