# C-3.0.8：CN1/CN3 编码器链路专项排查与闭环恢复计划

## Summary
- 本轮编号统一改为 `C-3.0.8`，从当前 `C-3.0.7` 已发布诊断基线继续，不回退 UNO、不改 `W/M/E/D/STOP` 控制语义、不重测开环真值表。
- 当前已知事实：`CN2/CN4` 在同一固件下有 `phase / TIM CNT / ENC delta`，`CN1/CN3` 电机能转但编码器 `phase=1/1`、`CNT=0`、`ENC=0`。
- 本轮目标不是调 PI，而是把 `CN1/CN3` 无反馈归因到：硬件链路、TIM/remap 初始化、软件读取链、或滤波/复用副作用，并尽量恢复非零计数。

## Key Changes
- 新建分支 `feature/c-3.0.8-openrf1-cn1-cn3-encoder-recovery`，只提交 `C-3.0.8` 诊断代码与文档。
- 新增文档 `docs/VERIFY/C-3.0.8_cn1_cn3_encoder_chain_debug.md`，把 `C-3.0.7` 结果作为起点，并记录本轮 `D/R/E` 实测与唯一归因。
- 保留现有控制协议：`W,a,b,c,d`、`M,<ch>,<pwm>`、`E,<ch>`、`D,<ch>`、`STOP`；不改树莓派、ROS、运动学。
- 保留现有 `R,<ch>` 诊断入口，并增强为底层专项诊断：
  - `R,0`：输出 `MAPR_RAW / MAPR_EFFECTIVE`，确认 `TIM2 full remap` 和 `SWJ_CFG=010` 的软件写入真值。
  - `R,1/R,3`：输出 TIM 寄存器，并补充 GPIO 模式/输入寄存器诊断，用于确认 `PA0/PA1`、`PA15/PB3` 是否真的处于输入状态且电平可变。
- 编码器滤波继续保持诊断口径：`CN1=0`、`CN3=0`、`CN2/CN4=10`；若 `CN1/CN3` 恢复计数，再另轮决定最终滤波值。
- 对照 vendor 只看两类文件：`y_encoder.c` 与 `app_motor.c`，只核对 `TIM5+PA0/PA1`、`TIM2 full remap+PA15/PB3`、时钟、GPIO、AFIO、TIM encoder mode，不照搬 vendor 应用层。

## Test Plan
- 编译烧录：
  - Keil 编译 `_local/openrf1_keil_work_3`，要求 `0 Error(s)`。
  - STLink 烧录，串口看到 `CSR_RF1_READY`。
- Remap 验证：
  - 发送 `STOP`、`R,0`。
  - 通过标准：`MAPR_EFFECTIVE` 显示 `SWJ_CFG=010`、`TIM2_REMAP=full remap`。
- CN1 专项：
  - 手拨 CN1 时连续发送 `D,1 / R,1`。
  - 短时 `M,1,700` 后发送 `D,1 / R,1 / E,1 / STOP`。
  - 判定：相位不变=硬件链路；相位变但 `TIM5->CNT` 不变=TIM5/GPIO 初始化；CNT 变但 `ENC` 不变=读取链。
- CN3 专项：
  - 手拨 CN3 时连续发送 `D,3 / R,3`。
  - 短时 `M,3,700` 后发送 `D,3 / R,3 / E,3 / STOP`。
  - 判定：相位不变=硬件链路或 PA15/PB3 未释放；相位变但 `TIM2->CNT` 不变=TIM2 full remap/输入模式/滤波；CNT 变但 `ENC` 不变=读取链。
- 对照回归：
  - 发送 `D,2/E,2`、`D,4/E,4`，确认 `CN2/CN4` 仍有计数，避免本轮诊断破坏已成立两路闭环基础。

## Assumptions
- 本轮 `C-3.0.8` 替代后续所有“C-3.0.7”口径；旧 `C-3.0.7` 仅作为历史诊断基线。
- 当前 `MAPR_RAW=04000300` 不再单独作为 SWJ 判据；以 `MAPR_EFFECTIVE=02000300` 作为自写层写入真值。
- 若 `CN1/CN3` 的 `phaseA/phaseB` 仍始终不变，本轮结论将锁定为硬件链路或板级输入路径问题，下一轮再决定是否查接插件、线序、供电、网表或客服确认。
- 若 `phaseA/phaseB` 会变但 `CNT` 不变，本轮继续在 TIM/remap/GPIO 初始化层修复，不改 PI 或上层协议。
