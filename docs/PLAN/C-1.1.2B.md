# C-1.1.2B 计划：自动模式风机智能控制模块化接入

## Summary
- 先完成当前未提交的前置基线，再从干净基线上新开 `feature/c-1.1.2b-fan-auto-control`。
- 前置基线包含两件事：完成 `C-1.1.1` 的未提交文档资产；把当前本地验证过的电机方向修正 `Tyler_1(1, 0, 1, 1, ...)` 同步到权威入口 `sketches/c002_uno_baseline`。
- `C-1.1.2B` 只在 `sketches/c002_uno_baseline` 集成风机逻辑，不把风机逻辑同步回 `libraries/Tyler_1/examples/Tyler_1/Tyler_1.ino`。
- 不改默认模式、不改模式切换链路、不改蓝牙协议、不改底盘运动语义；只新增独立风机模块、把它挂到现有自动模式流程，并补齐设计/接入/测试文档。

## Key Changes
- 前置清理与分支基线：
  - 先把当前 `README.md`、`docs/HARDWARE/*`、`LCEDA/`、`HardwarePicture/`、`docs/PLAN/C-1.1.1.md` 收敛为完整的 `C-1.1.1` 提交。
  - 把 `sketches/c002_uno_baseline/c002_uno_baseline.ino` 的 `Tyler_1` 构造参数改为 `1, 0, 1, 1`，使它与当前本地硬件修正一致。
  - 之后从该干净基线创建 `feature/c-1.1.2b-fan-auto-control`，再打 `checkpoint: before C-1.1.2B`。
- 新增模块与接口：
  - 新增 `libraries/CleanScoutFan/CleanScoutFan.h` 和 `libraries/CleanScoutFan/CleanScoutFan.cpp`。
  - 模块导出单一类 `CleanScoutFan`，状态枚举固定为：`DISABLED`、`COOLDOWN`、`RUNNING`、`PAUSED_BY_OBSTACLE`。
  - 固定接口：
    - `begin(uint8_t pin, bool activeHigh)`
    - `update(uint8_t currentMode, unsigned long nowMs)`
    - `pauseByObstacle()`
    - `notifyTurnCompleted()`
    - `forceOff()`
    - `bool isRunning() const`
    - `uint8_t getState() const`
    - `bool isPausedByObstacle() const`
  - 时间配置采用 sketch 侧常量 + 构造注入：`FAN_RELAY_PIN=A2`、`FAN_ACTIVE_HIGH=true`、`FAN_COOLDOWN_MS=3000UL`、`FAN_RUN_MS=6000UL`；`CleanScoutFan` 构造函数接收冷却和运行时长，`begin()` 只负责引脚和电平语义。
- 状态机行为固定：
  - 非 `AUTO` 模式：`update()` 立即关风机，状态转 `DISABLED`，清空累计计时。
  - `AUTO` 模式首次进入：从 `DISABLED` 进入 `COOLDOWN`，从该次 `update()` 的 `nowMs` 开始计 3 秒。
  - `COOLDOWN` 满 3 秒：开风机，进入 `RUNNING`，重新计时。
  - `RUNNING` 满 6 秒：关风机，回到 `COOLDOWN`，重新计时。
  - `pauseByObstacle()`：无论当前在 `COOLDOWN` 还是 `RUNNING`，立刻关风机、作废当前计时，进入 `PAUSED_BY_OBSTACLE`。
  - `notifyTurnCompleted()`：不立即开机；只标记“避障完成，允许重启冷却”。下一次 `update(AUTO, nowMs)` 才从 `PAUSED_BY_OBSTACLE` 进入新的 `COOLDOWN`，并从该 `nowMs` 重新计满 3 秒。
- 现有 sketch 的精确接入点：
  - 只改 `sketches/c002_uno_baseline/c002_uno_baseline.ino`。
  - 顶部新增 `#include <CleanScoutFan.h>`、风机常量、风机实例。
  - `setup()` 中调用 `fan.begin(FAN_RELAY_PIN, FAN_ACTIVE_HIGH)`；默认输出关闭。
  - `runMode()` 中保持现有 `cmdChar -> systemMode` 切换逻辑不变；在模式值确定后、进入 `manMode()/autoMode()` 前，调用一次 `fan.update(systemMode, millis())`。
  - `autoMode()` 中在 `frontDist < OK_DIST` 的分支里，先调用 `fan.pauseByObstacle()`，再执行现有 `stop()` 和 `autoTurn()`。
  - `autoTurn()` 中在 `turnR90()` 或 `turnL90()` 返回后、`return` 之前调用 `fan.notifyTurnCompleted()`。
  - `turnL90()` / `turnR90()` 继续只负责运动，不混入风机逻辑。
  - 主程序除模块初始化安全关闭外，不再出现散写的 `digitalWrite(A2, ...)` 业务控制。
- 构建与工具链同步：
  - 更新 `tools/compile_uno_cli.ps1`，不再只显式注入 `libraries/Tyler_1`；改为把受控库根 `libraries/` 作为统一库根，再显式保留 `Adafruit-Motor-Shield-library-master`，这样 `CleanScoutFan` 能被编译发现。
  - 更新 `tools/compile_uno.ps1` 的默认 sketch，从 `Tyler_1_Library/examples/Tyler_1/Tyler_1.ino` 切到 `sketches/c002_uno_baseline/c002_uno_baseline.ino`，保证回退构建验证的是同一个权威入口。
  - `.vscode/tasks.json` 任务名可保留，但默认编译结果必须覆盖 `c002_uno_baseline + CleanScoutFan`。
- 文档交付：
  - `docs/PLAN/C-1.1.2B.md`：目标、边界、禁止项、提交节奏、验收标准。
  - `docs/SOFTWARE/C-1.1.2B_fan_control_design.md`：A2 高电平触发、状态表、3 秒/6 秒时序、障碍暂停、转弯恢复、非自动模式关闭。
  - `docs/SOFTWARE/C-1.1.2B_integration_points.md`：`runMode()` 更新点、`autoMode()` 的 `pauseByObstacle()`、`autoTurn()` 的 `notifyTurnCompleted()`、禁止直接写 A2 的位置说明。
  - `docs/SOFTWARE/C-1.1.2B_test_cases.md`：任务书列出的 9 个行为场景，外加“默认模式/既有启动自动切换未变化”的回归项。
  - 文档里明确：本轮不改默认模式、不改模式层逻辑、风机智能控制只在自动模式中生效。
- 调试输出策略：
  - 默认不开启串口调试，不新增 `Serial.begin()` 到默认构建。
  - 模块中预留 `#define CLEANSCOUT_FAN_DEBUG 0` 的编译期开关；打开时才打印状态切换、障碍暂停、转弯完成和风机开关时刻。
  - 调试输出只能是旁路信息，不能参与状态判断。

## Public Interfaces
- 现有 Tyler API、模式常量、蓝牙命令协议不变。
- 新增库接口只有 `CleanScoutFan` 类及其状态枚举。
- 权威运行入口固定为 `sketches/c002_uno_baseline`；`libraries/Tyler_1/examples/Tyler_1/Tyler_1.ino` 本轮不再作为功能集成入口。

## Test Plan
- 前置基线验证：
  - `C-1.1.1` 未提交资产先收敛成干净提交。
  - `c002_uno_baseline` 中的 `Tyler_1(1,0,1,1,...)` 编译通过，作为后续风机功能基线。
- 编译验证：
  - `tools/compile_uno_cli.ps1` 能编译通过 `sketches/c002_uno_baseline`，并解析到 `Tyler_1`、`CleanScoutFan`、`AFMotor`。
  - `tools/compile_uno.ps1` 回退链路也能编译同一入口。
- 行为验证：
  - 自动模式且无障碍：进入 `AUTO` 后先关 3 秒，再开 6 秒，再回到 3 秒冷却循环。
  - 自动模式运行中遇障碍：立即关风机，状态转 `PAUSED_BY_OBSTACLE`。
  - 自动模式冷却中遇障碍：保持关闭，原计时作废。
  - 完成左右探测与实际转弯后：不立即开机，而是在下一次 `update()` 启动全新的 3 秒冷却。
  - 从 `AUTO` 切到 `MAN` 或 `GEST`：立即关风机，状态回 `DISABLED`。
  - 从 `MAN/GEST` 回 `AUTO`：重新走完整 3 秒冷却，不继承旧计时。
  - 连续障碍：每次都重新 `PAUSED`，每次转弯后都重新走完整冷却。
- 回归检查：
  - `systemMode` 默认值、`cmdChar` 处理、`runMode()` 的模式切换入口和已有自动切换行为均未改语义。
  - 主程序中除 `fan.begin()` 初始化外，不再出现 A2 的业务级 `digitalWrite()`。

## Assumptions
- 当前未提交的 `C-1.1.1` 文档资产和 `HardwarePicture/`、`LCEDA/` 先作为前置基线完成，不与 `C-1.1.2B` 混在同一个功能提交里。
- `sketches/c002_uno_baseline` 是后续唯一权威集成入口；库示例文件保留为历史参考，不在本轮同步风机功能。
- 当前本地电机方向修正 `Tyler_1(1,0,1,1,...)` 视为已经确认的基线，应先同步到 `c002_uno_baseline` 再做风机功能。
- 风机硬件口径沿用 `C-1.1.1`：`A2`、高电平触发、继电器只做总开关、不做调速。
