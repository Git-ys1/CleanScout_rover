# C-3.0.5：OpenRF1 最小自写底层 Bringup 重建计划

## Summary
- 本轮不回 `UNO`，也不再继续热修 `_local/openrf1_keil_work_2` 的混合运行逻辑。
- 当前实现改为：**只参考 vendor 结构，不复制 vendor 运行层代码**，自写 `CN1~CN4` 电机驱动、编码器读取和最小串口协议。
- 当前 `C-3.0.5` 的编译壳暂时继续复用 `_local/openrf1_keil_work_2/Project/RVMDK（uv5）/BH-STM32.uvprojx`，但编译输入已切到 `_local/openrf1_keil_work_3/User/` 的自写模块。
- 本轮唯一目标仍然是：把四路通道的驱动方向真值和编码器方向真值钉死，并让电脑串口工具可直接做单通道、多通道 bringup。

## Key Changes
### 1. 工作区与代码组织
- 新增自写运行层目录：`_local/openrf1_keil_work_3/User/`
- 本轮自写模块固定为：
  - `main.c / main.h`
  - `csr_board_map.c / .h`
  - `csr_motor_drv.c / .h`
  - `csr_encoder_drv.c / .h`
  - `csr_proto.c / .h`
- 不再复用 vendor 运行语义：
  - `app_motor_run()`
  - `motor_speed_set(float,float,float,float)`
  - `SPEED_PidCtlA/B/C/D`
  - 旧 `A/B/C/D -> L1/R1/L2/R2` 映射
- `Keil` 工程当前只负责编译壳，`USER/Components` 源文件列表已切换到自写模块。

### 2. 当前冻结的最小控制语义
- 运行时唯一真值编号：`CN1 / CN2 / CN3 / CN4`
- 当前物理轮位注释固定为：
  - `CN1 = 左后`
  - `CN2 = 左前`
  - `CN3 = 右后`
  - `CN4 = 右前`
- 方向修正只允许通过两组表完成：
  - `g_csr_motor_dir_sign[4]`
  - `g_csr_encoder_dir_sign[4]`
- 第一版初值：
  - `motor_dir_sign = {-1, +1, -1, +1}`
  - `encoder_dir_sign = {+1, +1, +1, +1}`
- 这只是 bringup 初值，不是最终冻结真值。

### 3. 最小协议与安全口径
- 串口固定：`USART2 @ 115200 8N1`
- 接收协议固定为：
  - `M,<ch>,<pwm>`
  - `E,<ch>`
  - `STOP`
- 板端输出固定为：
  - `CSR_RF1_READY`
  - `ACK:M`
  - `ACK:E`
  - `ACK:STOP`
  - `ERR:<reason>`
  - `ENC,<ch>,<count>,<delta>`
- 驱动量固定为有符号原始 PWM：
  - 输入范围 `[-1000,1000]`
  - 实际输出限幅到安全窗口 `300~700`
- 安全超时固定为 `2000ms`：
  - 超时后清零四路目标
  - 直接停车

## Test Plan
- 阶段 A：单通道驱动真值
  - `M,1,300 / M,1,-300 ... M,4,300 / M,4,-300`
  - 每条命令后执行 `STOP`
  - 记录每路正反转实际方向
- 阶段 B：单通道编码器真值
  - 在对应方向动作后发送 `E,<ch>`
  - 记录 `count/delta` 符号
- 阶段 C：多通道同号同值
  - 连续下发四条 `M` 命令，例如 `M,1,300` 到 `M,4,300`
  - 验证四轮是否能稳定同向转动
- 当前通过标准：
  - `CSR_RF1_READY` 正常输出
  - 四路都能稳定正反转
  - `E,<ch>` 能返回有效 `count/delta`
  - `STOP` 能可靠停车

## Assumptions
- 本轮不复制 vendor 例程目录，只参考其底层资源分配与工程结构。
- 当前 `Keil` 工程壳暂复用 `_local/openrf1_keil_work_2`，但其运行层编译输入已经切换到 `_local/openrf1_keil_work_3/User`。
- 本轮不做 PID、不做 m/s、不做 ROS、不做树莓派适配。
