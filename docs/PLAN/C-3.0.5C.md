# C-3.0.5C：OpenRF1 开环正向语义对齐

## Summary

- 本轮从 `C-3.0.5B` 的双向可控 checkpoint 继续。
- 目标只做一件事：将 `CN3/CN4` 的 `+pwm` 方向翻正，使四路都满足统一的开环正向语义。
- 本轮不改协议、不改编码器驱动、不改树莓派接口、不进闭环。

## Key Changes

- 保持：
  - `CN1`、`CN2` 当前驱动逻辑不变
  - 协议仍为 `M,<ch>,<pwm>` / `E,<ch>` / `STOP`
- 只修改：
  - `_local/openrf1_keil_work_3/User/csr_motor_drv.c`
  - `csr_motor_apply_cn3()`
  - `csr_motor_apply_cn4()`
- 目标状态：
  - `M,1,+700`、`M,2,+700`、`M,3,+700`、`M,4,+700` 都表示统一正向
  - 对应 `-700` 表示统一反向

## Test Plan

- 单通道验证：
  - `M,1,700` / `M,1,-700`
  - `M,2,700` / `M,2,-700`
  - `M,3,700` / `M,3,-700`
  - `M,4,700` / `M,4,-700`
  - 每步后 `STOP`
- 验收标准：
  - 四路都双向可控
  - 四路 `+700` 均为统一正向
  - 四路 `-700` 均为统一反向

## Assumptions

- `C-3.0.5B` 的 checkpoint 结论成立：当前只剩 `CN3/CN4` 正值方向错误。
- 本轮以“正式打通开环基线”为目标，为项目负责人验收准备最终提交。
