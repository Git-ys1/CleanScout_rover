# C-3.1.0：OpenRF1 CN1/CN3 板级关联与输入模式专项排查

## Summary
- 从 `C-3.0.8` 已发布诊断基线继续，新分支建议：`feature/c-3.1.0-openrf1-cn1-cn3-board-input-audit`。
- 本轮只排查 `CN1/CN3` 编码器无反馈，不改 `W/M/E/D/STOP` 既有语义，不调 PI，不改引脚，不回 UNO。
- 目标是把 `CN1/CN3` 明确归因到：板级链路、输入电气模式、TIM/remap 配置、滤波/模式组合，或软件读取链。

## Key Changes
- 新增全例程静态对照文档：`docs/VERIFY/C-3.1.0_openrf1_example_compare.md`
  - 扫描 `docs/STM32F103RCT6/03.源代码程序/` 全部例程。
  - 重点记录 `TIM2/TIM5`、`PA0/PA1/PA15/PB3`、`AFIO`、`GPIO_PinRemapConfig`、`GPIO_Mode_IN_FLOATING/IPU/IPD`、板级初始化、电源/复用/上拉相关代码。
  - 对照 `OpenRF1_开发速查.md`、网表资料、vendor `y_encoder.c/app_motor.c`，判断是否存在当前自写层遗漏的板级关联初始化。

- 在 `_local/openrf1_keil_work_3/User` 增加诊断能力，不改运行主协议。
  - 保留 `D,<ch>`、`R,<ch>`、`E,<ch>`。
  - 新增仅用于诊断的配置命令：`X,<ch>,<input>,<mode>,<filter>`。
  - `ch` 只允许 `1` 或 `3`。
  - `input` 取值：`F`、`IPU`、`IPD`。
  - `mode` 取值：`TI12`、`TI1`、`TI2`。
  - `filter` 取值：`0` 或 `10`。
  - `X` 执行后只重初始化目标编码器通道、清零计数器，并返回 `ACK:X`；不影响 `W/M/E/D/STOP` 定义。

- 新增板级关联审计文档：`docs/VERIFY/C-3.1.0_openrf1_board_association_audit.md`
  - 明确 `CN1 A/B` 是否应到 `PA0/PA1`。
  - 明确 `CN3 A/B` 是否应到 `PA15/PB3`。
  - 记录中间是否存在上拉、电平转换、复用、供电控制、接插件风险。
  - 未能从资料确认的项必须标为“需硬件实测/客服确认”，不写成已确认事实。

- 新增本轮矩阵结果文档：`docs/VERIFY/C-3.1.0_cn1_cn3_matrix_results.md`
  - 记录每组 `X + D/R/E` 的结果。
  - 最终输出唯一归因：硬件链路、输入电气模式、TIM/remap 配置、软件读取链，或多因素但下一刀明确。

## Test Plan
- 编译烧录：
  - Keil 编译 `_local/openrf1_keil_work_3`，要求 `0 Error(s)`。
  - STLink 烧录后，串口看到 `CSR_RF1_READY`。

- 基线确认：
  - `STOP`
  - `R,0`
  - 验证 `MAPR_EFFECTIVE` 中 `SWJ_CFG=010`，`TIM2_REMAP=full remap`。
  - `D,2/E,2`、`D,4/E,4` 确认 `CN2/CN4` 仍有计数。

- 输入模式矩阵：
  - 对 `CN1` 执行：`X,1,F,TI12,0`、`X,1,IPU,TI12,0`、`X,1,IPD,TI12,0`。
  - 对 `CN3` 执行：`X,3,F,TI12,0`、`X,3,IPU,TI12,0`、`X,3,IPD,TI12,0`。
  - 每组都执行手拨轮测试与短时 `M,<ch>,700` 测试，并记录 `D/R/E`。
  - 判定：浮空无相位、上拉有相位，优先归因输入电气模式/上拉问题；三种模式都无相位，优先归因板级链路或读错脚。

- 编码器模式矩阵：
  - 只在某个输入模式下 `phaseA/phaseB` 会变化时继续。
  - 对该输入模式测试 `TI12/TI1/TI2`。
  - 判定：单相模式有计数而 `TI12` 无计数，说明不是整路死，而是某一相或双相组合条件异常。

- 滤波矩阵：
  - 只在 `phaseA/phaseB` 会变化但 `CNT` 不稳定或不计数时继续。
  - 对有效输入模式和有效编码器模式测试 `filter=0` 与 `filter=10`。
  - 判定：`filter=0` 有数、`filter=10` 无数，归因滤波过重；两者都无数，继续归因 TIM/remap/板级链路。

## Assumptions
- `C-3.0.8` 结论有效：`CN2/CN4` 在同固件下有相位、计数和 `ENC delta`，`CN1/CN3` 当前为 `phase=1/1`、`CNT=0`、`ENC=0`。
- `X` 是诊断命令，不是正式底盘控制协议；它不改变 `W/M/E/D/STOP` 语义。
- 本轮不改引脚、不退回软件解码、不调 PI；只有当矩阵证明当前合法硬件编码器输入链不可用时，下一轮才讨论换脚或软件计数方案。
