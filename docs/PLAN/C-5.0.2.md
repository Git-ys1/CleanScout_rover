# C-5.0.2 机械臂总线舵机协议澄清与接口定版

## Summary

- 本轮不改 `C-5.0.1` 视觉追踪 demo 行为，只把机械臂协议层重新定版。
- 现有资料包里的 `#000P1500T1000!` 属于总线舵机 ASCII 文本指令，不是二进制帧。
- `P1500` 是协议字段，不应直接等同为裸 PWM 波形语义。
- 当前仓库里的机械臂官方 STM32 基线，代码层既有文本命令解析，也有本地 PWM 舵机输出和串口转发，因此它更像“文本协议前端 + 执行层适配”，而不是单一的裸总线帧发送器。
- 下一轮新任务书要先确认硬件接法，再决定是否继续沿用 `yh_pwm_text` 这种历史命名，还是拆成更清楚的“总线协议层 / 执行层”。

## Key Points

- 总线舵机资料表明，指令是以 ID、P、T 组成的 ASCII 文本帧。
- 目前仓库里没有看到独立的二进制总线帧主路径作为默认实现。
- 如果实际机械臂是 UART 总线舵机链，下一轮应优先确认 STM32 是“透传文本命令”还是“本地驱动 PWM 舵机”。
- 这条结论只是在协议语义上重新命名，不是功能性改动。

## Next Step

- 等用户下发新任务书后，再决定是否：
  - 直接烧录到下位机验证；
  - 调整 `arm_driver.py` 的命名和协议分层；
  - 或者补一份更明确的机械臂接线与协议映射文档。
