## Key Changes
- 新增硬件冻结文档组：
  - `docs/HARDWARE/C-1.1.1_hardware_freeze.md`
  - `docs/HARDWARE/C-1.1.1_pin_budget.md`
  - `docs/HARDWARE/C-1.1.1_power_and_relay_plan.md`
  - `docs/HARDWARE/assets/C-1.1.1_wiring_freeze.png`
- `C-1.1.1_hardware_freeze.md` 内容固定为：
  - 平台基线：UNO、AFMotor v1、4 路二线减速电机 + 麦克纳姆轮。
  - 硬件事实：当前代码未实现左右平移，这只是硬件安装事实冻结，不代表运动功能已实现。
  - 电机方向冻结：`M1=右后`、`M2=右前`、`M3=左前`、`M4=左后`，统一口径“车头超前为正方向”。
  - HC-06 冻结：当前是单向下行控制链路，冻结为 `D10` 输入侧；明确作废旧口径“HC-06 TX 到 D9”。
  - 旧口径作废说明：列出当前仓库中哪些历史来源保留了 `D9` 说法，并声明它们仅为历史基线，不再作为现行接线依据。
  - 版本冻结结论：C-1.1.1 只冻结硬件拓扑、引脚和供电边界，不代表任何新功能实现。
- `C-1.1.1_pin_budget.md` 内容固定为一张清晰分组表：
  - 已占用：`D11/D3/D5/D6` 电机 PWM，`D4/D7/D8/D12` AFMotor 锁存，`D9` SG90，`D10` HC-06 输入，`A0` TRIG，`A1` ECHO，`A2` HW-803 IN。
  - 预留：`A3/A4/A5`。
  - 可用未用：`D2`。
  - 不建议使用：`D13` 非优先扩展口，`D0/D1` 保留给 USB/上传/调试。
  - 附一节“引脚扩展压力”：说明当前余量有限，未来多传感器方案可能触发主控或 IO 重评估。
- `C-1.1.1_power_and_relay_plan.md` 内容固定为：
  - 主电源：3S 电池。
  - 主开关：船型开关串在主电源正极路径。
  - 电机与风机主电：AFMotor 电机电源与继电器风机电源共用主电源正极支路。
  - 逻辑供电：继电器 `DC+ / DC-` 由 `5V / GND` 供电。
  - 风机方案：两路风机负极直回系统 GND，正极由继电器 `COM/NO` 切换，仅做开关，不做调速。
  - 继电器冻结：`HW-803`、高电平触发、`IN=A2`，明确写出 `A2 = HIGH -> 继电器吸合 -> COM 与 NO 导通 -> 风机上电`。
  - 风险与待确认项：电源跳线是否安装、UNO 逻辑电源是否与电机/风机共源、两路风机并联总电流是否已测、上电瞬态/复位测试是否做过。
- 冻结图 `C-1.1.1_wiring_freeze.png` 直接按文字重画，不依赖现有图片：
  - 图中必须画出：3S 电池、船型开关、UNO + AFMotor、M1-M4、SG90、HC-06、HC-SR04、HW-803、两路风机。
  - 图中必须标出：`D9=SG90`、`D10=HC-06`、`A0=TRIG`、`A1=ECHO`、`A2=HW-803 IN`、`M1~M4` 的车体方位。
  - 图中不写旧 D9 蓝牙口径；旧口径只在文档“作废说明”里出现。
- README 更新：
  - 增加“当前硬件冻结版本为 `C-1.1.1`”。
  - 增加硬件入口链接，指向 `docs/HARDWARE/C-1.1.1_hardware_freeze.md`。
  - 写清当前风机方案仅为继电器开关版、当前蓝牙为单向接法、当前扩展余量有限。
