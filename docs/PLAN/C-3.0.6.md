# C-3.0.6：OpenRF1 开环基线进入闭环施工计划

## Summary
- 继续以 `_local/openrf1_keil_work_3/User` 的自写运行层为唯一施工基线，不回退到重测真值表，也不把当前工作区打散重来。
- 本轮只做三件事：最早期禁用 JTAG 保留 SWD、在现有开环真值基线上叠加四轮闭环、把协议改成目标值锁存 + 固定频率控制 + 超时停车。
- `csr_motor_drv.c` 里已经打通的四路独立 H 桥输出保持不动，不再用“统一公式 + sign 表”重写底层方向；闭环只包在其上层。
- 主协议升级为 `W,a,b,c,d`，含义固定为四个轮子的目标速度 `m/s`；`M / E / STOP` 保留为调试入口。

## Key Changes
### 1. 调试口与初始化顺序
- 在 `main()` 最早期加入 SWJ 配置：`RCC_APB2PeriphClockCmd(RCC_APB2Periph_AFIO, ENABLE);` 和 `GPIO_PinRemapConfig(GPIO_Remap_SWJ_JTAGDisable, ENABLE);`
- 这段逻辑必须放在 `SysTick`、电机、编码器、串口初始化之前执行。
- 从 `csr_encoder_drv.c` 的 `CN3` 初始化里移除“顺手禁用 JTAG”的隐式做法，只保留 `TIM2` remap；JTAG 释放改为全局早初始化真值。
- 验收口径固定为：SWD 仍可下载，`PA15/PB3/PB4` 被释放后，`CN3` 编码器链不再受默认 JTAG 占用影响。

### 2. 控制循环从单次执行改成持续闭环
- 保持当前 `csr_motor_set(channel, signed_pwm)` 作为唯一执行出口，不再改动四路 `csr_motor_apply_cnX()` 的开环真值。
- 在运行层新增“控制状态”而不是重写底层：
  - 四轮目标速度 `target_vel[4]`
  - 四轮实时速度 `measured_vel[4]`
  - 四轮控制输出 `output_pwm[4]`
  - 四轮积分状态与上次误差
  - 当前控制模式 `STOP / RAW / CLOSED_LOOP`
- 继续用 `SysTick` 毫秒基准，在 `while(1)` 中做协作式调度：
  - 控制循环固定 `20ms / 50Hz`
  - 遥测循环固定 `100ms / 10Hz`
- 控制循环每次执行固定流程：
  1. 对四路调用 `csr_encoder_read_and_reset()` 取本窗口 `delta`
  2. 依据固定系数把 `delta` 换算成 `m/s`
  3. 用 `target - measured` 做每轮独立 PI 计算
  4. 输出限幅到 `[-1000,1000]`
  5. 将非零输出交给当前已验证的 `csr_motor_set()`
- 速度换算口径固定为当前阶段默认值：
  - `CONTROL_HZ = 50`
  - `WHEEL_RESOLUTION = 1560.0`
  - `WHEEL_DIAMETER_M = 0.08`
  - `wheel_scale = PI * WHEEL_DIAMETER_M * CONTROL_HZ / WHEEL_RESOLUTION`
- 控制器先固定为 PI：
  - `kp[4]`、`ki[4]` 按轮独立可调
  - `kd[4] = 0`，本轮不启用 D
  - 必须有积分限幅、输出限幅、目标为 0 时积分清零、超时停车时积分清零
- `g_csr_motor_dir_sign[4]` 不再参与低层 H 桥真值，但允许在控制层作为“目标速度/输出符号修正”使用；`g_csr_encoder_dir_sign[4]` 继续只用于编码器符号修正。

### 3. 协议改成目标锁存 + 持续控制
- `csr_proto.c` 新增主命令：
  - `W,a,b,c,d`
  - `a~d` 固定解析为四个轮子的目标速度，单位 `m/s`
- `W` 的行为固定为：
  - 更新四轮 `target_vel`
  - 切换到 `CLOSED_LOOP` 模式
  - 刷新闭环看门狗
  - 返回 `ACK:W`
- 保留调试命令：
  - `M,<ch>,<pwm>`：进入 `RAW` 模式，直接锁存指定通道原始 PWM
  - `E,<ch>`：只返回缓存的 `count` 和 `last_delta`，不得再重置编码器窗口，避免干扰闭环
  - `STOP`：清零四轮目标、清零 PI 状态、立即停车
- 看门狗口径直接锁死为两档：
  - `W` 闭环模式：`250ms`
  - `M` 调试模式：`2000ms`
- 回传格式固定为：
  - `CSR_RF1_READY`
  - `ACK:W` / `ACK:M` / `ACK:E` / `ACK:STOP`
  - `ERR:<reason>`
  - `VEL,rt1,rt2,rt3,rt4,tg1,tg2,tg3,tg4`
  - `PWM,p1,p2,p3,p4`
  - `ENC,<ch>,<count>,<delta>` 仅在 `E,<ch>` 查询时输出
- 串口解析继续用行协议，`\n` 结尾，兼容 `\r\n`。`W` 的解析失败统一回 `ERR:parse_float` 或 `ERR:arg_count`。

### 4. 文档与留痕
- 新增 `docs/SOFTWARE/C-3.0.6_pid_close_loop_and_protocol_update.md`
- 文档必须明确写清：
  - JTAG 禁用、SWD 保留的初始化位置与原因
  - 当前控制周期 `50Hz`
  - 当前遥测周期 `10Hz`
  - `W` 的单位是 `m/s`
  - `M/E/STOP` 只保留为调试入口
  - `W` 使用 `250ms` 看门狗，`M` 使用 `2000ms` 调试看门狗
  - 当前 PI 默认参数、积分限幅、输出限幅
  - 当前仍待后续调优项：每轮 `kp/ki`、编码器符号、速度尺度校准
- 串口调试记录至少保留三组：
  - 单轮低速闭环持续转动
  - 四轮同速闭环持续转动
  - 停止刷新 `W` 后超时停车

## Public Interfaces
- 主控制协议固定为：`W,a,b,c,d`
  - `a~d` 为四轮目标速度，单位 `m/s`
- 调试协议继续保留：
  - `M,<ch>,<pwm>`
  - `E,<ch>`
  - `STOP`
- 板端输出固定为：
  - `CSR_RF1_READY`
  - `ACK:W` / `ACK:M` / `ACK:E` / `ACK:STOP`
  - `ERR:<reason>`
  - `VEL,rt1,rt2,rt3,rt4,tg1,tg2,tg3,tg4`
  - `PWM,p1,p2,p3,p4`
  - `ENC,<ch>,<count>,<delta>`
- 运行层唯一执行出口继续固定为 `csr_motor_set(channel, signed_pwm)`，不引入 vendor `app_motor_run()` 或 `motor_speed_set()` 回主线。

## Test Plan
- **初始化验证**
  - 烧录后 SWD 仍可连接
  - `CSR_RF1_READY` 正常输出
  - `CN3` 编码器在 JTAG 释放后不再表现为异常静默
- **RAW 调试回归**
  - `M,1,700`、`M,1,-700` 到 `M,4,700`、`M,4,-700`
  - 确认当前开环基线未被闭环施工破坏
- **闭环单轮验证**
  - 连续发送单轮 `W`，其余三轮目标为 `0`
  - 该轮低速持续转动，不再“动一下停一下”
  - `VEL` 与 `PWM` 连续更新
- **闭环四轮验证**
  - 连续发送同速 `W`
  - 四轮在命令保持期间持续运行
  - 同速命令下四轮输出接近，不再明显脉冲式抽动
- **协议与超时验证**
  - 连续发送 `W` 时持续转动
  - 停止发送后约 `250ms` 自动停车
  - `M` 调试模式单条命令可维持至 `2000ms` 看门狗
  - `E,<ch>` 不会打断闭环采样节拍，只返回缓存值
- **通过标准**
  - JTAG 已禁用、SWD 保留
  - 当前工作区可编译运行
  - `W` 已成为持续控制协议，不再是“收到一次执行一次就停”
  - 单轮和四轮在闭环下明显比当前开环抽动更平顺
  - 可为下一轮树莓派接入提供稳定底层接口

## Assumptions
- 本轮继续以 `_local/openrf1_keil_work_3/User` 为唯一运行层，不回到 `_work_2` 的 vendor 运行语义。
- `W` 的控制量默认采用 vendor 已验证的速度语义 `m/s`，并沿用当前可查到的 `1560 CPR + 0.08m 轮径 + 50Hz` 作为第一版换算常量。
- PI 优先于完整 PID；`Kd` 本轮固定为 `0`，后续若需要再单独启用。
- `M / E / STOP` 保留仅为 bringup 与诊断，不再承担正式底盘持续控制语义。
