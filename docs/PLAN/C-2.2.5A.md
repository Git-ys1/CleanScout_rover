## Key Changes
### 1. 冻结硬件映射与引脚真相
- 在 `C-2.2.5A_uno_encoder_architecture_freeze.md` 中明确写死两套映射：
  - **优先映射（A2 继电器线拔掉）**
    - `M1_A -> A0`
    - `M1_B -> A1`
    - `M2_A -> A2`
    - `M2_B -> A3`
    - `M3_A -> A4`
    - `M3_B -> A5`
    - `M4_A -> D9`
    - `M4_B -> D10`
  - **备选映射（A2 暂不释放）**
    - `M1_A -> A0`
    - `M1_B -> A1`
    - `M2_A -> A3`
    - `M2_B -> A4`
    - `M3_A -> A5`
    - `M3_B -> D2`
    - `M4_A -> D9`
    - `M4_B -> D10`
- 文档必须显式写明：
  - `SERVO1/2` 本质是 `D9/D10` 普通数字口
  - `A0~A5` 可以作为数字输入
  - `D0/D1` 不纳入当前编码器方案
  - 当前推荐先拔掉 `A2` 风机继电器控制线，以换取更整洁的 8 路编码器接入
  - 风机迁移本轮不做，也不混入编码器接入轮
- 本轮统一使用 `AFMotor` 通道号 `M1~M4` 作为编码器接线口径，不在本轮擅自冻结 `前左/前右/后左/后右` 轮位；文档中要明确下一轮实施前必须先完成 `M1~M4` 对物理车轮位置的一次性核对。

### 2. 冻结采样方向：PCINT，而不是 8 路外部中断
- 文档必须把实现方向写死为：
  - `UNO` 编码器采样优先走 **Pin Change Interrupt (PCINT)** 方案
  - 不走“给 8 路 A/B 相都配 `attachInterrupt()`”的假设
- 采样结构固定写成：
  - `A0~A5` 归在 `Port C / PCINT` 组
  - `D9/D10` 归在 `Port B / PCINT` 组
  - 备选方案中的 `D2` 只是备用输入，不是优先方向
- 文档中必须说明：
  - 这轮的技术难点不是“脚够不够”，而是“如何稳定读 8 路正交编码器”
  - 优先映射比备选映射更好，不是因为“多了脚”，而是因为 `Port C + Port B` 分组更整洁，PCINT 处理更统一

### 3. 冻结下一轮代码实施顺序
- 文档中的实施顺序必须固定为三步，不允许跳步：
  1. **原始采样阶段**
     - 只做四轮编码器原始计数
     - 输出 `tick / direction / wheel speed estimate`
  2. **闭环控制阶段**
     - 再做四轮速度 `PI/PID`
     - 初始建议先上 `PI`，`D` 项后置
  3. **协议对接阶段**
     - 最后再和树莓派/ROS 串口协议打通
     - 回传 `tick / wheel speed`，有余力再补 `battery/state`
- 文档里要给出建议频率：
  - 编码器边沿采样：事件驱动 `PCINT`
  - 轮速估计周期：`20ms (50Hz)` 起步
  - 轮速控制周期：`20ms (50Hz)` 起步
  - 树莓派正式速度命令频率：`10Hz` 起步，推荐向 `20Hz` 收敛
- 文档里必须明确：
  - 本轮不写 PID 代码
  - 本轮冻结的是“先采样、后闭环、再对接”的实施秩序

### 4. 冻结与树莓派 / ROS 的长期接口语义
- 文档必须承接 `C-2.2.4`，明确写死：
  - 当前 `1~9` 数字动作码仍只属于联调过渡协议
  - 长期正式接口语义不是动作码，而是速度语义
  - 对麦克纳姆底盘，长期冻结为 `vx / vy / wz`，对应 `Twist.linear.x / Twist.linear.y / Twist.angular.z`
- 文档必须写清：
  - 树莓派负责上层感知、规划、导航、OpenClaw 等
  - `UNO` 负责电机执行、编码器采样、下一轮轮速闭环、里程计基础量
  - 当前无编码器阶段只能定义为开环/半开环联调，不是正式导航底盘
  - 下一轮若要进入成熟 `PID / odom / 巡航 / SLAM`，霍尔编码电机是前置条件
