# C-2.0.5：上位介入一期硬件冻结线生成任务书

## 1. 文档定位

- 版本号：`C-2.0.5`
- 本轮性质：硬件冻结线生成任务书入库轮
- 本轮对象：`C` 线底盘 / 上位介入一期
- 本轮目标：在不更换当前 `Arduino UNO + AFMotor` 底盘控制基线的前提下，完成新的硬件边界收口，形成后续 `ROS2` 建图、导航、巡航的一期硬件冻结线

本文件是给本地端直接执行的正式任务书。  
本轮不要求产出接线图；本轮只要求产出任务书、硬件冻结说明、`BOM`、接线表、风险记录与照片/草图留痕。

## 2. 本轮冻结目标

本期必须冻结以下 4 类新增硬件与接口边界：

- 激光雷达：`RPLIDAR` 一只，挂树莓派 `USB`
- IMU：`MPU6050` 一只，挂树莓派 `I2C`
- 轮速编码器：左右侧轮速反馈方案明确
- 上位机链路：树莓派与 Arduino 的串口控制 / 反馈链路明确

当前冻结口径：

- 继续保留 `UNO + AFMotor` 作为当前底盘控制基线
- 树莓派作为上位介入一期主机
- 本轮先冻结趋势和接口边界，不直接替换现有底层控制基线

## 3. 本期明确不做

以下事项不纳入 `C-2.0.5`：

- 不迁移 `STM32`
- 不更换 `J` 线主抓取基线
- 不引入深度相机
- 不做 `ROS2` 全栈代码重构
- 不改变现有底盘基本机械布局，除非编码器选型逼迫必须局部调整
- 不画接线图，不生成连线示意图

## 4. 本地端必须完成的硬件核查

### 4.1 现有驱动电机实测建档

本地端必须对当前驱动电机形成实测记录，至少包括：

- 电机额定电压
- 空载电流
- 工作电流
- 若条件允许，记录堵转电流
- 电机外形尺寸
- 轴径
- 安装孔位
- 当前轮径
- 当前四轮 / 两轮驱动拓扑

### 4.2 AFMotor 兼容性核查

本地端必须明确：

- 新编码版电机工作电流是否落在 `AFMotor` 可接受区间
- 现有端口占用与编码器输入引脚预算是否足够
- 是否只对左右各选一只编码器电机，还是全驱动轮都需要升级

### 4.3 编码器方案二选一评估并冻结

本轮优先级冻结如下：

1. 方案 A：同规格编码版减速电机替换现有驱动电机
2. 方案 B：保留现有电机，外加轮侧 / 轴侧编码器
3. 方案 C：`Omron E6B2-CWZ6C` 工业轴编码器，本期明确不推荐

结论要求：

- 本地端必须在 `A / B` 之间收敛成一个主方案
- 若因实物条件不能立即定型，也必须明确当前主推荐路径与阻塞原因

## 5. 本轮推荐趋势冻结

### 5.1 激光雷达

- 方向：`RPLIDAR`
- 接口：树莓派 `USB`
- 要求：明确具体安装位和线缆走向趋势

### 5.2 IMU

- 方向：`MPU6050`
- 接口：树莓派 `I2C`
- 要求：明确 `I2C` 挂载方案与供电口径

### 5.3 编码器

- 优先级 1：编码版减速电机
  - 目标：与现有底盘机械兼容，直接输出 `AB` 相
  - 原因：改动最小，最适合当前基线
- 优先级 2：外置轮编码器
  - 目标：保留现有驱动系统
  - 原因：可维持底盘，但需要做好支架、对中、抗抖
- 不建议：`Omron E6B2-CWZ6C`
  - 原因：它是工业增量编码器，不是当前小车最小改造路径

默认推荐覆盖策略：

- 优先先做左右各一侧轮速反馈
- 只有当实测机械、采购或控制需求明确逼迫时，才升级为全驱动轮编码反馈

### 5.4 树莓派 <-> Arduino 链路

默认推荐：

- 冻结为串口控制 / 反馈链路

本地端必须明确并收敛成单一结论：

- 采用 `USB Serial`
  或
- 采用 `GPIO UART`

不得在文档中同时保留两个并行“想法”而不给结论。

## 7. 冻结验收标准

满足以下条件，才算新的硬件冻结线成立：

- 激光雷达型号、接口和安装位明确
- `MPU6050` 挂树莓派 `I2C` 的方案明确
- 左右轮编码器方案确定，且不是停留在“想法”
- 现有 `Arduino + AFMotor` 是否继续承载新编码器输入，结论明确
- 接线表、`BOM`、风险清单齐全
- 本轮之后，`C-2.0.X` 软件实现不再反复改硬件口径

## 9. 结论

`C-2.0.5` 的定位不是“立刻把系统迁到树莓派”，而是先把上位介入一期的硬件边界收住，为后续 `ROS2` 建图、导航、巡航和上位主机接管创造稳定前提。

一句话冻结：

- 本轮先冻结 `RPLIDAR + MPU6050 + 编码器 + 树莓派 <-> Arduino 串口链路` 的硬件边界；
- 不画接线图；
- 不改当前 `UNO + AFMotor` 控制基线；
- 电机型号允许暂未最终拍板，但选型趋势和验收要求必须写死。
