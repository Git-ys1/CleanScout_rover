# C-2.2.6A：UNO 四轮编码器 + PI 闭环桥接基线

## Summary
- 本轮本地只做 `UNO` 侧落地，不拉取树莓派云端 `ROS` 工作区；树莓派侧只作为后续持久串口上位机存在。
- 保留 [c002_uno_baseline.ino](/f:/Project/CleanScout_rover/sketches/c002_uno_baseline/c002_uno_baseline.ino) 作为已验证的手动联调回退线，新建 `sketches/c003_uno_encoder_pid_bridge/` 作为编码器/PID 主入口，并把编译脚本默认入口切到 `c003`。
- `Tyler_1` 不再承载编码器、PID、串口协议或麦轮运动学；它只补一层“原始四轮执行接口”。新增本地库 `libraries/CsrBaseController/` 承担编码器采样、四轮 `PI`、轮速目标管理和新串口协议。
- 本轮控制量冻结为 **每轮 `ticks/s`**，不等 `vx/vy/wz -> rad/s` 和正式 `odom` 才开工。长期 `vx/vy/wz` 语义继续保留，但本轮只把它作为下一阶段扩展位。
- 物理参数先按当前可用真相入配置，不阻塞闭环：
  - `D_MECH_MM = 80.0`
  - `D_EFF_MM = 78.0`（暂定，仅作未来换算占位，本轮闭环不用）
  - `WB_M = 0.16535`
  - `TW_M = 0.17850`
  - `K_M = 0.171925`
  - `CPR_X1_EST = 260.0`（先按卖家典型值占位，后续用 10 圈实测回填）

## Key Changes
### 1. Tyler 保持最小执行层，但必须补原始轮控接口
- 同步修改两套库副本：
  - `libraries/Tyler_1`
  - `Tyler_1_Library`
- 保留现有 motion-only 语义和 `255` 禁用语义不变。
- 新增最小原始执行接口，固定为：
  - `setWheelCommand(uint8_t wheel, int16_t signedPwm)`
  - `setWheelCommands(int16_t m1, int16_t m2, int16_t m3, int16_t m4)`
- 语义固定：
  - 输入范围 `-255..255`
  - 符号决定方向，绝对值决定 PWM
  - `0` 等价 `RELEASE`
  - 保留现有 `(1,0,1,1)` 轮向极性校正逻辑，原始轮控接口必须复用这套极性，不允许在新库里再硬编码第二套电机方向表
- 不向 Tyler 新增：
  - 编码器采样
  - PID
  - 串口协议
  - 麦轮逆解
  - `cmd_vel` 语义

### 2. 新增 `CsrBaseController` 本地库，采用 x1 编码器 + 四轮 PI
- 新建 `libraries/CsrBaseController/`，固定包含：
  - `WheelEncoder.h/.cpp`
  - `WheelPI.h/.cpp`
  - `SerialProtocol.h/.cpp`
  - `CsrBaseController.h/.cpp`
- 这轮不单独建 `MecanumKinematics` 文件；因为当前主协议先收口到四轮目标速度 `W,w1,w2,w3,w4`。`vx/vy/wz` 适配留到下一阶段，避免几何常数未完全实测时把运动学半成品塞进当前主线。
- `WheelEncoder` 固定实现：
  - `x1` 解码
  - 只有 A 相触发计数
  - ISR 中读取 B 相判方向
  - 维护 `volatile long encoder_ticks[4]`
  - 维护上一采样窗口 tick 快照，计算 `delta_ticks`
- 引脚逻辑固定为已接好的优先映射：
  - `wheel1: A0/A1`
  - `wheel2: A2/A3`
  - `wheel3: A4/A5`
  - `wheel4: D9/D10`
  - A 相：`A0 A2 A4 D9`
  - B 相：`A1 A3 A5 D10`
- 中断方向固定为：
  - `PCINT1_vect` 处理 `Port C` 上的 `A0/A2/A4`
  - `PCINT0_vect` 处理 `Port B` 上的 `D9`
  - 只对 A 相做边沿检测；推荐固定用上升沿计数，B 相只判方向
- `WheelPI` 固定为自写轻量类，不引入 `PID_v1/QuickPID`：
  - `begin(kp, ki, i_limit, out_limit, dt_s)`
  - `reset()`
  - `compute(target_ticks_s, measured_ticks_s)`
- `WheelPI` 行为固定：
  - 只做 `PI`
  - `Kd = 0`
  - 固定 `dt = 0.02s`
  - 积分限幅
  - 输出限幅 `-255..255`
  - 目标为 `0`、超时停车或模式切换时必须 `reset()`

### 3. `c003` 成为编码器/PID 主入口，`c002` 保留回退
- 新建：
  - `sketches/c003_uno_encoder_pid_bridge/c003_uno_encoder_pid_bridge.ino`
- `c003` 只做：
  - `Serial.begin(115200)`
  - 初始化 `Tyler_1(1, 0, 1, 1, 200, 255, 255, 255)`
  - 初始化 `CsrBaseController`
  - 上电打印 `CSR_UNO_READY`
  - 主循环中按固定 20ms 调 `controller.update()`
  - 处理串口输入、LED 调试、超时停车
- `c002` 不再扩写，只保留为字符动作码回退线。当前本地 4000ms 脏树改动不纳入这轮实施。
- 构建工具和入口文档统一切到 `c003`：
  - `tools/compile_uno_cli.ps1`
  - `tools/compile_uno.ps1`
  - 根 `README.md`
  - `docs/SOFTWARE/C-2.2.6A_uno_encoder_pid_execution.md`
- `c002` 在文档中明确标为：
  - 已验证手动联调回退入口
  - 非当前闭环主线

### 4. 新主协议冻结为四轮目标速度，`1~9` 不进入 `c003`
- 本轮新主协议固定为单行 ASCII：
  - `W,<w1>,<w2>,<w3>,<w4>\n`
- 字段语义固定：
  - `w1..w4` 是四个轮子的**有符号整数**目标速度
  - 单位固定为 `ticks/s`
- 当前 `c003` 不再实现 `1~9` 字符协议；兼容调试留给 `c002`。这样避免把开环动作码和闭环轮速控制混在同一入口里。
- `SerialProtocol` 固定输出：
  - 启动：`CSR_UNO_READY`
  - 有效帧：`ACK:W`
  - 非法帧：`ERR:<reason>`
  - 编码器/速度遥测：`ENC,t1,t2,t3,t4,v1,v2,v3,v4`
- `ENC` 语义固定：
  - `t1..t4`：累计 tick
  - `v1..v4`：当前窗口速度，单位 `ticks/s`
- `PID,...` 不作为本轮必需输出；若实现，放在 `CSR_DEBUG_PID` 编译开关后，默认关闭。
- 串口解析规则固定：
  - 行缓冲 + `\n` 结尾
  - 忽略 `\r`
  - 解析失败不刷新命令超时
- 命令超时固定为 `400ms`
  - 超时后四轮目标速度清零
  - 所有 PI 积分复位
  - 通过 Tyler 原始轮控接口停车
- 板载 `LED_BUILTIN` 在收到有效 `W` 帧时非阻塞闪烁一次

### 5. 本轮先做 ticks/s 闭环，不阻塞于物理量和 ROS 全接入
- 本轮控制环固定流程：
  - 读最新 `W` 帧目标
  - 每 20ms 读取四轮 `delta_ticks`
  - 换算四轮 `measured_ticks_s`
  - 四个 `WheelPI` 分别输出四路有符号 PWM
  - 通过 `Tyler_1::setWheelCommands()` 下发到 AFMotor
- 本轮不要求：
  - `vx/vy/wz -> 轮速(rad/s)` 全量正确落地
  - 正式 `odom`
  - 直接接 `ROS /cmd_vel`
- 但要把下一阶段扩展位设计好：
  - `CsrBaseController` 对外 API 固定先提供 `setWheelTargetsTicksPerSecond(w1,w2,w3,w4)`
  - 预留未来 `setBodyVelocity(vx, vy, wz)` 入口，但本轮不实现
- 物理量配置的冻结方式：
  - `CPR_X1_EST = 260.0` 只用于未来 `rpm/rad_s` 转换和调试显示准备，不影响本轮 ticks/s PI
  - `D_EFF_MM = 78.0`、`WB_M`、`TW_M`、`K_M` 先入配置常量区，作为下一阶段逆运动学和 odom 的唯一来源
  - 同时在执行文档中写死测量回填方法：
    - `CPR_x1`：10 圈实测平均
    - `D_eff`：长距离直行反推
    - `WB/TW`：轮轴中心距实测

## Test Plan
- 编译：
  - `c003` 通过 `arduino-cli` 和 `arduino-builder` 两套脚本编译
  - `c002` 回退线不被破坏
- 启动与串口：
  - 上电输出 `CSR_UNO_READY`
  - 有效 `W` 帧回 `ACK:W`
  - 非法帧回 `ERR:<reason>`
  - 有效帧时 `LED_BUILTIN` 闪烁
- 编码器：
  - 四个轮子的 tick 都会变化
  - `ENC,t1..t4,v1..v4` 持续稳定输出
  - 断开某一路 A/B 相时，可在 `ENC` 中观察到对应轮异常
- 闭环：
  - `w1..w4` 目标变化时，对应轮方向正确
  - `target=0` 时积分复位、PWM 回到 0
  - 命令超时 `400ms` 后四轮停车
  - 在同样目标下，闭环响应明显优于直接开环 PWM
- 结构与边界：
  - `Tyler_1` 只新增原始轮控接口，不引入编码器/PID/串口协议
  - 新逻辑集中在 `libraries/CsrBaseController`
  - `c003` 只做初始化与调度，不把控制算法写散进 `.ino`
