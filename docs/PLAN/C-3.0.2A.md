# C-3.0.2A：UNO 编码器 X1 升级到 X2 并更新底盘常数

## Summary
- 本轮只改 `UNO` 本地实现，不改 `W,w1,w2,w3,w4` 协议、不改树莓派桥接、不改 ACK/ERR 返回格式。
- 核心改动有两类：
  - `WheelEncoder` 从“只计 A 上升沿”的 `X1` 升级为“计 A 上升沿 + 下降沿”的 `X2`
  - `c003` 当前底盘常数统一更新为：
    - `WB_M = 0.1905`
    - `TW_M = 0.1800`
    - `K_M = 0.18525`
    - `D_EFF_MM = 78.0`
    - `CPR_X1_EST = 520.0`
- 物理接线真相保持不变，任何方向问题继续只通过 `ENC_SIGN_Wn` 修正，不再交换 A/B。

## Key Changes
### 1. 编码器解码升级为 X2
- 在 `libraries/CsrBaseController/WheelEncoder.cpp` 将当前“只在 A 上升沿计数”的逻辑替换为“只要 A 变化就计数”。
- 固定采用你给出的判向口径：
  - 比较 `a_now` 和 `a_prev`
  - `A` 不变则不计数
  - `A` 变化时读取 `B`
  - `delta = (a_now == b_now) ? -1 : 1`
  - `delta *= directionSign`
- `WheelEncoder` 的对外接口保持不变：
  - `begin(...)`
  - `readTicks()`
  - `sampleTicksPerSecond(...)`
- `RAW4` 探测模式自动继承新的 `X2` 计数逻辑，不再单独保留旧 `X1` 分支。

### 2. 常量更新，但接口命名先不重构
- 在 `sketches/c003_uno_encoder_pid_bridge/c003_uno_encoder_pid_bridge.ino` 更新当前冻结常量：
  - `WB_M = 0.1905`
  - `TW_M = 0.1800`
  - `K_M = 0.18525`
  - `D_EFF_MM = 78.0`
  - `CPR_X1_EST = 520.0`
- `CsrBaseControllerConfig.cprX1Est` 和 `CPR_X1_EST` 这轮只更新数值，不重命名字段；避免把本轮扩大成接口重构。
- `K_M` 继续明确按 `(WB_M + TW_M) / 2` 取值。

### 3. 方向与诊断口径保持一致
- `wheel1~wheel4` 当前 A/B 引脚映射不变：
  - `wheel1: A0/A1`
  - `wheel2: A2/A3`
  - `wheel3: A4/A5`
  - `wheel4: D9/D10`
- 若 `X2` 升级后某个轮子方向相反，只改 `ENC_SIGN_Wn`，不改接线、不改协议。
- `M3` 继续保留当前已验证的：
  - `ENC_SIGN_W3 = -1`
- `M4` 当前既然已定位过接触问题并恢复转动，本轮不重新引入新的 `Port B` 诊断目标，只保留 `RAW4` 作为回退诊断入口。

### 4. 文档同步到新冻结真相
- 新增本轮文档：
  - `docs/PLAN/C-3.0.2A.md`
  - `docs/SOFTWARE/C-3.0.2A_uno_x2_encoder_upgrade.md`
- 文档必须写清：
  - 当前从 `X1` 升级到 `X2`
  - 计数规则变成“A 的双边沿计数，B 判方向”
  - 新冻结底盘常量
  - 当前不改协议、不改树莓派桥
- 同步修正当前仍写着旧常量/旧 `X1=260` 的既有文档，至少覆盖：
  - `docs/SOFTWARE/C-2.2.6A_uno_encoder_pid_execution.md`
  - `docs/SOFTWARE/C-2.2.7A_uno_w_protocol_freeze.md`
  - `docs/SOFTWARE/C-2.2.7_serial_protocol_contract.md`
- 更新口径时只改“当前冻结值”和“当前编码倍率”相关表述，不回写历史结论。

## Test Plan
- 编译回归：
  - `c003` 正常模式通过 `arduino-cli` 和 `arduino-builder`
  - `c003` `RAW4` 模式也能编译
  - `c002` 回退线不受影响
- 编码器行为验证：
  - 四轮在低速 `W,80,80,80,80`、`W,120,120,120,120` 下持续转动
  - `ENC` 输出频率保持正常
  - 与 `X1` 相比，低速测速更连续、更平顺
  - 若个别轮方向反了，只通过 `ENC_SIGN_Wn` 修正
- 参数一致性验证：
  - `ENC/PID` 输出仍正常
  - `RAW4` 仍可用
  - 文档中所有“当前冻结 CPR/几何常量”不再出现旧值 `260 / 0.16535 / 0.17850 / 0.171925`

## Assumptions
- 本轮继续以 `c003_uno_encoder_pid_bridge` 为唯一闭环主线，`c002` 保持回退线，不纳入改造。
- 现有用户本地对 `c002` 的脏改动继续不碰。
- 本轮不做 `X4`，不改 `PI` 结构，不改 `W` 协议，也不把 `cmd_vel -> W` 提前带进来。
- 为避免无谓扩散，本轮保留 `CPR_X1_EST` / `cprX1Est` 这些旧命名，只把数值更新为 `520.0`；若后续需要统一命名，再单独起一轮重构。
