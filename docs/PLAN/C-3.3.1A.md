# C-3.3.1A：RF1 通道语义纠偏与六方向闭环收口计划

## Summary
- 这轮只处理 RF1 底层，不碰树莓派、ROS、建图、导航。
- 关键结论：不同增益不能“产生左右转”。左右转/横移来自 `W,a,b,c,d` 的四轮目标符号组合；增益只负责让每个轮子跟住自己的目标速度。
- 当前必须先把通道语义写死：`CN1=左后(LR)`、`CN2=左前(LF)`、`CN3=右后(RR)`、`CN4=右前(RF)`。
- 当前 `C-3.1.4D` 的 `[CN1,CN2]` vs `[CN3,CN4]` 分组本质是左右侧分组，不是前后轮分组；本轮先显式纠偏，再做六方向测试和参数收口。

## Key Changes
- 新建/继续自测分支：`feature/c-3.3.1a-rf1-channel-semantic-pid`。
- 在 RF1 代码中新增明确轮位别名，后续不再口头解释通道：
  - `CSR_WHEEL_LR = CSR_CHANNEL_CN1`
  - `CSR_WHEEL_LF = CSR_CHANNEL_CN2`
  - `CSR_WHEEL_RR = CSR_CHANNEL_CN3`
  - `CSR_WHEEL_RF = CSR_CHANNEL_CN4`
- 保持协议不变：
  - `W,a,b,c,d`
  - `M,<ch>,<pwm>`
  - `E,<ch>`
  - `D,<ch>`
  - `STOP`
- 保持 `W` 参数顺序不变：
  - `a=CN1/LR`
  - `b=CN2/LF`
  - `c=CN3/RR`
  - `d=CN4/RF`
- 先把 PI 与最小驱动参数从“误认为前后”的注释/实现中纠正为真实轮位语义。
- 本轮调参顺序固定：
  - 第一轮：四轮统一 PI 参数，验证六方向符号表是否正确。
  - 第二轮：若前进偏航，再按左右侧差分调参。
  - 第三轮：若横移/旋转中前后轮差异明显，再按前后轮差分补偿。
- 六方向 `W` 测试表按当前轮位顺序冻结：
  - 前进：`W,+v,+v,+v,+v`
  - 后退：`W,-v,-v,-v,-v`
  - 左移：`W,+v,-v,-v,+v`
  - 右移：`W,-v,+v,+v,-v`
  - 左转：`W,-v,-v,+v,+v`
  - 右转：`W,+v,+v,-v,-v`
- 测试速度分三档：
  - 低速：`v=0.05`
  - 中速：`v=0.15`
  - 较高速：`v=0.25`
- 安全策略：
  - 每组连续发送 `W`，测试 1.2 到 1.8 秒。
  - 每组结束立即 `STOP`。
  - 先架空测六方向，再落地测低速前进/转向/横移。

## Implementation Details
- `csr_board_map.h`
  - 增加轮位别名和注释，明确 `CN1/CN2/CN3/CN4` 与 `LR/LF/RR/RF` 的关系。
- `main.c`
  - 将 `g_kp/g_ki/g_kd` 注释改成轮位语义，不再写“前两个/后两个”。
  - 初始回到统一 PI 基线，建议从当前较稳值折中：
    - `kp = 1800.0f`
    - `ki = 160.0f`
    - `kd = 0.0f`
  - 若统一参数下六方向符号正确但前进偏航：
    - 只做左右侧差分：`CN1/CN2` 为左侧，`CN3/CN4` 为右侧。
  - 若左右侧已经平衡但横移或旋转明显前后不均：
    - 再做前后差分：`CN1/CN3` 为后轮，`CN2/CN4` 为前轮。
- `csr_motor_drv.c`
  - 保留分通道最小驱动能力，但注释改成真实轮位。
  - 不再把 `[0,1]` 写成前轮组、`[2,3]` 写成后轮组。
- 文档新增：
  - `docs/VERIFY/C-3.3.1A_rf1_channel_semantic_pid_closure.md`
  - 必须记录最终轮位表、六方向命令表、每档测试结果、最终参数。

## Test Plan
- 编译烧录：
  - Keil 编译 RF1 工程，要求 `0 Error(s)`。
  - STM32CubeProgrammer/STLink 烧录。
  - COM14 看到 `CSR_RF1_READY`。
- 单轮通道确认：
  - `M,1,300` 到 `M,4,300`
  - 确认四路仍对应 `LR/LF/RR/RF`。
- 六方向架空测试：
  - 对前进、后退、左移、右移、左转、右转逐项发送冻结 `W` 表。
  - 每项记录 `VEL` 和 `PWM`。
  - 通过标准：四个目标通道都有相应速度反馈，且物理方向与命令一致。
- 六方向落地测试：
  - 先用 `v=0.05`。
  - 必测：前进、后退、左转、右转。
  - 横移只做短测，避免卡地面导致误判。
- 调参验收：
  - 前进不再明显偏航。
  - 左转/右转可复现，不再只有前进可用。
  - 左移/右移至少在架空状态下符号正确。
  - `CN1/CN3` 不再被当成“前/后分组”误调。
- 回归：
  - `STOP` 可靠停车。
  - 停发 `W` 后仍按 watchdog 自动停车。
  - `M` raw 调试语义不变。

## Assumptions
- 本轮只做 RF1 固件闭环调参，不处理树莓派建图/导航。
- 当前轮位真值固定为：`CN1=左后`、`CN2=左前`、`CN3=右后`、`CN4=右前`。
- 六方向测试的符号表按常规麦轮安装假设冻结；如果实测发现“左移/右移”整体反了，只在文档中记录并在上位机运动学层修正，不改 `W` 协议。
- 不通过增益制造转向；转向必须来自正确的四轮目标符号组合。
