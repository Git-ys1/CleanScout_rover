# C-5.0.3 机械臂追踪路线核定

日期：2026-06-10

## 已核实事实

- `firmware/mechanical_arm_official_baseline/User/Components/y_kinematics/y_kinematics.c` 只对 4 个关节做逆运动学。
- `kinematics_move()` 最终只生成 `#000`~`#003` 的同步文本命令。
- `set_servo()` 是单舵机文本命令入口。
- `OrangePi/rk3588_ai/arm_tracking_demo/00_REFERENCE_AUDIT.md` 里的参考追踪不是完整 IK，而是固定姿态下只调少数关节。
- 仓库里没有找到单独的 A1 教程文件；这里的结论以可核实的参考审计和基线代码为准。

## 当前结论

- 第一版追踪应采用“固定前向姿态 + 局部视觉伺服”。
- `004/005` 先只做末端旋转和夹爪，不进追踪闭环。
- 完整 IK 只在后续需要末端位姿保持、到位搬运或抓取时使用。

## 2026-06-11 实机闭环结果

- Astra Pro 在 `/dev/video0` 以 `640x480 MJPG 30FPS` 正常工作。
- CH340 下位机链路为 `/dev/ttyUSB0 @ 115200`。
- `000` 偏转 100 PWM 后，末端相机画面约旋转 `-4.65°`，证明 yaw 真实动作。
- `003` 偏转 100 PWM 后，末端画面约旋转 `1.96°`、位移约 `17px`，证明腕部 pitch 真实动作。
- `001/002` 会造成大幅视野变化，属于主臂大关节，不应替代 `003` 做第一版细调追踪。
- 合成右下目标连续运行 14 个 10Hz 控制周期后：
  - `000` 从 `1492` 累积到 `1395`。
  - `003` 从 `1508` 累积到 `1605`。
  - 末端画面约旋转 `12.67°`，位移约 `92/-60px`。
  - 测试结束自动恢复 `000/003 = 1500` 并分别发送停止命令。
- 真实 YOLO 已识别到包类目标；人工把目标向左移动后，`000` 实际向右转动，证明“检测框横向误差 -> 串口命令 -> 底座 yaw”真实闭环已动作。

## 末端相机负载与安全预备姿态

- 末端安装 Astra Pro 后，原预备姿态 `001=1350、002=1750` 会让主臂持续向前伸展，`003` 承受较大静态力矩，实机曾出现低头后无法自行抬起。
- 官方运动学映射确认：当前机构中增大 `001` 可让肩部越过竖直后向后仰，增大 `002` 可继续加大肘部反折。
- 2026-06-11 分级实测得到安全折叠路径：
  - `001=1550、002=1850`
  - `001=1700、002=2000`
  - `003=1480`
- `002=2450` 已被实机否定：该方向会让末端明显下压，不是目标“002 回转、003 抬起”的姿态。
- 最终验证位置接近 `001=1692、002=2046、003≈1480`。末端相机重新看向前方，且机械臂不再长距离前伸。
- 启动顺序必须是先慢速反折 `001/002`，再允许 `003` 承担小范围视觉俯仰；追踪循环本身仍只连续改动 `000/003`。

## 新增实测工具

- `tools/bus_servo_probe.py`：总线命令收发和原始十六进制记录。
- `tools/arm_servo_tune.py`：远程桌面手动调舵机位置，支持 `read/set/nudge/save`。
- `tools/save_arm_start_pose.py`：一键把当前或指定 PWM 保存为视觉追踪启动姿态。
- `tools/test_camera_arm_motion.py`：用末端相机量化单舵机动作。
- `tools/test_visual_servo_motion.py`：用确定性目标框回归完整视觉伺服输出链。
- 三类工具与真实追踪程序均独占 `/dev/ttyUSB0`，禁止并发抢回包。

## 手动调姿与一键保存

远程桌面到香橙派后，进入交互调姿：

```bash
cd ~/rk3588_ai/arm_tracking_demo
~/rk3588_ai/rknn_lite_env/bin/python3 tools/arm_servo_tune.py
```

常用输入：

```text
read 0-5
set 2=2000,3=1480 1500
nudge 2=-50 1000
nudge 3=50 1000
save 1-3
quit
```

如果某个舵机能动作但 `PRAD` 偶发读空，例如本轮 `003`，可直接保存确认好的数值：

```bash
~/rk3588_ai/rknn_lite_env/bin/python3 tools/save_arm_start_pose.py \
  --values 1=1700,2=2000,3=1480
```

保存会改 `config/arm_track_config.yaml` 的 `tracking_pose_pwms`、`tracking_pose_stages`，
并在保存 `003` 时同步更新 `pitch_pwm_neutral`，避免视觉追踪启动后覆盖手动调好的腕部角度。
