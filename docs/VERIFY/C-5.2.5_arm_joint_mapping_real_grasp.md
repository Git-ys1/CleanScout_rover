# C-5.2.5 机械臂关节映射与真实分阶段抓取

## 目标

将 C-5.2.4 已通过的 RGB-D 固定视角坐标链，与本机 Servo000--005 的实测方向、
角度比例和夹爪边界接通。真实测试仍使用
`OPEN -> PRE_GRASP -> APPROACH -> CLOSE -> LIFT`，禁止针对单个瓶子位置手调 PWM。

## 冻结实测

| 舵机 | PWM 增大时的物理运动 | 抓取语义正方向 | PWM 符号 |
| --- | --- | --- | ---: |
| 000 | 向左/俯视逆时针 | 向左为正 | +1 |
| 001 | 主臂向后 | L1 向前为正 | -1 |
| 002 | 肘部收拢 | L2 相对 L1 向前折叠为正 | +1 |
| 003 | 末端抬头 | L3 相对 L2 抬头为正 | +1 |
| 004 | 本阶段不动 | 固定 `1500` | 不参与 IK |
| 005 | 夹爪收拢 | 闭爪方向 | 分阶段直接控制 |

厂家确认原始总线舵机标称能力为 `PWM 500..2700 <-> 0..270 deg` 线性映射，因此
Servo000 的量角器基准采用：

```text
K_PWM_PER_DEG = (2700 - 500) / 270
              = 8.148148148148149 PWM/deg
angle_deg = (pwm - zero_pwm) / (pwm_sign * K_PWM_PER_DEG)
```

但不能把 `8.148148 PWM/deg` 无条件套给所有安装后的关节。D435 桌面平面法实测
发现 Servo001--003 的机构有效斜率不同，旧代码统一比例正是 PRE_GRASP 跑到瓶子
上方的直接原因。运行时必须使用逐轴斜率：

| 舵机 | 有效斜率 PWM/deg | 证据 |
| --- | ---: | --- |
| 000 | `8.1481481481` | 厂家线性映射及 `1500=0 deg`、`1744~=+30 deg` 量角器复核 |
| 001 | `7.0908242948` | D435 桌面平面法，`+103 PWM -> 14.526 deg` |
| 002 | `7.9358274363` | D435 桌面平面法，`+100 PWM -> 12.601 deg` |
| 003 | `6.4780957391` | D435 桌面平面法，`+183 PWM -> 28.249 deg` |

本机定义 Servo000--003 的语义零位均为 `PWM1500`，不使用原始范围中点 `1600`
作为机械零位。Servo000 的实测校验点为 `1500=0 deg`、`1744~=+30 deg`，由同一
线性映射得到 `2233~=+90 deg`、`767~=-90 deg`。

厂家标称 `2700` 不是当前 STM32 固件可发送的上限。已烧录的官方控制器源码会
拒绝 `aim>2500`，并在更新环节钳到 `2490`，因此运行时命令范围冻结为
`500..2490`。实测发送 `2500` 曾回读为 `500`，不得再触碰该边界。

按照冻结系数重新计算参考姿态角，而不是沿用早期近似值：

| 舵机 | PWM | 语义角 |
| --- | ---: | ---: |
| 001 | 1909 | `-57.680 deg` |
| 002 | 1900 | `+50.404 deg` |
| 003（视觉追踪） | 900 | `-92.620 deg`，组合后 L3 接近水平 |
| 003（固定视角） | 620 | `-135.842 deg`，组合后向下观察桌面 |

夹爪实测：

| Servo005 PWM | 状态 |
| ---: | --- |
| 1000 | 接近物理最大张开，作为 `OPEN` |
| 1800 | 最大松动间距约 15 mm |
| 1900 | 最大松动间距约 5 mm，物理闭合起点 |
| 2000 | 最大松动间距约 4 mm，作为第一版 `CLOSE` |
| 2200 | 间距仍约 4 mm，只增加夹持力，不作为默认值 |

## 运动学修正

官方 STM32 例程继续提供平面几何结构，但不再直接沿用其舵机比例和符号：

- Servo000 偏航使用 `atan2(left, forward)` 的真实角度，不再乘 `270/pi`。
- `q1 = 90 deg - theta5`，并通过 Servo001 的负 PWM 符号实现“向前为正”。
- `q2 = theta4`。
- `q3 = alpha - theta5 + theta4`。
- 逆解和诊断正解共享同一 `zero_pwms/pwm_signs/pwm_per_deg_by_joint` 配置。

本机有效连杆仍使用 C-5.2.0 实测值：`L0=0.100 m`、`L1=0.130 m`、
`L2=0.065 m`、`L3=0.177 m`。这组参数用于固定腕部的分阶段抓取，不代表动态手眼
标定已经完成。

临时 `pitch_deg=18` 虽能让距底座约 `0.245 m` 的近目标获得 IK，但真实 PRE_GRASP
确认 L3 明显不水平，因此该方案只保留为诊断证据，不得用于正式抓取。正式配置冻结
`pitch_deg=0`，使 L3/TCP 水平接近瓶子；闭爪后的 80 mm LIFT 使用
`lift_pitch_deg=-11`。

扫描当前实测连杆、逐轴比例、PRE_GRASP `70 mm` 间距和 `500..2490` 控制器边界后，
水平完整计划要求瓶心距底座至少约 `0.340 m`。在该半径、当前目标高度与偏航下，
PRE_GRASP/瓶心/LIFT 的代表 PWM 分别为：

```text
PRE_GRASP [1481,1289,2486,1914,1500,1000]
CLOSE     [1481,1129,1977,1646,1500,2000]
LIFT      [1481,1106,1576,1410,1500,2000]
```

目标更近时必须沿已对正方向向外移动，不能通过下压 L3、改某个舵机 PWM 或绕过 IK
强行执行。

## 固定视角参考姿态换基

新参考姿态为：

```text
[1500,1909,1900,620,1500,1500]
```

C-5.2.4 的 11 点矩阵是在 Servo000=`1380` 时采集。曾按底座 Z 轴解析计算：

```text
q0_old = (1380 - 1500) / K = -14.7272727 deg
q0_new = 0 deg
delta  = +14.7272727 deg
T_candidate = Rz(delta) * T_C-5.2.4
```

但该候选矩阵在真实 PRE_GRASP 中把正前方目标预测为约 `+12.38 deg`，机构明显偏向
左侧；原始实测矩阵预测约 `-2.35 deg`，后续真实动作被用户确认左右对正。因此
`Rz(delta)` 解析换基已否决，当前临时采用：

```text
T_reference1500 = T_C-5.2.4
```

源矩阵、被否决的推导和物理证据记录在：

- `OrangePi/rk3588_ai/arm_grasp_pipeline/config/base_camera_report.c525_reference1500.json`
- `fixed_view_calibration.rebase_provenance`

源 11 点拟合的 RMSE `5.85 mm` 和最大误差 `9.71 mm` 仍是历史质量证据，但尚未在
`000=1500` 下做独立留出点复测。因此必须逐阶段观察，不能直接运行到 `LIFT`。

## 真实安全链

每次真实命令先执行：

1. 5 秒低速回到固定参考姿态。
2. 用 `PRAD` 回读 Servo000--005。`PRAD` 是 STM32 内部命令/当前 PWM 状态，不是
   舵机物理编码器反馈，不能单独证明机械关节真的到位。
3. 任一轴缺失或与目标相差超过 `40 PWM`，立即拒绝继续。
4. 锁定 bottle 的 aligned depth 与底座 XYZ。
5. 只预检到 `--max_stage` 请求的阶段，逐步检查 workspace、IK、逐轴安全 PWM 和
   Servo004=`1500`；未来阶段无解不能反向阻断当前合法阶段。
6. 只执行 `--max_stage` 允许的阶段并保持，等待人工观察。

阶段含义：

| 阶段 | 实际动作 |
| --- | --- |
| OPEN | Servo005 到 1000 |
| PRE_GRASP | 张爪保持，在瓶心前方 70 mm 停止 |
| APPROACH | 每 10 mm 一个路点，保持同一高度水平接近瓶心 |
| CLOSE | 到瓶心后 Servo005 到 2000 |
| LIFT | 闭爪保持并向上抬 80 mm |

## 自动测试

2026-07-17 本机与香橙派均通过：

- `python -m unittest discover -s arm_grasp_pipeline/tools -p 'test_*.py'`：31 项通过。
- `tools/test_official_kinematics.py`：`OFFICIAL_KINEMATICS_OK`。
- `tools/test_grasp_safety.py`：`GRASP_SAFETY_CHECK_OK`。
- 新增回归冻结逐轴角度映射、STM32 `2490` 上限、追踪/固定视角组合俯仰，以及
  已知瓶心 PRE_GRASP 的逐轴 PWM。

STM32 还有一个与慢速测试直接相关的坑：`current` 为 `uint16_t`，`increment` 为
`float`。小位移配过长时间会使每周期增量小于 `1 PWM` 并在整数写回时截断；因此
`003 +100 PWM / 6000 ms` 是无效探针，不得据此判断舵机卡住。标定工具会拒绝
低于 `1.05 PWM/tick` 的测试组合。
