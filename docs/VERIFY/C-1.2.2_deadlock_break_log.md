# C-1.2.2 死结解锁日志

## 1. 本轮目的

本轮专门处理 `WAIT_PICK_WINDOW` 死结：

- `OpenMV` 已经能识别并反复发送 `COLOR_FOUND`
- `F411` 在等 `STOP_ACK`
- `J` 在等 `PICK_WINDOW`
- 任一环掉链子，抓取就永远不会发生

本轮的处理方式不是删协议，而是：

- `J` 侧增加本地抓取回退
- `F411` 侧接住早到结果

## 2. 本轮代码结果

### 2.1 OpenMV

已新增：

- `LOCAL_PICK_FALLBACK_DELAY_MS = 1400`
- `POST_PICK_COOLDOWN_MS = 1800`
- `LOCAL_PICK_FALLBACK` 可视状态
- `WAIT_PICK_WINDOW` 期间定时重发 `COLOR_FOUND`
- 超过本地回退阈值且目标仍在视野中时，直接进入本地抓取
- 抓取成功/超时/异常后继续发 `PICK_DONE / PICK_TIMEOUT / ARM_FAIL`

- `diag_pick_done_count`

已修改：

- `WAIT_UNO_STOP_ACK` 阶段允许接收 `PICK_DONE / PICK_TIMEOUT / ARM_FAIL`
- 接住后直接转 `RESUME_REQ -> WAIT_UNO_RESUME_ACK`
