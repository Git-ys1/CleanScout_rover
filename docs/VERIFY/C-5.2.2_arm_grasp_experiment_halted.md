# C-5.2.2 机械臂抓取实验终止记录

## 结论

本轮没有完成 bottle 抓取，实机调试已按用户要求停止。当前代码只能作为实验分支基线，不能视为可用抓取算法，也不能直接用于无人值守自动执行。

停止时已向 `000--005` 分别发送 `PDST`，没有继续发送位置、闭爪或抬升命令。

## 已确认事实

- D435 RGB、对齐深度和 RKNN YOLO bottle 识别可运行。
- 当前 STM32 固件接收 `$KMS` 后，`000--003` 实测位置不变；该路径在本轮实机上属于静默不执行。
- Python 官方 IK 计算出的 `000--003` 目标改用总线舵机直接 PWM 后，四轴会真实运动。
- 旧计划沿完整“相机到目标”三维向量逼近，同时官方 IK 端口忽略了 `pitch_deg`，导致机械臂以明显俯视姿态向 bottle 接近。这条路线不满足从瓶身正前方张爪包夹的要求。
- 直接 PWM 的首个 PRE_GRASP 将中央深度从约 `0.288 m` 降到约 `0.172 m`，但画面仍是俯视，不能据此执行闭爪。
- 现场按 001 前送、003 上抬后可以得到更接近平视的 bottle 正面画面；该人工姿态尚未形成可靠、可重复的自动运动学方案。
- 005 张开命令 `P0600` 的实测回读约为 `1112`，闭合命令 `P2400` 的实测回读约为 `1897`。未完成空载端点、开口宽度、瓶径和夹持受阻位置标定，因此不能仅凭回读值判断已抓住。
- 本轮没有获得“瓶底离桌并稳定保持”的物理成功证据。

## 本轮保留的实验代码

- 分阶段执行：`OPEN / PRE_GRASP / APPROACH / CLOSE / LIFT`。
- 显式 `--allow_manual_seed` 实验开关和重复尝试后的视觉核验框架。
- 官方 IK 结果改为 `000--003` 原子直接 PWM，不再依赖本轮失效的 `$KMS` 执行链。
- `pitch_deg` 固定俯仰求解、水平面接近和独立 lift pitch 的初版实现。

上述实现均未通过完整实抓验收。配置已标记 `halted_unverified`。
