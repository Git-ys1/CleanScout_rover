# C-5.2.0 机械臂、夹爪与 D435 外参测量表

## 何时测量

只在以下条件全部满足后测量：

- 新相机支架完成并锁紧，后续不再改孔位、垫片和安装方向。
- D435、004 腕部和 005 夹爪均安装到最终位置。
- 机械臂底座固定，不再改变底座相对桌面的高度和朝向。
- 电池电量正常，000--005 均能读取位置并缓慢动作。

支架变化会改变完整 6DoF 外参，旧支架的数据不能沿用。

## 坐标系冻结

### 基座坐标 `B`

- 原点：000 竖直旋转轴与机械臂底座安装平面的交点。
- `+X_B`：车体/机械臂正前方。
- `+Y_B`：面向正前方时的左侧。
- `+Z_B`：竖直向上。
- Python 统一使用 `[forward, left, up] = [X_B, Y_B, Z_B]`，单位为米。
- 官方 `$KMS:x,y,z,time!` 使用 `[left, forward, up]`，单位为毫米。

`+Y_B` 对应 000 的实际转向必须用一次小幅动作确认，不能只靠代码符号猜测。

### 工具/TCP 坐标 `T`

- 原点：两爪内侧接触面之间、计划夹住物体高度处的中心点。
- `+X_T`：从腕部沿夹爪中心线指向待抓物体。
- `+Y_T`：从夹爪中心指向左爪。
- `+Z_T`：按右手系确定，正常抓取姿态下大致向上。

TCP 不是 003 轴心、相机光心或夹爪外壳中心。

### D435 彩色光学坐标 `C`

- 原点：D435 RGB 镜头光心。
- `+X_C`：图像向右。
- `+Y_C`：图像向下。
- `+Z_C`：镜头向前。

当前程序把深度对齐到彩色图后再反投影，因此需要的是
`T_tool_camera_color_optical`，不是左红外镜头对应的深度外参。

D435 出厂保存的 RGB、左右红外和深度模组内外参由 librealsense 自动读取，
`rs.align(rs.stream.color)` 已用这些参数完成 depth-to-color 对齐。这组相机内部
外参无需人工测量，也不能替代厂家未知的“机械臂 TCP 到整台 D435”安装外参。

## 第一组：必须先回答的安装关系

| 项目 | 实测/选择 |
| --- | --- |
| 相机支架固定在哪个刚体：003 后端、004 固定侧、004 转动侧、005 夹爪基座 | 待填 |
| 当前冻结：D435 位于 004 正上方，固定在 004 定子/不转外壳 | 已确认 |
| 相机是否随 004 输出轴旋转 | 否；只随 004 壳体和上游 000--003 整体移动 |
| 夹爪基座是否随 004 输出轴旋转 | 是 |
| 改变 005 开合时，相机是否随之移动 | 否 |
| 第一版抓取期间是否固定 `004=1500` | 是，程序在 OPEN 前主动锁定 |

相机位于 004 定子侧，TCP 位于 004 转子/005 侧，两者不是同一刚体。因此只有
`004=1500` 时能使用本轮标定的常量 `T_T_C`。未来允许 004 转动时，变换链必须改为：

```text
T_B_C = T_B_S4(q0..q3) * T_S4_C
T_B_T = T_B_S4(q0..q3) * T_S4_R4(q4) * T_R4_T
```

其中 `S4` 是 004 定子坐标，`R4` 是 004 转子坐标。005 只改变开口宽度，
第一版把对称夹爪中心视为不随 005 改变。

## 第三组：相机到 TCP 的人工初值

把 RGB 镜头光心在工具坐标 `T` 中的位置写成：

| 量 | 含义 | 实测值 |
| --- | --- | ---: |
| `t_x` | RGB 光心相对 TCP 沿 `+X_T` 的距离 | -60 mm（RGB 在闭合 TCP 后方） |
| `t_y` | RGB 光心相对 TCP 沿 `+Y_T` 的距离 | 32mm |
| `t_z` | RGB 光心相对闭合 TCP 的物理竖直高度差 | +13 mm（即以 RGB 光心为原点，闭合 TCP 竖直向下为 -13 mm） |

> 2026-07-20 紧急实测澄清：`-13 mm` 的原点是 RGB 光心，描述的是
> RGB 光心到闭合 TCP 的物理竖直落差；它不是 004/wrist 到 TCP 的 Z 分量，
> 也不是 RealSense 彩色光学坐标的 `+Z_C` 深度方向。旧的 `55 mm` 竖直记录由
> 本次 `13 mm` 实测取代。相机轴映射仍需填完后才能把该标量写进完整 4×4 手眼矩阵。

再记录相机方向：

| 量 | 含义 | 实测值 |
| --- | --- | ---: |
| yaw | 相机光轴相对 `+X_T` 的左右偏角 | 待填 deg |
| pitch | 相机光轴相对 `+X_T` 的上下偏角 | 待填 deg |
| roll | 图像水平线相对工具水平线的滚转角 | 待填 deg |

人工值只用于初始化 `T_T_C`；最终矩阵必须通过标定板多姿态求解。
最终结果优先直接写入配置的 `hand_eye.matrix_4x4`，避免欧拉角顺序和取值分支造成误差。

## 最终算法标定

人工尺寸完成后仍需：

1. 固定 AprilTag、ArUco 或棋盘格标定板，使其相对基座不动。
2. 机械臂在安全范围采集至少 15 个明显不同的 000--003 姿态。
3. 每个姿态保存 000--005 回读值、标定板位姿、RGB 图和对齐深度。
4. 求解并保存 `T_tool_camera_color_optical`。
5. 用未参与求解的 5 个姿态验证，统计 TCP 三维误差和图像重投影误差。
6. TCP 三维误差未达到抓取余量前，`hand_eye.calibrated` 和
   `tool_reference.calibrated` 保持 `false`。

矩阵链固定为：

```text
p_B = T_B_T(q) * T_T_C * p_C
```

其中 `T_B_T(q)` 来自连杆、零位和当前关节状态，`T_T_C` 来自最终支架手眼标定，
`p_C` 来自 D435 彩色对齐坐标下的深度反投影。
