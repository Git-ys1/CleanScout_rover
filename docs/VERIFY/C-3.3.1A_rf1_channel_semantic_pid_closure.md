# C-3.3.1A RF1 通道语义纠偏与六方向闭环收口

## 结论

本轮只处理 RF1 底层闭环，不涉及树莓派、ROS、建图或导航。

当前 RF1 通道语义已显式冻结为：

| 通道 | 轮位 | W 参数 |
| --- | --- | --- |
| CN1 | 左后 LR | a |
| CN2 | 左前 LF | b |
| CN3 | 右后 RR | c |
| CN4 | 右前 RF | d |

关键结论：左右转、横移、前进/后退由 `W,a,b,c,d` 的四轮目标符号组合决定；PI 增益只能改善每个轮子对目标速度的跟随能力，不能凭空产生转向能力。

## 本轮代码改动

- 在 `csr_board_map.h` 增加显式轮位别名：
  - `CSR_WHEEL_LR = CSR_CHANNEL_CN1`
  - `CSR_WHEEL_LF = CSR_CHANNEL_CN2`
  - `CSR_WHEEL_RR = CSR_CHANNEL_CN3`
  - `CSR_WHEEL_RF = CSR_CHANNEL_CN4`
- 在 `main.c` 中把 `g_kp/g_ki/g_kd` 的顺序注释固定为 `CN1/LR, CN2/LF, CN3/RR, CN4/RF`。
- 将 PI 恢复为四轮统一基线，不再沿用“前两个/后两个”的错误分组假设。
- 在 `csr_motor_drv.c` 中保留分通道 H 桥真值，并增加按 raw 正/负方向区分的起步补偿表。

## 当前参数

| 参数 | 当前值 | 说明 |
| --- | ---: | --- |
| `CSR_PI_KP_DEFAULT` | `1200.0` | 四轮统一 P |
| `CSR_PI_KI_DEFAULT` | `80.0` | 四轮统一 I |
| `CSR_PI_KD_DEFAULT` | `0.0` | 本轮不启用 D |
| `CSR_PI_PWM_STEP_LIMIT` | `40` | 限制闭环输出跃迁，降低抽动 |
| `CSR_EFFECTIVE_PWM_MIN` | `120` | 非零输出最小有效 PWM |
| `CSR_PI_OUTPUT_LIMIT` | `1000` | PI 输出限幅 |

驱动层保留方向相关 deadband 补偿：CN1 raw 正向、CN2 raw 负向、CN3/CN4 多数方向需要较高起步驱动；若降到 `850`，CN3/CN4 及部分反向会重新出现零速，因此本轮保留 `1000` 级起步地板。

## 六方向命令表

以下命令顺序均为 `W,CN1,CN2,CN3,CN4`，也就是 `W,LR,LF,RR,RF`。

| 动作 | 命令 |
| --- | --- |
| 前进 | `W,+v,+v,+v,+v` |
| 后退 | `W,-v,-v,-v,-v` |
| 左移 | `W,+v,-v,-v,+v` |
| 右移 | `W,-v,+v,+v,-v` |
| 左转 | `W,-v,-v,+v,+v` |
| 右转 | `W,+v,+v,-v,-v` |

## 本轮判断

- 六方向的目标符号与编码器反馈方向已经对齐，`W` 层不再是“只能前进”的协议状态。
- `v=0.15` 是当前最可用的调试速度档，四轮反馈基本覆盖六方向。
- `v=0.05` 能跑但低速仍有部分轮过冲或停转均值维持现象；这是 deadband 和起步补偿导致的低速 bang-bang，不是通道语义错误。
- `v=0.25` 高速下右侧负向偏快、左后/左前负向偏慢更明显；如果现场物理表现仍偏航，应优先做左右侧差分：`CN1/CN2 = left`，`CN3/CN4 = right`。
- 若左右侧平衡后横移/旋转仍前后不均，再做前后轮差分：`CN1/CN3 = rear`，`CN2/CN4 = front`。

## 下一步建议

1. 现场补充架空六方向物理观察，确认串口符号与实际轮向一致。
2. 落地只先测 `v=0.05` 前进、后退、左转、右转；横移短测，避免地面摩擦误判。
3. 若前进仍偏航，下一轮只做左右侧差分增益，不再把 `[CN1,CN2]` 误称为前轮组。
