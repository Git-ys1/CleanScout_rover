## 5. Result Classification

### Conclusion A

`vendor_claw_smoketest.py` moves the claw.

Interpretation:

- vendor PWM path is alive
- current issue is still above the raw PWM layer
- next round should return to `claw_runtime.py` / `main.py` call path inspection

### Conclusion B

`vendor_claw_smoketest.py` and `vendor_pwm_sweep.py` both fail to move the claw.

Interpretation:

- issue has escalated beyond Python abstraction logic
- suspected area becomes:
  - board-level `B10` route
  - claw servo wiring / power
  - vendor firmware / board mapping
  - IO multiplexing conflict

Final conclusion for this round:

- `PENDING HARDWARE RESULT`

- Is there any likely IO mux conflict on `MENGFEI_OPENMV4-STM32H7xx`?
