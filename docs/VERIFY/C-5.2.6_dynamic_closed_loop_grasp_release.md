# C-5.2.6 动态闭环抓取软件重构归档

## 版本定位

C-5.2.6 将 `OrangePi/rk3588_ai/arm_grasp_pipeline/` 整理为非 ROS 的动态
wrist-camera 抓取软件基线，保留 RKNN YOLO11、D435 RGB-D 对齐、总线舵机 ASCII
协议、PRAD 回读、dry-run、Servo004=1500 和阶段门控。

本版本只归档已经形成的软件、配置、测试、调参工具和操作文档；未新增实机抓取成功
结论。后续实机工作必须从遗留门禁继续，不能把模拟通过写成物理抓取通过。

## 已归档内容

- 以实时 Servo000～003 PRAD 经 FK 得到 `T_base_wrist`，再计算
  `T_base_camera = T_base_wrist × T_wrist_camera`。
- 区分 wrist、RGB optical camera、open TCP 和 closed TCP；最终规划使用
  closed TCP。
- 记录 L3=190 mm、wrist→closed TCP=135 mm、RGB 原点到 closed TCP 的物理
  竖直差 13 mm、基座平面到桌面 120 mm。
- PREGRASP 后强制新观测；APPROACH 每步 5～10 mm，并重新读取 RGB-D、目标关联和
  六轴 PWM。
- CLOSE、验证小抬升和完整 LIFT 分离；命令写入或 PRAD 到位不等同于抓取成功。
- 提供只读在线偏移调参器、配置迁移、自动测试、示例 JSONL、README 和分阶段命令。
- 旧 fixed-view 与应急 bottle demo 保留作历史诊断，但不属于默认真实路径。

## 发布前软件验证

在 Windows 开发机、无硬件访问条件下运行：

```text
python -m pytest -q --disable-warnings
119 passed
```

测试覆盖动态坐标链、wrist FK、hand-eye、open/closed TCP、目标跟踪、深度质量、
补偿顺序、新帧门禁、严格 PRAD、Servo004/005 相机不变性、状态机阶段截断、抓取验证、
旧模式隔离和调参工具。警告主要来自 NumPy matrix 与显式 legacy fixed-view 测试，
不影响本次退出码。

## 遗留问题

1. Servo005 最大安全闭合 PWM、闭合/张开内宽、标准 bottle 夹持直径和重量仍需实测；
   当前 `gripper_close_calibrated=false`，真实 CLOSE/VERIFY/LIFT 必须拒绝。
2. open TCP 仍是暂存几何，`tool_tcp.open_calibrated=false`；正式抓取继续只使用
   closed TCP。
3. 当前 hand-eye 旋转使用既有轴映射并通过三姿态静态目标验证；仍建议补直接 RPY
   测量和至少五个独立验证姿态。
4. C-5.2.6 未完成连续 10 次、成功至少 8 次的实物抓取验收；后续必须用独立日志
   记录每次失败分类。
5. `COARSE_CENTER`、多帧 DEPTH_LOCK 与 `RETRY_BACKOFF` 已进入代码/config 基线；
   retry 默认关闭，启用前仍需专项仿真和分阶段实机验收。

仍需测量的精确定义见
`OrangePi/rk3588_ai/arm_grasp_pipeline/KNOWN_PHYSICAL_VALUES_TO_MEASURE.md`。
