# C-3.0.8：CN1/CN3 编码器链路专项排查记录

## 1. 本轮目标

本轮从 `C-3.0.7` 诊断基线继续，只排查 `CN1 / CN3` 编码器链路。

本轮不改：

- `W / M / E / D / STOP` 协议语义
- PI 参数
- 树莓派 / ROS / 运动学
- 已冻结的开环方向真值表

## 2. 代码诊断增强

`R,<ch>` 寄存器诊断返回格式升级为：

```text
REG,<ch>,<MAPR_RAW>,<MAPR_EFFECTIVE>,<SMCR>,<CCMR1>,<CCER>,<CNT>,<CFG_A>,<CFG_B>,<LV_A>,<LV_B>,<GPIOA_CRL>,<GPIOA_CRH>,<GPIOA_IDR>,<GPIOB_CRL>,<GPIOB_CRH>,<GPIOB_IDR>
```

字段说明：

- `MAPR_RAW`：直接读回的 `AFIO->MAPR`，仅作参考。
- `MAPR_EFFECTIVE`：自写层影子配置，作为本轮 SWJ/TIM2 remap 写入真值。
- `CFG_A / CFG_B`：目标通道 A/B 输入脚的 GPIO 配置 nibble。
- `LV_A / LV_B`：目标通道 A/B 输入脚当前电平。
- `GPIOx_CRL / GPIOx_CRH / GPIOx_IDR`：用于核查整组 GPIO 配置与输入状态。

诊断口径：

- `CFG=4` 表示 STM32F1 输入浮空模式。
- `MAPR_EFFECTIVE=02000300` 表示自写层已写入 `JTAG disabled + SWD enabled + TIM2 full remap`。
- `MAPR_RAW=04000300` 不再作为 SWJ 唯一判据，因 STM32F1 `MAPR.SWJ_CFG` 读回可能误导。

## 3. 实测结果

### 3.1 全局 remap

```text
R,0
ACK:R
REG,0,04000300,02000300,0000,0000,0000,0000,0,0,0,0,44444B44,48833443,A68F,44484444,44444444,FF5B
```

结论：

- `MAPR_EFFECTIVE=02000300`，自写层已写入 `SWJ_CFG=010` 与 `TIM2_REMAP=full remap`。
- `MAPR_RAW=04000300` 仅作为读回参考，不用于否定当前 SWJ 配置。

### 3.2 CN1

短时驱动：

```text
M,1,700
D,1
DBG,1,1,1,0,0,0
R,1
REG,1,04000300,02000300,0003,0101,0001,0000,4,4,1,1,44444B44,48833443,A68F,44484444,44444444,FF5B
E,1
ENC,1,0,0
```

结论：

- `CFG_A=4 / CFG_B=4`，`PA0 / PA1` 已处于输入浮空模式。
- `LV_A=1 / LV_B=1`，短时驱动后电平仍固定。
- `TIM5` 已进入编码器模式：`SMCR=0003`，`CCMR1=0101`，`CCER=0001`。
- `CNT=0`，`ENC=0`。

### 3.3 CN3

短时驱动：

```text
M,3,700
D,3
DBG,3,1,1,0,0,0
R,3
REG,3,04000300,02000300,0003,0101,0001,0000,4,4,1,1,44444B44,48833443,B68F,44484444,44444444,FF5B
E,3
ENC,3,0,0
```

结论：

- `CFG_A=4 / CFG_B=4`，`PA15 / PB3` 已处于输入浮空模式。
- `LV_A=1 / LV_B=1`，短时驱动后电平仍固定。
- `TIM2` 已进入编码器模式：`SMCR=0003`，`CCMR1=0101`，`CCER=0001`。
- `CNT=0`，`ENC=0`。

## 4. 对照组

同一固件、同一测试方法下，`CN2 / CN4` 有有效计数。

### 4.1 CN2

```text
M,2,700
D,2
DBG,2,0,0,65432,1672,159
R,2
REG,2,04000300,02000300,0003,01A1,0001,FF92,4,4,0,0,44444B44,48833443,AE0F,44484444,44444444,FF5B
E,2
ENC,2,4987,167
```

### 4.2 CN4

```text
M,4,700
D,4
DBG,4,1,0,87,1405,133
R,4
REG,4,04000300,02000300,0003,01A1,0001,005D,4,4,0,0,44444B44,48833443,A6CF,44484444,44444444,FF1B
E,4
ENC,4,4183,141
```

结论：

- `CN2 / CN4` 的 GPIO 输入、电平变化、TIM 计数、ENC 读取链均有效。
- `CN1 / CN3` 的软件读取链不是全局失效。

## 5. 最终归因

本轮归因：`A 类：硬件链路或板级输入路径问题`。

依据：

- `CN1 / CN3` 的目标输入脚均已配置为输入浮空：`CFG=4`。
- `CN1 / CN3` 的定时器编码器寄存器均已配置：`SMCR=0003`、`CCMR1=0101`、`CCER=0001`。
- `CN1 / CN3` 的 A/B 输入电平在驱动后仍固定 `1/1`，说明信号没有进入当前 MCU 输入脚。
- `CN2 / CN4` 在同一固件下可正常计数，排除协议、公共计数读取链、控制循环全局失效。

## 6. 下一步建议

优先做硬件/板级验证：

- 确认 `CN1` 编码器 A/B 是否真实到达 `PA0 / PA1`。
- 确认 `CN3` 编码器 A/B 是否真实到达 `PA15 / PB3`。
- 用万用表或示波器在对应 MCU 输入脚附近测 A/B 相是否有跳变。
- 若客服坚持软件问题，应要求给出 OpenRF1 板上 `CN1/CN3` 编码器 A/B 到 MCU 引脚的精确原理图或网表。

如果后续确认 `PA0/PA1`、`PA15/PB3` 有波形但 `CNT` 不变，再回到 `TIM5/TIM2` 初始化层继续查。
