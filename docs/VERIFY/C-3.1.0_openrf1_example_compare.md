# C-3.1.0 OpenRF1 全例程编码器相关初始化对照

## 结论

本轮对 `docs/STM32F103RCT6/03.源代码程序/` 下全部 12 个例程做了关键词扫描，并重点对照 `y_encoder.c` 与 `app_motor.c`。

结论：

- 所有包含 `y_encoder.c` 的例程，其文件哈希一致，编码器初始化逻辑完全同源。
- vendor 编码器路径仍是：
  - `CN1 -> TIM5 + PA0/PA1`
  - `CN2 -> TIM3 + PA6/PA7`
  - `CN3 -> TIM2 full remap + PA15/PB3`
  - `CN4 -> TIM4 + PB6/PB7`
- 未在其他例程中发现针对 `CN1/CN3` 的独立编码器供电使能、接口复用使能、上拉使能或板级开关初始化。
- 多个例程会演示 `GPIO_Remap_SWJ_JTAGDisable`，但这只解释 `PA15/PB3` 释放，不解释 `CN1/TIM5/PA0/PA1` 无相位。
- 当前自写层与 vendor 可见源码在定时器和 GPIO 映射层面一致；`CN1/CN3` 当前更应按板级输入链、输入电气条件或实际接插件链路继续排查。

## vendor 关键源码对照

### `y_encoder.c`

- `Encoder_Init_TIM5()`：
  - 开启 `RCC_APB1Periph_TIM5`
  - 配置 `GPIOA Pin0 / Pin1`
  - 输入模式为 `GPIO_Mode_IN_FLOATING`
  - `TIM_EncoderInterfaceConfig(TIM5, TIM_EncoderMode_TI12, Rising, Rising)`
  - `TIM_ICFilter = 10`
  - 读取 `TIM5->CNT`

- `Encoder_Init_TIM2()`：
  - 开启 `RCC_APB1Periph_TIM2`
  - 开启 `GPIOA / GPIOB / AFIO`
  - 执行 `GPIO_PinRemapConfig(GPIO_FullRemap_TIM2, ENABLE)`
  - 配置 `PA15 / PB3`
  - 输入模式为 `GPIO_Mode_IN_FLOATING`
  - `TIM_EncoderInterfaceConfig(TIM2, TIM_EncoderMode_TI12, Rising, Rising)`
  - `TIM_ICFilter = 10`
  - 读取 `TIM2->CNT`

### `app_motor.c`

- vendor 的 `Wheel_C.RT` 来自 `ENCODER_C_GetCounter()`，即 `TIM5`。
- vendor 的 `Wheel_D.RT` 来自 `ENCODER_D_GetCounter()`，即 `TIM2`。
- `app_motor.c` 没有额外为 `TIM5/PA0/PA1` 或 `TIM2/PA15/PB3` 做板级供电、上拉或复用开关初始化。

## 对当前自写层的意义

- `C-3.1.0` 前的自写层没有明显遗漏 vendor 可见源码中的 TIM/GPIO 初始化步骤。
- 当前新增 `X,<ch>,<input>,<mode>,<filter>` 是为验证“vendor 的 FLOATING/TI12/filter10 是否不适配本板当前 CN1/CN3 实物链路”。
- 如果 `FLOATING/IPU/IPD` 都没有原始相位变化，问题不应继续停留在 TIM 参数层，应转入板级输入路径、接插件、编码器供电、线序或网表核查。
