## 目标与规划结果

流程使用 aligned depth 的目标框内部 ROI 中位深度，不使用检测框面积估距。相机前表面点
先变换到 `arm_base`，再沿水平接近轴补偿瓶子半径 `0.032 m`，得到瓶心。PRE_GRASP
位于瓶心前方 `0.070 m`，APPROACH 以 `0.010 m` 间距生成水平直线路点。

当前实测瓶子位置的一次有效结果约为：

```text
point_base_surface_m = [0.2140, -0.0120, 0.1454]
bottle_center_base_m = [0.2460, -0.0138, 0.1454]
pre_grasp_base_m     = [0.1761, -0.0099, 0.1454]
```

`pitch_deg=20` 时该水平路径不可达；数值可达性扫描后使用 `pitch_deg=35`，完成：

```text
OPEN -> PRE_GRASP -> 7 个 APPROACH 路点 -> CLOSE -> LIFT
```

所有阶段通过 workspace、IK、PWM 边界检查，Servo004 始终为 `1500`。LIFT 阶段
Servo002 规划 PWM 为 `2181`，距离当前上限 `2200` 仅余 `19`，因此真实测试时必须重点
观察该关节负载，不能直接跳过 PRE_GRASP 验收执行完整抓取。

## 本机连杆、TCP 与舵机角度标定

这不是继续为某个瓶子手调 PWM，而是让通用 IK 与实物一致：

- 连杆：相邻关节转轴之间的真实距离。
- TCP：两侧夹爪闭合时的实际抓取中心，不是相机中心，也不是舵机外壳中心。
- 舵机角度映射：每个关节的 PWM 增大方向、物理零位、PWM/角度比例和安全机械边界。

IK 先算物理关节角，再由舵机映射转成 PWM。相机 XYZ 即使正确，以上参数错误仍会让
夹爪偏离瓶心。用户现场观察“是否到瓶前、是否碰瓶、夹爪是否全开/闭合、能否抬起”
是分阶段验收的重要证据，但不能替代关节角度映射本身。
