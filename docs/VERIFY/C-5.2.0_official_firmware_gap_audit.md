## 官方代码实际提供的能力

| 能力 | 结论 |
| --- | --- |
| `$KMS:x,y,z,time!` | 有，只求解并控制 000--003 |
| 坐标顺序 | `x=横向`、`y=前向`、`z=离地高度`，单位 mm |
| 连杆参数 | `L0/L1/L2/L3 = 100/105/88/155 mm` |
| 末端俯仰 | 上位机不可指定；固件扫描 `Alpha=0..-135 deg` 并选最负可行值 |
| 004 腕部 | 不属于 `$KMS`，必须单独命令 |
| 005 夹爪 | 不属于 `$KMS`，必须单独命令 |
| 正运动学 | 没有 |
| TCP 定义 | 没有明确物理定义 |
| 关节编码器/零位反馈 | 没有 |
| 抓取成功反馈 | 没有 |
| 碰撞检测/路径规划 | 没有 |

## 发现的问题

1. 旧 Python 端把官方 IK 的 000--003 扩展成了六路 PWM，自行生成 004 和 005。
   这不是官方语义，已删除。
2. 旧 Python 端把自推正运动学当成当前 TCP 位姿。官方代码并不提供正运动学、
   关节零位或 TCP 定义，因此该结果现在只允许作为 dry-run 估计。
3. 旧几何层复制了参考 ROS2 工程的固定轴重排和外参种子。CleanScout 的安装结构
   不同，且 D435 深度已对齐到彩色坐标，固定轴重排和 `-7 mm` 修正均已移除。
   D435 内部 RGB/红外外参仍由 librealsense 正常使用；缺少的是厂家无法提供的
   `T_tool_camera_color_optical` 安装外参。
4. 官方全局停止分支 `pwmServo_stop_motion(255)` 读取了下标 255，存在越界风险。
   上位机继续逐路发送 `#000PDST!` 到 `#005PDST!`，不使用全局停止。
5. `$KMS` 失败只向 UART1 打印非结构化字符串，USART3 调用方拿不到可靠 ACK。
   当前 Python IK 只作前置可达性检查，后续仍需设计结构化回执。

## 本轮代码收口

- 真实运动命令改为官方 `$KMS`，且只控制 000--003。
- 004 保持当前值，005 由 OPEN/CLOSE 阶段独立控制。
- 硬件关系冻结为 D435 固定在 004 定子外壳、TCP 位于 004 转子/005 侧；
  第一版在抓取前强制 `004=1500`，只在该角度使用常量手眼矩阵。
- 真实抓取先回到完整参考姿态再采集目标；未标定动态正运动学前，不允许
  自动视觉居中与真实抓取同时运行。
- 新增 `$KMS` 坐标顺序、单位、时间范围测试。
- `HandEye` 明确定义为 `T_tool_camera_color_optical`。
- 新增 `tool_reference.calibrated` 执行锁；未标定 `T_B_T(q_ref)` 时拒绝真实抓取。
- 旧参考外参改为全零未标定占位，不再悄悄偏移目标。
- 自推正运动学重命名为 `estimate_tool_matrix_from_pwm()`，只用于 dry-run。
