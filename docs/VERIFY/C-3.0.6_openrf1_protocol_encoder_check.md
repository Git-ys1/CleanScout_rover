# C-3.0.6：OpenRF1 串口协议与编码器回传核查

## 1. 核查对象

本页记录 `C-3.0.6` 在 `_local/openrf1_keil_work_3/User` 自写运行层上的板端实测结论。

核查范围只包括：

- USART2 / COM14 串口协议
- `W` 持续控制协议
- `M / E / STOP` 调试协议
- CN1~CN4 编码器回传状态

本页不重新验证 `C-3.0.5` 开环方向真值表。

## 2. 已验证成立项

串口链路已成立：

- 上电可见 `CSR_RF1_READY`
- `STOP` 返回 `ACK:STOP`
- `M,<ch>,<pwm>` 返回 `ACK:M`
- `E,<ch>` 返回 `ACK:E` 与 `ENC,<ch>,<count>,<delta>`
- `W,a,b,c,d` 返回 `ACK:W`
- `VEL,...` 与 `PWM,...` 周期输出存在
- USART2 已改为 RX 中断环形缓冲，持续刷新 `W` 时不再因阻塞式遥测发送丢字节

持续控制已成立：

- `W` 命令会锁存四轮目标速度
- 控制循环按 `20ms / 50Hz` 执行
- 遥测按 `100ms / 10Hz` 输出
- 停止刷新 `W` 后，约 `250ms` 超时停车

## 3. 编码器实测状态

| 通道 | 编码器资源 | 当前回传状态 | 结论 |
| --- | --- | --- | --- |
| `CN1` | `TIM5 + PA0/PA1` | `ENC,1,0,0` | 当前无计数 |
| `CN2` | `TIM3 + PA6/PA7` | 有计数，`+pwm` 时速度为负 | 编码器链有效，需符号反相 |
| `CN3` | `TIM2 full remap + PA15/PB3` | `ENC,3,0,0` | 当前无计数 |
| `CN4` | `TIM4 + PB6/PB7` | 有计数，C-3.0.6 复测确认当前符号表下仍需保持 `+1` | 编码器链有效，当前不反相 |

新增诊断命令：

```text
D,<ch>
```

返回：

```text
DBG,<ch>,<phaseA>,<phaseB>,<timerCount>,<count>,<delta>
```

使用方式：

- 对 CN1/CN3，手拨轮或短时 `M` 驱动时连续发送 `D,1` / `D,3`
- 若 `phaseA/phaseB` 始终不变，优先查编码器 A/B 相线、供电、接插件
- 若 `phaseA/phaseB` 会变但 `timerCount` 不变，再回到 TIM5/TIM2 remap 配置排查
- 若 `timerCount` 会变但 `ENC` 不变，再查 `read_and_reset` 和符号回填

## 4. 当前代码回填

`_local/openrf1_keil_work_3/User/csr_board_map.c` 当前冻结为：

```c
int8_t g_csr_motor_dir_sign[CSR_CHANNEL_COUNT] = { 1, 1, 1, 1 };
int8_t g_csr_encoder_dir_sign[CSR_CHANNEL_COUNT] = { 1, -1, 1, 1 };
```

原因：

- `C-3.0.5` 已经把 `M` 原始调试命令统一成 `+pwm` 正向，因此 `W` 层不应再对电机输出做二次翻转。
- CN2 的编码器回传在 `+pwm` 下为负，因此闭环层先对 CN2 编码器符号反相。
- CN4 在本轮复测中确认不应继续反相，当前保持 `+1`。
- CN1/CN3 当前是“无计数”，不是“方向反了”；符号表不能修复无计数问题。

## 5. 与 vendor 初始化对照结论

对照 vendor 例程 `9.控制霍尔编码器电机/User/Components/y_encoder/y_encoder.c` 后，当前自写层未发现 CN1/CN3 软件映射上的明显缺项：

- CN1 仍为 `TIM5 + PA0/PA1`
- CN3 仍为 `TIM2 full remap + PA15/PB3`
- JTAG 已在 `main()` 最早期禁用，SWD 保留

因此，当前阶段不应继续怀疑 `W` 协议或串口链路。下一步应优先查：

- CN1/CN3 编码器线序
- CN1/CN3 接插件与焊点
- 编码器供电与地线
- OpenRF1 板级对 CN1/CN3 的专有差异
- 是否存在电机能转但编码器 A/B 相未接入的情况

## 6. CN2/CN4 闭环低速结论

本轮对 CN2/CN4 做了低速 `W` 单轮闭环测试：

- 加入速度低通后，`VEL` 不再瞬时大幅跳变
- 禁止反向刹车后，低速目标下不再出现明显前后反向抽动
- `0.020 m/s` 目标目前过低，起步后 `VEL` 很快高于目标，控制器会将 `PWM` 拉回 `0`

结论：

- CN2/CN4 的闭环链路已经可观测
- 当前还不能认为低速连续运行已经完成
- 下一步需要提高测试目标或加入更明确的起步保持/死区补偿策略

## 7. 下一步边界

在 CN1/CN3 编码器恢复计数前：

- 不应宣布“四轮闭环完成”
- 不应继续把 PI 参数调大来掩盖无反馈问题
- 不应重新推翻 `C-3.0.5` 开环真值
- 可以继续使用 `M` 做开环调试
- 可以针对 CN2/CN4 做闭环参数验证

## 8. 本轮冻结口径

用户本轮最新实测结论：

- `CN2 / CN4` 当前闭环表现“还算不错”，能向前转，但低速仍有顿挫。
- `CN1 / CN3` 电机能输出，但因为没有编码器回传，会一口气冲出，无法被闭环调节。

因此 `C-3.0.6` 发布口径固定为：

- `W` 持续控制协议已打通。
- 两轮闭环阶段性成立：`CN2 / CN4`。
- `CN1 / CN3` 不再归类为协议问题或电机开环问题，而是编码器反馈链路问题。
- 下一轮必须优先查 `CN1 / CN3` 的 A/B 相输入、TIM5/TIM2 计数与板级初始化差异。
