# C-1.2.5 J 线执行基线测试报告

## 1. 结论

`C-1.2.5` 已完成 `J` 线执行基线冻结。当前 `J` 线后续应以
`C-1.2.5_vendor_actuator_baseline` 作为三维执行控制基线，而不是继续在旧抓取/通信热修线上叠补丁。

本轮已确认：

- 云台左右动作链正常
- 云台上下动作链正常
- 夹爪张开/闭合动作链正常
- `MODE_DEMO_LOOP` 可稳定循环执行
- 输入层已与蓝牙硬件解耦，后续可由 `F411` 替换命令来源

## 2. 基线来源

- Vendor 模板来源：
  `J-jixiebi/introduction and examples/07 梦飞openmv4_大底座视觉识别追踪云台例程/07 蓝牙&wifi图传app控制AI视觉追踪综合案例`
- 运行时执行器：
  - `Servo(3)`：PAN
  - `Servo(4)`：TILT
  - `Servo(1)`：CLAW

## 3. 空间坐标口径

以板面、摄像头朝向车头正方向为前：

- `PAN +` = 左
- `PAN -` = 右
- `TILT 0` = 上方边界
- `TILT -90` = 下方边界（机头朝地）

## 4. 冻结边界

### 4.1 PAN

- 左边界：`+90`
- 右边界：`-90`
- 默认 / 回中：`0`

### 4.2 TILT

- 上边界：`0`
- 下边界：`-90`
- 默认 / 回中：`-85`

说明：

- `-90` 已接近或超出舒适运行范围，但保留为下边界参考
- 本轮默认姿态冻结为 `-85`，作为后续长期开发的统一默认位

### 4.3 CLAW

- 张开边界：`-60`
- 闭合边界：`40`

实测结论：

- `-60` 为完全张开状态
- `40` 可对约 `0.6 cm` 物体形成稳定夹持
- 高于 `40` 的更大闭合值会导致舵机发热风险上升，因此不继续上探

## 5. Demo Loop 验证结果

已验证 `MODE_DEMO_LOOP` 可正常运行：

1. `CENTER`
2. `TILT_UP x5`
3. `TILT_DOWN x10`
4. `CENTER`
5. `PAN_LEFT x5`
6. `PAN_RIGHT x10`
7. `CENTER`
8. `CLAW_CLOSE`
9. `CLAW_OPEN`

现场结果：

- 左右正常
- 上下正常
- 夹爪张/合正常
- 回中正常
- 循环稳定

当前应冻结为：

- `PAN_CENTER_DEG = 0`

- `PAN_MAX_DEG = 90`
- `TILT_CENTER_DEG = -85`
- `TILT_MIN_DEG = -90`
- `TILT_MAX_DEG = 0`
- `CLAW_OPEN_ANGLE = -60`
- `CLAW_CLOSE_ANGLE = 40`

## 7. 对后续任务的意义

本轮之后，`J` 线执行层已具备稳定基线。后续可以在此基础上继续做：

- `F411` 接管命令源

但这些都应建立在本轮冻结的执行边界之上，不再回到旧 `C-1.2.0_color_grasp_uart_poc` 热修线继续堆补丁。
