# C-3.3.1D：RF1 全正语义统一相位基线冻结

## 结论

本轮冻结的不是“最终闭环完美版”，而是当前可回退、可继续整定的统一基线：

- `g_csr_motor_dir_sign[4]` 固定为 `{1, 1, 1, 1}`
- 不再保留 `CN1/CN3 = -1` 这类控制层符号翻转
- `csr_motor_drv.c` 将四路电机的底层相位语义统一为：
  - `+pwm -> IN1=SET`
  - `-pwm -> IN1=RESET`
- 删除此前用于补偿不同相位的 `reset_phase_offset` 近似归一化逻辑
- 本轮不继续引入新的起步补偿、分段地板或按轮差异化 PI 热修

这版的目标只有一个：先把“四轮统一正向语义”和“底层相位统一”冻结下来，作为后续继续调同步的安全基线。

## 变更摘要

### 1. 控制层语义收口

当前板级轮位继续固定为：

- `CN1 = 左后`
- `CN2 = 左前`
- `CN3 = 右后`
- `CN4 = 右前`

当前控制层的电机方向表已收口为：

```c
int8_t g_csr_motor_dir_sign[CSR_CHANNEL_COUNT] = { 1, 1, 1, 1 };
```

也就是：

- `W,a,b,c,d` 的四个目标值不再通过 `motor_dir_sign` 在控制层被拆成正负两组
- “前进”语义就是四轮同号目标，不再依赖 `CN1/CN3` 取负来实现

### 2. 驱动层相位统一

本轮之前，驱动层存在“正向两套底层相位”的问题：

- 一部分通道正向走 `IN1=SET`
- 一部分通道正向走 `IN1=RESET`
- `IN1=RESET` 路径还叠加了额外的近似换算

这会导致：

- 同一个 signed PWM 落到不同通道时，对应的有效驱动并不一致
- 低速起转和中速同步表现明显分裂

本轮将其收口为统一相位模型：

- `+pwm -> IN1=SET`
- `-pwm -> IN1=RESET`

同时移除了：

- `reset_phase_offset = 1100`
- 基于 `Bit_RESET` 的额外“通道归一化”近似补偿

### 3. 本轮明确没有做的事

本轮刻意不做下面这些事：

- 不新增按轮起步补偿
- 不恢复分段地板
- 不继续按轮拆分 PI
- 不改协议 `W/M/E/D/STOP`
- 不改树莓派桥接

原因很简单：这轮目标是先冻结“统一语义”的底层，不再一边调驱动一边加额外补丁。

## 串口回归现象

本轮冻结时，使用 COM14 对当前板上镜像做了全正回归。

### 低速全正

测试命令：

```powershell
.\tools\openrf1_serial_probe.ps1 -WheelA 0.05 -WheelB 0.05 -WheelC 0.05 -WheelD 0.05
```

实际串口回传样例：

```text
VEL,-0.205,0.080,0.056,-0.247,0.050,0.050,0.050,0.050
PWM,250,110,126,250
```

观察结论：

- 当前全正低速下，`CN1/CN4` 的反馈符号仍为负，`CN2/CN3` 为正
- 说明“统一相位驱动”已经成立，但编码器/轮向整体一致性还未完全收口
- 低速已经可以持续转动，不再回到更早那种明显分裂卡顿
- 但四轮同步仍未完全达成，当前更像是“统一语义已建立，但符号/同步细节还待下一轮继续热修”

### 中速全正

测试命令：

```powershell
.\tools\openrf1_serial_probe.ps1 -WheelA 0.15 -WheelB 0.15 -WheelC 0.15 -WheelD 0.15
```

实际串口回传样例：

```text
VEL,-0.515,0.300,0.268,-0.551,0.150,0.150,0.150,0.150
PWM,510,289,310,510
```

观察结论：

- 四轮已经进入同一套全正语义控制，不再存在旧版那种 `CN1/CN3` 必须走控制层负号翻转的结构
- 但 `CN1/CN4` 仍明显偏强，`CN2/CN3` 相对偏弱
- 当前不同步的责任边界已进一步缩小到：
  - 编码器符号/测速一致性
  - 统一驱动后的轮间机械差异
  - 统一 PI 下的残余前馈与积分收口

## 当前归因

截至 C-3.3.1D，本轮已经可以把“不同步”的责任边界进一步收窄：

已排除：

- 控制层 `CN1/CN3 = -1` 的旧符号拆分
- 驱动层 `SET/RESET` 双正向路径
- `reset_phase_offset` 这类额外近似补偿

仍待后续继续整定的方向：

- 低速下个别轮的起转差异
- 当前统一 PI 在不同轮上的残余机械差异
- 编码器测速与前馈/积分之间的同步收口

## 适用结论

本轮可以正式作为 `C-3.3.1D` 冻结发布，但要明确其定位：

- 这是“统一正向语义”的 RF1 底层基线
- 不是“同步问题已彻底解决”的最终版
- 它适合继续作为后续闭环同步热修的安全起点
- 若后续新热修走偏，可直接回退到：
  - `39c481f`
  - 或本轮 `C-3.3.1D` 发布提交
