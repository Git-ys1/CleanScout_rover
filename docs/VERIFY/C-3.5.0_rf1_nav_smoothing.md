# C-3.5.0 RF1 导航低速平滑与防超调

## 当前目标

本轮只改 RF1 下位机速度执行链，不改树莓派、ROS、轮序、方向表，也不改变 `W/M/E/D/STOP` 协议语义。目标是让导航低速命令进入下位机后保持连续，避免小速度命令被瞬间放大成大 PWM，降低直线冲出、转弯过头和横移卡住后猛冲的概率。

## 代码改动

| 项目 | 当前值/行为 | 说明 |
|---|---|---|
| 串口命令目标 | `g_target_vel_cmd[4]` | `W,a,b,c,d` 收到的原始轮速目标 |
| PI 实际目标 | `g_target_vel_ramp[4]` | 控制周期内平滑追踪命令目标 |
| 目标轮速加速度 | `CSR_WHEEL_ACC_LIMIT_MPS2 = 0.20f` | 每 `20ms` 最大变化 `0.004m/s` |
| PWM 斜坡 | `CSR_PWM_STEP_LIMIT_NAV = 10` | 闭环输出每 `20ms` 最多变化 10 PWM |
| PI reset | 取消 `abs(delta)>0.08` 触发 | 常规导航微调不再反复清 PI |
| anti-windup | 条件积分 | 饱和且误差继续推向饱和时不继续积分 |
| watchdog | `CSR_W_COMMAND_TIMEOUT_MS = 250ms` | 保持通信丢失快速停车 |

## 遥测格式

保留原有格式，避免破坏树莓派侧解析：

```text
VEL,rt1,rt2,rt3,rt4,tg1,tg2,tg3,tg4
PWM,p1,p2,p3,p4
```

新增 10Hz 诊断遥测：

```text
NAVDBG,cmd1,cmd2,cmd3,cmd4,ramp1,ramp2,ramp3,ramp4,rt1,rt2,rt3,rt4,ff1,ff2,ff3,ff4,corr1,corr2,corr3,corr4,out1,out2,out3,out4,satMask
```

字段含义：

| 字段 | 说明 |
|---|---|
| `cmd*` | 上位机 `W` 命令目标轮速 |
| `ramp*` | 下位机实际送入 PI 的平滑目标 |
| `rt*` | 实测轮速 |
| `ff*` | 当前 feedforward PWM |
| `corr*` | 当前 PI correction |
| `out*` | 经过 PWM slew 后实际下发的 signed PWM |
| `satMask` | bit0~bit3 对应 CN1~CN4 是否出现输出饱和 |

## 建议测试命令

测试前先架空，再落地短测。每组连续发送 1~2 秒，结束 `STOP`。

| 动作 | 命令示例 |
|---|---|
| 前进 | `W,0.05,0.05,0.05,0.05` |
| 后退 | `W,-0.05,-0.05,-0.05,-0.05` |
| 左移 | `W,0.05,-0.05,-0.05,0.05` |
| 右移 | `W,-0.05,0.05,0.05,-0.05` |
| 左转 | `W,-0.05,-0.05,0.05,0.05` |
| 右转 | `W,0.05,0.05,-0.05,-0.05` |

速度建议按 `0.03 -> 0.05 -> 0.08 -> 0.10m/s` 逐步升高。横移优先观察是否从“卡住后猛冲”变成“缓慢爬升”。

## 验收重点

1. `NAVDBG` 中 `ramp` 应按每 tick 约 `0.004m/s` 追踪 `cmd`。
2. `PWM` 每个控制周期最大变化不超过 10。
3. `satMask` 不应长期满位；若长期饱和，说明目标速度、摩擦或 feedforward 参数仍需下一轮处理。
4. 小速度导航不应再出现突然大 PWM 起步。
5. 本轮不做每轮每方向 `KS/KV` 标定；该项留作下一轮精细化。
