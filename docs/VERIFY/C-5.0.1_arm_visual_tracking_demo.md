## 关键判断

当前机械臂官方 STM32 基线默认不是 ROS2 `0x90` 二进制帧，而是 PWM 舵机文本命令：

```text
#000P1500T0200!
#000PDST!
#003PDST!
```

注意：官方固件 `$DST!` 会进入 `pwmServo_stop_motion(255)`，当前源码中该分支疑似存在 `pwmServo_angle[255]` 越界风险。因此 demo 默认只对 Servo0/Servo3 发单舵机停止命令，等固件确认后再考虑全停止。

因此 C-5.0.1 demo 默认采用 `yh_pwm_text` 后端。ROS2 参考项目中的 `0x80/0x90` 帧仅作为可选兼容后端保留，不默认启用。

## 当前限制

- 当前 OrangePi 能看到 `/dev/video0` USB 摄像头。
- 当前未看到 `/dev/ttyUSB*` 或 `/dev/ttyACM*`。
- 当前只看到 `/dev/ttyS7`。
- 当前 `~/rk3588_ai/rknn_lite_env` 缺 `pyserial`。
- 因此真实机械臂输出需要下一轮接线与环境确认。

## 验证命令

```bash
cd ~/rk3588_ai/arm_tracking_demo
~/rk3588_ai/rknn_lite_env/bin/python3 -m py_compile \
  arm_driver.py target_selector.py visual_servo.py yolo_arm_track.py \
  tools/scan_serial.py tools/test_arm_driver_dryrun.py \
  tools/test_one_joint_yaw.py tools/test_one_joint_pitch.py
```

```bash
~/rk3588_ai/rknn_lite_env/bin/python3 tools/scan_serial.py
~/rk3588_ai/rknn_lite_env/bin/python3 tools/test_arm_driver_dryrun.py --print_cmd
```

实际板端结果：

- `py_compile` 通过。
- `tools/scan_serial.py` 只看到 `/dev/ttyS7`。
- `pyserial` 当前不可用：`No module named 'serial'`。
- `test_arm_driver_dryrun.py` 输出 `#000P1500T0200!#003P1500T0200!`，并分别输出 `#000PDST!`、`#003PDST!`。
- `yolo_arm_track.py --no_show --max_frames 5` 成功打开 `/dev/video0`、加载 RKNN、完成推理并释放资源。
