## `X` 命令定义

`X` 是本轮新增的诊断命令，不改变 `W/M/E/D/STOP` 既有语义。

格式：

```text
X,<ch>,<input>,<mode>,<filter>
```

字段：

- `ch`：只允许 `1` 或 `3`
- `input`：`F` / `IPU` / `IPD`
- `mode`：`TI12` / `TI1` / `TI2`
- `filter`：`0` / `10`

行为：

- 先停车
- 只重初始化目标编码器通道
- 清零目标通道计数
- 返回 `ACK:X`

## EXTI 补测：外部中断边沿探测

本轮追加了纯诊断命令：

```text
Y,<ch>,<input>
Q,<ch>
```

用途：

- `Y,<ch>,<input>`：停车后把 `CN1/CN3` 的 A/B 相切到 EXTI 双边沿中断探测。
- `Q,<ch>`：读取 EXTI A/B 边沿计数与当前相位。
- 该路径不使用 TIM 编码器接口，不参与闭环，只验证 MCU 对应 GPIO/EXTI 是否能看到边沿。

回传格式：

```text
EXTI,<ch>,<edgeA>,<edgeB>,<phaseA>,<phaseB>,<pending>
```

### 静止状态

```text
Y,1,IPU -> ACK:Y
Q,1 -> EXTI,1,0,0,1,1,00000000

Y,3,IPU -> ACK:Y
Q,3 -> EXTI,3,0,0,1,1,00000000
```

### 正向带动

测试方式：

```text
Y,1,IPU
M,1,700
Q,1
STOP

Y,3,IPU
M,3,700
Q,3
STOP
```

结果：

```text
M,1,700 -> PWM,700,0,0,0
Q,1 -> EXTI,1,0,0,1,1,00000000

M,3,700 -> PWM,0,0,700,0
Q,3 -> EXTI,3,0,0,1,1,00000000
```

### 反向带动

测试方式：

```text
Y,1,IPU
M,1,-700
Q,1
STOP

Y,3,IPU
M,3,-700
Q,3
STOP
```

结果：

```text
M,1,-700 -> PWM,-700,0,0,0
Q,1 -> EXTI,1,0,0,1,1,00000000

M,3,-700 -> PWM,0,0,-700,0
Q,3 -> EXTI,3,0,0,1,1,00000000
```

判定：

- `Y/Q` 诊断命令可用，说明 EXTI 诊断固件已运行。
- `M` 命令下 PWM 已实际下发。
- CN1/CN3 在正向和反向带动下，EXTI A/B 边沿计数仍为 `0/0`。
- 这进一步排除“只是 TIM 编码器模式不计数”的可能，优先指向端子到 MCU 输入脚之间没有可见边沿。

## 当前唯一归因

本轮结果满足判定 B：

> GPIO 原始相位在 `FLOATING / IPU / IPD` 下始终不恢复。

因此，当前不再优先怀疑：

- `W/M/E/D/STOP` 协议
- PI 参数
- 编码器读取函数
- TIM `TI12/TI1/TI2` 模式选择
- `TIM_ICFilter=10`
- JTAG 未释放

当前优先归因为：

```text
A 类：硬件链路 / 板级输入路径 / 接插件或线序 / 编码器供电地 / 读错脚
```

EXTI 补测后，归因进一步收紧为：

```text
CN1/CN3 的 A/B 相在 MCU 对应 GPIO/EXTI 层也没有边沿。
当前不应继续调 TIM 编码器模式、滤波或 PI；下一刀应查端子信号、导通、供电地、线序和实际 MCU 引脚路径。
```

## 下一步建议

下一轮不要继续调 TIM 参数，优先做硬件链路验证：

- 万用表测 `CN1.2/CN1.5`、`CN3.2/CN3.5` 是否有编码器 5V/GND。
- 测 `CN1.3/CN1.4` 与 MCU `PA0/PA1` 的导通。
- 测 `CN3.3/CN3.4` 与 MCU `PA15/PB3` 的导通。
- 手拨轮时用示波器或逻辑分析仪看 `CN1.3/CN1.4`、`CN3.3/CN3.4` 是否有 A/B 脉冲。
- 若端子处有脉冲但 MCU 引脚无变化，按板级输入链或焊接/走线问题处理。
