## 4. 当前测试流程

1. 串口上电后确认 `CSR_RF1_READY`
2. 单通道驱动测试：
   - `M,1,300` / `M,1,-300`
   - `M,2,300` / `M,2,-300`
   - `M,3,300` / `M,3,-300`
   - `M,4,300` / `M,4,-300`
3. 每步测试后执行：
   - `STOP`
4. 编码器查询：
   - `E,1` / `E,2` / `E,3` / `E,4`
5. 将结果回填到本表，不再靠口头记忆

## 5. C-3.0.5 失败回溯说明

- 当前最小协议已经打通：
  - `CSR_RF1_READY`
  - `ACK:M`
  - `ACK:E`
  - `ACK:STOP`
- 当前问题不在串口协议，而在单通道底层真值未建立。
- `C-3.0.5B` 的施工原则是：
  - 保留这份文档作为失败证据
  - 新分支只重建单通道真值
  - 在真值完整前，不回填方向表，不恢复高层语义

- 当前自写驱动已摆脱“统一公式 + sign 表”的错误抽象，改为四路独立驱动函数。
