# C-5.1.1 D430/D435 深度抓取管线与 ROS 预留接口

## 路线修正

普通 USB 相机 / YOLO 的 2D 视觉追踪继续作为“搜索/居中阶段”。抓取阶段使用 RealSense 深度，不使用图像面积估距。

本阶段暂不接 ROS，是为了 OrangePi 上快速开发、频繁迭代和便于排障；但新增代码保留 `ros_compat.py`，后续可把 `GraspEventMsg` / `ArmTargetMsg` 转成 ROS2 或 ROS Noetic 的 topic/action/service。

## D430 今日测试边界

D430 没有 RGB，因此今天只测：

```text
D430 depth frame
→ depth scale 转米
→ depth intrinsics
→ 中心/指定 ROI 深度中位数
→ depth_pixel_to_camera()
→ camera xyz 输出
```

今天不做：YOLO RGBD 对齐、不做真实抓取、不做手眼闭环验收。

## D435 到货后的主链路

```text
YOLO color bbox
→ D435 depth aligned to color
→ bbox 中心 ROI depth median
→ pixel/depth to camera XYZ
→ 相机轴重映射
→ hand-eye
→ base frame target
→ IK
→ RF1 串口文本协议
→ 抓取状态机
```

## 树莓派 ROS2 学习包迁移点

- `dynamic_space_grab.py`：RGBD 同步、目标深度、hand-eye、抓取顺序。
- `transform.py`：`depth_pixel_to_camera()`、齐次矩阵工具。
- `arm_ik_sdk.py`：`matrix_hand_to_cam`、`move_arm()` 中的 `z -= 0.1034`、hand/roll 语义。
- `Kinematics.py`：5DoF 几何 IK；移植时修掉全局 DH 状态被正运动学累计修改的问题。

## RF4 学习包迁移点

RF4 证明同厂家总线舵机执行层可迁移到 RF1。迁移重点是：

- `parse_action()` 文本命令解析。
- `$`、`#`、`{}`、`<>` 等定界收帧能力。
- `y_servo.c` 里的舵机目标限幅、方向、执行入口。

RF1 不继承上位机 IK；RF1 只继承执行层。

## C-5.1.1A RealSense 预检补丁

新增：

```bash
~/rk3588_ai/rknn_lite_env/bin/python3 tools/realsense_env_check.py
```

用途：

- 打印当前 Python 与虚拟环境。
- 打印 `lsusb`，显式判断是否检测到 Intel/RealSense USB 设备。
- 打印 `/dev/video*`。
- 检查 `pyrealsense2` 是否可导入。
- 若运行库缺失，D430/D435 smoke 会给出明确错误提示，不再直接抛 traceback。

2026-07-09 板端复测结果：

```text
host: 192.168.8.148 / orangepi5max
python: 3.8.10
/dev/video0, /dev/video1: present
CH340 serial: present
Intel RealSense USB: not detected
pyrealsense2: missing
```

当前判断：

- 普通 RGB 摄像头仍可继续跑 C-5.0.9 追踪。
- D430 需要先插到香橙派并在 `lsusb` 中出现 Intel/RealSense 设备。
- RealSense runtime / `pyrealsense2` 缺失时，不能进入 D430/D435 深度 smoke。
