# C-3.1.4C：OpenRF1 CN1/CN3 闭环方向热修记录

## 结论

本轮定位到 `CN1/CN3` 在 `W` 闭环语义下的两个问题：

- `CN1/CN3` 作为当前底盘后轮通道，和 `CN2/CN4` 的车体前进方向相反；如果 `W,a,b,c,d` 四轮同号下发，不做输出方向修正，后轮会与前轮对抗。
- `CN3` 之前看起来“不被闭环控制”，首因不是协议未接入，而是当前 raw 正向下 `CN3` 编码器反馈为负，导致闭环误判方向；在后轮输出方向修正后，`CN3` 的 raw 负向反馈为正，可以进入有效闭环。

本轮修复方式：

- 保持 `M,<ch>,<pwm>` 为原始单通道调试命令，不改变底层 H 桥真值。
- 只在 `W` 闭环层通过 `g_csr_motor_dir_sign` 修正车体语义方向。
- 按 raw 负向实测结果修正 `g_csr_encoder_dir_sign`，使 `W+` 语义下的反馈速度为正。

## 实测依据

### 1. C-3.1.4B 最终收敛测试

在新电机、彩虹线、7.4V 条件下，`CN1/CN3` 原生定时器反馈均成立。

`CN1` raw 正向：

```text
M,1,500
DBG,1,1,0,37,25584,102
ENC,1,27127,102
```

`CN3` raw 正向：

```text
M,3,500
DBG,3,1,1,65483,-3579,-132
ENC,3,-5568,-133
```

这说明：

- `CN1` raw 正向反馈为正。
- `CN3` raw 正向反馈为负。

### 2. C-3.1.4C raw 负向补测

本轮为了确认闭环方向表，不直接猜测，而是补测 raw 负向。

`CN1` raw 负向：

```text
M,1,-500
DBG,1,0,1,65461,-4956,-199
ENC,1,-7961,-201
```

`CN3` raw 负向：

```text
M,3,-500
DBG,3,1,1,59,3760,155
ENC,3,6863,156
```

这说明：

- `CN1` raw 负向反馈为负。
- `CN3` raw 负向反馈为正。

## 修复口径

当前车体语义要求：

- `W+` 应代表四轮同一车体前进语义。
- `CN1/CN3` 与 `CN2/CN4` 物理安装方向相反，因此 `W+` 对 `CN1/CN3` 应输出 raw 负向 PWM。

据此冻结方向表：

```c
int8_t g_csr_motor_dir_sign[CSR_CHANNEL_COUNT] = { -1, 1, -1, 1 };
int8_t g_csr_encoder_dir_sign[CSR_CHANNEL_COUNT] = { -1, -1, 1, 1 };
```

解释：

- `CN1`：`W+` 使用 raw 负向；raw 负向反馈为负，所以编码器符号设为 `-1`。
- `CN2`：沿用既有 `-1`，控制组已成立。
- `CN3`：`W+` 使用 raw 负向；raw 负向反馈为正，所以编码器符号保持 `+1`。
- `CN4`：沿用既有 `+1`，控制组已成立。

## 平顺性热修

方向热修后，高速短测证明 `CN1/CN3` 方向与闭环接入正确，但用户观察到仍有“一卡一卡”的脉冲感。

代码侧根因：

- `csr_motor_drv.c` 原始最小实际驱动过高，非零 PWM 会被提升到较高占空区间。
- `main.c` 原默认 PI 增益较大，低速目标下一推就超速，随后反向制动保护把输出压到 0，形成“脉冲-滑行”。

本轮追加平顺性参数热修：

```c
#define CSR_EFFECTIVE_PWM_MIN           120
#define CSR_PI_KP_DEFAULT               1800.0f
#define CSR_PI_KI_DEFAULT               160.0f
#define CSR_PI_PWM_STEP_LIMIT           60
```

底层实际驱动最小值从 `1200` 下调并折中为：

```c
#define CSR_DRIVE_DOMINANT_MIN 950U
```

同时按当前轮间差异设置每轮独立 PI：

```c
static float g_kp[CSR_CHANNEL_COUNT] = {
    1200.0f,
    1200.0f,
    2600.0f,
    2600.0f
};

static float g_ki[CSR_CHANNEL_COUNT] = {
    100.0f,
    100.0f,
    260.0f,
    260.0f
};
```

解释：

- `CN1/CN2` 当前偏快，降低 PI 增益，减少起步大脉冲。
- `CN3/CN4` 当前偏慢，提高 PI 增益，让后轮更容易跨过静摩擦并持续计数。
- `CSR_DRIVE_DOMINANT_MIN=650U` 曾导致 `CN3/CN4` 在低速/中速下无法稳定起转，因此不采用。

## 待上板验证

本轮已完成烧录与上板串口验证。

### 1. 低速四轮同向验证

连续发送：

```text
STOP
W,0.05,0.05,0.05,0.05
```

观测结果：

```text
VEL,0.144,0.140,0.147,0.153,0.050,0.050,0.050,0.050
PWM,0,0,0,0
...
VEL,0.172,0.171,0.073,0.073,0.050,0.050,0.050,0.050
PWM,0,0,0,0
```

结论：

- `VEL` 四路均为正，`CN1/CN3` 不再与 `CN2/CN4` 反馈方向相反。
- 低速目标下实测速度常高于 `0.05 m/s`，PI 输出会进入 `0 PWM` 滑行段，这是当前低速调参问题，不是 `CN3` 未闭环。

### 2. 单轮闭环接入验证

逐轮发送单轮 `W` 目标：

```text
W,0.08,0,0,0
W,0,0.08,0,0
W,0,0,0.08,0
W,0,0,0,0.08
```

观测结论：

- 四个通道分别只有对应 `VEL` 变化，说明 `W` 目标锁存按通道生效。
- `CN3` 在 `W,0,0,0.08,0` 下出现正向 `VEL`，并出现 `PWM` 输出脉冲：

```text
VEL,0.000,0.000,0.073,0.000,0.000,0.000,0.080,0.000
PWM,0,0,-16,0
...
VEL,0.000,0.000,0.067,0.000,0.000,0.000,0.080,0.000
PWM,0,0,-32,0
```

这说明 `CN3` 已进入闭环链路；此前“CN3 像是没被控制”的现象不是协议漏控，而是方向/反馈符号与低速输出策略叠加造成的误判。

### 3. 高速四轮同向短测

连续发送：

```text
W,0.15,0.15,0.15,0.15
```

首帧输出：

```text
VEL,0.000,0.000,0.000,0.000,0.150,0.150,0.150,0.150
PWM,-120,120,-120,120
```

这符合当前方向表预期：

- `CN1/CN3` raw PWM 为负，用于抵消后轮安装方向差异。
- `CN2/CN4` raw PWM 为正。

后续回传示例：

```text
VEL,0.120,0.122,0.134,0.163,0.150,0.150,0.150,0.150
PWM,-103,86,-61,0
...
VEL,0.124,0.207,0.146,0.154,0.150,0.150,0.150,0.150
PWM,-65,0,0,0
```

结论：

- 高速短测下四路 `VEL` 均保持正向。
- `CN3` 不再出现“目标为正但测量为负导致持续顶输出”的闭环失效现象。
- 当前仍存在低速/中速段的脉冲-滑行特征，下一轮应单独做 PI 参数、起步补偿和输出保持策略调参。

### 4. 平顺性热修后复测

低速：

```text
W,0.05,0.05,0.05,0.05
```

典型回传：

```text
VEL,0.056,0.052,0.039,0.054,0.050,0.050,0.050,0.050
PWM,0,0,-30,0
...
VEL,0.036,0.034,0.048,0.049,0.050,0.050,0.050,0.050
PWM,-15,17,-7,3
```

中速：

```text
W,0.15,0.15,0.15,0.15
```

典型回传：

```text
VEL,0.120,0.120,0.103,0.112,0.150,0.150,0.150,0.150
PWM,-32,32,-127,102
...
VEL,0.137,0.163,0.102,0.108,0.150,0.150,0.150,0.150
PWM,-9,0,-146,125
```

复测结论：

- 四路仍保持同向正反馈。
- `CN3/CN4` 不再出现第二版调参时的长期 `VEL=0`。
- 低速脉冲感比初始方向热修版本缓和，但 `CN1/CN2` 仍偏快、`CN3/CN4` 仍偏慢。
- 当前已经适合作为 C-3.1.4C 热修基线；下一轮若继续优化，应做系统性轮间标定，而不是再改方向表。

## 当前结论

`C-3.1.4C` 将 `CN1/CN3` 的后轮安装方向差异收口到闭环方向表，而不是改底层 raw 驱动。这样可以保留 `M` 命令作为硬件调试入口，同时让 `W` 成为车体语义一致的持续控制协议。

本轮热修后的直接结论：

- `CN1/CN3` 方向问题已在 `W` 闭环语义层修正。
- `CN3` 已确认进入闭环链路。
- 当前剩余问题不是“是否闭环”，而是闭环平顺性，需要后续围绕 PI、低速起步补偿、最小保持 PWM 和滤波继续整定。
- 本轮已追加一版平顺性热修，明显改善后轮不起转和大脉冲问题，但还不是最终标定。
