# C-3.1.4B：OpenRF1 定时器主线最终收敛测试结论

## 结论

本轮完成了 `C-3.1.4A` 之后的最终收敛测试，结论已经可以写死：

- OpenRF1 的原生定时器主线仍然成立。
- `CN1 -> TIM5 + PA0/PA1` 与 `CN3 -> TIM2 full remap + PA15/PB3` 在当前固件、当前 7.4V 电池条件下，均已拿到非零编码器反馈。
- 这几轮里出现的“有时有数、有时没数”，主因已经收敛到电机编码器链、线簇/线序、接触状态，而不是原生 TIM 主线软件本身。
- 本轮最终状态已经达到“可结题归档”的标准：`CN1` 与 `CN3` 均恢复到可进入原生闭环链的反馈状态。

## 本轮测试目的

本轮不是重新怀疑协议，也不是重开软编码器绕行。

本轮只做一件事：

在 `C-3.1.4A` 已确认原生 TIM 主线成立的基础上，继续用“新电机 + 不同线簇 + 不同电池电压”做最后收敛测试，判断问题到底跟着：

- 原生 TIM 通道走
- 电机走
- 线簇/线序/接触链走

## 关键对照过程

### 1. 先前已确认的事实

`C-3.1.4A` 已经确认：

- 控制组 `CN2/CN4` 一直有稳定计数。
- 当 `CN1/CN3` 接入已确认正常的测试编码器电机后，`CN1/CN3` 也能恢复非零 `DBG/ENC`。

这一步已经说明：

- 原生 TIM 主线软件不是根因。
- `CN1/CN3` 通道并非天然不可用。

### 2. 新电机 + 12V 测试

后续测试中，把新电机接入 `CN1/CN3`，并把电池换成 `12V` 后，得到的结果是：

- `CN1` 出现历史累计值，但无新增 `delta`
- `CN3` 仍为 `0`
- 说明 `12V` 本身没有把反馈救回来

这一步排除了“只是 7.4V 供电不够”的单一解释。

### 3. 交叉测试：新电机移到控制组

随后做交叉测试：

- 原 `CN2` 正常电机 -> `CN1`
- 新电机 -> `CN2`

结果：

- `CN1` 立刻恢复计数
- `CN2` 接入新电机后仍无计数

这一步把问题进一步收敛到：

- 问题不是 `CN1` 通道软件
- 问题更像跟着“新电机那套编码器链/线簇/线序”走

### 4. 最终收敛测试：新电机改用彩虹线，回到 7.4V

本轮最终状态：

- `CN1` 新电机改为彩虹线
- `CN3` 新电机改为彩虹线
- 电池恢复为 `7.4V`

这一步的目标是验证：

如果换掉可疑白线线簇后，`CN1/CN3` 都恢复反馈，那么问题就应明确归因到“线簇/线序/接触链”，而不是再继续怀疑 TIM 软件。


### CN1：新电机 + 彩虹线 + 7.4V

静态：

```text
DBG,1,0,0,0,22828,0
ENC,1,22828,0
```

短时驱动后：

```text
M,1,500
DBG,1,1,0,37,25584,102
ENC,1,27127,102
```

结论：

- `phaseA/phaseB` 有变化
- `timerCount` 非零
- `ENC delta = 102`
- `VEL` 也被连续估计出来，说明 `CN1` 已重新进入有效反馈链

### CN3：新电机 + 彩虹线 + 7.4V

静态：

```text
DBG,3,0,1,0,1,0
ENC,3,1,0
```

短时驱动后：

```text
M,3,500
DBG,3,1,1,65483,-3579,-132
ENC,3,-5568,-133
```

结论：

- `phaseA/phaseB` 有变化
- `timerCount` 非零
- `ENC delta = -133`
- `CN3` 也已经恢复有效反馈

负号只代表当前方向定义与实测方向的符号关系，不代表链路失败。

## 收敛后的唯一判断

本轮完成后，当前最强结论如下：

### 1. 已被排除的原因

以下解释已经不成立：

- 原生 TIM 主线软件本身有系统性错误
- `CN1` / `CN3` 板级定时器输入天然不可用
- 7.4V 电池天然导致 `CN1/CN3` 无反馈
- 必须依赖软编码器绕行才能恢复 `CN1/CN3`

### 2. 当前最合理归因

当前多轮对照测试共同指向：

- 问题主要跟着电机编码器链、线簇/线序、接触状态变化
- 尤其是此前使用的白线线簇高度可疑
- 当改用彩虹线后，`CN1` 与 `CN3` 均恢复计数，说明 TIM 主线并没有坏

### 3. 对正式主线的影响

从本轮起，可以继续坚持：

- 正式主线仍是 `C-3.0.6` 的原生 TIM 编码器架构
- `C-3.1.3` 软编码器分支只保留为止损/证据，不再作为正式主线

## 最终结题结论

`C-3.1.4B` 的最终结论是：

OpenRF1 原生定时器编码器主线已通过最终收敛测试；`CN1` 与 `CN3` 在更换为彩虹线线簇并恢复 7.4V 供电后，均恢复了非零编码器反馈。当前问题首因已经从“TIM 软件/板级通道可疑”收敛为“电机编码器链与线簇/线序/接触问题”。

后续若继续推进 RF1 闭环整定，应建立在这条原生 TIM 主线上，而不是继续扩大软编码器绕行方案。
