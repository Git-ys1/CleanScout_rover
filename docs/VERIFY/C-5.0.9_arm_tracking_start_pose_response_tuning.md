# C-5.0.9：机械臂三轴追踪启动位姿与响应调参

## 当前基线

C-5.0.9 阶段目标仍固定为 `bottle`，启用：

- `000 yaw`：左右追踪。
- `003 pitch`：上下快速细调。
- `001 lift`：只在上下误差较大时低频粗调。

启动追踪前默认回标准初始化位姿：

```text
0=1500,1=1907,2=1900,3=900,4=1500,5=1500
```

## 2026-07-06 垂直方向基线

```yaml
driver:
  pitch_pwm_sign: -1

visual_servo:
  invert_pitch: false
```

注意：曾短暂误改为 `pitch_pwm_sign = 1`，该改动已撤销。后续若怀疑上下方向异常，必须先做真实动作复核和单轴 dry-run 对照，不要直接改驱动层符号。

## 响应测试工具修正

`tools/arm_tracking_response_test.py` 会先回标准位姿，再施加小扰动，再运行追踪评分。

本轮修正一个测试漏洞：扰动后运行 `yolo_arm_track.py` 时，不再发送一次初始轴位姿覆盖扰动：

```text
--send_initial_axis_pose false
```

建议测试命令：

```bash
cd ~/rk3588_ai/arm_tracking_demo
~/rk3588_ai/rknn_lite_env/bin/python3 tools/arm_tracking_response_test.py \
  --real \
  --no_show \
  --track_class bottle \
  --perturb_pose 0=1540,1=1887,3=930 \
  --max_frames 90
```

如果机械臂或相机姿态不安全，先不用自动扰动测试，直接跑三轴真实追踪并人工观察方向。

判断：

- 003 pitch 能参与，但也存在小幅跨零。

## 第二轮保守调参

目标：降低抖动，不改方向、不改总线舵机协议、不改 002/004/005。

当前参数：

```yaml
driver:
  duration_ms: 150

visual_servo:
  dead_zone_px: 42
  control_rate_hz: 12
  max_yaw_delta: 0.018
  max_lift_delta: 0.006
  max_pitch_delta: 0.012
  error_filter_alpha: 0.6
  kp_yaw: 0.0009
  kp_lift: 0.00045
  kp_pitch: 0.00065
  combined_lift_error_px: 80
  combined_lift_rate_divider: 4
  invert_pitch: false

driver:
  pitch_pwm_sign: -1
```

第二轮日志目录：

```text
/home/orangepi/rk3588_ai/debug_logs/arm_tracking_eval/20260706-173547/
```

结果：

```text
score = 188.4431
target_frames = 73 / 90
final_abs_error_x = 14.06
final_abs_error_y = 4.65
mean_abs_error_x = 88.87
mean_abs_error_y = 11.27
sign_changes_x = 1
sign_changes_y = 3
shake_penalty = 32.0
```

结论：

- 抖动明显下降：`shake_penalty 112.0 -> 32.0`。
- 最终误差明显下降：`x 33.89 -> 14.06`，`y 29.61 -> 4.65`。
- 第二轮前 15 帧没有锁定 bottle，导致 `target_ratio` 下降；这更像扰动姿态/检测窗口问题，不直接代表控制参数变差。
- 当前建议先用第二轮参数继续实测观察。

下一轮如果仍觉得慢：

- 优先把 `max_yaw_delta` 从 `0.018` 小步升到 `0.020`。
- 不要直接升 `kp_yaw`，否则靠近中心时更容易抖。
- 如果上下仍抖，先把 `max_pitch_delta` 保持 `0.012`，只调 `dead_zone_px` 或 `error_filter_alpha`。
