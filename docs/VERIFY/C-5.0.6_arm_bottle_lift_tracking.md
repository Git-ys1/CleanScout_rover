# C-5.0.6：OrangePi bottle 001 lift-only 上下追踪阶段

## 目标

在 C-5.0.5 已验证 `bottle` + 000 yaw 左右追踪后，本阶段只做 001 号舵机的上下追踪。

本阶段不做：

- 不控制 002。
- 不控制 003 末端 pitch。
- 不改变 12、23、34 当前夹角关系。
- 不做抓取、IK、ROS 融合或多舵机联动。

## 当前姿态约束

当前机械臂是靠 001/002/003 的姿态把末端相机架起来：

- 001/002 让 12、23 形成向后钝角，使 23 更靠近底座上方，减小前伸力矩。
- 003 让 34 接近水平，使相机看向前方。
- 004/005 当前只保持末端旋转和夹爪姿态。

因此 C-5.0.6 的上下追踪只允许动 001。002/003/004/005 继续保持当前人工调出的姿态，避免算法把机械臂重新伸到前方或让 003 抬不住相机。

## 当前实现

- 新增视觉控制轴：`lift`。
- `lift` 只映射到 001 号舵机。
- `yaw` 仍只映射到 000 号舵机。
- `pitch` 保留给后续 003 末端俯仰阶段。
- `yolo_arm_track.py --control_axes lift` 只发送 001 命令，退出时只发 `#001PDST!`。

主要参数：

```yaml
visual_servo:
  control_axes: [lift]
  lift_init: -0.93
  lift_min: -1.2
  lift_max: -0.65
  kp_lift: 0.0007
  ki_lift: 0.0
  kd_lift: 0.0
  max_lift_delta: 0.01
  invert_lift: true

driver:
  lift_servo_index: 1
  lift_pwm_neutral: 1912
  lift_pwm_per_rad: 500
  lift_pwm_sign: 1
  stop_servo_indices: [1]
```

如果上下方向反了，只改一处：

- 优先改 `visual_servo.invert_lift`。
- 不要同时改 `invert_lift` 和 `driver.lift_pwm_sign`。
