# C-3.0.4B：OpenRF1 底层驱动实测真值冻结

## 1. 文档目的

本文件只记录已经通过实机观察拿到的底层真值，不写推断，不写目标状态。

本文件用于向项目负责人提交当前 `C-3.0.4` 的阶段性结果，并申请对该项进行审批。

## 2. 当前已确认的板口与轮位映射

用户实测冻结为：

| 协议位 | 板口 | 物理轮位 |
| --- | --- | --- |
| `A` | `CN1` | 左后 |
| `B` | `CN2` | 左前 |
| `C` | `CN3` | 右后 |
| `D` | `CN4` | 右前 |

## 3. 当前关键测试数据

| 板口 | 物理轮位 | 正值测试结果 | 负值测试结果 | 冻结结论 |
| --- | --- | --- | --- | --- |
| `CN1` | 左后 | 无响应 | 实际为负方向 | 当前只在负值下动作 |
| `CN2` | 左前 | 实际为正方向 | 无响应 | 当前只在正值下动作 |
| `CN3` | 右后 | 无响应 | 实际为正方向 | 当前只在负值下动作 |
| `CN4` | 右前 | 实际为负方向 | 无响应 | 当前只在正值下动作，但方向为负 |

说明：

- “无响应”表示在本轮测试中未观察到该板口在该极性下出现稳定、可确认的动作。
- 以上数据以用户最终实测为准，用于审批，不再与此前推断性轮位语义混用。

## 4. 阶段性结论

当前 OpenRF1 bring-up 的阶段性结论是：

- 四轮控制极性没有统一起来
- 四轮同向同转目标未达成
- 统一的前进/后退控制语义未建立
- 当前协议化施工结果不具备继续作为稳定主线扩写的条件

## 5. 审批建议

建议将当前 `C-3.0.4` OpenRF1 协议化底盘 bring-up 按以下口径提交项目负责人审批：

- 结论：`阶段性失败`
- 处理建议：`弃项或暂停，待重新立项`
- 当前保留物：本文件与相关运行时说明文档，作为失败证据链

## 6. 后续若重开，必须遵守的顺序

1. 重新固定 `CN1~CN4` 单轮正反真值
2. 先打通四轮统一正反控制
3. 再接协议
4. 最后再谈编码器与闭环

在这一步之前，不应继续把当前实现当成可直接扩写的底盘基线。
