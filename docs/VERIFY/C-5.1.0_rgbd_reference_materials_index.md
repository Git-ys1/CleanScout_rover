# C-5.1.0 RGB-D 与机械臂参考资料索引

## 结论

`docs/00最新参考/` 是下一阶段 RGB-D 相机 + 机械臂开发的本地参考资料库，不是当前 CleanScout 主线源码。该目录已经加入 `.gitignore`，原因是它包含约 1,183 个文件、两个接近 200 MB 的压缩包、ROS2/vendor 依赖、Keil/STM32 编译产物和大量第三方源码，直接提交会污染主仓。

后续开发可以学习其中的架构和算法思路，但不要直接把整个目录入库。需要移植时，按功能摘出最小代码、重写成当前 OrangePi / ROS Noetic / RKNN / 总线舵机链路可维护的实现。

| RF4 默认姿态 | `rf4_25.12.22/rf4_25.12.22/Core/Src/usart.c` | 参考启动/复位姿态 `[1500,1839,2490,1160,1500,1500]`，但当前 CleanScout 已有实测启动姿态，不能直接覆盖 |

## 下一阶段建议路线

1. D435 到货后先只做相机 bringup：确认彩色图、深度图、`CameraInfo`、深度单位、RGB/Depth 对齐。
2. 在 OrangePi 当前 `yolo_arm_track.py` 基础上加入 depth read，不急着上完整 IK。
3. 第一版只输出目标中心点的 `u/v/depth/x/y/z`，并保存 debug overlay，确认深度没有跳变。
4. 第二版再把 3D 目标映射到当前三轴视觉伺服策略：yaw 负责水平居中，lift/pitch 负责垂直居中和末端俯仰。
5. 最后才考虑借鉴 `arm_ik_sdk.py` 做全臂 IK；在此之前不要把 ROS2 工程整体搬进 Noetic。

## 注意事项

- 当前参考源码是 ROS2，不是明天要直接部署的 ROS Noetic 包；它的价值是话题结构、深度处理、手眼外参和 IK 流程。
- 当前 CleanScout 机械臂已经形成总线舵机文本协议与 OrangePi 视觉伺服主线，不能被 RF4 的二进制串口帧直接替换。
- 任何从 `docs/00最新参考/` 摘出的代码，都要先写清楚来源和改写边界，再进入正式目录。
