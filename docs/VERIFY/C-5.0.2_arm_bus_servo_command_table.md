# C-5.0.2 机械臂总线舵机指令表 Markdown 摘录

这份文件是对 `docs/001-总线舵机资料/1.使用手册/附件1《总线舵机指令表》.docx` 的可检索整理。

## 先记住三件事

- 这是总线舵机 ASCII 文本协议，不是裸 PWM 波形。
- 单个动作命令格式是 `#IDPxxxxTxxxx!`。
- 多个舵机一起发时，需要用 `{...}` 包起来。

## 字段规则

- `ID` 固定 3 位，范围 `000-254`，广播 ID 是 `255`。
- `P` 固定 4 位，常见范围 `0500-2500`。
- `T` 固定 4 位，单位毫秒，范围 `0000-9999`。

## 典型用法

单舵机动作：

```text
#000P1500T1000!
```

多舵机动作：

```text
{#000P1500T1000!#001P1500T1000!}
```

读取当前位置：

```text
#000PRAD!
```

读取 ID、版本、温度电压：

```text
#000PID!
#000PVER!
#000PRTV!
```

## 本项目的使用原则

- 先按这份表判断协议语义，再去看 OrangePi 或 STM32 代码。
- 如果读回没出来，先查串口端口、线序、DTR/RTS、转接链路，再查固件。
- 不要再把这套协议先误读成 PWM 舵机协议。
- 当前官方 STM32 基线会同时做 UART 总线转发和本地 6 路 PWM 解析。
- 2026-06-11 实测只有 `000/001/002` 在当前总线接线上返回版本/位置；`003/004/005` 无总线回包，但不能据此否定其本地 PWM 动作。
- 任何时刻只允许一个程序独占 CH340 `/dev/ttyUSB0`。
