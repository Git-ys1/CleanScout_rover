# C-3.1.4D：OpenRF1 四轮闭环平顺性热修记录

## 结论

`C-3.1.4C` 已经解决 `CN1/CN3` 后轮方向和 `CN3` 闭环接入问题，但实车表现仍有“一卡一卡”的脉冲感。

本轮继续在同一条原生 TIM 闭环主线上调参，不改协议、不改方向表、不改编码器链路。

最终结论：

- `W,0.05` 低速下四轮均有反馈，不再只剩部分轮子动。
- `W,0.15` 中速下四轮速度基本贴近目标。
- `W,0.25` 较高速下四轮持续正向，前轮接近目标，后轮略低但稳定。
- `W,0.35` 高速短测通过，四轮均保持正向闭环，未出现反向、掉反馈或超时停车异常。

当前剩余问题是轮间精标定，不是方向、协议或编码器问题。

## 修改范围

本轮只修改 RF1 自写运行层：

- `_local/openrf1_keil_work_3/User/main.c`
- `_local/openrf1_keil_work_3/User/csr_motor_drv.c`

不修改：

- `W/M/E/D/STOP` 协议
- `csr_board_map.c` 的方向表
- `csr_encoder_drv.c` 的 TIM2/3/4/5 原生编码器读取
- Keil 工程结构

## 问题分析

上一版的脉冲感来自两个因素：

1. 全局最小驱动不适合四轮差异。
   `CN1/CN2` 更容易起转，过高最小驱动会导致低速超速；`CN3/CN4` 更弱，需要更高补偿才能跨过静摩擦。

2. 单一 PI 参数不能兼顾前后轮。
   前轮偏快，后轮偏慢，所以继续使用同一组 `kp/ki` 会让前轮脉冲、后轮不足。

## 最终参数

### 每轮 PI

```c
static float g_kp[CSR_CHANNEL_COUNT] = {
    800.0f,
    800.0f,
    4200.0f,
    4200.0f
};

static float g_ki[CSR_CHANNEL_COUNT] = {
    60.0f,
    60.0f,
    420.0f,
    420.0f
};
```

含义：

- `CN1/CN2`：降低增益，减少低速大脉冲。
- `CN3/CN4`：提高增益，补偿后轮偏弱。

### 每轮最小有效 PWM

```c
static const uint16_t g_effective_pwm_min[CSR_CHANNEL_COUNT] = {
    30U,
    30U,
    120U,
    120U
};
```

含义：

- `CN1/CN2` 允许更小闭环输出，减少一推就超速。
- `CN3/CN4` 保持较高最小输出，避免低速不起转。

### 每轮底层实际驱动最小值

```c
static const uint16_t g_drive_dominant_min[CSR_CHANNEL_COUNT] = {
    550U,
    550U,
    1000U,
    1000U
};
```

含义：

- `CN1/CN2` 降低底层最小占空。
- `CN3/CN4` 保持较高底层起步补偿。

## 实测记录

### 低速

命令：

```text
W,0.05,0.05,0.05,0.05
```

典型回传：

```text
VEL,0.051,0.047,0.050,0.047,0.050,0.050,0.050,0.050
PWM,0,1,-1,13

VEL,0.070,0.064,0.053,0.056,0.050,0.050,0.050,0.050
PWM,0,0,0,0
```

结论：

- 四轮均有正向速度反馈。
- `CN3/CN4` 不再长期为 0。
- 低速仍存在轻微脉冲，但比 C-3.1.4C 初版明显缓和。

### 中速

命令：

```text
W,0.15,0.15,0.15,0.15
```

典型回传：

```text
VEL,0.131,0.140,0.145,0.153,0.150,0.150,0.150,0.150
PWM,-15,7,-22,0

VEL,0.145,0.145,0.148,0.147,0.150,0.150,0.150,0.150
PWM,-2,3,-11,13
```

结论：

- 中速档四轮最接近目标。
- 四路闭环均工作，未见 `CN1/CN3` 方向问题复发。

### 较高速

命令：

```text
W,0.25,0.25,0.25,0.25
```

典型回传：

```text
VEL,0.247,0.252,0.217,0.220,0.250,0.250,0.250,0.250
PWM,-2,0,-141,133

VEL,0.260,0.228,0.220,0.219,0.250,0.250,0.250,0.250
PWM,0,17,-136,141
```

结论：

- 前轮接近目标。
- 后轮略低，但稳定、同向、持续闭环。
- 后续若继续精调，应优先继续提高 `CN3/CN4` 的中高段补偿，而不是改方向。

### 高速短测

命令：

```text
W,0.35,0.35,0.35,0.35
```

典型回传：

```text
VEL,0.333,0.333,0.352,0.380,0.350,0.350,0.350,0.350
PWM,-14,13,0,0

VEL,0.363,0.360,0.304,0.310,0.350,0.350,0.350,0.350
PWM,0,0,-198,172
```

结论：

- 高速短测下四路均保持正向。
- 没有出现反转、反馈丢失或闭环掉线。
- `CN3/CN4` 高速后段略低于目标，但已经进入可继续标定状态。

## 当前状态

`C-3.1.4D` 已经把问题从“卡顿明显”推进到“可运行但还需轮间标定”。

推荐下一步：

- 继续用当前分通道 PI/最小驱动结构。
- 不再改方向表。
- 若要进一步优化，增加按目标速度分段的 feed-forward 表，而不是继续只靠 PI。
