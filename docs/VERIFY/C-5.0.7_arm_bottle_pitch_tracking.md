# C-5.0.7：OrangePi bottle 003 pitch-only 末端俯仰追踪阶段

## 目标

在 C-5.0.5 已验证 `bottle` + 000 yaw 左右追踪、C-5.0.6 已验证 001 lift 上下追踪后，本阶段只做 003 号舵机的末端俯仰追踪。

本阶段不做：

- 不控制 000 yaw。
- 不控制 001 lift。
- 不控制 002。
- 不控制 004/005。
- 不做 `yaw + lift + pitch` 联动。
- 不做抓取、IK、ROS 融合或手眼标定。

## 当前姿态与轴分工

当前机械臂可理解为三层职责：

- 000 yaw：底座左右转，让目标在画面水平中心附近。
- 001 lift：整臂粗抬升/粗下压，负责让 12、23、34 组合姿态处在安全力矩区间。
- 003 pitch：只改变 34 末端夹角，负责相机视线的细俯仰。

因此 `lift` 和 `pitch` 后续组合时，不应该都直接抢同一个 `err_y`：

- `pitch/003`：优先做快速、小幅、细的视线补偿。
- `lift/001`：只做慢速、低频、粗姿态补偿，例如目标长期偏上/偏下，或 003 已接近行程边界。
- 组合前必须先分别验证 `lift-only` 和 `pitch-only` 的方向、限幅和稳定性。

这也是本阶段继续保持 `pitch-only` 的原因：先证明 003 单独动作正确，再谈多轴协同。

## 当前实现

- 默认视觉控制轴切换为：`pitch`。
- `pitch` 只映射到 003 号舵机。
- `yolo_arm_track.py --control_axes pitch` 只发送 003 命令，退出时只发 `#003PDST!`。
- 000 yaw 与 001 lift 的命令仍可通过 `--control_axes yaw` / `--control_axes lift` 回归。

主要参数：

```yaml
visual_servo:
  control_axes: [pitch]
  pitch_init: 1.2
  pitch_min: 0.9
  pitch_max: 1.5
  kp_pitch: 0.0006
  ki_pitch: 0.0
  kd_pitch: 0.0
  max_pitch_delta: 0.01
  invert_pitch: true

driver:
  pitch_servo_index: 3
  pitch_pwm_neutral: 884
  pitch_pwm_per_rad: 500
  pitch_pwm_sign: -1
  stop_servo_indices: [3]
```

如果上下方向反了，只改一处：

- 优先改 `visual_servo.invert_pitch`。
- 不要同时改 `invert_pitch` 和 `driver.pitch_pwm_sign`。

## 通过标准

- `--control_axes pitch` 时，启动、追踪、退出都只出现 003 号舵机命令。
- 目标上下移动时，003 朝能减小 `err_y` 的方向动作。
- 000/001/002/004/005 不被追踪循环改写。
- 目标进入垂直死区后，003 不明显来回抖动。
- 000 yaw 和 001 lift 回归命令仍可单独控制对应舵机。
