# C-3.1.0 OpenRF1 CN1/CN3 板级关联审计

> C-3.1.3 更新：本文审计对象是原板 `CN1=PA0/PA1` 与 `CN3=PA15/PB3` 编码器输入链路。C-3.1.3 已采用循迹口飞线止损方案：`CN1_A->PC4(X1)`、`CN1_B->PC5(X2)`、`CN3_A->PB0(X3)`、`CN3_B->PC14(X4)`；旧 `PB10` 记录不作为循迹口真值。

## 结论

依据 `docs/STM32F103RCT6/OpenRF1_开发速查.md`、电机接口表、当前网表整理和 vendor 例程源码，当前板级关联应按下表理解。

当前审计结论：

- 文档与 vendor 源码都指向 `CN1 -> PA0/PA1 -> TIM5`、`CN3 -> PA15/PB3 -> TIM2 full remap`。
- 当前资料未发现需要额外软件使能的“编码器供电开关”或“CN1/CN3 输入复用开关”。
- `CN1/CN3` 在 `C-3.1.0` 实测中，输入模式已能切到 `FLOATING/IPU/IPD`，但原始相位仍固定为 `1/1`。
- 因此当前更像板级输入链或接插件/线序/编码器供电问题，而不是 `W` 协议、PI、读数函数或统一方向表问题。

## CN1 审计

- 预期链路：`CN1.3 -> PA0/TIM5_CH1`，`CN1.4 -> PA1/TIM5_CH2`。
- JTAG/SWD：`PA0/PA1` 不属于默认 SWJ/JTAG 占用脚，因此不能用 JTAG 解释 CN1 无相位。
- 当前自写层：
  - 已开启 `RCC_APB1Periph_TIM5`
  - 已配置 `GPIOA Pin0/Pin1`
  - 可通过 `X,1,F/IPU/IPD,TI12/TI1/TI2,0/10` 重新初始化
  - `R,1` 可确认 GPIO 配置值和 `TIM5` 寄存器
- C-3.1.0 实测：
  - `FLOATING` 配置值为 `4`
  - `IPU/IPD` 配置值为 `8`
  - 三种输入模式下 `phaseA=1, phaseB=1, CNT=0, ENC=0`
- 当前归因：优先查硬件输入链、CN1 编码器 A/B 是否真实进入 PA0/PA1、CN1 编码器供电与地、接插件压接或线序。

## CN3 审计

- 预期链路：`CN3.3 -> PA15/TIM2_CH1`，`CN3.4 -> PB3/TIM2_CH2`。
- JTAG/SWD：必须禁用 JTAG、保留 SWD，释放 `PA15/PB3`。
- 当前自写层：
  - `R,0` 显示 `MAPR_EFFECTIVE=02000300`
  - 该值表示自写层写入了 `SWJ_CFG=010` 与 `TIM2 full remap`
  - 可通过 `X,3,F/IPU/IPD,TI12/TI1/TI2,0/10` 重新初始化
- C-3.1.0 实测：
  - `FLOATING` 配置值为 `4`
  - `IPU/IPD` 配置值为 `8`
  - 三种输入模式下 `phaseA=1, phaseB=1, CNT=0, ENC=0`
- 当前归因：若 `MAPR_EFFECTIVE` 可信，CN3 当前更像 PA15/PB3 板级输入链没有看到编码器波形，而不是简单的 JTAG 未释放。

## 需硬件实测或客服确认项

- CN1 的 `CN1.3/CN1.4` 到 MCU `PA0/PA1` 的导通。
- CN3 的 `CN3.3/CN3.4` 到 MCU `PA15/PB3` 的导通。
- CN1/CN3 编码器 5V 与 GND 是否在电机插头处稳定存在。
- CN1/CN3 编码器 A/B 输出是否为开漏/开集电极，板上是否有外部上拉。
- CN1/CN3 是否存在网表未体现的跳线、焊桥、接口方向、排线压接方向或端子定义差异。

## 当前下一刀

如果后续手拨或示波器/万用表仍证明 `CN1/CN3` 的 MCU 引脚电平不变化，应停止继续调 TIM 参数，转入：

- 电机插头 6Pin 定义复核
- CN1/CN3 编码器供电实测
- CN1/CN3 A/B 到 MCU 引脚导通实测
- 与客服确认 CN1/CN3 编码器口是否需要板级跳线、焊接或额外上拉
