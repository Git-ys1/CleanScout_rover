目前本地后端已经具备这种模式，但默认本地模板还不是 edge-relay 模式。

现在有三套后端 profile：

local-lan：本地旧链路，backend -> rosbridge -> ROS

模板：vline-backend.local-lan.env.example
当前默认：ROS_TRANSPORT=rosbridge
适合之前那套 ws://10.117.77.84:9090 联调
public-cloud：公网普通部署，不直连 ROS

当前默认：ROS_TRANSPORT=mock
public-edge：新链路，ROS/edge-relay -> backend

模板：vline-backend.public-edge.env.example
当前默认：ROS_TRANSPORT=edge-relay
暴露 /edge/ros
适合你说的“ROS 找后端”
也就是说：代码已经支持“ROS 主动找后端”，但本地测试时你要用 public-edge profile 启动本地 backend，而不是继续用 local-lan。

本地 edge-relay 测试建议这样配：

APP_PROFILE=public-edge
PORT=3000
DATABASE_URL="file:./dev.db"
JWT_SECRET="至少32位随机字符串"
JWT_EXPIRES_IN=7d
CORS_ALLOWED_ORIGINS=http://127.0.0.1:5173,http://localhost:5173

ROS_ENABLED=true
ROS_TRANSPORT=edge-relay

EDGE_RELAY_ENABLED=true
EDGE_RELAY_PATH=/edge/ros
EDGE_DEVICE_AUTH_REQUIRED=true
EDGE_HELLO_TIMEOUT_MS=5000
EDGE_HEARTBEAT_TIMEOUT_MS=15000
EDGE_SERVER_PING_INTERVAL_MS=25000
EDGE_ALLOWED_DEVICE_IDS=csrpi-001
EDGE_DEVICE_BOOTSTRAP_ID=csrpi-001
EDGE_DEVICE_BOOTSTRAP_TOKEN=这里填32位以上设备token
然后本地启动 backend 后，树莓派 edge-relay 应该连：

ws://10.117.77.190:3000/edge/ros
以后云端则改成：

wss://api.hzhhds.top/edge/ros
注意两个关键点：

EDGE_DEVICE_BOOTSTRAP_TOKEN 只用于 seed 初始化设备，Pi 侧实际连接时必须用同一个 token 发 hello。
如果你已经 seed 过一次，后面只改 env token 不会自动改数据库里的 token hash；要么保持原 token，要么重置/更新 EdgeDevice 记录。
