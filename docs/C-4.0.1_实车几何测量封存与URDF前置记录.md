# C-4.0.1 实车几何测量封存与 URDF 前置记录

## 1. 当前采用的 base_link 假设

当前建议继续采用：

```text
base_link = 四个轮子围成的矩形中心，在地面上的投影点对应的车体中心坐标系
```

这是当前导航、编码器 odom、footprint 和后续 URDF 最稳妥的选择。

原因：

1. 当前 `rf1_vel_to_odom.py` 的运动学反解，本质上默认车体转动中心接近轮系几何中心。
2. 当前 `footprint`、costmap、激光外参也更适合围绕这个参考点定义。
3. 如果把 `base_link` 改成“实际碰撞体积中心”，会导致：
   - 当前里程计解释口径改变
   - 雷达 / IMU 外参重算
   - `footprint` 重算
   - 旧参数与当前链路口径不统一

结论：

**你这次按“四轮矩形中心 = base_link”来算，是对的，而且当前阶段应该继续这样。**

## 4. IMU 平面位置推导校验

### 4.1 左右方向

你给出的计算：

- `IMU 左侧到达底盘中心距离 = 21.7 / 2 - 2.6 - 0.3 = 7.95 cm`

校验：

- `10.85 - 2.6 - 0.3 = 7.95`

**这一步是正确的。**

如果 IMU 宽度为 1.5 cm，并且从左向右摆放，则：

- `IMU 右侧到达底盘中心距离 = 7.95 - 1.5 = 6.45 cm`

校验：

- `7.95 - 1.5 = 6.45`

**这一步也是正确的。**

则 IMU 中心的 y 坐标可取：

- `y_imu_center = (7.95 + 6.45) / 2 = 7.20 cm = 0.072 m`

按当前坐标约定，IMU 在车体左侧，所以：

- `imu y = +0.072 m`

### 4.2 前后方向

以四轮前沿位置作为 `x = +13.5 cm`：

- IMU 前侧到车轮前沿距离 = 3.2 cm
  - `x_imu_front = 13.5 - 3.2 = 10.3 cm`
- IMU 后侧到车轮前沿距离 = 5.2 cm
  - `x_imu_back = 13.5 - 5.2 = 8.3 cm`

则 IMU 中心 x 坐标：

- `x_imu_center = (10.3 + 8.3) / 2 = 9.3 cm = 0.093 m`

## 5. 雷达平面位置推导

### 5.1 左右方向

左侧车体最外边界到雷达中心 = 10.2 cm。

若车体左侧最外边界对应 `y = +10.85 cm`，则：

- `y_laser = 10.85 - 10.2 = 0.65 cm = 0.0065 m`

这意味着雷达几乎在中线附近，略偏左。

### 5.2 前后方向

你已确认：

- 雷达中心在四轮前边缘的后方 6.05 cm

以四轮前沿位置作为 `x = +13.5 cm`：

- `x_laser = 13.5 - 6.05 = 7.45 cm = 0.0745 m`

这一步现已确认为正式结论。
