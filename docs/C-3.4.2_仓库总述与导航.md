# C-3.4.2：CleanScout_rover 仓库总述与导航

## 1. 这是什么仓库

`CleanScout_rover` 当前不应再被理解成“一个单独的小车工程”，而应被理解成：

- 以车体 `C` 线为核心的实验室清扫/巡检机器人主仓
- 同时承载机械臂 `J` 线实验副本与交接资料
- 保留底层下位机、树莓派上位机、传感器接入、清扫执行与交接文档的统一证据链

一句话说清：

**这是“实验室清扫巡检系统”当前阶段的统一工程仓，核心主线是 `Raspberry Pi 4B + OpenRF1 + ROS + 激光雷达/IMU + 四轮编码底盘 + 双风机清扫执行`。**

## 2. 当前实物硬件情况

当前车体实物硬件已经收敛到下面这套组合：

- `Raspberry Pi 4B`
  - 当前上位机
  - 负责 ROS 工作区、建图导航、底盘桥接、风机控制、边缘通信与多模块协调
- `OpenRF1 (STM32F103RCT6)`
  - 当前下位机
  - 负责四路编码电机底层控制、编码器采样、串口协议、闭环执行
- 四个 6 线霍尔编码电机 + 四个麦克纳姆轮
  - 当前轮位语义冻结为：
    - `CN1 = 左后`
    - `CN2 = 左前`
    - `CN3 = 右后`
    - `CN4 = 右前`
- `思岚 A3` 激光雷达
  - 当前主激光感知
  - ROS 中提供 `/scan`
- `MPU6050`
  - 当前姿态辅助传感器
  - ROS 中提供 `/imu/data`
- 双 6 线吸尘风机
  - 继电器做主电源开关
  - 两路 PWM 分别调速
  - FG 线做速度回传
- 风机顶盖舵机
  - 当前冻结控制脚为 `GPIO12`
- `OpenMV` 机械臂
  - 当前近场识别/抓取候选子系统
- 3D 打印外壳与机械支架

更细的车体硬件总述可参考：

- [OpenRF1_开发速查](./STM32F103RCT6/OpenRF1_开发速查.md)
- [OpenClaw_树莓派4B_项目交接书](./OpenClaw_树莓派4B_项目交接书.md)

## 3. 当前实物软件情况

当前软件主线已经分成三层：

### 3.1 下位机固件层

位置：

- [`_local/openrf1_keil_work_3/User/`](../_local/openrf1_keil_work_3/User/)

职责：

- 四路电机驱动
- 四路编码器采样
- `W,a,b,c,d` 串口速度协议
- `M/E/D/STOP` 调试协议
- 四轮 PI 闭环与遥测回传

### 3.2 树莓派 ROS 层

位置：

- [`Raspberrypi/`](../Raspberrypi/)

职责：

- RF1 串口桥
- `/cmd_vel -> RF1` 速度桥接
- 雷达接入
- IMU 接入
- 建图导航
- 风机与顶盖 ROS 控制
- 边缘通信与交接脚本

### 3.3 机械臂实验副本层

位置：

- [`jixiebi/`](../jixiebi/)

职责：

- OpenMV 卖家程序受控实验副本
- actuator 基线
- vision grasp 基线
- 抓取/分拣实验留痕

## 4. 当前仓库结构应该怎么理解

当前顶层目录里，最重要的是下面这些：

### 4.1 `docs/`

统一文档区。  
应当优先在这里找“项目事实”“冻结结论”“验证记录”“交接说明”。

子目录含义：

- `docs/PLAN/`
  - 每一轮任务书
- `docs/SOFTWARE/`
  - 软件结论、协议冻结、架构说明
- `docs/VERIFY/`
  - 实测与验证记录
- `docs/HARDWARE/`
  - 硬件冻结、引脚、供电与接线资料
- `docs/STM32F103RCT6/`
  - OpenRF1 参考资料、vendor 例程、开发手册

### 4.2 `_local/`

本地工作副本区。  
这里包含实际施工中的 OpenRF1 工程、Keil 工作目录、vendor 对照副本等。

当前最重要的是：

- [`_local/openrf1_keil_work_3/`](../_local/openrf1_keil_work_3/)
  - 当前 RF1 下位机主施工副本
- [`_local/openrf1_keil_work_2/`](../_local/openrf1_keil_work_2/)
  - 当前实际 Keil 工程、输出与构建路径

### 4.3 `Raspberrypi/`

树莓派侧 ROS 工作区、发布物、脚本和说明入口。

### 4.4 `jixiebi/`

机械臂受控实验副本。  
这是当前可追踪、可二开的 `J` 线内容。

### 4.5 `J-jixiebi/`

原始资料/脏目录保留区。  
它不是当前主线开发入口，不能和 `jixiebi/` 混为一谈。

### 4.6 `tools/`

仓库脚本入口，包括：

- OpenRF1 串口测试脚本
- UNO 构建脚本
- 其他本地自动化脚本

## 5. 当前主线判断

## 5.1 C 线主线

当前 `C` 线正式主线已经不是旧的 `UNO + AFMotor`，而是：

- 上位机：`Raspberry Pi 4B`
- 下位机：`OpenRF1`
- 软件栈：`ROS Noetic + RF1 serial bridge`

但旧 `UNO` 线仍保留为历史证据与回退资料，不应直接删除。

## 5.2 J 线主线

当前 `J` 线应理解为：

- `jixiebi/` 中的受控实验副本是可继续维护的工程入口
- `J-jixiebi/` 是原始卖家资料与历史脏目录，不是当前规范主线

## 5.3 文档主线

当前文档主线应以：

- 冻结文档
- 验证文档
- 交接书
- 总述文档

作为优先事实来源，而不是仅靠分支名和零散提交猜当前状态。
