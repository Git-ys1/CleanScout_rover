# C-1.1.6 J 线系统适配评估

## 1. 评估目的

本文件不回答“卖家给了多少资料”，而回答：

- 这套机械臂是否适合作为实验室清扫巡检系统中的操作子系统
- 它当前最有价值的能力是什么
- 它离真正并入 `C` 线主线还差什么

说明：

- 本轮所有结论都来自本地资料分析
- 不代表已经完成实机验证

## 2. 平台判断

当前 J 线应定性为：

- 半开放二开平台
- 接近可作为子系统接入的平台
- 但仍带明显卖家固件、板级定义和 APP 依赖

它不是纯黑盒，依据包括：

- 存在卖家用户手册、连线说明、调试手册
- 存在板级配置与固件文件
- 存在可直接修改的 MicroPython 例程
- 存在 UART 控外部小车的协同样例

它也不是低风险成熟模块，依据包括：

- 固件版本口径不统一
- 协议语义偏演示流程
- 板级与引脚资源明显绑定卖家平台
- 未看到面向系统协同的稳定任务接口

## 3. 当前已证明的能力

基于例程和说明文档，当前已证明的平台能力包括：

- 视觉追踪抓取
- 红黄蓝目标分拣
- 手势触发夹爪动作
- OpenMV 视觉识别与舵机控制
- UART 控外部小车的协同雏形

对应本地证据：

- `本地来源：J-jixiebi/.../claw_catch.py`
- `本地来源：J-jixiebi/.../aoto_catch_rby.py`
- `本地来源：J-jixiebi/.../gesture_recognition.py`
- `本地来源：J-jixiebi/.../uart1_car_control.py`

## 4. 对主线有价值的能力

如果 J 线并入实验室清扫巡检系统，当前最有价值的不是“它能演示很多模式”，而是以下几类能力：

- 小件拾取
- 遗落物清理
- 拨动 / 试探性局部操作
- 视觉引导近场处置

这些能力与 `C` 线可以形成的系统级协同是：

- `C` 线负责到位、停靠、接近和安全承载
- `J` 线负责局部执行动作
- 异常点由 `C` 线带到可操作位置，再由 `J` 线做近场操作

## 5. 展示价值大于主线价值的内容

以下内容当前更多体现卖家展示能力，而不是主线优先方向：

- 手势演示
- 卖家 APP 模式切换流程
- 泛化云台 / YOLO / UI 展示案例

原因：

- 它们能证明平台“能做很多事”
- 但不能直接证明它们对实验室清扫巡检主线具有不可替代价值

## 6. 最值得保留的二开方向

当前最值得保留并进入后续二开的方向应冻结为：

- 视觉引导抓取链路
- 颜色 / 目标分拣
- 与 `C` 线的串口 / 任务协同接口

原因：

- 这三类能力最接近“能为主线解决问题”的系统价值
- 也最容易从卖家演示能力过渡到我方受控实现

## 7. 当前最大风险

### 7.1 缺少 `C <-> J` 系统级任务 / 状态协议

当前看到的协议主要是：

- 手机到 OpenMV 的单字节蓝牙命令
- OpenMV 到外部 MCU 的零散 UART 帧

但仍缺少：

- 主线任务下发
- 状态回传
- 执行结果上报
- 异常与失败处理

### 7.2 卖家固件版本与板级定义绑定

当前资料直接包含：

- `openmv.bin`
- `omv_boardconfig.h`
- `mpconfigboard.h`

同时又存在固件版本差异要求。

结论：

- 后续若迁移固件、迁移板卡或回归标准 OpenMV 体系，都会产生额外成本

### 7.3 APP / 蓝牙协议语义偏演示，不是主线任务接口

蓝牙文档中的单字节语义同时承担：

- 方向控制
- 模式切换
- 抓爪开合

这类协议适合展示，但不适合系统级调度。

### 7.4 通信、LCD、WiFi、机械爪存在 IO 资源冲突

当前资料显示：

- 抓爪、LCD、WiFi、串口等都在争有限引脚资源

结论：

- 在最小硬件形态冻结前，不应假设可以同时加载所有卖家附件

### 7.5 视觉稳健性依赖阈值与场景调参

多个例程都要求：

- 标定颜色阈值
- 调整镜像 / 翻转
- 按场景调 PID 或测距参数

结论：

- 当前“能跑通例程”不等于“能稳定完成实验室任务”

## 8. 并线前必补项

如果 J 线要真正并入 `C` 线主线，下一步最需要补的不是新花样，而是基础接口与边界：

- 任务接口
- 状态回传
- 动作原语抽象
- 最小硬件形态冻结
- 稳定性验证

具体来说，应至少解决：

- `C` 如何向 `J` 下发动作任务
- `J` 如何回传完成 / 失败 / 中断状态
- 哪些动作是稳定可复用的动作原语
- 哪些外设必须保留，哪些应从最小形态中拿掉
- 在不同光照、目标和供电条件下，抓取链路是否稳定

## 9. 当前阶段结论

J 线当前不是“不能做”，也不是“已经能直接接主线”。

更准确的判断是：

- 它已经证明卖家平台具备视觉驱动三自由度机械臂的二开潜力
- 它最有价值的是近场操作能力，而不是卖家演示流程本身
- 它能否并入主线，关键不在“再跑一个例程”，而在“把平台能力变成受控的系统子接口”

因此，J 线在 `C-1.1.6` 之后的正确方向，不是继续堆展示例程，而是进入：

- 并线接口定义
- 最小硬件形态冻结
- 最小可复用动作链路验证
