## 5. 资料冲突与不稳定项

### 5.1 固件版本口径不稳定

- `固件说明.txt` 推荐使用 `6.1.0`
- 但部分 UI / 综合案例要求 `6.5.5`

结论：

- 卖家案例并不稳定运行在单一统一固件版本上

### 5.2 板级与引脚口径为卖家专用口径

- 资料多次出现 `MENGFEI_OPENMV4`
- 舵机、LCD、WiFi、串口等功能复用在同一批引脚上

结论：

- 这是卖家深度二开的板级平台，不应当按标准 OpenMV 官方板口径直接理解

### 5.3 蓝牙 / APP 语义绑定卖家演示流程

- 蓝牙协议中的单字节指令同时承担方向、模式切换、机械爪动作等含义
- APP 操作流程围绕卖家预设的展示模式编排

结论：

- 现有协议更像卖家展示链路，不是现成的系统级任务接口

### 5.4 引脚资源存在复用冲突风险

- 抓爪、LCD、WiFi、串口等都可能占用有限引脚
- 机械爪、图传、通信同时接入时存在明显资源竞争

结论：

- 并线前必须冻结最小硬件形态，不能无约束叠加外设

## 6. 本轮归纳结论

- J 线资料覆盖的不只是机械臂动作脚本，而是一整套卖家 OpenMV 平台生态
- 其中真正对主线有判断价值的，不是通用 OpenMV 教材，而是：
  - 机械臂安装 / 接线 / 调试口径
  - 蓝牙与串口控制路径
  - 追踪抓取、颜色分拣、手势控制、串口控车四类例程
  - 卖家板级与固件资料
- 因此，J 线本轮后续应该进入“系统适配判断”，而不是直接进入功能开发
