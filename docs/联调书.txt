发现一个实际问题，小车在左右转的时候没法正常运动，前进后退和平移都是正常，但是左右转的时候左前和右后始终与正常逻辑相反，以左转为例：需要左前、左后负，右前、右后正；而实际上是左前右前正、左后右后负。你先来查一下是不是你这里的速度给错了？如果你这里的速度没有错，那就要去查树莓派那边对应节点是不是速度给错了

V 线这里没有把左右转速度给错。

当前 backend preset 是：

turn_left:
linear.x = 0
linear.y = 0
angular.z = +0.8

turn_right:
linear.x = 0
linear.y = 0
angular.z = -0.8
edge-relay 下发帧也是原样：

{
  "op": "manual_control",
  "vx": 0,
  "vy": 0,
  "wz": 0.8,
  "holdMs": 400
}
所以 V 线只是在发标准 Twist 语义：

左转 = wz 正
右转 = wz 负
你描述的错误现象：

期望左转：左前/左后负，右前/右后正
实际：左前/右前正，左后/右后负
这不是 V 线发错 vx/vy/wz，而是树莓派侧把 wz 分解到四轮时，轮位映射或左右/前后轮编号语义错了。尤其“左前右前同号、左后右后同号”更像是把“前后轴”当成“左右侧”在做旋转分解。

给树莓派侧排查口径：

V 线左转只发 manual_control: vx=0, vy=0, wz=+0.8；右转只发 wz=-0.8。若左转时实际轮向变成前排同号、后排同号，问题在 Pi 侧 cmd_vel -> 四轮目标速度分解或轮位编号映射，不在前端/backend。请检查 RF1 当前轮位顺序是否仍为 CN1/LR、CN2/LF、CN3/RR、CN4/RF，以及 wz 项在 mecanum/omni 分解公式里的符号分配。
