# C-1.2.0 C↔J 通信协议冻结

## 1. 协议分层

本轮协议故意拆成两层：

1. `UNO <-> F411`
   极简 ASCII 行协议，目标是补丁式接入和现场可观察
2. `F411 <-> J(OpenMV)`
   定界帧 + `CRC-8/ATM`，目标是真正的全双工实验协议

## 2. UNO ↔ F411

- 物理口：`UNO A3/A4` 软件串口 <-> `F411 USART2(PA2/PA3)`
- 串口参数：`9600 8N1`
- 传输形式：ASCII 行协议，`
` 作为结束符

### 2.1 命令集合

- `PING`
- `PONG`
- `STOP_REQ`
- `STOP_ACK`
- `RESUME_REQ`
- `RESUME_ACK`

### 2.2 语义

- `STOP_REQ`
  F411 请求 UNO 立即停车并进入等待态
- `STOP_ACK`
  UNO 已停车并拉起外部暂停闸门
- `RESUME_REQ`
  F411 请求 UNO 释放等待态，恢复默认流程
- `RESUME_ACK`
  UNO 已释放等待态
- `PING/PONG`
  仅用于链路健康检查

### 2.3 超时

- `STOP_ACK` 等待：`1000 ms`
- `RESUME_ACK` 等待：`1000 ms`

## 3. F411 ↔ J(OpenMV)

- 物理口：`F411 USART1(PA9/PA10)` <-> `OpenMV UART1(A9/A10)`
- 串口参数：`9600 8N1`
- 协议端点：本轮直接冻结为 `OpenMV UART1`，独立 `J-STM32` 收口层留到下一轮

### 3.1 帧格式

```text
0x43 0x4A | ver:u8 | seq:u8 | type:u8 | len:u8 | payload | crc8
```

字段说明：

- `0x43 0x4A`
  固定帧头，对应 `CJ`
- `ver`
  版本号，本轮固定 `0x01`
- `seq`
  发送序号，单字节循环
- `type`
  消息类型
- `len`
  `payload` 字节数
- `crc8`
  `CRC-8/ATM`，覆盖 `ver..payload`

### 3.2 J -> F411 消息类型

- `0x10 COLOR_FOUND(color_id)`
- `0x11 ARM_BUSY(state)`
- `0x12 PICK_DONE(color_id)`
- `0x13 PICK_TIMEOUT(color_id)`
- `0x14 ARM_FAIL(reason)`
- `0x15 HEARTBEAT(state)`

### 3.3 F411 -> J 消息类型

- `0x80 ACK(acked_seq, acked_type)`
- `0x81 PICK_WINDOW(timeout_ms_le16)`
- `0x82 ABORT(reason)`
- `0x83 PING`

### 3.4 冻结规则

- 同一时刻只允许一个机械臂活动任务
- `COLOR_FOUND` 只有在 `IDLE` 状态下有效
- `PICK_WINDOW` 本轮固定下发 `10000 ms`
- `ABORT` 本轮只要求在 `WAIT_PICK_WINDOW` 阶段保证生效
- `C-1.2.2` 起，`PICK_DONE / PICK_TIMEOUT / ARM_FAIL` 在 `WAIT_UNO_STOP_ACK` 阶段也必须被接住，不能直接丢弃

## 4. 端到端时序

```text
J: COLOR_FOUND
F411 -> UNO: STOP_REQ
UNO -> F411: STOP_ACK
F411 -> J: PICK_WINDOW(10000)
J -> F411: ARM_BUSY
J -> F411: PICK_DONE / PICK_TIMEOUT / ARM_FAIL
F411 -> UNO: RESUME_REQ
UNO -> F411: RESUME_ACK
```

### 4.1 C-1.2.2 死结解锁补充

`C-1.2.2` 没有推翻上面的主优先路径，而是增加了 bring-up 过渡策略：

1. `J(OpenMV)` 仍然先上报 `COLOR_FOUND`
2. `J(OpenMV)` 仍然优先等待 `PICK_WINDOW`
3. 若在本地回退阈值内未收到 `PICK_WINDOW`，且目标仍在视野中，则允许进入一次本地抓取回退
4. 本地回退抓取后的结果仍然必须回发：
   - `PICK_DONE`
   - `PICK_TIMEOUT`
   - `ARM_FAIL`
5. `F411` 在以下两个状态都允许接收 J 线终态结果：
   - `WAIT_UNO_STOP_ACK`
   - `WAIT_J_RESULT`

这一步的目的不是废弃通信主线，而是解除当前“J 等放行 / F411 等 STOP_ACK”的联锁死结。

## 5. F411 状态机

- `IDLE`
- `WAIT_UNO_STOP_ACK`
- `WAIT_J_RESULT`
- `WAIT_UNO_RESUME_ACK`
- `FAULT`

超时定义：

- J 本地抓取超时：`10000 ms`
- F411 外层 watchdog：`12000 ms`

## 6. 当前实现与下一轮遗留

当前仓内已落地：

- UNO ASCII 行协议解析
- F411 状态机、环形缓冲区和定界帧协议源码骨架
- J 侧 `COLOR_FOUND / PICK_WINDOW / PICK_DONE / PICK_TIMEOUT / ARM_FAIL / HEARTBEAT` 实验链路
- `C-1.2.2` 起，J 侧增加“优先等待放行，超时后本地回退抓取”的过渡策略

下一轮遗留：

- F411 真实板级初始化和烧录验证
- J 侧由 OpenMV 直出过渡到 `J-STM32` 统一对外接口
- 超时重发和日志输出的硬件级验证
