# C-1.2.1 F411 平台层补齐说明

## 1. 本轮目标

本轮不改 `cj_bridge.c` 的桥接状态机，只补 `F411` 的最小板级平台层，让现有状态机第一次具备真实串口收发能力。

冻结口径：

- `USART2(PA2/PA3)` 对接 `UNO`
- `USART1(PA9/PA10)` 对接 `OpenMV UART1`
- `9600 8N1`
- `HSI 16MHz`
- `PB2` 继续作为 bring-up 心跳与状态灯

## 2. 已补齐的平台项

### 2.1 UART GPIO 与时钟

已在 [main.c](/f:/Project/CleanScout_rover/firmware/cj_bridge_f411_cubeide/Core/Src/main.c) 中补齐：

- `GPIOA` 时钟
- `USART1` / `USART2` 时钟
- `PA2/PA3/PA9/PA10` 复用到 `AF7`
- `PA3`、`PA10` 作为接收脚启用上拉

### 2.2 UART 最小初始化

平台层已实现：

- `platform_init_uart_uno()`
- `platform_init_uart_j()`
- `platform_send_uno_line()`
- `platform_send_j_bytes()`

首版 TX 采用阻塞发送，RX 采用中断收字节。

### 2.3 IRQ 与接收路径

已补齐：

- `USART1_IRQHandler`
- `USART2_IRQHandler`
- 启动文件中的 `USART1/USART2` 向量项

中断处理策略：

- `RXNE` 置位后读取 `DR`
- `USART2` 字节推入 `usart2_rx_isr()`
- `USART1` 字节推入 `usart1_rx_isr()`
- 错误位累计到诊断计数器

### 2.4 SysTick 与 keepalive

已保留并继续使用 `SysTick` 毫秒节拍。

在 `IDLE` 状态下新增最小 keepalive：

- 若 `UNO` 链路未确认，每 `1000 ms` 发送一次 `PING\n`
- 若 `J` 链路未确认，每 `1000 ms` 发送一次 `PING` 帧

这一步仅用于 Gate1-Gate3 bring-up，不改变现有主协议集合。

## 3. 现场可观测性

### 3.1 PB2 灯码

`PB2` 保留为 active-low 指示脚。

灯码定义：

- `IDLE`：1 短闪
- `WAIT_UNO_STOP_ACK`：2 短闪
- `WAIT_J_RESULT`：3 短闪
- `WAIT_UNO_RESUME_ACK`：4 短闪
- `FAULT`：快速连续闪

### 3.2 诊断变量

`main.c` 内新增 `volatile` 诊断变量，供 ST-LINK 观察：

- 当前桥接状态
- 最近状态迁移时间
- 最近日志原因
- 最近 fault 原因
- `USART1/USART2` RX/TX 字节计数
- `USART1/USART2` 错误计数
- `UNO` `PONG/STOP_ACK/RESUME_ACK` 计数
- `J` `ACK/HEARTBEAT/frame` 计数
- `UNO/J` 链路确认标志
