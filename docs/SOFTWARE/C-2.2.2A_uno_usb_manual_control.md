# C-2.2.2A UNO USB 手动控制极简说明

## 当前口径
- 主控草图：`sketches/c002_uno_baseline/c002_uno_baseline.ino`
- 通信入口：`USB Serial`
- 波特率：`115200`
- 控制模式：单模式手动上位控制
- 失联自动停车：`400ms`

## 当前支持字符命令
- `8`：前进
- `2`：后退
- `4`：左转
- `6`：右转
- `5`：停止
- `7`：左前
- `9`：右前
- `1`：右后
- `3`：左后

## 当前已裁掉的功能
- `AUTO` 自动模式
- `GEST` 体感模式
- 蓝牙入口 `softSerial(9, 2)`
- 桥接串口 `bridgeSerial(A3, A4)`
- 外部暂停桥接协议
- 风机自动逻辑与 `CleanScoutFan`
- 超声自动避障与相关转向流程
- 头部舵机初始化与转头调用

## 当前释放出来的引脚
- `D2`
- `D9`
- `D10`
- `A0`
- `A1`
- `A2`
- `A3`
- `A4`

## 当前为兼容 Tyler_1 构造保留的占位脚
- `A5`：超声 trig 占位
- `D13`：超声 echo / 舵机占位

## 当前不是最终库边界收口版本
本轮只裁主控草图，不改 `Tyler_1` 库。为了不动库 API，草图通过 `A5` 与 `D13` 作为占位脚，绕开原先对 `A0`、`A1`、`D10` 的直接依赖。

## 后续库改造要求（必须立项）
后续必须单独修改 `Tyler_1` 库，不能长期停留在占位脚绕开方案。项目负责人需要将以下内容立项：
- 把超声相关 `trig/echo` 从 `Tyler_1` 构造函数中拆出
- 把头部舵机接口从 `Tyler_1` 库内部耦合中拆出
- 取消库构造阶段对超声相关引脚的 `pinMode()` 副作用
- 形成真正面向 `C-2.x` 的纯底盘执行库边界

原因很明确：当前 `Tyler_1` 构造函数仍会对超声相关引脚做 `pinMode()`，库内部仍内置舵机与超声接口。本轮只是通过占位脚绕开，不是根治。
