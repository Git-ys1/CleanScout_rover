# C-3.6.0：OpenRF1 正式固件工程收口

## 1. 本轮目标

将长期存在的“`work2` 提供 Keil 工程、跨目录编译 `work3/User`”模式，
收口为仓库内可独立编译、可追溯、可阅读的正式 RF1 固件工程：

```text
firmware/openrf1_motion_controller/
```

本轮以 `_local/car_move_lihaotian/2-1 STM32工程模板` 的最新增量 PI 版本
为行为来源，但不原样提交其中的模板污染和构建产物。

## 2. 原工程审计结论

### 2.1 目标芯片配置错误

原工程配置为：

- Device：`STM32F103C8`
- 启动文件：`startup_stm32f10x_md.s`
- Flash：64KB
- SRAM：20KB

实际 OpenRF1 为 `STM32F103RCT6`。正式工程已更正为：

- Device：`STM32F103RC`
- 宏：`STM32F10X_HD`
- 启动文件：`startup_stm32f10x_hd.s`
- Flash：256KB
- SRAM：48KB

### 2.2 同一模块存在两套源码真值

原工程根目录和 `User/` 同时存在 `csr_board_map.c/h` 等文件，且 Keil：

- 编译根目录 `csr_*.c`
- 编译 `User/main.c`
- `User/main.c` 包含 `User/csr_board_map.h`
- 根目录驱动包含根目录 `csr_board_map.h`

两份头文件的轮径、分辨率、PWM、PI 参数不同；两份
`csr_board_map.c` 的电机和编码器方向表也相反。

正式工程只保留：

```text
User/csr_board_map.c
User/csr_board_map.h
```

参数采用最新 `User/` 版本，方向表采用原 Keil 实际编译的根目录版本，
从而保持原固件实际行为，同时建立单一源码真值。

### 2.3 模板与构建污染

原目录包含：

- `Objects/`
- `Listings/`
- 多份 `uvguix`
- 重复应用源码
- 完整但未使用的 SPL 模块

正式工程只保留实际使用的 CMSIS、GPIO/RCC/TIM/USART/NVIC 驱动。
本机构建产物统一进入 `Build/` 并由 Git 忽略。

## 3. 控制代码整理

本轮保持最新增量 PI 控制策略：

```text
delta_pwm = Kp * (error - previous_error) + Ki * error * dt
pwm       = clamp(previous_pwm + delta_pwm)
```

当前参数：

| 参数 | 值 |
| --- | ---: |
| 控制周期 | 20ms / 50Hz |
| 编码器分辨率 | 1768 |
| 轮径 | 0.06m |
| 目标加速度上限 | 2.5m/s² |
| Kp | 200 |
| Ki | 2500 |
| PWM 上限 | 1000 |
| 速度低通 alpha | 0.35 |

同时修正以下实现问题：

1. 删除未参与控制的第二套 `g_smooth_target_vel`。
2. 控制和 `NAVDBG` 统一使用 `g_target_vel_ramp`。
3. STOP/模式切换会同步清零斜坡目标和增量 PI 累计输出。
4. RAW 模式在控制周期中明确保持原始 PWM。
5. 删除未使用的 `Kd`、旧位置式 PI 参数和失效的 PWM slew 函数。
6. 中文注释统一为 UTF-8，并补充轮序、参数单位、硬件映射和安全边界。

## 4. 工程入口

Keil 工程：

```text
firmware/openrf1_motion_controller/OpenRF1_Motion.uvprojx
```

编译：

```powershell
.\firmware\openrf1_motion_controller\scripts\build.ps1
```

烧录：

```powershell
.\firmware\openrf1_motion_controller\scripts\flash.ps1
```

## 5. 验证结果

本轮在 Keil ARMCC 5.06 update 5 下完成独立编译：

```text
Program Size: Code=16896 RO-data=552 RW-data=144 ZI-data=1376
0 Error(s), 0 Warning(s)
```

生成文件：

```text

```

本轮只完成工程结构、源码真值和可读性收口；新的 HEX 尚需在实车上完成
串口、四轮方向、编码器速度和六方向闭环回归后，再作为新的板端发布基线。

## 6. 后续纪律

1. RF1 正式开发只修改 `firmware/openrf1_motion_controller/User/`。
2. `_local/openrf1_keil_work_2`、`work3` 和 `car_move_lihaotian` 仅作历史证据。
3. 不再跨目录编译，不再维护重复 `csr_*` 文件。
4. 新参数必须写清单位、控制含义和实测依据。
5. 构建产物、个人 Keil 配置不得进入 Git。
