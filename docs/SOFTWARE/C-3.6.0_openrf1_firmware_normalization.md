# C-3.6.0：OpenRF1 正式固件工程收口

## 目标

把长期存在的“旧 Keil 工程跨目录编译 `_local` 源码”收口为仓库内唯一、
自包含的正式工程：

```text
firmware/openrf1_motion_controller/
```

本轮是工程迁移和可读性整理，不是控制算法改版。已收敛的增量 PI、目标平滑、
方向表、PWM 真值、协议和看门狗行为均按实际编译源码保留。

## 行为来源

审计对象：

```text
_local/car_move_lihaotian/2-1 STM32工程模板/Project.uvprojx
```

原工程实际编译的是：

- `User/main.c`
- 工程根目录的 `csr_board_map.c`
- 工程根目录的 `csr_motor_drv.c`
- 工程根目录的 `csr_encoder_drv.c`
- 工程根目录的 `csr_proto.c`

其中 `main.c` 使用 `User/csr_board_map.h`，根目录驱动使用根目录头文件。
正式工程消除了这套重复目录结构，但参数和方向真值按原 Keil 实际输入组合保留。

已对 9 组应用源码执行“去注释、去空白后逐字符比较”，结果全部等价：

```text
main.c                  PASS
csr_board_map.c/.h      PASS
csr_motor_drv.c/.h      PASS
csr_encoder_drv.c/.h    PASS
csr_proto.c/.h          PASS
```

因此，本轮新增中文注释、UTF-8 转码和目录收口没有改动 C 语句。

## 工程配置修正

旧模板的目标配置误写为 STM32F103C8、64KB Flash、20KB SRAM 和中密度启动
文件。OpenRF1 实际 MCU 为 STM32F103RCT6，正式工程使用：

- Device：`STM32F103RC`
- 宏：`STM32F10X_HD`
- 启动文件：`startup_stm32f10x_hd.s`
- Flash：256KB
- SRAM：48KB

这属于目标芯片配置纠正，不改变应用控制策略。

## 后续死代码清理

在确认运行路径后，后续清理仅删除未参与运行计算的兼容项：

- 只保留 `g_target_vel_ramp` 作为 PI 的平滑目标，不再保留第二套空目标数组。
- 删除未调用的 `g_kd`、`csr_slew_i16()`、`csr_ramp_target()`。
- 删除未使用的前馈、修正量、积分限幅和 PWM step 宏。
- `NAVDBG` 收口为实际数据：`cmd/ramp/rt/out`。
- `g_integral_state` 仍保存当前增量 PI 的累计 PWM，算法不变。
- RAW 模式仍由 `M` 命令直接输出，不进入速度闭环。

## 目录治理

- 当前 RF1 主线：`firmware/openrf1_motion_controller/`
- 早期 RF1 overlay：`firmware/archive/openrf1_keil_overlay/`
- `_local/`：本地实验和历史证据，不再作为发布或编译入口

## 编译与烧录

```powershell
.\firmware\openrf1_motion_controller\scripts\build.ps1
.\firmware\openrf1_motion_controller\scripts\flash.ps1
```

如需把告警也视为失败：

```powershell
.\firmware\openrf1_motion_controller\scripts\build.ps1 -StrictWarnings
```

清理后的目标是 `0 Error(s), 0 Warning(s)`；若 Keil 仍产生告警，应先确认告警
是否来自实际运行路径，不得为了消除告警修改已收敛控制算法。

## 后续纪律

1. RF1 正式源码只在 `firmware/openrf1_motion_controller/User/` 维护。
2. 不再从 `_local/openrf1_keil_work_2`、`work3` 或 `car_move_lihaotian` 发布。
3. 注释和新源码统一使用 UTF-8。
4. 工程整理任务不得夹带 PI、PWM、方向或协议行为改动。
5. 任何控制逻辑调整必须单独设 checkpoint、编译、烧录并完成实车回归。
