# C-1.2.6 Motion Mapping Notes

## 1. Runtime Truth Source

The only runtime truth for motion in C-1.2.6 is the actuator boundary set frozen in C-1.2.5:

- `PAN_CENTER_DEG = 0`
- `PAN_MIN_DEG = -90`
- `PAN_MAX_DEG = 90`
- `TILT_CENTER_DEG = 85`
- `TILT_MIN_DEG = 0`
- `TILT_MAX_DEG = 90`
- `CLAW_OPEN_ANGLE = -60`
- `CLAW_CLOSE_ANGLE = 40`

## 2. Vendor Angle Mapping Replaced

Vendor primary script uses:

- initial `pan = 90`
- initial `tilt = 30`
- runtime tilt clamp `[-80, 80]`
- claw release/catch `-70 / 50`

C-1.2.6 replaces them with:

- search pose `pan = 0, tilt = 85, claw = open`
- tilt clamp `[0, 90]`
- claw open/close `-60 / 40`

## 3. Vendor Sleeps Normalized

Confirmed vendor timing problems:

- `time.sleep(2000)` after initial pan move
- multiple `time.sleep(1000)` and `time.sleep(500)` in grab/drop path

These are treated as millisecond intents written with second API calls.

Runtime replacements:

- `time.sleep_ms(CLAW_SETTLE_MS)`
- `time.sleep_ms(LIFT_SETTLE_MS)`
- `time.sleep_ms(DROP_SETTLE_MS)`
- `time.sleep_ms(RESET_SETTLE_MS)`

## 4. Distance Proxy Freeze

Distance proxy remains derived from vendor main script:

- `distance_proxy = 13500 / (blob.w * 2)`
- grasp window: `90 <= distance_proxy <= 110`

## 5. F411 Pause Note

F411 joint-debug is intentionally paused in C-1.2.6.
The vision/grasp baseline must be validated locally first, then future command-source takeover can resume from this frozen baseline.
