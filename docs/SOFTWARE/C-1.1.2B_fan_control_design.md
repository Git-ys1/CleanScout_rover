# C-1.1.2B Fan Control Design

## 1. 目标

`C-1.1.2B` 的目标是在不改默认模式、不改模式系统逻辑的前提下，为自动模式新增风机智能控制能力，并将该能力封装为独立模块。

当前硬件冻结口径沿用 `C-1.1.1`：

- 控制引脚：`A2`
- 继电器：`HW-803`
- 触发方式：高电平触发
- 电平语义：
- `A2 = HIGH`：继电器吸合，风机启动
- `A2 = LOW`：继电器释放，风机关闭

## 2. 模块职责

本轮新增模块：

- `libraries/CleanScoutFan/CleanScoutFan.h`
- `libraries/CleanScoutFan/CleanScoutFan.cpp`

模块负责：

- `A2` 输出引脚初始化
- 风机开关输出
- 风机状态管理
- 3 秒冷却计时
- 6 秒运行计时
- 障碍物中断暂停
- 转弯完成后的重新冷却
- 非自动模式下强制关闭
- 最小状态查询

模块不负责：

- 模式切换逻辑
- 蓝牙协议
- 底盘运动控制
- 障碍物判定策略

## 3. 状态机

状态枚举固定为：

| 状态 | 含义 | 输出 |
| --- | --- | --- |
| `DISABLED` | 当前不在自动模式 | 风机关闭 |
| `COOLDOWN` | 自动模式下等待 3 秒冷却 | 风机关闭 |
| `RUNNING` | 自动模式下运行 6 秒 | 风机开启 |
| `PAUSED_BY_OBSTACLE` | 障碍物触发暂停，等待转弯完成 | 风机关闭 |

## 4. 状态迁移

| 当前状态 | 触发条件 | 下一状态 | 说明 |
| --- | --- | --- | --- |
| `DISABLED` | 进入 `AUTO` 并调用 `update()` | `COOLDOWN` | 重新开始完整 3 秒冷却 |
| `COOLDOWN` | 计时满 3000 ms | `RUNNING` | 打开风机 |
| `RUNNING` | 计时满 6000 ms | `COOLDOWN` | 关闭风机并重新冷却 |
| `COOLDOWN` / `RUNNING` | `pauseByObstacle()` | `PAUSED_BY_OBSTACLE` | 立刻关闭风机，原计时作废 |
| `PAUSED_BY_OBSTACLE` | `notifyTurnCompleted()` 后下一次 `update(AUTO, nowMs)` | `COOLDOWN` | 不立即启动，重新完整冷却 |
| 任意状态 | 当前模式不是 `AUTO` | `DISABLED` | 立即关闭，清空计时 |

## 5. 时序冻结

自动模式下的正常循环：

1. 风机关闭 3 秒
2. 风机运行 6 秒
3. 风机关闭 3 秒
4. 重复循环

障碍物打断规则：

1. 一旦检测到障碍物，立即调用 `pauseByObstacle()`
2. 风机立即关闭
3. 原有冷却或运行计时作废
4. 只有在完成左右探测、完成实际转弯，并调用 `notifyTurnCompleted()` 后，模块才允许在后续 `update()` 中重新进入 `COOLDOWN`
5. 恢复后仍需重新等待完整 3 秒，不能立即启动

## 6. 非自动模式行为

以下模式中风机统一关闭：

- 遥控模式
- 体感模式

一旦当前模式不是 `AUTO`：

- 风机立即关闭
- 状态回到 `DISABLED`
- 不保留旧计时
- 不做后台计时

## 7. 计时实现要求

模块内部统一使用：

- `millis()`

禁止使用：

- 基于 3 秒或 6 秒风机逻辑的长时间 `delay()`

说明：

- 当前自动避障逻辑内部仍存在既有 `delay()`，这是原有模式实现的一部分。
- 本轮新增的风机控制逻辑本体不采用阻塞式计时。

## 8. 调试策略

模块预留编译期开关：

- `#define CLEANSCOUT_FAN_DEBUG 0`

当开关打开时，可输出：

- 当前状态切换
- 障碍暂停事件
- 转弯完成事件
- 风机开关时刻

默认保持关闭，不新增 `Serial.begin()`，不让调试输出成为业务依赖。
