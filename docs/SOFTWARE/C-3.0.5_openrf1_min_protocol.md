# C-3.0.5：OpenRF1 最小协议与自写底层说明

## 1. 当前定位

本轮是 OpenRF1 的最小自写底层 bringup，不做闭环，不做运动学，不做树莓派接入。

当前唯一目标：

- 用最小串口协议直接控制 `CN1~CN4`
- 直接读取四路编码器 `count/delta`
- 建立可靠的方向真值表

## 2. 当前自写模块

当前自写运行层位于：

- `_local/openrf1_keil_work_3/User/`

模块角色：

- `csr_board_map`：通道编号、轮位注释、方向符号表
- `csr_motor_drv`：四路 H 桥直驱
- `csr_encoder_drv`：四路编码器初始化、读取、清零
- `csr_proto`：`USART2` 最小协议解析与回传
- `main`：初始化、命令分发、超时停车

## 3. 最小协议

### 上位机 -> OpenRF1

#### 单通道驱动

```text
M,<ch>,<pwm>
```

约束：

- `ch ∈ {1,2,3,4}`
- `pwm ∈ [-1000,1000]`
- 目标是持久的，直到收到新的同通道命令或 `STOP`

示例：

```text
M,1,300
M,1,-300
M,3,0
```

#### 查询编码器

```text
E,<ch>
```

示例：

```text
E,1
E,4
```

#### 全部停车

```text
STOP
```

### OpenRF1 -> 上位机

#### 启动完成

```text
CSR_RF1_READY
```

#### 合法命令

```text
ACK:M
ACK:E
ACK:STOP
```

#### 非法命令

```text
ERR:<reason>
```

典型错误：

- `ERR:bad_prefix`
- `ERR:bad_channel`
- `ERR:arg_count`
- `ERR:parse_int`
- `ERR:range`
- `ERR:line_too_long`

#### 编码器回传

```text
ENC,<ch>,<count>,<delta>
```

说明：

- `count`：软件累计计数
- `delta`：本次查询窗口增量，查询后自动清零硬件计数器

## 4. 当前安全口径

- 串口：`USART2 @ 115200 8N1`
- 超时：`2000ms`
- 超时动作：清零四路目标并停车
- 输入 `pwm` 不是直接满量程输出：
  - 输入 `[-1000,1000]`
  - 当前实际输出限幅到 `300~700`
- 目的不是线性速度，而是先保证单通道可控、可观察、可停下
