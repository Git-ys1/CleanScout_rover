# C-2.2.8A UNO 侧 M3/M4 诊断说明

## 1. 本轮定位

`C-2.2.8A` 只处理 `UNO` 本地诊断，不改树莓派协议，不推进 `cmd_vel -> W`，不做 `odom`。

本轮唯一目标是把：

- `M3` 的方向问题
- `M4` 的 `Port B / D9,D10` 输入链问题

拆开诊断清楚。

## 2. 当前起始事实

当前本地基线已经确认：

- `UNO` 协议继续为 `W,w1,w2,w3,w4`
- `PID,...` 输出已经在代码里存在
- 你本地实测表明：`ENC_SIGN_W3 = -1` 后，`M3` 与 `M1/M2` 的速度明显更接近
- `W4` 当前已确认不是 `D9/D10`/`PCINT0` 先天不支持，而是电机驱动板接触不良导致该轮不转
- 接触问题排除后，执行 `W,500,500,500,500` 时，四轮已经可以齐转
- 当前联调参数已经推进到：
  - `WHEEL_KP = 2.0`
  - `WHEEL_KI = 0.5`

因此本轮不再把“补 PID 输出”当任务，也不再把“交换 M3 的 A/B”当优先修法。

## 3. M3 处理口径

### 3.1 物理真相保持不变

`wheel3` 继续保持：

- `pinA = A4`
- `pinB = A5`

不采用长期 A/B 交换方案。

### 3.2 方向修正

当前冻结：

- `ENC_SIGN_W3 = -1`

这代表 `M3` 当前优先按“方向符号错误”处理，而不是“引脚定义错误”处理。

### 3.3 验证方法

当前只用低速：

- `W,80,80,80,80`
- `W,120,120,120,120`

验收看的是：

- `ENC` 中 `v3` 与 `v1/v2` 是否同号
- `PID` 中 `meas3/out3` 是否还出现明显反向放大
- 低速下 `M3` 是否还明显暴冲

## 4. M4 处理口径

### 4.1 先 raw，后闭环

本轮新增编译期开关：

```cpp
#define CSR_WHEEL4_RAW_PROBE 1
```

raw 模式下：

- 不跑整车闭环
- 不跑四轮 `PI`
- 只初始化 `wheel4`
- 持续输出：

```text
RAW4,ticks,currentA,currentB
```

其中：

- `ticks`：当前累计 tick
- `currentA`：D9 实时电平
- `currentB`：D10 实时电平

### 4.2 当前物理真相

`wheel4` 继续保持：

- `pinA = D9`
- `pinB = D10`

当前不允许先把 `D9/D10` 判成“Uno 不支持”，也不允许跳过 raw 直接调 `PI`。

### 4.3 结论分支

#### A. raw 无 tick

若正拨/反拨 `wheel4` 时：

- `ticks` 不变化

则本轮结论写为：

- `Port B 输入链未通过`

下一步优先查：

- `D9` 到 Servo #1 信号针脚导通
- `D10` 到 Servo #2 信号针脚导通
- 接头/焊点/插接稳定性

#### B. raw 有 tick

若正拨/反拨时：

- `ticks` 有变化

则说明：

- `Port B / PCINT0` 路径是通的

下一步再回闭环模式，优先测试：

- `ENC_SIGN_W4 = -1` 是否更合理

### 4.4 当前阶段结论

当前 `wheel4` 的首因已经明确为：

- 电机驱动板接触不良

因此本轮阶段性结论更新为：

- `RAW4` 探测入口仍保留，作为后续复发时的快速隔离工具
- 但当前 `wheel4` 已不再停留在“Port B 输入链未通过”的待定状态
- 当前四轮均已能转动，`M4` 的主故障已从“编码器/中断能力猜测”收敛到“硬件接触问题已修复”

## 5. 当前协议边界

当前正常闭环模式的正式返回帧仍然是：

- `CSR_UNO_READY`
- `ACK:W`
- `ERR:<reason>`
- `ENC,...`
- `PID,...`

本轮新增的：

- `RAW4,...`

只属于本地诊断模式，不属于正式跨端协议契约。

## 6. 当前 timeout 与 PI 口径

当前继续保留：

- 正式 timeout：`400ms`
- 联调 timeout：较长 debug timeout

当前联调参数已经推进为：

- `WHEEL_KP = 2.0`
- `WHEEL_KI = 0.5`
- `D` 仍未启用

当前阶段现象已经确认：

- 发送 `W,500,500,500,500`
- 四轮可以齐转

这代表本地 `UNO` 端已经拿到“四轮编码器 + PI 闭环第一版可运行”的阶段性成果。

但当前仍不把它写成最终冻结值，后续还需要：

- 继续低速验证 `ENC/PID` 曲线稳定性
- 再决定是否进一步调 `Kp/Ki`

## 7. 本轮验收标准

### 7.1 M3

- 保持 `A4/A5` 物理真相
- 通过 `ENC_SIGN_W3 = -1` 修正方向
- 低速下 `ENC/PID` 表现与 `M1/M2` 接近

### 7.2 M4

- 当前主故障已经定位为驱动板接触不良
- 接触问题排除后，`wheel4` 已恢复转动
- `RAW4` 探测入口继续保留，作为后续复发时的快速定位工具

## 8. 当前阶段成果

截至本轮，当前可对项目负责人明确汇报的阶段性结果为：

- `M3` 已通过 `ENC_SIGN_W3 = -1` 明显改善，与 `M1/M2` 更接近
- `M4` 的首因已定位为电机驱动板接触不良，不是 `D9/D10`/`PCINT0` 先天不可用
- 接触问题排除后，四轮均已可转动
- 当前 `PI` 联调参数为 `Kp=2.0`、`Ki=0.5`
- 在 `W,500,500,500,500` 指令下，四轮已实现齐转
- 这标志着当前小车已经完成“轮级闭环第一版可运行实现”的阶段性里程碑

## 9. 下一轮是否继续查 Port B

当前规则很简单：

- 若 `wheel4` 再次异常：优先用 `RAW4` 入口复核 `Port B` 输入链
- 若 `wheel4` 继续稳定：下一轮重点应转向更细的 `ENC/PID` 曲线观测与 `PI` 调参

也就是说，`M4` 本轮的首要目标不是“立刻跑通”，而是先把锅从“中断能力猜测”收敛成“Port B 物理链”还是“方向/控制链”的明确结论。
