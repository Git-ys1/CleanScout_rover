# C-3.0.4：OpenRF1 下位机迁移分析说明

## 1. 迁移结论

当前底盘下位机正式迁移判断已经锁定：

- `OpenRF1 (STM32F103RCT6)` 取代 `UNO` 成为新的底盘下位机主线
- `UNO` 不再继续承担当前四轮编码电机闭环主线优化
- `UNO` 仅保留历史事实、联调结论和必要回退证据

这不是“多开一条探索线”，而是当前底盘下位机的正式换代判断。

## 2. 为什么不再继续死磕 UNO

当前停止继续深挖 `UNO` 的原因，不是协议没有打通，也不是编码器采样完全不可用，而是平台性价比已经不足。

当前 `UNO` 线上已经暴露出的现实问题包括：

- 四轮编码器采样、轮速闭环、串口协议和后续树莓派上位通信全部堆在 `ATmega328P` 上，资源边界过紧
- `X1 / X2` 解码、轮间差异、单轮驱动异常、PI 整定互相缠绕，调试成本持续抬升
- 即使能继续做出原型，也很难成为后续树莓派上位接管、底盘闭环和稳定轮速控制的长期底座

因此，`UNO` 当前的定位应改写为：

- 已完成阶段性验证
- 不再继续作为底盘闭环主线平台

## 3. 为什么 OpenRF1 更适合当前任务

当前本地参考资料位于：

- [docs/STM32F103RCT6](f:/Project/CleanScout_rover/docs/STM32F103RCT6)

从开发手册和例程 9 的源码可确认，`OpenRF1` 具备以下与当前任务强相关的条件：

- 主控为 `STM32F103RCT6`
- 板上已有四路编码电机接口
- 板上已有四路电机驱动
- 例程 9 已具备四轮闭环骨架
- 编码器采样采用 `TIM2 / TIM3 / TIM4 / TIM5` 编码器接口模式
- 电机控制节拍已固定在 `20ms / 50Hz`
- 轮速接口已经直接使用 `m/s`

这意味着当前最有价值的并不是继续把 `UNO` 拉到更复杂的闭环深度，而是切换到已经更接近正式底盘控制器形态的 `OpenRF1`。

## 4. 迁移策略：底层复用，应用层重写

本轮不采用“整包继承 vendor 工程”的策略，也不采用“从零重写底层”的策略。

当前固定路线是：

- 复用 vendor 已验证的底层模块
- 重写我们自己的应用层和串口桥接层

底层复用范围冻结为：

- `y_encoder/*`
- `y_motor/*`
- `y_timer/*`
- `y_usart/*`

当前由我方接管的运行层范围冻结为：

- `app_motor.c/.h`
- `app_csr_bridge.c/.h`
- `app_chassis.c/.h`
- `app_telemetry.c/.h`

同时冻结当前轮位编号真相，避免继续把协议轮位和驱动通道混为一谈：

- `A = 左前 = M3`
- `B = 右前 = M2`
- `C = 左后 = M4`
- `D = 右后 = M1`

后续凡是讨论方向、测速、遥测和单轮测试，都必须先经过这层映射。

这样做的原因有三点：

1. 编码器、PWM 驱动、定时器节拍和串口底层已经在 vendor 例程中跑通，继续重写这一层收益不高。
2. vendor 的 `app_motor` 与其他应用层逻辑耦合过重，不适合作为当前极简底盘协议基线，所以这层要接管。
3. 公开仓库不能直接把 vendor 全家桶原样并入主线，因此需要一层自研 overlay 来承接当前实施线。

## 5. 为什么本轮不上树莓派、不上 ROS

当前迁移轮唯一目标是：

- 在电脑串口工具控制下，让四个轮子持续、平顺、可控、可回传

因此本轮明确不做：

- 不接树莓派
- 不接 ROS
- 不接 OpenMV
- 不接机械臂
- 不接额外传感器
- 不兼容旧 `UNO` 的 `ticks/s` 协议

原因很直接：

- 现在的主故障与主收益都在底盘下位机本体
- 如果在下位机尚未稳定前就把树莓派和上层协议混入，只会把问题源继续放大

## 6. 当前公开仓库的入库策略

本轮公开仓库只跟踪：

- 自研应用层骨架
- 迁移说明文档
- 接口契约
- 集成说明

不跟踪：

- vendor 原工程整包
- vendor 的可编译工作副本

当前公开安全目录冻结为：

- [firmware/archive/openrf1_keil_overlay](../../firmware/archive/openrf1_keil_overlay)

本地可编译工作副本冻结为忽略路径：

- `_local/openrf1_keil_work/`

## 7. 当前阶段的真正验收标准

本轮不是看“能不能和树莓派对上”，也不是看“能不能先兼容 ROS”。

当前唯一真正验收标准是：

1. 四轮都能持续转动
2. 四轮都有实时速度回传
3. 电脑串口可下发四轮目标速度
4. 板子可稳定回传目标速度、实时速度和 PWM
5. 超时后能自动停车

在这五条达成之前，不应继续扩展功能。
