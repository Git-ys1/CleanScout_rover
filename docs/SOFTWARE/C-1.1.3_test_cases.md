# C-1.1.3 Test Cases

## 1. 目标

本文件用于冻结 `C-1.1.3` 的协议改造、风机控制权交接和 IDE 编译兼容的验收口径。

## 2. 构建测试

| 编号 | 场景 | 预期结果 |
| --- | --- | --- |
| BT-01 | 运行 `.\tools\compile_uno_cli.ps1` | 权威入口编译通过 |
| BT-02 | 运行 `.\tools\compile_uno.ps1` | 回退入口编译通过 |
| BT-03 | 运行 `.\tools\install_sketchbook_links.ps1` 后，用 Arduino IDE 直接 Verify `sketches/c002_uno_baseline` | 编译通过，不再出现 `CleanScoutFan.h: No such file or directory` |
| BT-04 | 编译 `libraries/Tyler_1/examples/Tyler_1/Tyler_1.ino` | 同步后的历史示例也能通过 |

## 3. 协议测试

| 编号 | 场景 | 预期结果 |
| --- | --- | --- |
| PT-01 | 发送 `M` | 进入手动模式并停车 |
| PT-02 | 手动模式下第一次发送 `G` | `A2` 拉高，风机开启 |
| PT-03 | 手动模式下第二次发送 `G` | `A2` 拉低，风机关闭 |
| PT-04 | 手动模式下连续多次发送 `G` | 风机按次翻转，无重复抖动 |
| PT-05 | 发送 `T` | 成功进入体感模式 |
| PT-06 | 体感模式下发送 `G` | 风机按次翻转 |
| PT-07 | 自动模式下发送 `G` | 风机控制不被人工翻转打断 |
| PT-08 | 一次 `G` 处理完成后继续循环 | 不会因 `cmdChar` 常驻而连续翻转 |
| PT-09 | 先发送运动指令，再发送一次 `G` | 风机翻转后当前运动保持，不被强制改成 `5` |

## 4. 控制权交接测试

| 编号 | 场景 | 预期结果 |
| --- | --- | --- |
| OT-01 | `MAN/GEST -> AUTO` | 手动锁存被释放，自动风机从完整 3 秒冷却重新开始 |
| OT-02 | `AUTO -> MAN/GEST` | 自动控制立即失效，人工域默认风机关 |
| OT-03 | `AUTO -> MAN/GEST` 后再次发送 `G` | 风机重新由人工翻转打开 |
| OT-04 | `MAN <-> GEST` 互切 | 手动风机状态保留 |

## 5. 回归测试

以下行为必须保持不变：

- 默认启动模式未改
- 自动避障流程未改
- 底盘动作语义未改
- `MAN / AUTO / GEST` 三模式体系仍然存在
- 主程序和历史示例中都没有散写 `digitalWrite(A2, ...)`
