# C-3.0.2A UNO 编码器 X2 升级说明

## 1. 本轮定位

`C-3.0.2A` 只处理 `UNO` 本地实现。

本轮明确不改：

- `W,w1,w2,w3,w4` 协议
- `ACK:W / ERR / ENC / PID` 返回格式
- 树莓派桥接结构

本轮唯一目标是：

- 把四轮编码器从 `X1` 升级到 `X2`
- 同步更新底盘几何常数与当前 `CPR` 占位值

## 2. 编码器口径更新

### 2.1 旧口径

此前 `WheelEncoder` 采用的是：

- 只在 `A` 相上升沿计数
- `B` 相只用于判方向

这属于 `X1`。

### 2.2 新口径

本轮升级后，统一采用：

- `A` 相上升沿计数
- `A` 相下降沿也计数
- `B` 相继续用于判方向

也就是：

- `A` 有变化就计数
- `A` 不变就不计数

当前判向逻辑冻结为：

```cpp
if (a_now != a_prev) {
  int8_t delta = (a_now == b_now) ? -1 : 1;
  delta *= directionSign;
  ticks += delta;
}
```

冻结结论：

- 当前编码器口径已经从 `X1` 升级到 `X2`
- 本轮不上 `X4`
- 不改物理接线真相

## 3. 当前冻结常量

本轮统一更新为：

- `WB_M = 0.1905`
- `TW_M = 0.1800`
- `K_M = 0.18525`
- `D_EFF_MM = 78.0`
- `CPR_X1_EST = 520.0`

说明：

- `K_M` 明确按 `(WB_M + TW_M) / 2` 取值
- `D_EFF_MM` 本轮继续沿用 `78.0`
- `CPR_X1_EST` 字段名暂不重构，但当前数值已经按 `X2` 口径提升到 `520.0`

## 4. 接线与方向修正口径

当前四轮 A/B 映射保持不变：

- `wheel1: A0/A1`
- `wheel2: A2/A3`
- `wheel3: A4/A5`
- `wheel4: D9/D10`

当前规则继续写死：

- 不交换 A/B 线
- 若个别轮方向相反，只改 `ENC_SIGN_Wn`

当前已验证保留：

- `ENC_SIGN_W3 = -1`

`M4` 当前既然已经定位过接触问题并恢复转动，本轮不重新引入新的 `Port B` 诊断目标，只保留 `RAW4` 作为回退诊断入口。

## 5. 当前影响范围

本轮升级后，最直接的影响是：

- 相同机械转速下，tick 计数密度比 `X1` 更高
- 低速测速会比 `X1` 更连续
- `ENC/PID` 观察会更细

本轮不直接宣称：

- `odom` 已经可用
- `cmd_vel -> W` 已经完成
- 所有 `PI` 参数已经最终冻结

## 6. 本轮验收口径

本轮最少通过以下检查：

- `c003` 正常模式可编译
- `c003` `RAW4` 模式可编译
- `c002` 回退线不受影响
- 四轮在低速 `W,80,80,80,80`、`W,120,120,120,120` 下持续转动
- `ENC` 输出持续正常
- 与 `X1` 相比，低速测速更连续、更平顺
- 若个别轮方向相反，只通过 `ENC_SIGN_Wn` 修正

## 7. 当前结论

截至 `C-3.0.2A`：

- `UNO` 编码器口径已经从 `X1` 升级到 `X2`
- 当前几何常数已经按新冻结值更新
- 协议层、树莓派桥、返回帧格式保持不变
- 本轮的意义是先把底层编码器分辨率和底盘常量对齐，为后续继续调 `PI` 和进入下一阶段做准备
