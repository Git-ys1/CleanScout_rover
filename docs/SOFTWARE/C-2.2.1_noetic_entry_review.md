# Noetic 镜像接管入口审查

## 1. 审查定位

本文件基于当前已归档的树莓派 `Noetic` 审计日志，整理当前镜像中最关键的启动入口、驱动假设与底层依赖。

本轮只做审查，不改镜像代码。

## 2. 当前主工作空间

当前主接管入口是：

- `/home/clbrobot/catkin_ws`

当前同时存在：

- `/home/clbrobot/goolge_ws`

当前结论：

- `catkin_ws` 是主工作空间
- `goolge_ws` 仅在审计日志中出现，当前视为“存在但角色未定”的次级 workspace
- 本轮不把 `goolge_ws` 当成主接管入口

## 3. 关键 launch 入口

### 3.1 bringup.launch

当前审查结论：

- `bringup.launch` 包含 `rosserial_python serial_node.py`
- 包含 IMU relay / `imu_calib`
- 包含 `imu_filter_madgwick`
- 包含 `base_footprint -> imu_link` 静态 TF
- 包含原始里程计发布
- 包含 `robot_localization` 的 Odom-IMU EKF，并把 `odometry/filtered` remap 到 `odom`

这说明 `bringup.launch` 的底层假设不是“简单串口控车”，而是：

- 串口底盘
- 原始里程计
- IMU
- EKF 融合

### 3.2 navigate.launch

当前审查结论：

- `navigate.launch` 是 `clbrobot` 主导航入口之一
- 按当前包结构与命名，它处于 `bringup` 之上的导航层
- 它应被视为“在底层 bringup 成立后叠加导航栈”的入口，而不是裸底盘入口

### 3.3 slam/lidar_slam.launch

当前审查结论：

- `slam/lidar_slam.launch` 是当前基于激光雷达的 SLAM 路线入口
- 它属于“`bringup + lidar + slam`”组合层，而不是单一雷达驱动入口

## 4. RPLIDAR 入口假设

当前镜像内存在两套路径：

- `clbrobot/launch/lidar/rplidar.launch`
- `rplidar_ros` 官方包内自带 launch

### 4.1 clbrobot 自带 rplidar.launch

根据 grep 审查：

- 串口默认：`/dev/clblidar`
- 默认波特率：`115200`（A1/A2）
- 注释保留：`256000`（A3）

这说明当前 `clbrobot` 自带 launch 的默认硬件假设更接近 `A1/A2` 旧口径。  
若当前主线改为 `RPLIDAR A3`，后续必须显式改正这一路径，不能直接照抄。

### 4.2 rplidar_ros 官方包

当前 `rplidar_ros` 审查结论：

- 包 README 明确区分 `A1/A2` 与 `A3`
- `node.cpp` 默认 `serial_port = /dev/ttyUSB0`
- `serial_baudrate` 默认 `115200`，注释中说明 `A3` 需改 `256000`

## 5. 当前可直接接管的上层 / 中层

当前可直接作为中上层接管对象的内容：

- `rplidar_ros`
- `imu_filter_madgwick`
- `imu_calib`
- `move_base` / navigation 相关 launch 组织方式
- `robot_localization` 的整体思路与参数组织方式

这些内容可以作为树莓派侧中上层参考或直接复用对象。

## 6. 明显依赖更完整底层的部分

以下部分明显依赖一个比当前 Tyler/AFMotor 更完整的底层：

- `rosserial_python` 串口底盘接口
- `raw_odom -> odom` 里程计链
- Odom-IMU EKF 融合链
- `clbrobot` 自带底盘假设

原因：

- 当前现实车尚无现成编码器闭环
- 当前 Tyler/AFMotor 现实车并没有镜像默认假设的 `raw_odom`、`imu/data`、`odom` 话题基础

## 7. 当前接管结论

当前镜像并不是“不可用”，而是：

- 上层 / 中层结构可以接管
- 底层能力假设不能直接照抄

因此当前正确路径是：

1. 先完成镜像接管归档与入口审查
2. 再把现实车底层能力逐步映射到这些入口要求
3. 不在本轮直接推进导航、SLAM、EKF 实现
