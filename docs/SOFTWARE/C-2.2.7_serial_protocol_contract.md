# C-2.2.7 串口协议契约

## 1. 当前唯一联调真相协议

### 1.1 上位机 -> UNO

当前唯一有效输入协议：

```text
W,w1,w2,w3,w4
```

说明：

- `w1..w4` 为四个轮子的有符号目标速度
- 单位固定为 `ticks/s`

### 1.2 UNO -> 上位机

当前唯一有效返回帧：

```text
CSR_UNO_READY
ACK:W
ERR:<reason>
ENC,t1,t2,t3,t4,v1,v2,v3,v4
PID,target1,target2,target3,target4,meas1,meas2,meas3,meas4,out1,out2,out3,out4
```

说明：

- 上述帧是当前正常闭环模式下的正式联调返回帧。
- `RAW4,ticks,currentA,currentB` 仅属于 `C-2.2.8A` 的本地诊断输出，不属于当前正式跨端协议契约。

## 2. 旧协议与当前协议的关系

旧树莓派桥协议：

```text
CMD,vx,vy,wz
```

当前定义：

- `CMD,vx,vy,wz` 是树莓派旧桥协议
- 它不是当前联调真相协议
- 当前联调真相协议已经切换为 `W,w1,w2,w3,w4`

## 3. 当前通过标准

本轮只认下面三条：

1. `Pi` 发 `W,...`
2. `UNO` 回 `ACK:W`
3. 底盘动作与 `ENC/PID` 数据一致变化

以下都不算通过：

- 只收到 `READY`
- 只看到树莓派写串口成功
- 只看到 `UNO` 本地打印
- 只看到 `CMD,...` 自己逻辑自洽

## 4. 下一阶段接口说明

### 4.1 下一阶段

下一阶段才做：

- `cmd_vel -> W`
- `vx / vy / wz -> 轮速`
- 正式 `odom`
- `EKF / SLAM / 导航` 接入

### 4.2 当前几何参数角色

当前物理参数：

- `D_MECH = 80`
- `D_EFF = 78`
- `WB = 0.16535`
- `TW = 0.17850`
- `K = 0.171925`
- `CPR_X1_EST = 260`

当前只服务下一阶段，不作为本轮联调失败判责依据。

## 5. 联调前 Checklist

### 5.1 UNO 端

联调前必须回填：

- 当前分支名
- 当前 commit hash
- 当前协议前缀
- 当前返回帧
- 当前 timeout（正式 / 联调）
- 当前控制周期

### 5.2 树莓派端

联调前必须回填：

- 当前分支名
- 当前 commit hash
- 当前发送帧
- 当前发送频率
- 当前等待 `READY` 规则
- 当前解析哪些返回帧

没有这张表，不准开始跨端联调。
