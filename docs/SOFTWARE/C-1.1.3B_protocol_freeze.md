# C-1.1.3B Protocol Freeze

## 1. 目标

`C-1.1.3B` 用于彻底冻结当前蓝牙协议，明确模式命令和事件命令已经解耦。

## 2. 当前协议表

| 字符 | 类型 | 含义 | 说明 |
| --- | --- | --- | --- |
| `A` | 模式命令 | `AUTO` | 进入自动模式 |
| `M` | 模式命令 | `MAN` | 进入手动模式 |
| `T` | 模式命令 | `GEST` | 进入体感模式 |
| `G` | 事件命令 | `FAN TOGGLE` | 只翻转风机，不改模式 |
| `1/2/3/4/5/6/7/8/9` | 运动命令 | 人工运动控制 | 仅用于人工控制域 |

## 3. 冻结结论

- `G` 不再承担体感模式入口职责。
- `T` 是唯一的蓝牙体感模式入口。
- `G` 只能表示一次风机开关翻转事件。
- 处理 `G` 时不得修改 `systemMode`。
- 自动模式下 `G` 无效，但必须立即消费，不能残留到后续循环。

## 4. 旧协议对照

本轮允许在本节保留历史对照，以便明确问题来源：

| 协议项 | 历史口径 | 现行口径 |
| --- | --- | --- |
| 体感模式入口 | `G -> GEST` | `T -> GEST` |
| `G` 的语义 | 复用为手势入口 | `G -> FAN TOGGLE EVENT` |
