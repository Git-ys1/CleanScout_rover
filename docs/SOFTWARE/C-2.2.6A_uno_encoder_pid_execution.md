# C-2.2.6A UNO 编码器 + PI 执行说明

## 1. 本轮定位

`C-2.2.6A` 是 `UNO` 侧第一次把四轮霍尔编码器采样、四路 `PI` 闭环和新的串口桥接协议真正落到本地代码。

本轮只处理 `UNO` 本地控制层，不拉取树莓派云端 `ROS` 工作区，也不在本轮直接落地 `vx / vy / wz -> 轮速(rad/s)` 和正式 `odom`。

当前主线目标是：

- 四个编码器 `x1` 采样可工作
- 每轮速度先用 `ticks/s` 统一表达
- 每轮 `PI` 可独立闭环
- 上位机可通过新协议持续下发四轮目标速度

## 2. 分层口径

### 2.1 Tyler_1

`Tyler_1` 当前只保留原始电机执行层：

- 历史 `motion-only` 语义

## 3. 编码器映射与采样方向

### 3.1 当前接线冻结

- `wheel1: A0 / A1`
- `wheel2: A2 / A3`
- `wheel3: A4 / A5`
- `wheel4: D9 / D10`

其中：

- A 相：`A0 A2 A4 D9`
- B 相：`A1 A3 A5 D10`

### 3.2 解码方式

当前固定为 `x1` 解码：

- 只对 A 相做中断/变化检测
- 在 ISR 里读取 B 相判断方向

### 3.3 中断实现

当前固定走 `PCINT`：

- `PCINT1_vect` 处理 `Port C` 上的 `A0/A2/A4`
- `PCINT0_vect` 处理 `Port B` 上的 `D9`

不采用“8 路都走 `attachInterrupt()`”的方案。

## 4. 控制周期与 PI 口径

### 4.1 控制周期

当前固定：

- `20ms`
- 即 `50Hz`

### 4.2 控制量

当前冻结为：

- 目标量：每轮 `ticks/s`
- 反馈量：每轮 `ticks/s`
- 输出量：有符号 `PWM`

输出范围：

- `-255 .. 255`

符号决定方向，绝对值决定 PWM。

### 4.3 WheelPI

当前 `WheelPI` 是自写轻量类，不引入第三方 PID 库。

当前固定口径：

- `Kp` 启用
- `Ki` 启用
- `Kd = 0`
- 积分限幅
- 输出限幅
- 目标为 `0`、超时停车或模式切换时重置积分

建议调参顺序：

1. 先 `Ki = 0`
2. 先把 `Kp` 调到能跟上目标且不明显振荡
3. 再逐步加入 `Ki`
4. 本轮不加 `D`

## 5. 串口协议

### 5.1 波特率

- `115200`

### 5.2 新主协议

当前闭环主协议固定为：

```text
W,<w1>,<w2>,<w3>,<w4>\n
```

含义：

- `w1..w4` 是四个轮子的有符号目标速度
- 单位统一为 `ticks/s`

### 5.3 当前输出

启动：

```text
CSR_UNO_READY
```

收到有效速度帧：

```text
ACK:W
```

收到非法帧：

```text
ERR:<reason>
```

编码器与测速遥测：

```text
ENC,t1,t2,t3,t4,v1,v2,v3,v4
```

其中：

- `t1..t4`：累计 tick
- `v1..v4`：当前窗口速度，单位 `ticks/s`

### 5.4 旧协议口径

`1~9` 动作码不再进入 `c003` 主线。

如果需要历史动作码联调，回到 `c002`。
