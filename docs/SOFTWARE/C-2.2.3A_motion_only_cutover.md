# C-2.2.3A Motion-Only Cutover

## 当前阶段定位
- 目标：把 `Tyler_1` 从历史综合库裁成 `C-2.x` 当前实施线可用的运动最小核。
- 当前下层：`Arduino UNO + AFMotor`
- 当前上位接管入口：`USB Serial@115200`
- 当前草图：`sketches/c002_uno_baseline/c002_uno_baseline.ino`

## 当前串口协议
- 波特率：`115200`
- 输入命令：`1/2/3/4/5/6/7/8/9`
- 上电输出：`CSR_UNO_READY`
- 有效命令回显：`ACK:<char>`
- 非法命令回显：`ERR:<char>`
- 超时停车：`400ms`

## 当前支持命令
- `8`：前进
- `2`：后退
- `4`：左转
- `6`：右转
- `5`：停止
- `7`：左前
- `9`：右前
- `1`：右后
- `3`：左后

## 当前已裁掉的历史能力
- `AUTO` 自动模式
- `GEST` 体感模式
- `M/A/T/G` 模式字符
- 蓝牙入口 `softSerial(9, 2)`
- 串口桥接 `bridgeSerial(A3, A4)`
- 风机逻辑与 `CleanScoutFan`
- 超声自动避障与相关自动转向流程
- 头部舵机初始化、扫描与转头逻辑

## Tyler 库如何解耦超声/舵机
### 禁用语义
- `trigPin = -1 或 255`：禁用超声
- `echoPin = -1 或 255`：禁用超声
- `servoPin = -1 或 255`：禁用舵机

### 当前主程序构造口径
- `Tyler_1 tyler_1(1, 0, 1, 1, 200, 255, 255, 255);`

### 当前实现行为
- 构造函数始终完成电机初始化。
- 只有超声启用时，才会对 `trig/echo` 做 `pinMode()`。
- `headServoIni()`：舵机禁用时直接 no-op。
- `setHeadPos()`：舵机禁用时直接 no-op；启用时允许懒 attach。
- `getHeadPos()`：舵机禁用时返回 `-1`。
- `getDistance()`：超声禁用时返回 `-1`。
- 所有运动函数 `forward/backward/turnL/turnR/forwardL/forwardR/backwardL/backwardR/stop` 都不依赖超声或舵机初始化状态。

## 静态审查结论
- 当前库内未发现 `forward()`、`turnL()` 等运动 API 直接依赖超声/舵机结果来阻断运动。
- 现有耦合点实质上是：
  - 构造函数对超声引脚的 `pinMode()` 副作用
  - 传感器/舵机接口长期和运动接口共存于同一库
- 本轮解耦的是这些副作用与接口耦合，不是去掉“运动函数内部的避障判断”，因为库里本来没有这类判断。

## 当前引脚口径
- 仍保留业务使用：
  - `D0/D1`：USB Serial
  - AFMotor 原有电机相关占用保持不变
- 从主流程释放：
  - `D2`
  - `D9`
  - `D10`
  - `A0`
  - `A1`
  - `A2`
  - `A3`
  - `A4`
  - `A5`
- `D13`：仅作为本轮板载 LED 调试口，不作为业务扩展口

## 当前调试输出格式
- `CSR_UNO_READY`
- `ACK:8`
- `ACK:5`
- `ERR:X`

## 当前结论
- 本轮完成后，`Tyler_1` 已满足当前 `C-2.x` 实施线所需的“无超声、无舵机”运动最小核要求。
- 历史示例能力仍在库接口中保留，但当前实施线默认走 motion-only 口径。
- 仓库内两套库副本 `libraries/Tyler_1` 与 `Tyler_1_Library` 已按相同口径同步，避免后续继续分叉。
