# C-1.1.3 Fan Control Ownership

## 1. 目标

`C-1.1.3` 把风机控制统一冻结为“单模块、双控制源”模型，避免自动逻辑和蓝牙手动开关同时争抢 `A2`。

## 2. 控制源

| 控制源 | 适用模式 | 控制方式 |
| --- | --- | --- |
| `AUTO_SOURCE` | `AUTO` | 3 秒冷却 / 6 秒运行状态机 + 障碍暂停 |
| `MANUAL_SOURCE` | `MAN` / `GEST` | 蓝牙 `G` 事件翻转手动锁存 |

冻结结论：

- `A2` 仍是唯一风机控制脚。
- `A2 = HIGH` 表示风机开启。
- `A2 = LOW` 表示风机关闭。
- 任何时候都只能有一个控制源生效。

## 3. AUTO_SOURCE 规则

自动模式下：

- 风机控制权属于 `AUTO_SOURCE`
- `G` 无效
- 自动状态机继续使用 `C-1.1.2B` 的节奏：
- 3 秒关闭冷却
- 6 秒开启运行
- 障碍时立即暂停
- 转弯完成后重新开始完整 3 秒冷却

## 4. MANUAL_SOURCE 规则

手动模式和体感模式下：

- 风机控制权属于 `MANUAL_SOURCE`
- `G` 每收到一次翻转一次手动锁存
- 手动锁存为 `ON` 时，`A2` 输出 `HIGH`
- 手动锁存为 `OFF` 时，`A2` 输出 `LOW`

本轮不允许：

- 在 sketch 中散写 `digitalWrite(A2, ...)`
- 绕开 `CleanScoutFan` 直接控制继电器

## 5. 模式切换交接

### 5.1 `MAN/GEST -> AUTO`

- 立即退出手动控制权
- 清空手动锁存
- 关闭当前输出
- 风机控制转交给 `AUTO_SOURCE`
- 从切换时刻重新开始完整 3 秒冷却

### 5.2 `AUTO -> MAN/GEST`

- 立即退出自动控制权
- 进入人工控制域
- 默认风机为关闭
- 必须再次收到 `G` 才能打开

### 5.3 `MAN <-> GEST`

- 保留当前手动锁存
- 不要求切换时自动关风机
- 因为两者同属人工控制域

## 6. 边界

- 自动模式下 `G` 不是全局抢占指令。
- `G` 只在人工控制域有效。
- 自动障碍暂停和转弯恢复逻辑只在 `AUTO_SOURCE` 中生效。
