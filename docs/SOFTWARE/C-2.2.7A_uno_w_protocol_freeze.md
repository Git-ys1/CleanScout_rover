# C-2.2.7A UNO W 协议冻结说明

## 1. 本轮定位

`C-2.2.7A` 不是协议回退轮，而是当前 `UNO` 侧 `W,w1,w2,w3,w4` 协议的联调增强与真相冻结轮。

本轮目标：

- 保持 `UNO` 只接受 `W,w1,w2,w3,w4`
- 保持 `ACK:W + ENC,...` 主线不变
- 补足联调所需的 `PID,...` 观测
- 区分正式 timeout 与联调 timeout

## 2. 当前 UNO 协议真相

### 2.1 输入

当前 `UNO` 只接受：

```text
W,w1,w2,w3,w4
```

含义：

- `w1..w4` 是四个轮子的有符号目标速度
- 单位固定为 `ticks/s`

`CMD,vx,vy,wz` 不是当前 `UNO` 真相协议。

### 2.2 输出

上电输出：

```text
CSR_UNO_READY
```

收到合法命令：

```text
ACK:W
```

收到非法帧：

```text
ERR:<reason>
```

周期输出编码器与测速：

```text
ENC,t1,t2,t3,t4,v1,v2,v3,v4
```

周期输出 PI 调试：

```text
PID,target1,target2,target3,target4,meas1,meas2,meas3,meas4,out1,out2,out3,out4
```

其中：

- `target1..target4`：四轮目标速度，单位 `ticks/s`
- `meas1..meas4`：四轮实测速度，单位 `ticks/s`
- `out1..out4`：四路当前有符号 PWM 输出，范围 `-255..255`

## 3. 当前 timeout 口径

### 3.1 正式模式

正式模式默认值保持为：

- `COMMAND_TIMEOUT_MS = 400`

这是后续持续流式控制模式下的正式口径，不因当前联调需要而被回退或重写。

### 3.2 联调模式

本轮额外引入联调常量：

- `COMMAND_TIMEOUT_DEBUG_MS = 2000`

并通过显式布尔开关控制当前编译所使用的 timeout。

当前代码口径：

- `USE_DEBUG_TIMEOUT = true`

因此当前联调编译默认使用：

- `2000ms`

这只是联调观察参数，不是长期正式控制口径。

## 4. 当前 PI 参数

当前冻结值：

- `CONTROL_PERIOD_MS = 20`
- `WHEEL_KP = 2.0`
- `WHEEL_KI = 0.5`
- `WHEEL_I_LIMIT = 150.0`
- `WHEEL_OUTPUT_LIMIT = 255.0`

本轮继续保持：

- `Ki = 0`
- 不引入 `D`

原因很简单：

- 当前优先目标是把 `目标轮速 / 实测轮速 / PWM 输出` 三者关系跑顺
- 不在这一轮提前把积分和微分一起混进来

## 5. 当前编码器映射

当前已接线映射：

- `wheel1: A0 / A1`
- `wheel2: A2 / A3`
- `wheel3: A4 / A5`
- `wheel4: D9 / D10`

其中：

- A 相：`A0 A2 A4 D9`
- B 相：`A1 A3 A5 D10`

当前解码方式：

- `x2`
- A 相双边沿计数
- B 相判方向
- `PCINT` 实现

## 6. 当前物理参数口径

当前写入配置区的参数：

- `D_MECH_MM = 80`
- `D_EFF_MM = 78`
- `WB_M = 0.1905`
- `TW_M = 0.1800`
- `K_M = 0.18525`
- `CPR_X1_EST = 520`

当前必须明确：

- 这些参数本轮只算预配置和下一阶段扩展位
- 它们不是本轮 `W` 协议联调失败的判责依据
- 本轮验收的核心仍然是 `ACK:W + ENC + PID + 动作一致`

## 7. 本轮边界

本轮不做：

- 协议回退到 `CMD,vx,vy,wz`
- 几何参数大改
- `odom` 正式发布
- `ROS` 端逻辑侵入 `UNO`
- `D` 项 PID

本轮唯一目标是把 `UNO` 当前真实协议稳定、可观测地冻结下来，供树莓派端对齐。
