# C-1.1.2B Test Cases

## 1. 目标

本文件用于冻结 `C-1.1.2B` 的行为验证口径，确保风机智能控制只在自动模式中生效，并且不破坏现有模式系统。

## 2. 功能测试场景

| 编号 | 场景 | 预期结果 |
| --- | --- | --- |
| TC-01 | 自动模式，无障碍，进入 `AUTO` | 风机初始关闭，先进入 3 秒冷却 |
| TC-02 | 自动模式，无障碍，冷却满 3 秒 | 风机开启，状态进入 `RUNNING` |
| TC-03 | 自动模式，无障碍，运行满 6 秒 | 风机关闭，重新进入 `COOLDOWN` |
| TC-04 | 自动模式，风机运行中遇到障碍 | 风机立即关闭，状态进入 `PAUSED_BY_OBSTACLE` |
| TC-05 | 自动模式，风机冷却中遇到障碍 | 风机保持关闭，冷却计时作废，状态进入 `PAUSED_BY_OBSTACLE` |
| TC-06 | 障碍处理完成并完成实际转弯 | 调用 `notifyTurnCompleted()` 后，不立即启动风机，而是重新开始完整 3 秒冷却 |
| TC-07 | 自动模式切到遥控模式 | 风机立即关闭，状态进入 `DISABLED` |
| TC-08 | 自动模式切到体感模式 | 风机立即关闭，状态进入 `DISABLED` |
| TC-09 | 遥控或体感切回自动模式 | 风机重新开始完整 3 秒冷却，不继承旧计时 |
| TC-10 | 连续障碍场景 | 每次障碍都立即关闭风机；每次转弯完成后都重新开始完整 3 秒冷却 |
| TC-11 | 默认启动行为回归 | 默认模式与既有自动切换行为不变 |

## 3. 工程验收点

以下条件必须同时满足：

- 风机控制已独立封装为 `CleanScoutFan` 模块
- 主程序中不存在散乱 A2 业务控制
- 风机 3 秒 / 6 秒计时采用非阻塞方案实现
- `arduino-cli` 编译通过当前权威入口
- `arduino-builder` 回退编译通过当前权威入口
- 本轮没有混入蓝牙协议、模式系统或底盘运动语义改动

## 4. 文档验收点

以下文档必须存在：

## 5. 特别回归说明

本轮必须明确验证以下事项没有被改坏：

- 默认启动模式未被改动
- 模式切换入口未被改动
- 现有自动避障总策略未被重写
- 非自动模式下风机始终关闭
