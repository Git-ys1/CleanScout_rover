# C-1.1.3 Bluetooth Protocol Freeze

## 1. 目标

`C-1.1.3` 冻结当前蓝牙字符协议，明确体感模式入口迁移和人工风机开关入口扩展。

本轮只改协议入口，不删除任何既有模式枚举和模式分支。

## 2. 新协议表

| 字符 | 含义 | 说明 |
| --- | --- | --- |
| `A` | `AUTO` | 进入自动模式，行为保持现状 |
| `M` | `MAN` | 进入手动模式，行为保持现状 |
| `T` | `GEST` | 进入体感模式，新冻结入口 |
| `G` | `FAN TOGGLE` | 人工风机翻转命令，不再切换模式 |
| `1/2/3/4/5/6/7/8/9` | 运动控制 | 保持现有人工控制语义 |

## 3. 旧协议与新协议对照

| 协议版本 | `G` 的含义 | 体感模式入口 |
| --- | --- | --- |
| 旧协议 | `G -> GEST` | `G` |
| `C-1.1.3` | `G -> FAN TOGGLE` | `T` |

冻结结论：

- `G` 从本轮起不再进入体感模式。
- `T` 从本轮起成为唯一的蓝牙体感模式入口字符。
- `GEST` 模式本体保留，不删除、不合并。

## 4. G 事件处理规则

`G` 是一次性事件，不是持续态命令。

固定要求：

- 一次 `G` 只触发一次风机翻转。
- 处理完后必须立即消费该事件。
- 不允许让 `cmdChar == 'G'` 在主循环里常驻。
- 本轮实现采用独立事件位，保留当前人工运动命令，不把运动指令强制改回 `5`。

## 5. 生效范围

| 当前模式 | `G` 是否有效 | 说明 |
| --- | --- | --- |
| `MAN` | 有效 | 人工控制域，可手动切风机 |
| `GEST` | 有效 | 体感模式仍属于人工控制域 |
| `AUTO` | 无效 | 自动模式下风机仍由自动状态机接管 |

## 6. 备注

- 本轮不改蓝牙串口底层。
- 本轮不改默认模式初始化逻辑。
- 本轮不改自动避障总策略。
