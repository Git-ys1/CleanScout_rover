# C-1.1.2B Integration Points

## 1. 权威接入入口

本轮唯一权威运行入口：

- `sketches/c002_uno_baseline/c002_uno_baseline.ino`

本轮不把风机功能回写到：

- `libraries/Tyler_1/examples/Tyler_1/Tyler_1.ino`

该示例保留为历史参考，不作为本轮功能集成入口。

## 2. 模块初始化位置

在 `setup()` 中完成：

- `fan.begin(FAN_RELAY_PIN, FAN_ACTIVE_HIGH);`

要求：

- 默认输出为关闭
- 不在 `setup()` 中直接写风机业务逻辑

## 3. 周期更新位置

在 `runMode()` 中：

- 保持现有 `cmdChar -> systemMode` 逻辑不变
- 在模式值已经确定之后、进入 `manMode()` 或 `autoMode()` 之前，调用：

```cpp
fan.update(systemMode, millis());
```

目的：

- 非自动模式时立即关闭风机
- 自动模式时推进 3 秒冷却 / 6 秒运行状态机

## 4. 障碍暂停接入点

在 `autoMode()` 的障碍分支中，顺序固定为：

1. `fan.pauseByObstacle();`
2. `tyler_1.stop();`
3. `autoTurn();`

说明：

- 风机必须先暂停，再继续沿用当前已有的避障流程。
- 不允许把障碍暂停逻辑放到 `loop()` 的散乱分支里。

## 5. 转弯完成恢复接入点

在 `autoTurn()` 中：

- 完成左右探测
- 完成实际 `turnR90()` 或 `turnL90()` 动作后
- 在 `return` 之前调用：

```cpp
fan.notifyTurnCompleted();
```

说明：

- `notifyTurnCompleted()` 只表示“允许重新开始冷却”。
- 它不会立即启动风机。
- 风机会在后续 `update()` 中重新进入完整 3 秒冷却。

## 6. 禁止直接操作 A2 的位置

除模块初始化安全关闭之外，以下位置禁止直接写业务级 A2 控制：

- `loop()`
- `runMode()`
- `autoMode()`
- `autoTurn()`
- `turnL90()`
- `turnR90()`

禁止出现散乱的：

```cpp
digitalWrite(A2, HIGH);
digitalWrite(A2, LOW);
```

风机业务控制必须统一通过 `CleanScoutFan` 模块完成。

## 7. 模式层边界

本轮只允许读取当前模式：

- `MAN`
- `AUTO`
- `GEST`

本轮不允许改动：

- 默认模式值
- 模式切换入口
- 蓝牙命令解析
- 启动后的既有自动切换行为
