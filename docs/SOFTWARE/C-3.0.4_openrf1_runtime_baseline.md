# C-3.0.4：OpenRF1 当前运行时基线与失败边界说明

## 1. 当前定位

`C-3.0.4` 原目标是把 OpenRF1 建成新的底盘下位机最小可用基线。

截至 2026-04-16，当前最重要的结论不是“已经打通”，而是：

- 已拿到一组可信的 `CN1~CN4` 实机响应真值
- 当前四轮同向同转和统一正反控制未达标
- 当前实现不应冻结为正式运行基线

因此本文件从“运行时基线说明”调整为“当前运行时事实 + 失败边界说明”。

## 2. 当前已确认的物理映射真值

用户实测冻结如下：

| 协议位 | 板口 | 物理轮位 |
| --- | --- | --- |
| `A` | `CN1` | 左后 |
| `B` | `CN2` | 左前 |
| `C` | `CN3` | 右后 |
| `D` | `CN4` | 右前 |

这组映射优先级高于此前文档中任何关于 `A/B/C/D`、`L1/R1/L2/R2`、`M1~M4` 的推断性描述。

## 3. 当前关键测试数据

本轮提交给项目负责人的关键测试真值如下：

| 板口 | 物理轮位 | 接受正值 | 接受负值 | 当前事实 |
| --- | --- | --- | --- | --- |
| `CN1` | 左后 | 无响应 | 实际为负方向 | 正值不工作，负值才动作 |
| `CN2` | 左前 | 实际为正方向 | 无响应 | 仅正值可驱动 |
| `CN3` | 右后 | 无响应 | 实际为正方向 | 正值不工作，负值才动作 |
| `CN4` | 右前 | 实际为负方向 | 无响应 | 仅正值可驱动，但方向为负 |

这里的“无响应”含义固定为：

- 在本轮串口/直驱测试中，未观察到该板口在该极性下出现稳定、可确认的轮子动作

## 4. 当前能与不能

### 当前能确认的事实

- OpenRF1 板、CH340 串口、ST-Link 烧录链路可用
- `CN1~CN4` 单轮测试可观察到底层行为差异
- 当前问题不是“完全没有任何驱动输出”，而是四路方向/极性真值不一致

### 当前不能宣称成立的能力

- 不能宣称“四轮同向同转”已经打通
- 不能宣称“统一正值前进、统一负值后退”已经成立
- 不能宣称 `W,va,vb,vc,vd` 已形成可靠主协议
- 不能把当前实现继续包装成树莓派接入前的稳定下位机基线

## 5. 当前项目判断

基于以上实测，当前建议是：

- 将 `C-3.0.4` 当前这条 OpenRF1 协议化 bring-up 线提交项目负责人审批
- 审批建议口径为：`阶段性失败，建议弃项或暂停` 
- 保留本文件和验证文档作为失败证据链
- 不再把当前版本继续当作“已经接近完成”的底盘基线扩写

## 6. 若后续重开，唯一正确起点

如果项目负责人后续批准重开 OpenRF1 线，必须按下面顺序重新开始：

1. 固定 `CN1~CN4` 的单轮正反真值
2. 先证明四轮都具备双向、同语义驱动
3. 再接 `W,va,vb,vc,vd` 协议
4. 最后才考虑编码器、闭环、树莓派适配

当前不能再从“半完成协议版”继续堆补丁。
