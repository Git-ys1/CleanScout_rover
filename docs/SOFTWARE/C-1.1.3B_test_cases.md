# C-1.1.3B Test Cases

## 1. 目标

本文件用于冻结 `C-1.1.3B` 的协议修复验收口径，重点验证 `G` 已经从模式入口彻底解耦为纯风机事件。

## 2. 协议行为

| 编号 | 场景 | 预期结果 |
| --- | --- | --- |
| TB-01 | 当前在 `MAN`，点击一次 `G` | 风机开启，模式仍为 `MAN` |
| TB-02 | 当前在 `MAN`，再次点击 `G` | 风机关停，模式仍为 `MAN` |
| TB-03 | 当前在 `MAN`，连续多次点击 `G` | 风机按次翻转，不进入 `GEST` |
| TB-04 | 点击 `T` | 系统进入 `GEST` |
| TB-05 | 当前在 `GEST`，点击 `G` | 只切风机，不改模式 |
| TB-06 | 当前在 `AUTO`，点击 `G` | 不进入 `GEST`，模式保持 `AUTO` |
| TB-07 | 点击 `A/M/T` | 三个模式入口仍可正常切换 |
| TB-08 | 当前在手动遥控状态，点击 `G` | 不再出现被顶出手动模式的现象 |

## 3. 事件消费

| 编号 | 场景 | 预期结果 |
| --- | --- | --- |
| TB-09 | 收到一次 `G` 后继续循环 | 不会重复翻转 |
| TB-10 | `cmdChar` 保持旧运动指令，再收到一次 `G` | 风机翻转，当前运动命令保持不变 |
| TB-11 | `AUTO` 下收到 `G` | 事件被消费，但不改变模式，不抢占自动风机控制 |

## 4. 构建验证

以下编译必须通过：

- `.\tools\compile_uno_cli.ps1`
- `.\tools\compile_uno.ps1`
- `libraries/Tyler_1/examples/Tyler_1/Tyler_1.ino`

## 5. 审计要求

在现行代码和现行文档范围内：

- 不存在由 `G` 触发的模式切换路径
- 不存在 `G` 进入体感模式的现行口径
- 不存在新的散写 `digitalWrite(A2, ...)` 业务控制

允许旧口径保留的位置仅有：

- `docs/SOFTWARE/C-1.1.3B_protocol_freeze.md` 中的历史对照说明
- `Tyler_1_Library/` 上游保留目录
