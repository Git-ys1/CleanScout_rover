# V-1.7.0 计划：/edge/ros 长连接入口 + edge-relay transport + 云端联调配套

## Summary
- 本轮只做静态施工与本地模拟验证，不做真实 Pi/ROS 实验室联调；真实 `edge-relay(Pi) -> ROS -> OpenRF1` 验收顺延到 C 线可联调时执行。
- 保留现有本地链路：`前端 -> backend -> rosbridge -> ROS`，不破坏 `ROS_TRANSPORT=rosbridge`。
- 新增云端链路：`前端 -> backend <- WSS /edge/ros <- edge-relay(Pi) -> ROS`，作为第三种 transport：`mock | rosbridge | edge-relay`。
- 前端仍只调用现有 `/api/ros/*` 与 `/api/integrations/ros/status`，不直连树莓派、rosbridge 或 `/edge/ros`。
- Git 建议：先提交 `checkpoint: before V-1.7.0`，新分支 `feature/v-1.7.0-edge-relay-cloud-transport`，按 `V-1.7.1` 到 `V-1.7.7` 拆分提交，最后只 supplement merge `vue3/` 到 `main`。

## Implementation Changes
- 后端启动层拆分：
  - `backend/src/app.js` 改为只创建并 export Express app，不再 `listen`。
  - 新增 `backend/src/server.js`，先加载 runtime env，再创建共享 HTTP server，挂载 Express REST 与 `/edge/ros` WebSocket upgrade。
  - `backend/package.json` 的 `main/start/dev`、systemd `ExecStart`、部署脚本统一切到 `node src/server.js`，避免部署仍跑旧入口。
- edge-relay WebSocket 与认证：
  - 使用现有 `ws` 依赖和共享 HTTP server，路径固定 `EDGE_RELAY_PATH=/edge/ros`。
  - 不使用 `verifyClient`；upgrade 只做路径分发，设备认证在首帧 `hello` 完成。
  - socket 建立后 5 秒内必须发送合法 `hello`，否则关闭；认证前忽略 heartbeat/telemetry/control。
  - `hello` 校验 `deviceId + token`，token 用 bcrypt 对比数据库 `EdgeDevice.tokenHash`；同一设备重复连接时关闭旧 session。
- Prisma 与设备模型：
  - 新增 `EdgeDevice`：`id`、`deviceId @unique`、`tokenHash`、`isEnabled`、`transport`、`topicsJson`、`capabilitiesJson`、`lastSeenAt`、`lastHelloAt`、`lastDisconnectAt`、`createdAt`、`updatedAt`。
  - `DeviceCache` 继续作为最新摘要缓存；edge telemetry 写入内存状态并 upsert `DeviceCache.summaryJson`。
  - `seed.js` 支持通过 `EDGE_DEVICE_BOOTSTRAP_ID` 和 `EDGE_DEVICE_BOOTSTRAP_TOKEN` 初始化设备；未配置 token 时不创建默认设备，避免明文/弱 token 入库。
- ROS integration 扩展：
  - `getRosRuntimeConfig()` 支持 `ROS_TRANSPORT=edge-relay`。
  - ROS transport 扩成 `mock | rosbridge | edge-relay`；`edge-relay` 复用现有 `sendRosCmdVel()`、`sendRosManualPreset()`、`getRosTelemetrySummary()`。
  - edge-relay 模式下，`cmd-vel/manual-preset` 转成下行 WS 帧：非零命令发 `manual_control`，零速或 hold 到期发 `stop`。
  - `/api/integrations/ros/status` 保持原接口并新增字段：`edgeRelayConnected`、`edgeDeviceId`、`lastTelemetryAt`、`lastRelayError`。
- 协议落地：
  - 入站 `hello`、`heartbeat`、`telemetry` 按任务书 JSON shape 实现。
  - 出站 `manual_control` 与 `stop` 按任务书 JSON shape 实现，`seq` 由 backend 递增生成。
  - telemetry 只接收摘要，不接收全量 LaserScan；`scanSummary` 映射到现有 summary，不新增全量 scan 存储。
- profile 与部署：
  - runtime profile 支持 `local-lan | public-cloud | public-edge`。
  - 新增 `deploy/env/vline-backend.public-edge.env.example`，固定 `ROS_TRANSPORT=edge-relay`、`EDGE_RELAY_ENABLED=true`、`EDGE_RELAY_PATH=/edge/ros`、`EDGE_ALLOWED_DEVICE_IDS=csrpi-001`。
  - `public-cloud` 继续 `ROS_TRANSPORT=mock`，`local-lan` 继续 `ROS_TRANSPORT=rosbridge`。
  - Nginx `api.hzhhds.top` 模板新增 `/edge/ros` WebSocket 反代，显式透传 `Upgrade` / `Connection`，并设置长 `proxy_read_timeout`。
- 前端最小可视化：
  - `src/stores/ros.js` 默认状态扩展 edge 字段。
  - 首页 ROS 卡片显示 `transport`、在线状态、`deviceId`、最近心跳。
  - 管理员页接入状态区显示 edge-relay 在线状态、最近 telemetry 时间、当前下行 transport。
  - 前端不新增 `/edge/ros` 客户端，不改 `/api/ros/*` 调用方式。

## Test Plan
- 静态与构建：
  - `node --check backend/src/app.js`
  - `node --check backend/src/server.js`
  - `node --check` 覆盖新增 edge-relay 模块。
  - `cmd /c npm.cmd run build:h5`
  - `cmd /c npm.cmd run build:mp-weixin:production`
- Prisma：
  - `npx prisma generate`
  - 使用临时 SQLite `DATABASE_URL` 跑 `npx prisma db push`，确认 `EdgeDevice` schema 可落库。
  - 使用临时 env 跑 seed，确认 `EDGE_DEVICE_BOOTSTRAP_TOKEN` 会写入 bcrypt hash，而不是明文。
- 本地 rosbridge 旧链不坏：
  - `APP_PROFILE=local-lan` 时确认 `/api/integrations/ros/status` 仍返回 `transport=rosbridge`。
  - 不要求实验室 ROS 在线；若 `10.117.77.84:9090` 离线，只记录不可达，不用 mock 冒充通过。
- edge-relay 本地模拟：
  - 使用临时 `public-edge` env 和临时 SQLite 启动 backend。
  - 用本地 Node WS 模拟 Pi 连接 `/edge/ros`，发送 `hello`、`heartbeat`、`telemetry`。
  - 验证 `/api/integrations/ros/status` 返回 `transport=edge-relay`、`edgeRelayConnected=true`、`edgeDeviceId=csrpi-001`。
  - 验证 `/api/ros/telemetry/summary` 从模拟 telemetry 更新。
  - 调用 `/api/ros/manual-preset`，确认模拟 WS 客户端收到 `manual_control`，hold 到期收到 `stop`。
  - 非法 token、禁用设备、非白名单 deviceId、未 hello 超时必须被拒绝。
- 部署模板：
  - Nginx 配置包含 `/api/` REST 反代与 `/edge/ros` WebSocket 反代。
  - 文档明确正式链路为 `https://api.hzhhds.top/api` 与 `wss://api.hzhhds.top/edge/ros`。
  - release 文档明确本轮未做真实 Pi/ROS 云端联调，只完成 backend 静态与模拟配套。

## Assumptions
- 本轮不做真实实验室 ROS 联调，因为实验室关闭；不能宣称 Pi 已收到真实 `/cmd_vel`。
- `edge-relay` 不替代 `rosbridge`；`local-lan` 仍然是 rosbridge，`public-edge` 才启用 edge-relay。
- 第一版不做设备管理 UI/API；设备 token 通过部署 env + seed 初始化。
- 第一版只支持单个在线目标设备；若同一 `deviceId` 重连，backend 关闭旧连接并使用新连接。
- `EDGE_ALLOWED_DEVICE_IDS` 非空时必须命中白名单；为空则仍要求 DB 中存在并启用该设备。
- `scanSummary` 只保存摘要，禁止全量 LaserScan 入库或直推前端。
