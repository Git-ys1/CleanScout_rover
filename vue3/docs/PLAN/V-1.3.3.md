# V-1.3.3 计划：以 `main` 为基线的第一版 ROS 接入层施工

## Summary
- 当前 `vue3` 已有 `OpenClaw` adapter、管理员工作台和 `integrations/*` 路径形状；本轮 ROS 接入沿用同一架构，不让前端直连树莓派、ROS master 或 OpenClaw。
- 已核实远端 `main` 可用，当前公开树莓派工作区确实带 `Raspberrypi/catkin_ws/src`；实际可见的关键 launch 是 `bench_full_stack.launch`、`slam/lidar_slam_pi.launch`，以及导航相关的 `nav_base_stack.launch` / `desk_map_navigation.launch`。文档按这些真实文件名冻结，不继续沿用失配的 `navigate.launch` 口径。
- 已核实 `main` 的 `Raspberrypi/catkin_ws/src` 中未发现 `rosbridge_suite`、`rosbridge_websocket`、`roslibjs`；因此本轮 V 线补的是 `backend` 侧 ROS adapter，不是假设树莓派已自带 Web 接入层。
- `bench_full_stack.launch` 已能证明当前现实基线是 `RPLIDAR + MPU6050 chain + csrpi_base_bridge + enc_to_raw_vel + EKF`；`csrpi_base_bridge` 里也能看到 `/cmd_vel -> wheel_targets` 的 `cmdvel_to_wheels.py`，但其 README 明确写了它还不是默认外部接入入口，因此 V 线仍按 `rosbridge` 作为优先外桥施工。

## Key Changes
- Git / 发布
  - 自测分支建议固定为 `feature/v-1.3.3-ros-adapter-main-baseline`。
  - 提交拆分固定为：
    - `V-1.3.3: scaffold backend ros adapter against main catkin_ws baseline`
    - `V-1.3.4: add ros status and cmd_vel api shell`
    - `V-1.3.5: add ros console on admin page and dashboard ros status card`
    - `V-1.3.6: freeze ros/openclaw boundary and main-merge release docs`
  - 发布顺序固定为：先自测分支冻结，再把 `vue3/` 作为 supplement merge 补进 `main`，不覆盖、不碰 C 线源码。

- Backend ROS adapter
  - 新增 `backend/src/integrations/ros/`，文件固定为 `index.js`、`transport.js`、`rosbridgeClient.js`、`mapper.js`、`stateCache.js`、`dto.js`。
  - 适配层接口固定为统一服务形状：`getStatus()`、`getTelemetrySummary()`、`sendCmdVel()`、`sendManualPreset()`。
  - `transport.js` 同时提供 `mock` 与 `rosbridge` 两种 transport；默认先落 `mock`，`rosbridge` 为真实接入位。
  - `rosbridgeClient.js` 使用 backend Node 侧 WebSocket 客户端直连 `rosbridge` JSON 协议，不在 backend 引入浏览器向的 `roslibjs`。
  - `stateCache.js` 负责缓存连接状态、最后心跳、最后错误、最后 `odom / imu / scan` 时间戳，以及摘要速度/位姿。
  - `dto.js` 固定内部统一命令 DTO：`ManualControlCommand = { source, linear, angular, holdMs, metadata }`；管理员按钮与后续 OpenClaw 解析结果都归一到这个 DTO。
  - 环境变量固定新增：
    - `ROS_ENABLED=true`
    - `ROS_TRANSPORT=mock`
    - `ROSBRIDGE_URL=ws://127.0.0.1:9090`
    - `ROS_CMD_VEL_TOPIC=/cmd_vel`
    - `ROS_ODOM_TOPIC=/odom`
    - `ROS_IMU_TOPIC=/imu/data`
    - `ROS_SCAN_TOPIC=/scan`
    - `ROS_CMD_REPEAT_HZ=10`
    - `ROS_CMD_DEFAULT_HOLD_MS=400`
    - `ROS_RECONNECT_DELAY_MS=1000`
  - 当前默认 topic 按 `main` 基线与 ROS 常规约定冻结为 `/cmd_vel`、`/odom`、`/imu/data`、`/scan`；后续若树莓派端变更，仅通过 env 改，不改前端接口。
  - 管理员固定控制交互固定采用你已拍板的 `backend holdMs` 方案：前端点一次 preset，backend 以 `holdMs + repeatHz` 做短持续发布并自动补一次 `stop`。

- Backend API / auth
  - 新增 `GET /api/integrations/ros/status`，需登录、不限管理员，返回：
    - `enabled`
    - `transport`
    - `connected`
    - `rosbridgeUrl`
    - `lastHeartbeatAt`
    - `lastError`
    - `cmdVelTopic`
    - `odomTopic`
    - `imuTopic`
    - `scanTopic`
  - 新增 `POST /api/ros/cmd-vel`，仅管理员，输入按你冻结的 `Twist` DTO，`holdMs` 允许省略，省略时取 `ROS_CMD_DEFAULT_HOLD_MS`。
  - 新增 `POST /api/ros/manual-preset`，仅管理员，请求体固定为 `{ preset, holdMs? }`，preset 仅允许：
    - `forward`
    - `backward`
    - `turn_left`
    - `turn_right`
    - `strafe_left`
    - `strafe_right`
    - `stop`
  - 新增 `GET /api/ros/telemetry/summary`，需登录、不限管理员，返回：
    - `odomAvailable`
    - `imuAvailable`
    - `scanAvailable`
    - `lastOdomAt`
    - `lastImuAt`
    - `lastScanAt`
    - `latestLinearSpeed`
    - `latestAngularSpeed`
    - `latestPose2D`
  - 控制类接口统一响应 `accepted / transport / command / scheduledStopAt`；mock transport 也必须打通，不允许只留空壳。
  - 本轮不把 ROS 挂进聊天接口，对话页仍只承载 `mock / OpenClaw` 自然语言平面。

- Frontend
  - 新增 `src/api/ros.js` 与 `src/stores/ros.js`，不要把 ROS 状态塞进 `deviceStore` 或 `adminStore`。
  - 首页新增只读 ROS 状态卡片，展示 `transport / connected / lastHeartbeatAt`。
  - 管理员页在现有工作台内扩展 ROS 控制台，不新增一级导航；推荐把现有“接入状态”区扩成“接入状态 / ROS 控制”，同时展示 `OpenClaw` 状态和 `ROS` 状态。
  - ROS 控制台按钮固定为：
    - 前进
    - 后退
    - 左转
    - 右转
    - 左平移
    - 右平移
    - 停止
  - 前端按钮只发 preset，不做前端按下/抬起遥控；管理员页显示本次下发结果与当前 transport。
  - 对话页不加入 ROS 固定控制 UI，只保留自然语言平面。

- 文档 / release
  - 新增 `docs/releases/V-1.3.3/README.md`，明确写：
    - `main` 现已包含 `Raspberrypi/catkin_ws`
    - 当前接入基线按真实文件名冻结为 `bench_full_stack.launch`、`slam/lidar_slam_pi.launch`、`nav_base_stack.launch`、`desk_map_navigation.launch`
    - `main` 当前没有现成 `rosbridge_suite / roslibjs` Web 接入层
    - V 线补的是 `backend ROS adapter`
    - `ROS` 负责固定控制与状态摘要，`OpenClaw` 负责自然语言
    - 本轮完成自测分支冻结后，再 supplement merge 到 `main`
  - 根 `README.md` 同步补一段 ROS / OpenClaw 边界和 `main = 统一真相分支` 规则。

## Test Plan
- 仓库真相与文档
  - release 文档必须引用 `main` 中真实存在的 launch 名，不再写失配的 `navigate.launch`。
  - 文档必须明确“未发现现成 rosbridge Web 接入层”。
- Backend mock mode
  - `ROS_TRANSPORT=mock` 时，`GET /api/integrations/ros/status` 返回 `transport=mock`。
  - `POST /api/ros/cmd-vel` 能返回 `accepted=true` 与标准 command DTO。
  - `POST /api/ros/manual-preset` 能把 preset 映射成 `ManualControlCommand` 并自动补 stop。
  - `GET /api/ros/telemetry/summary` 至少能返回结构完整的 mock/null 摘要。
  - 非管理员调用 `/api/ros/cmd-vel` 与 `/api/ros/manual-preset` 返回 `403`。
- Frontend
  - 首页可见 ROS 状态卡片。
  - 管理员页可见 ROS 控制台和 ROS 状态。
  - 聊天页不出现 ROS 固定控制按钮。
- Rosbridge mode
  - 当 `ROS_TRANSPORT=rosbridge` 且 `ws://<pi>:9090` 可达时，status 返回 `connected=true`。
  - 收到 `odom / imu / scan` 后，`telemetry/summary` 的时间戳与速度摘要会更新。
  - 控制接口会按 `cmd_vel` 发布，并在 `holdMs` 到期后自动发一次零速 stop。

## Assumptions
- 文档与实现都以当前抓到的 `main` 真相为准：`bench_full_stack.launch`、`slam/lidar_slam_pi.launch`、`nav_base_stack.launch`、`desk_map_navigation.launch`；若树莓派端后续又改，V 线下一轮继续跟 `main`，不跟旧 feature 分支。
- 本轮不把 `rosbridge_suite` 源码 vendoring 到仓库，只把它视为树莓派运行环境补充依赖。
- `manual-preset` 的默认速度先用保守值冻结：前后 `±0.20 m/s`、平移 `±0.15 m/s`、转向 `±0.35 rad/s`、默认 `holdMs=400`；这些值写进文档作为“待树莓派负责人确认的运行参数”，后续只通过配置调。
- `connected` 语义按“当前选中 transport 是否健康”处理；因此 mock transport 可视为 `connected=true`，真实 ROS 是否接入由 `transport=mock|rosbridge` 区分，不让前端在 mock 阶段永久显示断开。
