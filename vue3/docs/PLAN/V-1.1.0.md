# V-1.1.0 实际业务开发第一轮计划

## Summary
- 基于现有 `vue3/` 基线扩展，不推翻 `src/*` 结构；前端继续使用当前 uni-app Vue3 CLI 工程，首页壳子在现有 `src/pages/index/index.vue` 上升级，不另起新目录体系。
- 本轮强制验收平台锁定为 `H5 本地联调`；前端通过 `npm run dev:h5`，后端本地 Express 提供 `http://127.0.0.1:3000/api`，不把微信小程序联调纳入本轮通过条件。
- 后端从 `vue3/backend/` 正式落地，技术栈固定为 `Express + Prisma + SQLite + bcrypt + jsonwebtoken`，工程制式固定为 `JavaScript + ESM`。
- 本轮只打通：登录、注册、默认管理员、角色保护、首页 mock 摘要、聊天 mock、管理员 mock 指令、前后端接口壳子；不接真实 openclaw、树莓派、真实设备控制或复杂 UI。

## Implementation Changes
- Git 流程固定为：
  - 开工前先做 `checkpoint: V-1.1.0 开工前基线`
  - 分支固定为 `feature/v-1.1.0-auth-shell`
  - 提交至少 5 笔，统一中文标题：
    - `V-1.1.0A: 搭建 backend 鉴权壳子与 Prisma SQLite`
    - `V-1.1.0B: 搭建前端鉴权页面与 Pinia auth store`
    - `V-1.1.0C: 搭建首页聊天管理员占位页与 API 壳子`
    - `V-1.1.0D: 冻结 V-1.1.0 文档`
    - `V-1.1.0E: 回填 published_at_commit 并发布云端镜像`
- 前端页面固定新增并注册到 `src/pages.json`：
  - `src/pages/auth/login.vue`
  - `src/pages/auth/register.vue`
  - `src/pages/chat/index.vue`
  - `src/pages/admin/index.vue`
  - `src/pages/profile/index.vue`
  - 保留 `src/pages/index/index.vue` 作为首页/总览页且继续作为首个页面；未登录时首页 `onLoad/onShow` 直接跳转登录页。
- 前端页面行为固定为：
  - 登录页：账号、密码、登录、跳注册；成功后 `reLaunch` 到首页。
  - 注册页：用户名、密码、确认密码、注册；成功后提示并跳登录页，不自动登录。
  - 首页：显示当前用户、mock 设备摘要、最近消息摘要、进入聊天/管理员/个人页入口；普通用户不显示管理员入口。
  - 聊天页：展示 `chatStore.messages`，发送后调用 mock API，回填用户消息与 mock 回复。
  - 管理员页：进入即做 `role===admin` 校验，非管理员 `redirectTo/reLaunch` 回首页并提示无权限。
  - 个人页：显示用户名、角色、token 登录态摘要；退出登录后清空本地状态并回登录页。
- Pinia 接入固定为：
  - 根包新增 `pinia`
  - `src/main.js` 中创建 `pinia` 并 `app.use(pinia)`
  - 新增 4 个 store：`auth.js`、`app.js`、`chat.js`、`device.js`
  - `authStore` 负责 `token/userInfo/role/isLoggedIn` 与 `login/logout/register/fetchMe/setToken/restoreSession`
  - `appStore` 负责 `appReady/currentTab/loading/networkStatus`
  - `chatStore` 负责 `messages/sending/draftText`
  - `deviceStore` 负责 `deviceOnline/deviceSummary/lastTelemetry/adminConsoleResult`
  - 登录态持久化统一使用 `uni.setStorageSync/uni.getStorageSync`
- 前端 API 层固定为：
  - `src/api/config.js`：`API_BASE_URL='http://127.0.0.1:3000/api'`、超时、占位 `WS_BASE_URL`
  - `src/api/http.js`：统一 `uni.request` 封装、Bearer token 注入、错误包装
  - `src/api/auth.js`、`device.js`、`chat.js`、`admin.js`
  - `src/api/ws.js`：只保留 `SocketTask` 预留接口与 TODO，不建真实连接
  - 页面禁止直接写死请求地址
- 后端工程固定为 `vue3/backend/`，目录固定为：
  - `src/routes`、`controllers`、`middleware`、`services`、`utils`、`app.js`
  - `prisma/schema.prisma`、`prisma/seed.js`
  - `package.json`、`.env.example`、本地 `.env`
- 后端依赖与脚本固定为：
  - 运行依赖：`express`、`cors`、`dotenv`、`bcrypt`、`jsonwebtoken`、`@prisma/client`
  - 开发依赖：`prisma`、`nodemon`
  - 脚本：`dev`、`start`、`prisma:generate`、`prisma:migrate`、`prisma:seed`
- 后端配置固定为：
  - `PORT=3000`
  - `DATABASE_URL="file:./dev.db"`，数据库文件落在 `backend/prisma/`
  - `JWT_SECRET=<local-dev-secret>`
  - `JWT_EXPIRES_IN=7d`
  - H5 本地联调开启 CORS，放通 `http://127.0.0.1:5173` 和 `http://localhost:5173`
- Prisma 模型固定为：
  - `User`: `id String @id @default(cuid())`、`username String @unique`、`passwordHash String`、`role Role`、`createdAt`、`updatedAt`
  - `DeviceCache`: `id String @id @default(cuid())`、`deviceId String @unique`、`summaryJson String`、`updatedAt`
  - `MessageCache`: `id String @id @default(cuid())`、`userId String`、`role String`、`content String`、`createdAt`
  - `Role` 枚举固定为 `user`、`admin`
- Seed 规则固定为：
  - `prisma db seed` 可重复运行
  - 若 `admin` 已存在则跳过
  - 默认管理员：`username=admin`、`password=123456`、`role=admin`
  - 密码使用 `bcrypt`，salt rounds 固定为 `10`
  - 同时可初始化一条默认 `DeviceCache(deviceId='mock-rover-001')`
- 鉴权与 mock 接口固定为：
  - `POST /api/auth/register`：注册普通用户，校验唯一用户名，默认 `role=user`
  - `POST /api/auth/login`：校验 bcrypt，返回 JWT 与用户信息
  - `GET /api/auth/me`：Bearer token 鉴权返回当前用户
  - `POST /api/auth/logout`：返回成功，前端负责清 token
  - `GET /api/device/summary`：返回 `online/battery/taskStatus/lastUpdate`
  - `GET /api/chat/history`：按当前用户返回消息历史，无数据时返回欢迎消息
  - `POST /api/chat/send`：持久化用户消息并生成一条 mock 回复
  - `POST /api/admin/command`：仅 `admin` 可调用，返回 `command/accepted/result`
  - `GET /api/system/health`：返回服务、数据库、时间戳状态
- 统一响应格式固定为：
  - 成功：`{ success: true, data: ... }`
  - 失败：`{ success: false, message: "...", code?: "..." }`
- 中间件固定为：
  - `authRequired`：解析 Bearer token，挂载 `req.user`
  - `adminOnly`：校验 `req.user.role === 'admin'`
  - `errorHandler`：统一 JSON 错误输出

## Public Interfaces
- 前端状态接口：
  - `authStore.userInfo = { id, username, role }`
  - `authStore.token` 保存 JWT 原文
  - `chatStore.messages` 元素固定为 `{ id, role, content, createdAt, pending? }`
  - `deviceStore.deviceSummary` 固定为 `{ deviceId, online, battery, taskStatus, lastUpdate }`
- JWT payload 固定为：`{ sub: user.id, username: user.username, role: user.role }`
- 前端路由保护策略固定为：
  - 首页、聊天页、个人页、管理员页都要求登录
  - 管理员页额外要求 `role=admin`
  - 登录/注册页在已登录时直接跳首页

## Test Plan
- 基线验证：
  - 当前前端 `build:h5` 已可通过；V-1.1.0 完成后再次执行 `cmd /c npm.cmd run build:h5`
  - 后端执行 `npm install`、`npx prisma generate`、`npx prisma migrate dev`、`npx prisma db seed`、`npm run dev`
- 鉴权链：
  - `admin / 123456` 可登录，`/api/auth/me` 返回 `role=admin`
  - 普通用户可注册并登录，返回 `role=user`
  - 错误密码返回统一失败 JSON
- 页面链：
  - 未登录访问首页时跳登录
  - 注册成功后跳登录
  - 登录成功后进入首页
  - 首页能拉到 mock 设备摘要
  - 聊天页能拉历史并发送 mock 消息
  - 普通用户进入管理员页被拦截
  - 管理员进入管理员页可看到命令区并成功调用 mock 接口
  - 个人页退出登录后回登录页
- 发布链：
  - 新增 `docs/releases/V-1.1.0/README.md`
  - 文档写明 `based_on_branch`、`published_to_root`、`published_at_commit`
  - 云端镜像继续发布到当前最新主施工分支根目录的 `vue3/`

## Assumptions
- 本轮强制验收仅针对 `H5 本地联调`；微信小程序端不作为 V-1.1.0 通过条件。
- 后端采用 `JavaScript + ESM`，不在本轮引入 TypeScript。
- `backend/.env` 作为本地运行文件存在，仓库追踪 `backend/.env.example`；真实密钥不进 Git。
- `published_at_commit` 采用本地 release/publish 提交哈希回填，因此 V-1.1.0 至少需要 5 笔提交而不是仅 4 笔。
- 真实设备链、真实 SocketTask、复杂 RBAC、多设备管理、验证码与美化 UI 全部明确排除在本轮之外。
