# V-1.0.0 立项冻结计划

## Summary
- 当前 `f:\Project\CSc——uniapp\vue3` 按“V 线本地主仓”处理；云端主仓的 `vue3/` 作为后续镜像发布目录，不在本地仓内混用。
- 目录冻结按现有 uni-app Vue3 CLI 实际结构执行：源码统一放在 `src/` 下，根目录只保留工程配置与文档，不在立项轮做模板重排。
- 本轮只做“立项、基线、文档、纪律、可运行性验证”，不做业务功能、不接真实设备、不引入真实 transport。
- 现状需先补齐的事实阻塞：当前目录还不是 Git 仓；`node_modules` 不存在；`npm.ps1` 被 PowerShell 执行策略拦截；默认模板页和项目元信息尚未替换。

## Implementation Changes
- 初始化本地主仓并冻结 Git 纪律：在当前目录建立独立 Git 仓，首轮分支名固定为 `feature/v-1.0.0-uniapp-init`，首轮提交信息固定为 `V-1.0.0: freeze uni-app vue3 project initiation`。
- 验证本地可运行性：使用 `npm.cmd install` 安装依赖，再用 `npm.cmd run build:h5` 作为最小通过验证；若补充人工验证，则再跑 `npm.cmd run dev:h5`。文档里明确 Windows 下优先使用 `npm.cmd`，避免 `npm.ps1` 执行策略报错。
- 规范化模板基线但不进入业务开发：把 `package.json` 项目名从模板默认值改为 `vue3`；把首页与工程描述改成 V 线“设备前端控制端”占位文案；清掉明显的模板默认/乱码注释；不新增业务页面、不写通信逻辑。
- 冻结目录口径为：
  - 根目录：`README.md`、`docs/releases/V-1.0.0/README.md`、`package.json`、`vite.config.js`、`.gitignore`
  - 源码目录：`src/pages`、`src/components`、`src/stores`、`src/api`、`src/composables`、`src/utils`、`src/types`、`src/mocks`
  - 工程关键文件继续保留在 `src/`：`App.vue`、`main.js`、`pages.json`、`manifest.json`、`uni.scss`
- 文档里明确“任务书示意目录”与“CLI 实际目录”的映射关系：任务书中的 `pages/`、`stores/`、`api/` 等，落地时统一解释为 `src/pages/`、`src/stores/`、`src/api/`。
- 冻结接口与类型边界为“文档级 contract”，本轮不落业务代码：`TransportAdapter`、`ConnectionState`、`DeviceState`、`ConversationMessage`、`AdminCommand`、`TelemetrySnapshot`。文档只定义职责和边界，不发明最终 wire schema。
- Pinia 在本轮只冻结为指定状态管理方案，不在 V-1.0.0 接入空 store、也不把状态层开发混进本轮。

## Documentation Freeze
- `README.md` 用中文完整写清：项目定位、技术基线、当前不立后端的原因、补后端触发条件、目录规划、页面规划、通信抽象原则、Git/发布纪律、运行验证方式。
- `docs/releases/V-1.0.0/README.md` 记录本轮冻结结论、当前环境现状、验证命令、通过标准、发布要求，以及“未同步到云端主仓不算冻结完成”。

## Test Plan
- `npm.cmd install` 成功完成，依赖可解析。
- `npm.cmd run build:h5` 成功通过；如做补充验证，`npm.cmd run dev:h5` 能启动默认壳层页。
- 根目录存在 `README.md` 与 `docs/releases/V-1.0.0/README.md`，且覆盖任务书要求的全部章节。
- `src/pages.json` 保持首页注册有效，文档明确“未注册页面会被忽略”的纪律。
- 本地主仓完成首轮提交后 `git status` 为 clean。
- 云端主仓最新分支根目录完成镜像同步后，至少包含 `vue3/`、`vue3/README.md`、`vue3/docs/releases/V-1.0.0/README.md`。

## Assumptions
- 当前工作目录就是 V 线本地主仓，不是云端主仓 checkout。
- 云端主仓镜像同步会在单独的主项目仓中完成；只有同步完成后，V-1.0.0 才算正式冻结。
- 本轮不接入真实树莓派/openclaw，不实现 mock/ws/http transport，不实现 Pinia store，不开始 UI 设计轮。
- 所有冻结文档统一使用中文；代码仍保持 uni-app Vue3 CLI 默认的 JavaScript 形态，不在本轮转 TypeScript。
