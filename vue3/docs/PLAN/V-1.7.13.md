# V-1.7.13 计划：首页块状重构 + 聊天伪流式气泡化 + 中文状态语义统一

## Summary

当前实际基线是：

- 首页还是纵向一条条堆叠：`OpenMV / ROS / 设备摘要 / 最近消息`
- ROS 控制台还在管理员页
- 聊天页还是单列卡片消息，不是左右气泡，也不具备流式展示壳子
- backend 聊天接口仍是一次性返回完整回复，不是真流式协议

本轮冻结为纯前端交互重构轮，**不改现有 backend 聊天协议**，先把首页信息架构、控制入口、聊天显示模型和中文状态语义做对，为后续接 OpenClaw 真流式留壳。

已拍板实现方向：

- 首页：两列块状模块
- 首页控制：搬到主页面，且**仅管理员可操作**
- 聊天：先做**前端伪流式**
- 首页模块组织：**总览 + 交互混合**
- 聊天继续作为独立 tab，不把聊天输入器搬回首页

## Key Changes

### 1. 首页改成两列块状模块，不再纵向清单式展示

首页重构为固定两列卡片布局，默认按手机端也保持“一行两个模块”的块状感，不再把各段落一条条往下排。

首页固定四块：

- `前视画面`：显示 OpenMV 缩略预览、图传状态、最近错误；提供“查看大图”入口，进入新的前视详情页
- `设备状态`：显示在线、电量、任务状态、最近更新时间，只读摘要，不承载控制
- `ROS 接入状态`：显示当前链路、连接状态、最近心跳、最近遥测，只读摘要
- `快捷控制台`：直接放在首页，承载固定控制按钮与最后一次控制结果

交互规则固定为：

- `前视画面` 卡片：点击进入新详情页，展示更大画面和更完整的预览状态
- `设备状态`、`ROS 接入状态`：保持首页只读摘要，不再塞一堆细项文字
- `快捷控制台`：直接在首页内交互，不跳管理员页
- 聊天仍通过底部 `对话` tab 进入，不再给首页再塞一个聊天输入区

### 2. ROS / 设备控制从管理员页挪到首页

当前管理员页里的 `ROS 控制台` 整块迁到首页。

权限固定为：

- 所有人都能在首页看到状态块
- 只有 `admin` 能看到可点击的控制按钮
- 非管理员只显示“控制受限 / 仅管理员可操作”的只读态，不允许误触发

首页控制台固定承载：

- 前进
- 后退
- 左转
- 右转
- 左平移
- 右平移
- 停止
- 最近一次下发结果
- 当前下行 transport

管理员页本轮收口为：

- 保留 `用户管理`
- 保留 `系统开关`
- 保留 `接入状态` 只读信息
- 移除 `ROS 控制按钮区`

### 3. 聊天页改成伪流式左右气泡，为 OpenClaw 真流式预留消息模型

本轮不改 backend `/api/chat/send` 的一次性返回模式，先做前端消息展示壳。

聊天页固定改为：

- 用户消息：右侧气泡
- 助手消息：左侧气泡
- 系统消息：居中或靠下的小字提示，不做大气泡
- 顶部 transport 状态条保留，但改成中文彩色状态标签，不再直接显示英文原值

前端消息模型新增本地展示态：

- `kind: user | assistant | system`
- `streaming: boolean`
- `displayText: string`

发送流程固定为：

1. 用户发送后，立即插入右侧用户气泡
2. 同时插入一个左侧助手占位气泡，初始 `displayText=''`
3. backend 返回完整回复后，前端按打字机方式把完整文本逐步填入 `displayText`
4. 打字完成后，把该条消息状态从 `streaming=true` 改为 `false`

系统消息只在前端本地生成，不要求本轮后端持久化，主要用于：

- 当前链路切换提示
- OpenClaw 回退 mock 提示
- 连接异常 / 重试提示
- 欢迎语或会话提示的轻量化显示

聊天页本轮明确不做：

- SSE
- chunked streaming
- WebSocket 聊天流
- OpenClaw 真流式协议

### 4. 全局英文状态词改成带颜色的中文语义标签

新增统一的前端状态展示映射层，所有页面不再直接展示英文原始值。

至少统一替换这些词：

- `mock` -> `模拟链路`
- `openclaw` -> `OpenClaw 链路`
- `rosbridge` -> `ROS 桥接`
- `edge-relay` -> `边缘中继`
- `healthy` -> `正常`
- `degraded` -> `降级`
- `disabled` -> `未启用`
- `error` -> `异常`
- `connected` -> `已连接`
- `disconnected` -> `未连接`
- `online` -> `在线`
- `offline` -> `离线`
- `available` -> `正常`
- `missing` -> `缺失`
- `ready` -> `就绪`
- `loss` -> `丢失`

展示规则固定为：

- 所有 transport / 状态 / 遥测关键词统一走彩色标签，不直接裸文本输出英文
- 颜色语义统一：
  - 正常/在线/就绪：绿色系
  - 降级/模拟/待接入：橙黄色系
  - 异常/丢失/未连接：红色系
  - 普通信息/未启用：灰蓝系

### 5. 新增前视详情页，但不扩张更多新页面

本轮只新增一个和首页直接相关的详情页：

- `OpenMV 前视详情页`

用途固定为：

- 放大显示前视画面
- 展示图传模式、刷新周期、图传地址、最后状态
- 不承载设备控制
- 不承载聊天

其余交互页保持现状：

- 聊天继续用现有 `对话` 页
- 用户/系统配置继续留在管理员页
- 不再新增设备详情页或 ROS 详情页

## Public Interfaces / Frontend Contracts

本轮不改现有 backend 外部接口：

- `POST /api/chat/send` 保持一次性返回
- `GET /api/integrations/openclaw/status` 保持不变
- `GET /api/integrations/ros/status` 保持不变
- `POST /api/ros/manual-preset` 保持不变

前端新增或调整的展示层契约：

- 聊天页本地消息视图模型新增：`kind`、`streaming`、`displayText`
- 首页新增统一状态映射工具，所有英文状态统一转中文标签
- OpenMV 首页卡片新增“查看大图”跳转到前视详情页

## Test Plan

- 首页在手机宽度下显示为两列块状模块，不再是四段纵向清单
- 首页固定出现四块：前视画面、设备状态、ROS 接入状态、快捷控制台
- 管理员登录时首页控制按钮可点；普通用户只能看到只读态
- 管理员页不再出现 ROS 控制按钮区，但用户管理/系统开关/接入状态仍正常
- 聊天页消息改为左右气泡，用户右侧、助手左侧、系统消息小字
- 聊天发送后，助手回复以伪流式逐字显示，不再整段瞬间出现
- 当前 transport、状态、遥测关键词全部显示为中文彩色标签，不再直接出现 `healthy / ready / loss / connected` 等英文词
- 现有 backend 接口不改动时，H5 与小程序页面仍能正常联调
- OpenMV 首页卡片可进入前视详情页，详情页能复用现有预览链路

## Assumptions

- 本轮“流式”定义为前端伪流式，不要求 backend 真流式输出
- 首页控制权限按现有安全边界冻结为“仅管理员可操作”
- 聊天仍是独立 tab，不把聊天输入器搬到首页
- 管理员页本轮继续保留后台管理职责，不再承载设备控制职责
- 首页块状布局优先保证两列视觉结构，不再为了兼容旧样式退回单列清单式展示
