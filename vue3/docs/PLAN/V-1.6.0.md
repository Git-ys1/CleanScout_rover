# V-1.6.0 计划：runtime profile 收口 + 第二次 ROS 真联调 + public-cloud 交付包 + edge-relay RFC

## Summary
- 当前本地源仓和镜像仓 `main` 已含 V-1.5.6；本地 `src/api/config.js` 已是 `VITE_*`，但 V-1.6.1 仍要做“main 漂移收口”的回归确认与文档冻结，避免公开端再出现旧代码口径。
- 本轮不推翻已跑通的 `backend -> rosbridge -> ROS` 链路；第二次联调继续使用 `ws://10.117.77.84:9090`、`/cmd_vel /odom /imu/data /scan`。
- 本轮云端部署范围按“仓库交付”执行：交付 `public-cloud` profile、Nginx/Certbot/systemd 配置模板和文档，不实际登录 VPS 部署。
- “ROS 主动连后端 / backend 中心化”只写 RFC，不替换当前 rosbridge transport，不要求 C 线本轮实现。

## Key Changes
- Git 流程：新建 `checkpoint: before V-1.6.0`，分支 `feature/v-1.6.0-runtime-profiles-public-rfc`，提交拆分按 `V-1.6.1` 到 `V-1.6.6`。
- 前端 profile：保留 `VITE_API_BASE_URL / VITE_WS_BASE_URL`，补齐 `.env.mp-weixin.production`、`.env.app.production`；生产 API 固定为 `https://api.hzhhds.top/api`，WS 继续留空。
- 前端脚本：明确区分 local / production，补 `dev:h5:local`、`build:mp-weixin:local`、`build:mp-weixin:production`、`build:app`、`build:app:production`；保留现有通用脚本但文档注明默认用途。
- backend profile loader：新增最小启动层，支持 `APP_PROFILE=local-lan|public-cloud` 和 `ENV_FILE=/path/to/env`；启动时打印 `APP_PROFILE`、实际 env file、`ROS_TRANSPORT`、`ROSBRIDGE_URL`、`OPENCLAW_ENABLED`。
- backend 模板：继续沿用 `deploy/env/vline-backend.local-lan.env.example` 和 `deploy/env/vline-backend.public.env.example`，但让启动流程和文档真正引用它们；`public-cloud` 保持 `ROS_TRANSPORT=mock`，不直连局域网树莓派。
- ROS 第二次联调：用 `local-lan` profile 启动 backend，再验证 `transport=rosbridge`、`connected=true`、telemetry 持续更新、管理员低速预设命令成功；结果写入 `docs/releases/V-1.6.0/ros-second-local-test.md`。
- 公网部署交付包：新增 `api.hzhhds.top` 的 Nginx server block 示例、Certbot 命令、systemd/env 落地说明；文档明确微信小程序配置的是 `request 合法域名`，不是 `CORS_ALLOWED_ORIGINS`。
- 联合 RFC：新增 `docs/releases/V-1.6.0/backend-centric-ros-relay-rfc.md`，定义可选 `edge-relay` transport 草案：Pi 侧 relay 主动连 backend、上送遥测摘要和心跳、接收 `ManualControlCommand` 后本地转 `/cmd_vel`。
- Release 文档：新增 `docs/releases/V-1.6.0/README.md`，记录 `based_on_branch`、`published_to_root`、`published_at_commit`、已完成项、未完成项和 C 线待评审项。

## Public Interfaces / Config
- 前端新增生产环境文件：`.env.mp-weixin.production`、`.env.app.production`。
- backend 新增运行参数：`APP_PROFILE`、`ENV_FILE`。
- backend profile 固定值：`local-lan` 用 `ROS_TRANSPORT=rosbridge` 与 `ROSBRIDGE_URL=ws://10.117.77.84:9090`；`public-cloud` 用 `ROS_TRANSPORT=mock`。
- 公网域名冻结：API 使用 `https://api.hzhhds.top/api`；未来 H5 使用 `https://h5.hzhhds.top`；未来 admin web 使用 `https://admin.hzhhds.top`。
- RFC 新增但不启用的 transport 名称：`edge-relay`。

## Test Plan
- 代码漂移回归：确认 `src/api/config.js` 不再出现硬编码 `127.0.0.1:3000/api`；确认生产构建产物含 `https://api.hzhhds.top/api`。
- 前端构建：跑 `build:h5`、`build:mp-weixin:local` 或等价 local 构建、`build:mp-weixin:production`，并检查产物 API 地址分别正确。
- backend profile：分别用 `APP_PROFILE=local-lan` 和 `APP_PROFILE=public-cloud` 启动，确认日志打印 profile、env file、ROS/OpenClaw 关键配置。
- ROS 第二次真联调：`GET /api/integrations/ros/status` 返回 `transport=rosbridge` 与 `connected=true`；`GET /api/ros/telemetry/summary` 时间戳持续更新；`POST /api/ros/manual-preset` 低速命令成功。
- 公网交付包：检查 Nginx 配置只反代到 `127.0.0.1:3000`，文档明确 HTTPS、微信合法域名、ICP 条件和 CORS 边界。
- 发布：先推自测分支，再只把 `vue3/` supplement merge 到 `main`，不改 `Raspberrypi/` 或 C 线目录。

## Assumptions
- V-1.6.4 只做仓库级交付，不实际登录服务器部署。
- 当前 `10.117.77.84:9090` 仍由 C 线保持可用；如果联调时不可达，本轮不能用 `mock` 冒充通过，只能记录阻塞。
- 当前 backend 仍不提供真实 `/ws`，所以所有前端 `VITE_WS_BASE_URL` 继续留空。
- `edge-relay` 只是 RFC 草案，需 C 线负责人评审后才能进入施工轮。
