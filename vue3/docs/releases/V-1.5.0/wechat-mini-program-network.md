# V-1.5.0 微信小程序网络规则

## 当前结论

微信小程序的网络访问规则必须拆成两套口径：

1. 本地调试
2. 正式发布 / 真机正常访问

这两套规则不能混写，也不能和 backend 的 `CORS_ALLOWED_ORIGINS` 混为一谈。

## 本地调试

本地调试阶段，可以让微信小程序指向：

```text
http://10.117.77.190:3000/api
```

前提是：

- 微信开发者工具已开启“不校验合法域名、TLS 版本以及 HTTPS 证书”
- 如果走手机调试，也必须按官方调试说明开启相应调试态

当前推荐做法：

- H5 本地调试继续走 `http://127.0.0.1:3000/api`
- 微信小程序本地调试走 `http://10.117.77.190:3000/api`
- backend 本机继续连局域网树莓派 `ws://10.117.77.84:9090`

## 正式发布 / 真机正常访问

正式发布后，必须满足：

- 使用公网可访问的 `HTTPS` 域名
- 该域名已配置进微信小程序后台的 `服务器域名`
- 不能使用 `localhost`
- 不能使用 IP
- 在微信后台配置域名时不写端口

说明：

- 实际请求 URL 可以带端口，前提是你的反向代理或部署方案允许
- 如果最终 API 统一收敛到 `443`，则请求 URL 可不显式带端口

## 和 CORS 的关系

`CORS_ALLOWED_ORIGINS` 只用于浏览器 H5 / admin web 的 Origin 白名单，不用于小程序服务器域名配置。

也就是说：

- 小程序地址不要填进 `CORS_ALLOWED_ORIGINS`
- 小程序正式可访问与否，关键看微信后台 `服务器域名`
- 当前本地联调阶段，`CORS_ALLOWED_ORIGINS` 只需保证 H5 浏览器可用

## 当前冻结口径

- 本地联调：`http://10.117.77.190:3000/api`
- 正式发布：`https://api.<你的域名>/api`
- 本轮不要求真公网打通，但必须先把规则写清
