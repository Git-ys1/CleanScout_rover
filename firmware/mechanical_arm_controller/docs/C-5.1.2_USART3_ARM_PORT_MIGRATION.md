# C-5.1.2 USART3 Arm Port Migration

日期：2026-07-09

## 结论

C-5.1.2 已将 RF1 机械臂控制入口固定到 OpenRF1 蓝牙串口 USART3。

冻结后的串口资源为：

| 串口 | OpenRF1 接口 | 用途 |
|---|---|---|
| USART2 / 115200 8N1 | 用户串口 H5，PA2_TX2 / PA3_RX2 | 保留给未来树莓派底盘 50Hz 控制链路 |
| USART3 / 115200 8N1 | 蓝牙串口 H6，PB10_TX3 / PB11_RX3 | 香橙派机械臂视觉追踪与后续抓取控制入口 |
| UART5 | 总线接口 | RF1 到总线舵机等下游设备，不作为香橙派入口 |

本轮没有修改 `firmware/openrf1_motion_controller/`，没有改变底盘协议，没有引入 ROS、D430/D435 或抓取逻辑。

## 代码改动

工作目录：

```text
firmware/mechanical_arm_controller/
```

本轮先从 `firmware/mechanical_arm_official_baseline/` 复制官方可烧录 Keil 工程到自研目录，再只修改自研副本。

关键改动：

1. `User/app_uart.c`
   - 保留 `uart1_init(115200)` 用于调试/兼容。
   - 移除 `uart2_init(115200)`，USART2 不再参与机械臂链路。
   - 保留 `uart3_init(115200)`，作为香橙派 CH340 到 OpenRF1 H6 的机械臂入口。
   - 保留 `uart5_init(115200)`，作为下游总线接口。
2. `User/Components/y_global/y_global.c`
   - `zx_uart_send_str()` 不再向 USART2 转发机械臂帧。
   - 防止 USART2 未初始化时调用 `uart2_send_str()` 卡在 TXE 等待。
3. `.gitignore`
   - 新增 `mechanical_arm_controller` 的 Keil 本地生成文件忽略规则。
   - 只保留 `Output/template.hex` 和 `Output/template.build_log.htm` 作为可复现/可烧录证据。

## 验收结论

通过项：

1. RF1 已烧录 `firmware/mechanical_arm_controller/Output/template.hex`。
2. 机械臂自研工程入口已固定到 USART3。
3. USART2 不再初始化，也不再被机械臂转发函数调用。
4. 香橙派能扫描到 CH340 `/dev/ttyUSB0`。
5. 单舵机真实 yaw 命令已通过 `/dev/ttyUSB0` 发出。
6. 普通相机 + YOLO11 能识别 `bottle`。
7. 80 帧短跑中 `bottle` 命中 73 帧，真实机械臂命令发出 37 帧。
8. `firmware/openrf1_motion_controller/` 未被本轮修改。

未做项：

1. 未做 D430/D435。
2. 未做深度测距。
3. 未做抓取。
4. 未做 ROS。
5. 未做底盘与机械臂协议合并。
